# CHECKS — 외관 완성 루프 게이트

## 검증 명령 (must-pass 기준선)

```powershell
cd "D:\_Project\xd-drawing-system"
npm test            # Vitest — 현 기준선 49 PASS (M4 완료 시점, 2026-06-24). 직전 M3=43, M2=39, M1=34.
npm run build       # tsc + vite build
git diff --check    # 공백 오류 없음
```

UI 렌더 검증(검증팀):
```powershell
npm run dev         # 로컬 dev 서버 (ES module은 file:// 불가 → HTTP 서버 필요)
# chrome-devtools MCP 또는 verify/run 스킬로 화면 이동·스크린샷·콘솔 확인
```

## Gate 1 — Must-pass (이진, 하나라도 실패 = 미완)

- `npm run build` 성공(타입체크 포함).
- `npm test` 전부 PASS.
- 브라우저 콘솔 에러 0.
- 시크릿/키 노출 없음.
- HUMAN_GATE 경계 침범 없음(영속화·실연동·유료 SDK·배포 등 도입 안 함).

## Gate 2 — Measured (수치)

- 신규/수정 화면 폭 1920·2560에서 셸 가로 오버플로 ≤ 0 (넓은 표는 전용 스크롤 컨테이너 내부만 허용).
- 각 마일스톤 대상 화면이 카탈로그에 기록한 영역/컨트롤 항목 커버리지(예: Build 홈 개요 위젯 N종 중 N종 렌더).

## Gate 3 — Evaluated (1~5, 근거+개선액션 필수)

- 카탈로그 대비 정보구조 정합성(영역 배치·탭·빈상태가 원본과 같은 의미인가).
- ACC 외관 근접성(레이아웃·아이콘·컬럼 의미 — 픽셀 일치는 비대상).
- 셸 상호작용 자연스러움(탭 전환·모달 열고닫기·빈상태 메시지).
- 코드 유지보수성(중복 셸 컴포넌트 정리, 기존 스타일 토큰 재사용).

스트레이트 만점 금지 — 항목별로 점수를 누르는 구체적 약점을 지목하거나 없음을 명시. 만점이면 "더 올릴 다음 한 가지"를 개선액션으로.

## Real user flow (해피패스 너머)

- 각 신규 화면 진입/이탈, 탭 전환, 모달 열고 닫기(취소·X·제출)가 동작.
- 빈/긴 입력, 빠른 더블클릭, 잘못된 값에서 셸이 깨지지 않음.
- Hub ↔ Project Admin ↔ Build 레벨 분리 유지(잘못된 컨텍스트 누수 없음).

## 적용 메모

- 이 루프는 외관 전용 → Gate 1 + Gate 2 + 실제 user flow가 핵심. Gate 3는 카탈로그 근접성 위주로 운영.
- 적용 안 되는 게이트 항목은 빈칸으로 두지 말고 "해당 없음 + 이유"로 명시(예: DB 마이그레이션 없음).
