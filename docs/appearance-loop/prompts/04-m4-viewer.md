## Co-design log (frozen decisions — 2026-06-24, AskUserQuestion)

1. **범위 = §E·§F·§G·§H 4영역 전부 한 사이클(affordance 깊이).** 뷰어는 한 화면이고 도구들이 한 셸에 공존하므로 함께 채운다. §F(2장)·§G(3장)·§H(1장)는 M3에서 이월된 이슈/양식과 달리 **캡처가 분명히 존재** → "추정 구현 금지" 원칙에 안 걸림. → LOOP Done-When **FR-FS-007/008/009/010을 본 M4가 커버**(M4를 또 쪼개지 않음). 깊이는 affordance로 눌러 한 사이클에 담음.
2. **마크업 캔버스 = 충실한 정적 외관(이미 그려진 모습).** 캔버스에 텍스트박스·클라우드·폴리라인·도형·다각형 마크업이 "이미 그려진" 모습으로 정적 배치(캡처 175535~175643 재현) + 속성 패널에 실제 값 + 마크업 로그 항목 리스트 + 이슈 핀. 도구 전환 시 활성 상태/속성 패널 변화. **새로 그리는 인터랙션(클릭/드래그 드로잉)은 없음** — 외관은 완성형, 코드는 정적. (휘발성 인터랙티브 드로잉은 비용↑ + 영속 게이트 인접 → 비대상으로 명시 확정.)
3. **측정/교정·시트비교·이슈연계 = 풀 affordance.** 측정=교정 패널(축척·단위·측정타입 4종)+"교정을 만드시겠습니까?" 확인 모달 / 비교=A/B 선택 모달(버전 드롭다운)+정적 결과 오버레이(색상 토글·빨강 이전/파랑 현재 데모 diff) / 이슈=우측 "검색 및 추가" 패널(Clash·Quality·Coordination). 캡처 외관 충실 재현, 실연산 없음.
4. **코드 구조 = 하위 컴포넌트 분할.** `src/build/viewer/` 하위로 추출. `SheetViewerShell.tsx`는 헤더+그리드 레이아웃+상태 오케스트레이션 셸로 남기고 각 하위 패널/모달을 import. M3의 화면별 분할 일관성 유지(테스트·유지보수 용이).

- **우측 속성 패널(`MarkupPropertyPanel`)**: 캔버스의 데모 마크업을 클릭(또는 마크업 목록에서 선택)하면 우측에 해당 **타입별 속성 폼** 표시. 텍스트=내용·채우기색·글꼴·크기·테두리·불투명도·정렬·작성자/날짜(175535) / 클라우드=테두리색·선두께·스타일·불투명도(175551) / 폴리라인=선색·두께·스타일·화살표머리·불투명도(175610) / 다각형=채우기·테두리·불투명도·작성자/날짜(175643). 폼 컨트롤은 외관(값 표시·비활성 또는 로컬 state 변경 무영속).
