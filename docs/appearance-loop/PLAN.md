# PLAN — 외관 완성 마일스톤

> 마일스톤 순서. 각 마일스톤은 ai-loop 한 사이클: 메타프롬프트 공동설계(AskUserQuestion로 실제 선택지 노출)→freeze→구현→별도 검증팀 채점(구조 체크리스트+브라우저 스크린샷)→기록→체크인 정지.
> 현재 구현 파일: `src/App.tsx`(Hub/My Home/템플릿/프로젝트 목록/작성 모달), `src/ProjectAdminView.tsx`, `src/BuildSheetsView.tsx`(Build 전 셸 + 2D 뷰어 셸). 셸은 이미 "기본 수준"으로 존재 → 카탈로그 수준으로 끌어올리는 작업.

## M1 — Hub 표면 (catalog A + B-생성)

대상: My Home, 프로젝트 목록, 프로젝트 템플릿(목록·샘플 갤러리·작성 2단계 모달).
주요 갭: My Home 위젯 충실도(할당 필터칩·지도·책갈피·최근항목 + 온보딩), 템플릿 샘플 갤러리 카드·"사용하여 생성", 허브 템플릿 빈상태/목록.
근거: 카탈로그 §A, §B(173312·173328·173339), FR-FS-001/002.
파일: `src/App.tsx`.

## M2 — Project Admin + 템플릿 상세 (catalog B-상세 + FR-FS-004)

대상: 구성원 접근(기존) + 측면 네비 구분 화면(회사·브리지·액티비티·알림·위치·설정) + 템플릿 상세 영역(템플릿 설정: 구성·템플릿 구성원 / 프로젝트 설정: 프로젝트 구성원·회사·알림 매트릭스).
주요 갭: 측면 네비 항목별 distinct 화면, 템플릿 설정 셸(게시 토글·프로젝트 만들기/사본/보관), 알림 매트릭스(도구×이벤트×주기×권한) 대형 표.
근거: 카탈로그 §B(173409~173637), FR-FS-004.
파일: `src/ProjectAdminView.tsx`.

## M3 — Build 비뷰어 표면 (catalog C·D·H·I)

대상: 홈(개요/종합 탭 + 위젯 + 분석 차트), 시트 목록(컬럼·검색/필터·보기토글·행 메뉴), 파일(폴더트리·업로드 모달·테이블 컬럼), 이슈(목록·작성 모달·인스펙터), 양식(placeholder), 사진(앨범/갤러리/맵), 구성원/브리지/설정.
주요 갭: Build 홈 개요/종합 탭 분리 + 분석 차트 6종, 시트 컬럼/행메뉴 충실도, 파일 폴더트리 + 업로드 모달 + 풀 컬럼, 이슈 인스펙터·필터 탭.
근거: 카탈로그 §C·§D·§H·§I, FR-FS-005/006/011/012/013/014/015.
파일: `src/BuildSheetsView.tsx`(+ 필요 시 분할).

## M4 — 2D 뷰어 + 마크업/측정/비교/이슈 (catalog E·F·G)

대상: 마크업 도구군(텍스트·도형·클라우드·폴리라인·다각형 + 우측 속성 패널 + 마크업 로그), 측정/축척 교정(패널 + 확인 모달), 시트 비교(A/B 선택 모달 + 차이 오버레이 결과), 이슈 연계 패널, 하단 뷰 컨트롤·우측 툴레일·필름스트립.
주요 갭: 현재 뷰어 셸은 정적 시트 + 빈 마크업/이슈 패널뿐 → 마크업 작성 affordance·속성 패널·측정 교정 패널·시트 비교 모달/결과·마크업 로그.
근거: 카탈로그 §E·§F·§G, FR-FS-007/008/009/010. **영속화/실제 diff는 HUMAN_GATE — affordance만.**
파일: `src/BuildSheetsView.tsx`(SheetViewerShell) 또는 신규 `src/SheetViewer*.tsx`.

## M5 — 횡단 레이아웃 호환 + 최종 reconcile (FR-LC)

대상: M1~M4에서 추가된 모든 신규 화면을 FHD(1920)·4K(2560)·macOS 폰트 분기에서 무파손 재확인(가로 클리핑·겹침·폴백 파손 0). 넓은 표는 전용 스크롤 컨테이너 안에서만 가로 스크롤.
마지막: 신선한 비평가가 LOOP.md Done-When 전 항목을 MET/NARROWED/UNMET + 증거등급으로 reconcile → `EVIDENCE.md`. NARROWED/UNMET면 HUMAN_GATE.
근거: FR-LC-001~003, FR-FS-016.

## 검증팀 구성 (각 마일스톤 Phase 4)

별도 서브에이전트 2~4명, 각자 다른 렌즈로 "문제를 찾도록" 지시:
- **구조 비평가**: 메타프롬프트 acceptance 체크리스트를 항목별 PASS/FAIL.
- **브라우저 렌더 검증자**: dev 서버 띄우고 chrome-devtools로 해당 화면 스크린샷 → 카탈로그 원본과 영역/컨트롤 대조, 콘솔 에러·가로 오버플로 확인.
- **레이아웃/접근성 렌즈**(필요 시): 1920·2560 폭 오버플로, aria/포커스.
스트레이트 만점은 합격이 아님 — 근거+개선액션 동반.
