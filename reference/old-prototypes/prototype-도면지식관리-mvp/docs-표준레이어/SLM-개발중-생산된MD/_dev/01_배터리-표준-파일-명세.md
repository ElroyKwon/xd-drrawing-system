## 📋 개요

### 목적
XD-HUB에서 수집한 원본 데이터를 **배터리 분석 전용 표준 포맷**으로 변환하여 저장.
이 파일은 이후 모든 분석(수학 공식, AI 프롬프트)의 **유일한 입력 소스**.

### 설계 원칙
1. **용량 최소화**: ZSTD 압축으로 85% 절감
2. **속도 최적화**: Polars 스트리밍 읽기 지원
3. **타입 안전**: 스키마 강제로 오류 방지
4. **쿼리 효율**: Row Group 기반 시간 범위 필터링

## 📐 파일 스키마 (확정)

### PyArrow 스키마 정의

```python
# config/battery_schema.py
import pyarrow as pa

BATTERY_STANDARD_SCHEMA_V1 = pa.schema([
    # ========================================
    # 시간 정보
    # ========================================
    ('timestamp', pa.timestamp('ms', tz='UTC')),  # UTC 기준 밀리초

    # ========================================
    # 식별 정보 (Dictionary 압축)
    # ========================================
    ('rack_id', pa.dictionary(pa.int16(), pa.string())),    # "RACK_001" ~ "RACK_160"
    ('module_id', pa.dictionary(pa.int8(), pa.string())),   # "MODULE_01" ~ "MODULE_12"

    # ========================================
    # 전압 데이터 (Volt)
    # ========================================
    ('v_avg', pa.float32()),   # 평균 전압
    ('v_min', pa.float32()),   # 최소 전압
    ('v_max', pa.float32()),   # 최대 전압
    ('v_std', pa.float32()),   # 전압 표준편차

    # ========================================
    # 온도 데이터 (Celsius)
    # ========================================
    ('t_avg', pa.float32()),   # 평균 온도
    ('t_min', pa.float32()),   # 최소 온도
    ('t_max', pa.float32()),   # 최대 온도
    ('t_std', pa.float32()),   # 온도 표준편차

    # ========================================
    # 전류/충전 상태
    # ========================================
    ('current', pa.float32()),  # 전류 (A), 방전(-), 충전(+)
    ('soc', pa.float32()),      # 충전율 (State of Charge, %)

    # ========================================
    # 품질 플래그
    # ========================================
    ('quality_flag', pa.int8()),  # 0: 정상, 1: 경고, 2: 오류, 3: 센서 이상
])

# 버전 정보
SCHEMA_VERSION = "1.0"
SCHEMA_CREATED_DATE = "2025-12-11"
```

## 📊 컬럼 상세 명세

### 1. 시간 정보

| 컬럼명 | 타입 | 필수 | 범위 | 설명 | 예시 |
|--------|------|------|------|------|------|
| `timestamp` | timestamp(ms) | ✅ | 2020-01-01 ~ | UTC 기준 시간 (밀리초 정밀도) | `2025-11-30T14:30:00.123Z` |

**제약 조건:**
- NULL 불가
- 시간 순서 정렬 필수
- 중복 허용 (같은 시간에 여러 Rack 데이터)

---

### 2. 식별 정보

| 컬럼명 | 타입 | 필수 | 값 범위 | 설명 | 예시 |
|--------|------|------|---------|------|------|
| `rack_id` | dictionary(string) | ✅ | RACK_001 ~ RACK_160 | Rack 고유 식별자 | `RACK_042` |
| `module_id` | dictionary(string) | ✅ | MODULE_01 ~ MODULE_12 | 모듈 고유 식별자 | `MODULE_05` |

**제약 조건:**
- NULL 불가
- 정규식 패턴:
  - `rack_id`: `^RACK_\d{3}$`
  - `module_id`: `^MODULE_\d{2}$`
- Dictionary 압축으로 메모리 90% 절감

---

### 3. 전압 데이터 (LFP 배터리 기준)

| 컬럼명 | 타입 | 필수 | 물리 범위 | 정상 범위 | 단위 | 설명 |
|--------|------|------|-----------|-----------|------|------|
| `v_avg` | float32 | ✅ | 0.0 ~ 5.0 | 2.5 ~ 3.65 | V | 평균 셀 전압 |
| `v_min` | float32 | ✅ | 0.0 ~ 5.0 | 2.5 ~ 3.65 | V | 최소 셀 전압 |
| `v_max` | float32 | ✅ | 0.0 ~ 5.0 | 2.5 ~ 3.65 | V | 최대 셀 전압 |
| `v_std` | float32 | ⚠️ | 0.0 ~ 1.0 | 0.0 ~ 0.1 | V | 전압 표준편차 (밸런싱 지표) |

**제약 조건:**
- `v_min <= v_avg <= v_max` 보장
- `v_std >= 0` 보장
- NULL 허용 (센서 오류 시)
- 범위 초과 시 `quality_flag = 3` 설정

**물리적 의미:**
- LFP 셀 전압 범위: 2.5V (방전 완료) ~ 3.65V (충전 완료)
- `v_std > 0.1V`: 셀 밸런싱 이상 의심

---

### 4. 온도 데이터

| 컬럼명 | 타입 | 필수 | 물리 범위 | 정상 범위 | 단위 | 설명 |
|--------|------|------|-----------|-----------|------|------|
| `t_avg` | float32 | ✅ | -40 ~ 100 | -10 ~ 50 | °C | 평균 온도 |
| `t_min` | float32 | ✅ | -40 ~ 100 | -10 ~ 50 | °C | 최소 온도 |
| `t_max` | float32 | ✅ | -40 ~ 100 | -10 ~ 50 | °C | 최대 온도 |
| `t_std` | float32 | ⚠️ | 0.0 ~ 50 | 0.0 ~ 10 | °C | 온도 표준편차 |

**제약 조건:**
- `t_min <= t_avg <= t_max` 보장
- `t_std >= 0` 보장
- NULL 허용 (센서 오류 시)
- `t_max > 60°C`: 열폭주 위험 (CRITICAL)

**물리적 의미:**
- 배터리 작동 온도: -10°C ~ 50°C
- `t_max > 60°C`: 열폭주 위험 구간

---

### 5. 전류/충전 상태

| 컬럼명 | 타입 | 필수 | 물리 범위 | 정상 범위 | 단위 | 설명 |
|--------|------|------|-----------|-----------|------|------|
| `current` | float32 | ✅ | -1000 ~ 1000 | -500 ~ 500 | A | 전류 (방전 음수, 충전 양수) |
| `soc` | float32 | ✅ | 0.0 ~ 100.0 | 10.0 ~ 90.0 | % | 충전 상태 (State of Charge) |

**제약 조건:**
- `current`: NULL 불가
- `soc`: 0.0 ~ 100.0 범위 강제
- `soc < 10%` 또는 `soc > 90%`: 경고

**물리적 의미:**
- `current < 0`: 방전 (배터리 → 부하)
- `current > 0`: 충전 (전원 → 배터리)
- `soc = 0%`: 완전 방전 (위험)
- `soc = 100%`: 완전 충전

---

### 6. 품질 플래그

| 컬럼명 | 타입 | 필수 | 값 범위 | 설명 |
|--------|------|------|---------|------|
| `quality_flag` | int8 | ✅ | 0, 1, 2, 3 | 데이터 품질 상태 |

**플래그 정의:**

| 값 | 상태 | 설명 | 조건 |
|----|------|------|------|
| `0` | 정상 (NORMAL) | 모든 센서 정상 | 모든 값이 정상 범위 내 |
| `1` | 경고 (WARNING) | 값이 주의 범위 | `soc < 10%` or `t_max > 50°C` |
| `2` | 오류 (ERROR) | 값이 위험 범위 | `t_max > 60°C` or `v_avg < 2.5V` |
| `3` | 센서 이상 (SENSOR_FAULT) | 물리적으로 불가능한 값 | `v_min > v_max` or NULL 많음 |

## 🗂️ 파일 구조

### 파일 명명 규칙

```
battery_data_{YYYYMMDD}_{HHMMSS}.parquet

예시:
battery_data_20251130_140000.parquet  # 2025년 11월 30일 14시 데이터
battery_data_20251201_000000.parquet  # 2025년 12월 01일 00시 데이터
```

### 디렉토리 구조

```
data/
└── standard/              # 배터리 표준 파일 저장소
    └── YYYY/              # 연도별
        └── MM/            # 월별
            └── battery_data_YYYYMMDD_HHMMSS.parquet

예시:
data/standard/2025/11/battery_data_20251130_140000.parquet
data/standard/2025/12/battery_data_20251201_000000.parquet
```

### 압축 설정

```python
# modules/storage/parquet_writer.py
import pyarrow.parquet as pq

PARQUET_WRITE_OPTIONS = {
    'compression': 'zstd',          # ZSTD 압축
    'compression_level': 3,         # Level 3 (속도/압축 균형)
    'row_group_size': 1_000_000,    # 100만 행/그룹
    'data_page_size': 1_048_576,    # 1MB 페이지
    'use_dictionary': True,         # Dictionary 압축 활성화
    'write_statistics': True,       # Min/Max 통계 저장
    'version': '2.6',               # Parquet 2.6
}
```

### Row Group 전략

```python
# 시간 기반 Row Group 분할
1개 Row Group = 1시간 데이터

예시 (160 Rack, 1분 주기):
- 1시간 = 60분 × 160 Rack = 9,600 행
- 1일 = 24 Row Group = 230,400 행
- 1개월 = 720 Row Group = 6,912,000 행

쿼리 최적화:
- "14:00 ~ 15:00 데이터만" → 1개 Row Group만 읽기
- 나머지 23개 Row Group 스킵 → 95% 빠름
```

## 📏 데이터 품질 규칙

### 필수 검증 (Validation)

```python
# modules/validators/battery_validator.py

class BatteryDataValidator:
    """배터리 표준 파일 검증기"""

    @staticmethod
    def validate_row(row: dict) -> tuple[bool, str]:
        """행 단위 검증

        Returns:
            (is_valid, error_message)
        """

        # 1. 전압 범위 검증
        if not (2.0 <= row['v_avg'] <= 4.0):
            return False, f"v_avg out of range: {row['v_avg']}"

        # 2. 전압 순서 검증
        if not (row['v_min'] <= row['v_avg'] <= row['v_max']):
            return False, "v_min > v_avg or v_avg > v_max"

        # 3. 온도 순서 검증
        if not (row['t_min'] <= row['t_avg'] <= row['t_max']):
            return False, "t_min > t_avg or t_avg > t_max"

        # 4. SOC 범위 검증
        if not (0.0 <= row['soc'] <= 100.0):
            return False, f"soc out of range: {row['soc']}"

        # 5. Rack ID 패턴 검증
        if not re.match(r'^RACK_\d{3}$', row['rack_id']):
            return False, f"Invalid rack_id: {row['rack_id']}"

        return True, "OK"
```

## 🔄 변환 프로세스

### XD-HUB CSV → 배터리 표준 Parquet

```python
# scripts/convert_to_standard.py
import polars as pl
from config.battery_schema import BATTERY_STANDARD_SCHEMA_V1

def convert_xdhub_to_standard(
    csv_path: str,
    output_path: str
) -> dict:
    """XD-HUB CSV를 배터리 표준 Parquet로 변환

    Args:
        csv_path: XD-HUB 원본 CSV 경로
        output_path: 출력 Parquet 경로

    Returns:
        {'status': 'success', 'rows': 6912000, 'size_mb': 155}
    """

    # 1. CSV 읽기 (Lazy)
    df = pl.scan_csv(csv_path)

    # 2. 컬럼 매핑 및 타입 변환
    df = df.select([
        pl.col('time').str.to_datetime().alias('timestamp'),
        pl.col('rack_name').alias('rack_id'),
        pl.col('module_name').alias('module_id'),
        pl.col('voltage_avg').cast(pl.Float32).alias('v_avg'),
        pl.col('voltage_min').cast(pl.Float32).alias('v_min'),
        pl.col('voltage_max').cast(pl.Float32).alias('v_max'),
        pl.col('voltage_std').cast(pl.Float32).alias('v_std'),
        pl.col('temp_avg').cast(pl.Float32).alias('t_avg'),
        pl.col('temp_min').cast(pl.Float32).alias('t_min'),
        pl.col('temp_max').cast(pl.Float32).alias('t_max'),
        pl.col('temp_std').cast(pl.Float32).alias('t_std'),
        pl.col('current').cast(pl.Float32).alias('current'),
        pl.col('soc').cast(pl.Float32).alias('soc'),
        pl.lit(0).cast(pl.Int8).alias('quality_flag'),  # 기본값 0
    ])

    # 3. 품질 검증 및 플래그 설정
    df = df.with_columns([
        pl.when(
            (pl.col('t_max') > 60) | (pl.col('v_avg') < 2.5)
        ).then(2)  # ERROR
        .when(
            (pl.col('soc') < 10) | (pl.col('t_max') > 50)
        ).then(1)  # WARNING
        .when(
            (pl.col('v_min') > pl.col('v_max'))
        ).then(3)  # SENSOR_FAULT
        .otherwise(0)  # NORMAL
        .alias('quality_flag')
    ])

    # 4. Parquet 쓰기
    df.collect().write_parquet(
        output_path,
        compression='zstd',
        compression_level=3,
        row_group_size=1_000_000,
    )

    return {
        'status': 'success',
        'rows': len(df.collect()),
        'size_mb': os.path.getsize(output_path) / 1024 / 1024
    }
```
