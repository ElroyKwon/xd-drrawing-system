## 1. 개요

### 1.1 배경

API로 수집된 데이터는 **서버에 저장**되어 있습니다:

```
서버 (원격 또는 로컬)
├── GEM/data/source/api/
│   ├── 2025-12-20/
│   │   ├── bsc_ch_01_realtime.parquet
│   │   └── bsc_ch_02_realtime.parquet
│   └── 2025-12-21/
│       └── bsc_ch_01_batch.parquet
│
└── 브라우저는 서버의 파일에 직접 접근 불가!
```

**해결책:**
- **프롬프트 명령어 시스템** 구현
- YAML 기반 명령어 정의
- SLM이 자연어 명령을 인식하여 자동 처리
- 서버 파일 시스템 직접 접근

### 2.1 YAML 기반 명령어 관리

**v1.0 (하드코딩)**
```python
PATTERNS = {
    'analyze_date': r'(\d{4}-\d{2}-\d{2})\s*(데이터|자료)?\s*(분석|실행)',
    'analyze_relative': r'(어제|오늘|그제|지난주|이번주)\s*(데이터|자료)?\s*(분석|실행)',
    # ... 코드에 하드코딩
}
```

**v2.0 (YAML 설정)**
```yaml
# GEM/config/system_commands.yaml
commands:
  - id: analyze_date
    name: "특정 날짜 데이터 분석"
    category: "analysis"
    patterns:
      - "(\\d{4}-\\d{2}-\\d{2})\\s*(데이터|자료)?\\s*(분석|실행|analyze)"
    parameters:
      - name: date
        type: date
        required: true
    examples:
      - "2025-12-25 데이터 분석"
```

## 3. 시스템 아키텍처

### 3.1 명령어 처리 흐름

```
사용자 입력
  ↓
APIDataCommandHandler
  ↓
1. YAML에서 명령어 로드
  ↓
2. 패턴 매칭
  ├─ 매칭 성공 → 3단계
  └─ 매칭 실패 → "알 수 없는 명령어" 응답
  ↓
3. 파라미터 추출
  ├─ 필수 파라미터 완전 → 4단계
  └─ 파라미터 누락 → 재질문 응답
  ↓
4. 명령어 핸들러 실행
  ↓
5. 최종 확인 프롬프트 생성
  ↓
사용자에게 응답 반환
```

### 3.2 명령어 우선순위

```
사용자 입력
  ↓
1순위: 시스템 명령어 (system_commands.yaml)
  ├─ 매칭 성공 → 실행
  └─ 매칭 실패 → 2순위로
  ↓
2순위: 사용자 정의 명령어 (향후 지원)
  ├─ 매칭 성공 → 실행
  └─ 매칭 실패 → 3순위로
  ↓
3순위: 알 수 없는 명령어
  └─ 안내 메시지 표시
```

### 5.2 명령어 정의 스키마

```yaml
commands:
  - id: <명령어_ID>               # 고유 식별자
    name: <명령어_이름>            # 사용자 친화적 이름
    category: <카테고리>           # system/collection/analysis/insight/quality/report
    patterns:                      # 매칭 패턴 (정규식)
      - <패턴1>
      - <패턴2>
    description: <설명>            # 명령어 설명
    parameters:                    # 필수 파라미터
      - name: <파라미터_이름>
        type: <타입>               # date/string/number
        required: <true/false>     # 필수 여부
        pattern: <추출_패턴>        # 파라미터 추출 정규식
        ask_if_missing: <재질문_메시지>  # 파라미터 누락 시 질문
    examples:                      # 사용 예시
      - <예시1>
      - <예시2>
    response_template: <응답_템플릿>  # 응답 포맷

### 6.2 명령어 핸들러 v2.0

```python
class APIDataCommandHandler:
    """YAML 기반 API 데이터 분석 명령어 핸들러"""

    def __init__(self, config_path: Optional[Path] = None):
        """YAML 설정 로드"""
        self.commands = []
        self.categories = {}
        self._load_commands()

    def _load_commands(self):
        """YAML에서 명령어 설정 로드"""
        with open(self.config_path, 'r', encoding='utf-8') as f:
            config = yaml.safe_load(f)
        self.commands = config.get('commands', [])
        self.categories = config.get('categories', {})

    def handle(self, user_input: str) -> Optional[str]:
        """명령어 처리 메인 엔트리포인트"""
        result = self.match_command(user_input)

        if result is None:
            # 알 수 없는 명령어
            return self._handle_unknown_command(user_input)

        cmd, params = result

        # 파라미터 검증
        missing_params = self._check_missing_parameters(cmd, params)
        if missing_params:
            return self._ask_for_parameters(cmd, missing_params, params)

        # 핸들러 호출
        handler_method = f"_handle_{cmd['id']}"
        if hasattr(self, handler_method):
            return getattr(self, handler_method)(cmd, params)
        else:
            return self._handle_generic_command(cmd, params)

    def _check_missing_parameters(self, cmd: Dict, params: Dict) -> List[Dict]:
        """필수 파라미터 누락 확인"""
        missing = []
        for param_def in cmd.get('parameters', []):
            if param_def.get('required') and params.get(param_def['name']) is None:
                missing.append(param_def)
        return missing

    def _ask_for_parameters(self, cmd: Dict, missing_params: List[Dict], current_params: Dict) -> str:
        """누락된 파라미터에 대한 재질문 생성"""
        param_def = missing_params[0]
        ask_message = param_def.get('ask_if_missing')

        msg = f"❓ {ask_message}\n\n"

        # 최근 데이터 표시 (날짜 파라미터인 경우)
        if param_def.get('type') == 'date':
            msg += self._show_recent_dates()

        # 예시 표시
        examples = cmd.get('examples', [])
        if examples:
            msg += "💡 사용 예시:\n"
            for example in examples[:3]:
                msg += f"  - \"{example}\"\n"

        return msg
```

**핵심 개선:**
1. ✅ **YAML 기반 명령어 관리** - 코드 수정 없이 명령어 추가/수정
2. ✅ **파라미터 자동 검증** - 필수 파라미터 누락 시 자동 재질문

## 8. SLM 통합

### 8.1 처리 흐름

```
사용자 입력
  ↓
SLM (qwen2.5:14b)
  ↓
시스템 프롬프트 확인 (ai_prompts.yaml)

APIDataCommandHandler 호출
  ↓
YAML 명령어 로드 (system_commands.yaml)
  ↓
패턴 매칭
  ├─ 매칭 성공 → 파라미터 검증
  │              ├─ 완전 → 최종 확인 프롬프트
  │              └─ 누락 → 재질문
  └─ 매칭 실패 → 알 수 없는 명령어 안내
```

### 8.2 ai_prompts.yaml 설정

```yaml
chat_system_prompt: |
  당신은 배터리 안전 분석 전문 엔지니어링 어시스턴트입니다.

  **특수 명령어 처리 (YAML 기반 시스템):**
  사용자 입력을 APIDataCommandHandler에 먼저 전달하세요.

  명령어 핸들러는 GEM/config/system_commands.yaml에 정의된 패턴을 확인합니다.

  핸들러 처리 흐름:
  1. 명령어 패턴 매칭
  2. 필수 파라미터 검증
  3. 파라미터 부족 시 재질문
  4. 파라미터 완전 시 최종 확인 프롬프트 표시
  5. 사용자 확인 후 실행 코드 제공

  주요 명령어 카테고리:
  - 시스템: "/", "명령어 목록", "도움말"
  - 데이터 수집: "수집 상태", "실패 요청 현황"
  - 데이터 분석: "YYYY-MM-DD 데이터 분석", "어제 데이터"
  - 인사이트 분석: "배터리 온도 급상승", "전압 강하", "SOC 이상", "셀 불균형"
  - 품질 관리: "데이터 품질 확인", "누락 데이터"
  - 리포트: "일일 요약", "주간 트렌드"

  - 핸들러가 응답을 반환하면 그대로 사용자에게 전달
  - 알 수 없는 명령어 → 안내 메시지 표시
  - None 반환 → 일반 대화로 처리
```
