## Ollama 설치 및 설정

### 1. Ollama 다운로드 및 설치

**Windows:**
```
https://ollama.com/download/windows
설치 후 재부팅 권장
```

**Linux:**
```bash
curl -fsSL https://ollama.com/install.sh | sh
```

**macOS:**
```bash
brew install ollama
```

### 2. Ollama 서버 시작

**Windows:**
```bash
# 새 터미널 열기
ollama serve
```

**Linux/Mac:**
```bash
# 백그라운드 실행
ollama serve &
```

### 3. 모델 다운로드

#### 개발 환경 (VRAM 8-12GB)

```bash
# 빠른 대화용 (2-3GB VRAM, 40-55 tokens/s)
ollama pull qwen2.5:3b

# 인사이트 테스트용 (8GB VRAM, 15-20 tokens/s)
ollama pull qwen2.5:14b
```

#### 운영 환경 (VRAM 24GB+)

```bash
# 빠른 대화용
ollama pull qwen2.5:3b

# Phase 3 인사이트 생성 (20GB VRAM, 20-25 tokens/s) ✅ 권장
ollama pull qwen3-coder:32b-q5_k_m

# 향후 상향 검토 (32GB VRAM, A100 필요)
# ollama pull qwen3-coder:32b-q8_0
```

### 4. 모델 설치 확인

```bash
ollama list
```

**예상 출력:**
```
NAME                        SIZE    MODIFIED
qwen2.5:3b                  2.0GB   1 hour ago
qwen2.5:14b                 8.5GB   1 hour ago
qwen3-coder:32b-q5_k_m      20GB    1 hour ago
```

## 문제 해결

### 1. Ollama 연결 실패

**증상:**
```
Connection refused: Ollama 서버가 실행되지 않았습니다.
```

**해결:**
```bash
# 1. Ollama 서버 실행 확인
ollama serve

# 2. 모델 다운로드 확인
ollama list

# 3. 포트 확인 (기본: 11434)
netstat -ano | findstr 11434  # Windows
lsof -i :11434                # Linux/Mac
```

---

### 2. GPU 메모리 부족 (CUDA out of memory)

**증상:**
```
CUDA error: out of memory
```

**해결:**

**방법 1: 더 작은 모델 사용**
```bash
# qwen2.5:32b → qwen2.5:14b
# qwen2.5:14b → qwen2.5:7b
# qwen2.5:7b → qwen2.5:3b
```

**방법 2: .env 파일 수정**
```bash
ENVIRONMENT=development  # 경량 모델 자동 선택
```

**방법 3: CPU 모드 강제**
```yaml
# config/ai_model_config.yaml
use_gpu: false
gpu_layers: 0
```

---

### 3. 모델 다운로드 느림

**해결:**
```bash
# Ollama 프록시 설정 (한국)
export OLLAMA_HOST=https://ollama.kr  # 예시

# 또는 직접 다운로드
wget https://huggingface.co/.../qwen2.5-3b-q4.gguf
ollama create qwen2.5:3b -f Modelfile
```

---

### 4. 패키지 설치 실패

**증상:**
```
ERROR: Could not find a version that satisfies the requirement...
```

**해결:**
```bash
# pip 업그레이드
python -m pip install --upgrade pip

# 캐시 삭제 후 재설치
pip cache purge
pip install -r requirements.txt --no-cache-dir

# 개별 설치
pip install polars pyarrow numpy
pip install fastapi uvicorn websockets
pip install ollama jinja2 pyyaml python-dotenv
```

---

### 5. 포트 충돌 (7860 already in use)

**해결:**

**방법 1: 다른 포트 사용**
```python
# main.py 수정
if __name__ == "__main__":
    import uvicorn
    uvicorn.run(app, host="0.0.0.0", port=8080)  # 7860 → 8080
```

**방법 2: 기존 프로세스 종료**
```bash
# Windows
netstat -ano | findstr 7860
taskkill /PID <PID> /F

# Linux/Mac
lsof -ti:7860 | xargs kill -9
```
