## 개요

GEM v2.0은 기존 시스템에 **Multi-Site 지원**을 추가하고, 폴더 구조를 개선하여 확장성과 유지보수성을 향상시켰습니다.

### 주요 변경 사항

1. **Multi-Site 지원**: 여러 배터리 사이트의 데이터를 동시에 관리
2. **폴더 구조 개선**: `data/tags/` → `data/map/{site}/`
3. **Phase 명명 개선**: `phase2/` → `phase2_validate/` 등
4. **사이트 관리 자동화**: SiteManager 클래스 추가

## Multi-Site 지원

### 1. SiteManager 클래스 (신규)

**파일:** `GEM/api_collector/site_manager.py`

**기능:**
- `scan_sites()`: 사이트 자동 스캔
- `generate_tag_files()`: Excel → 태그 파일 자동 생성
- `get_channel_list()`: 사이트별 채널 목록 조회
- `get_tag_file_path()`: 사이트+채널 태그 파일 경로 반환

**사용 예시:**
```python
from GEM.api_collector import SiteManager

manager = SiteManager()

# 모든 사이트 스캔
sites = manager.scan_sites()
# [
#   {
#     'name': 'jeju_bess',
#     'excel_file': '제주장주기밧데리맵.xlsx',
#     'channels': ['BSC_CH_01', 'BSC_CH_02', ...],
#     'tag_count': 4860,
#     'ready': True
#   },
#   ...
# ]

# 채널 목록 조회
channels = manager.get_channel_list('jeju_bess')
# ['BSC_CH_01', 'BSC_CH_02', 'BSC_CH_03', ...]
```

### 2. DataStorage 클래스 수정

**파일:** `GEM/api_collector/storage.py`

**변경사항:**
```python
# 기존 (v1.x)
class DataStorage:
    def __init__(self, base_dir=None):
        self.tag_file = project_root / "data" / "tags" / "bsc_ch_01_tags.txt"

# 새 버전 (v2.0)
class DataStorage:
    def __init__(self, base_dir=None, site="jeju_bess"):
        self.site = site
        self.map_dir = project_root / "data" / "map"

    def load_tag_order(self, channel):
        tag_file = self.map_dir / self.site / f"{channel.lower()}_tags.txt"
```

**기본값:** `site="jeju_bess"` (기존 시스템과 호환)

### 3. 사이트 추가 방법

1. **폴더 생성**
   ```bash
   mkdir GEM/data/map/busan_bess
   ```

2. **Excel 파일 복사**
   ```bash
   cp /path/to/부산BESS맵.xlsx GEM/data/map/busan_bess/
   ```

3. **자동 태그 생성**
   ```python
   from GEM.api_collector import SiteManager

   manager = SiteManager()
   sites = manager.scan_sites()  # 자동으로 태그 파일 생성
   ```

4. **완료!**
   ```
   GEM/data/map/busan_bess/
   ├── 부산BESS맵.xlsx
   ├── bsc_ch_01_tags.txt  (자동 생성)
   ├── bsc_ch_02_tags.txt  (자동 생성)
   └── ...
   ```

## 새로운 기능

### 1. 사이트 선택 UI (WebUI)

```
┌─────────────────────────────────────────────────┐
│  [사이트 선택]                   🆕 Multi-Site  │
│  ● 제주 BESS (jeju_bess) - 6채널, 4860태그     │
│  ○ 부산 BESS (busan_bess) - 4채널, 3200태그    │
│                                                  │
│  [채널 선택]  (선택된 사이트: 제주 BESS)        │
│  ☑ BSC_CH_01 (810개 태그)                      │
│  ☑ BSC_CH_02 (810개 태그)                      │
│  ...                                             │
└─────────────────────────────────────────────────┘
```

**동작 순서:**
1. 사이트 선택 (jeju_bess, busan_bess 등)
2. 선택한 사이트의 채널 목록 표시
3. 채널 선택
4. 데이터 수집/분석 시작

### 2. 자동 태그 파일 생성

**기존 방식:**
- 수동으로 Excel 파싱
- 태그 파일 직접 작성
- 에러 발생 가능성 높음

**새 방식 (v2.0):**
```python
# Excel 파일만 있으면 자동으로 태그 파일 생성
manager = SiteManager()
sites = manager.scan_sites()

# Excel 파일이 있지만 태그 파일이 없는 경우
# → generate_tag_files() 자동 실행
# → bsc_ch_01_tags.txt, bsc_ch_02_tags.txt 등 생성
```

## 마이그레이션 가이드

### 기존 시스템 (v1.x)에서 v2.0으로 업그레이드

#### 1. 폴더 구조 마이그레이션

**자동 마이그레이션 스크립트 사용:**
```bash
cd GEM
python scripts/migrate_folder_structure.py
```

**수동 마이그레이션:**
```bash
# 1. data/map/jeju_bess/ 폴더 생성
mkdir -p data/map/jeju_bess

# 2. Excel 파일 이동
mv data/source/제주장주기밧데리맵.xlsx data/map/jeju_bess/

# 3. 태그 파일 이동
mv data/tags/bsc_ch_01_tags.txt data/map/jeju_bess/
mv data/tags/bsc_ch_02_tags.txt data/map/jeju_bess/
mv data/tags/bsc_ch_03_tags.txt data/map/jeju_bess/
mv data/tags/bsc_ch_04_tags.txt data/map/jeju_bess/
mv data/tags/bsc_ch_05_tags.txt data/map/jeju_bess/
mv data/tags/bsc_ch_06_tags.txt data/map/jeju_bess/
mv data/tags/all_channels_tags.json data/map/jeju_bess/

# 4. 기존 data/tags/ 폴더 제거
rm -rf data/tags/
```

#### 2. 코드 업데이트

**api_collector 사용 코드 수정:**

기존 코드 (v1.x):
```python
from GEM.api_collector import DataStorage

storage = DataStorage()
tags = storage.load_tag_order('BSC_CH_01')
```

새 코드 (v2.0):
```python
from GEM.api_collector import DataStorage

# 기본 사이트 (jeju_bess) 사용
storage = DataStorage()
tags = storage.load_tag_order('BSC_CH_01')

# 또는 명시적으로 사이트 지정
storage = DataStorage(site='jeju_bess')
tags = storage.load_tag_order('BSC_CH_01')
```

**변경 불필요:**
- 기본값이 `site="jeju_bess"`이므로 기존 코드 그대로 동작

#### 3. 설정 파일 업데이트

**config/global/styles/report_style_guide.yaml:**
```yaml
data_sources:
  active_source: "jeju_bess"  # ← jeju_vpp에서 변경

  sources:
    jeju_bess:  # ← jeju_vpp에서 키 이름 변경
      name: "제주 BESS 프로젝트"  # ← "제주 VPP"에서 변경
      site_code: "JEJU_BESS_01"   # ← "JEJU_VPP_01"에서 변경
      # ... 나머지 동일
```

#### 4. 검증

```python
from GEM.api_collector import SiteManager, DataStorage

# 1. 사이트 스캔 확인
manager = SiteManager()
sites = manager.scan_sites()
print(f"Found {len(sites)} site(s)")
assert len(sites) >= 1, "No sites found!"

# 2. 태그 로드 확인
storage = DataStorage(site='jeju_bess')
tags = storage.load_tag_order('BSC_CH_01')
print(f"Loaded {len(tags)} tags")
assert len(tags) == 810, f"Expected 810 tags, got {len(tags)}"

print("✓ Migration successful!")
```

## 호환성

### 하위 호환성 (Backward Compatibility)

**v1.x 코드는 v2.0에서도 동작:**
- `DataStorage()` 기본값 `site="jeju_bess"`
- 기존 API는 변경 없음
- 기존 데이터 포맷 유지

**주의사항:**
- `data/tags/` 폴더는 더 이상 사용 안 함
- 마이그레이션 후 해당 폴더 삭제 권장

### 권장 업그레이드 경로

1. **Phase 1**: 폴더 구조 마이그레이션
2. **Phase 2**: SiteManager 사용하여 시스템 검증
3. **Phase 3**: (선택) 새 사이트 추가
4. **Phase 4**: 문서 및 설정 업데이트
