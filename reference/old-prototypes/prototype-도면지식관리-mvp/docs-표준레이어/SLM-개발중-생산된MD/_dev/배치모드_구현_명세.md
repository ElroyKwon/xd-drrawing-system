# 배치 모드 구현 명세

**작성일**: 2026-02-12
**버전**: 1.0
**목적**: 과거 데이터 자동 수집 및 분석 배치 시스템 구현

---

## 1. 요구사항 정리

### 1.1 UI 변경사항
- **CSV 업로드 모드**: 배포용으로 숨김 (삭제 X, CSS로만 숨김)
- **실시간 수집 모드**: 배포용으로 숨김 (삭제 X, CSS로만 숨김)
- **과거 데이터 수집 모드**: 유일하게 표시되는 모드

### 1.2 배치 시간 설정 추가
- 실시간 수집의 "자동 분석 시각" 기능을 과거 데이터 수집으로 이동
- 위치: "수집 날짜" **위**에 배치
- 체크박스로 사용 여부 선택 가능

### 1.3 설정 파일 저장
- 자동 분석 시각을 설정 파일에 저장
- 재시작 시에도 설정 유지
- 파일 위치: `GEM/config/batch_config.json`

### 1.4 배치 모드 동작
**두 가지 모드:**

#### A. 수동 모드 (기본값)
- 자동 분석 시각 사용 안함
- 사용자가 날짜 선택 → 즉시 수집 → 분석
- 기존 방식 유지

#### B. 자동 배치 모드 (신규)
- 자동 분석 시각 사용
- 매일 설정된 시간에 자동 실행
- **하루 전 데이터** 자동 수집 (예: 2월 12일 09:00 실행 → 2월 11일 데이터 수집)
- 수집 → Phase1~4 자동 진행 → 리포트 생성
- **하루에 한 번만** 실행 (중복 방지)

## 2. 배치 충돌 방지

### 2.1 현재 상황
- **실시간 수집 배치**: 이미 존재 (실시간 모드의 자동 분석 시각 사용)
- **과거 데이터 수집 배치**: 신규 추가

### 2.2 충돌 방지 전략
1. **배치 락(Lock) 메커니즘**: 한 번에 하나의 배치만 실행
2. **독립적 설정 관리**: `realtime_batch`, `historical_batch` 분리
3. **실행 시간 체크**: 각 배치의 설정 시간 확인 (±5분 허용)
4. **중복 실행 방지**: 오늘 이미 실행한 배치는 재실행 안 함

## 3. 설정 파일 구조

**파일 위치**: `GEM/config/batch_config.json`

```json
{
    "realtime_batch": {
        "enabled": false,
        "analysis_time": "09:00",
        "last_run_date": null,
        "channels": ["CH_01"]
    },
    "historical_batch": {
        "enabled": false,
        "analysis_time": "21:00",
        "last_run_date": null,
        "channels": ["CH_01", "CH_02", "CH_03", "CH_04", "CH_05", "CH_06"]
    },
    "batch_lock": {
        "is_running": false,
        "current_job": null,
        "started_at": null
    }
}
```

### 3.1 필드 설명
- `enabled`: 배치 활성화 여부
- `analysis_time`: 자동 분석 시각 (HH:MM 형식)
- `last_run_date`: 마지막 실행 날짜 (YYYY-MM-DD)
- `channels`: 수집할 채널 목록
- `batch_lock`: 배치 동시 실행 방지용 락

```python
from pathlib import Path
import json

@app.post("/api/batch/config")
async def save_batch_config(config: dict):
    """배치 설정 저장"""
    config_path = Path("GEM/config/batch_config.json")
    config_path.parent.mkdir(parents=True, exist_ok=True)

    with open(config_path, 'w', encoding='utf-8') as f:
        json.dump(config, f, indent=2, ensure_ascii=False)

    return {"status": "success"}

@app.get("/api/batch/config")
async def get_batch_config():
    """배치 설정 로드"""
    config_path = Path("GEM/config/batch_config.json")

    # 기본값
    default_config = {
        "realtime_batch": {
            "enabled": False,
            "analysis_time": "09:00",
            "last_run_date": None,
            "channels": ["CH_01"]
        },
        "historical_batch": {
            "enabled": False,
            "analysis_time": "21:00",
            "last_run_date": None,
            "channels": ["CH_01", "CH_02", "CH_03", "CH_04", "CH_05", "CH_06"]
        },
        "batch_lock": {
            "is_running": False,
            "current_job": None,
            "started_at": None
        }
    }

    if config_path.exists():
        with open(config_path, 'r', encoding='utf-8') as f:
            return json.load(f)

    return default_config

def load_batch_config():
    """내부용 설정 로드"""
    import asyncio
    loop = asyncio.new_event_loop()
    return loop.run_until_complete(get_batch_config())

def save_batch_config_sync(config: dict):
    """내부용 설정 저장 (동기)"""
    config_path = Path("GEM/config/batch_config.json")
    with open(config_path, 'w', encoding='utf-8') as f:
        json.dump(config, f, indent=2, ensure_ascii=False)
```

### 5.2 배치 실행 체크 로직

```python
from datetime import datetime, timedelta

def can_start_batch(batch_type: str) -> bool:
    """배치 시작 가능 여부 확인

    Args:
        batch_type: 'realtime_batch' or 'historical_batch'

    Returns:
        bool: 실행 가능 여부
    """
    config = load_batch_config()

    # 1. 다른 배치가 실행 중인지 확인
    if config['batch_lock']['is_running']:
        print(f"[배치] 이미 실행 중: {config['batch_lock']['current_job']}")
        return False

    # 2. 해당 배치가 활성화되어 있는지
    if not config[batch_type]['enabled']:
        return False

    # 3. 오늘 이미 실행했는지
    today = datetime.now().strftime('%Y-%m-%d')
    if config[batch_type].get('last_run_date') == today:
        return False

    # 4. 설정된 시간인지 확인 (±5분 허용)
    now = datetime.now()
    target_time = datetime.strptime(config[batch_type]['analysis_time'], '%H:%M').time()
    current_time = now.time()

    time_diff = abs((datetime.combine(datetime.today(), current_time) -
                     datetime.combine(datetime.today(), target_time)).total_seconds())

    if time_diff > 300:  # 5분(300초) 이상 차이나면 실행 안 함
        return False

    return True

def acquire_batch_lock(batch_type: str):
    """배치 락 획득"""
    config = load_batch_config()
    config['batch_lock'] = {
        'is_running': True,
        'current_job': batch_type,
        'started_at': datetime.now().isoformat()
    }
    save_batch_config_sync(config)
    print(f"[배치] 락 획득: {batch_type}")

def release_batch_lock():
    """배치 락 해제"""
    config = load_batch_config()
    batch_type = config['batch_lock'].get('current_job', 'unknown')

    config['batch_lock'] = {
        'is_running': False,
        'current_job': None,
        'started_at': None
    }
    save_batch_config_sync(config)
    print(f"[배치] 락 해제: {batch_type}")

def update_last_run_date(batch_type: str):
    """마지막 실행 날짜 업데이트"""
    config = load_batch_config()
    config[batch_type]['last_run_date'] = datetime.now().strftime('%Y-%m-%d')
    save_batch_config_sync(config)
    print(f"[배치] 실행 날짜 업데이트: {batch_type} - {config[batch_type]['last_run_date']}")
```

### 5.3 배치 스케줄러 (APScheduler)

```python
from apscheduler.schedulers.asyncio import AsyncIOScheduler
from apscheduler.triggers.cron import CronTrigger

# 스케줄러 인스턴스
scheduler = AsyncIOScheduler()

async def run_batch_job():
    """통합 배치 작업 (매분마다 실행 여부 체크)"""

    # 1. 실시간 배치 체크
    if can_start_batch('realtime_batch'):
        await execute_batch('realtime_batch')
        return

    # 2. 과거 데이터 배치 체크
    if can_start_batch('historical_batch'):
        await execute_batch('historical_batch')
        return

async def execute_batch(batch_type: str):
    """실제 배치 실행

    Args:
        batch_type: 'realtime_batch' or 'historical_batch'
    """
    try:
        # 1. 락 획득
        acquire_batch_lock(batch_type)

        # 2. 설정 로드
        config = load_batch_config()
        batch_config = config[batch_type]

        # 3. 하루 전 날짜 계산
        yesterday = (datetime.now() - timedelta(days=1)).strftime('%Y-%m-%d')

        print(f"\n{'='*80}")
        print(f"[배치-{batch_type}] 자동 분석 시작")
        print(f"  - 실행 시각: {datetime.now().strftime('%Y-%m-%d %H:%M:%S')}")
        print(f"  - 수집 날짜: {yesterday}")
        print(f"  - 채널: {', '.join(batch_config['channels'])}")
        print(f"{'='*80}\n")

        # 4. 가상 세션 생성
        session = SessionManager.Session(session_id=f"batch_{batch_type}_{yesterday}")

        # 5. API 데이터 수집
        await collect_api_data_batch(session, yesterday, batch_config['channels'])

        # 6. Phase1~4 자동 실행
        await run_full_analysis_batch(session)

        # 7. 완료 처리
        update_last_run_date(batch_type)

        print(f"\n{'='*80}")
        print(f"[배치-{batch_type}] 자동 분석 완료")
        print(f"  - 완료 시각: {datetime.now().strftime('%Y-%m-%d %H:%M:%S')}")
        print(f"{'='*80}\n")

    except Exception as e:
        print(f"\n{'='*80}")
        print(f"[배치-{batch_type}] 오류 발생")
        print(f"  - 오류 내용: {e}")
        print(f"{'='*80}\n")
        import traceback
        traceback.print_exc()
    finally:
        # 8. 락 해제
        release_batch_lock()

# 스케줄러 설정
@app.on_event("startup")
async def startup_event():
    """FastAPI 시작 시 스케줄러 시작"""

    # 매 1분마다 배치 실행 여부 체크
    scheduler.add_job(
        run_batch_job,
        CronTrigger(minute='*'),  # 매분마다 체크
        id='batch_job',
        replace_existing=True,
        max_instances=1  # 동시 실행 방지
    )

    scheduler.start()
    print("[배치] 스케줄러 시작됨 (매 1분마다 체크)")

@app.on_event("shutdown")
async def shutdown_event():
    """FastAPI 종료 시 스케줄러 종료"""
    scheduler.shutdown()
    print("[배치] 스케줄러 종료됨")
```

## 8. 주의사항

### 8.1 UI 숨김
- **절대 삭제하지 말 것**: CSS로만 숨김
- 나중에 다시 보이게 할 수 있어야 함
- `.hide-for-deploy` 클래스 주석 해제하면 복원

### 8.2 배치 충돌
- 두 배치가 같은 시간에 설정되지 않도록 주의
- 락 메커니즘 반드시 사용
- 예외 발생 시 반드시 `finally`에서 락 해제

### 8.3 에러 처리
- 배치 실행 중 에러 발생 시 락 해제 필수
- 에러 로그 상세히 기록
- 사용자 알림 필요 (이메일, 슬랙 등) - 추후 구현

## 9. 향후 개선 사항

1. **알림 기능**: 배치 성공/실패 시 이메일 또는 슬랙 알림
2. **웹 대시보드**: 배치 실행 이력 조회 UI
3. **재시도 로직**: 실패 시 자동 재시도 (최대 N회)
4. **로그 파일**: 배치 실행 로그를 별도 파일로 저장
5. **모니터링**: Prometheus 등으로 배치 상태 모니터링
