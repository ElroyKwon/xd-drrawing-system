## 🔄 프롬프트 관리 방식

### ✅ 현재 구현: Plain Text 템플릿 (.txt)

**구현 위치**: `config/prompts/*.txt` (5개 파일)

**사용 방법**:
```python
# 1. 템플릿 로드
def load_prompt_template(template_name: str) -> str:
    template_path = Path("config/prompts") / f"{template_name}.txt"
    with open(template_path, 'r', encoding='utf-8') as f:
        return f.read()

# 2. Ollama 호출
response = ollama.chat(
    model="qwen2.5:32b",
    messages=[
        {"role": "system", "content": prompt_template},
        {"role": "user", "content": json.dumps(data, ensure_ascii=False)}
    ],
    options={"temperature": 0.1}  # 일관성 보장
)
```

**장점**:
- ✅ 간단하고 직관적
- ✅ 도메인 전문가도 수정 가능 (Python 코드 몰라도 됨)
- ✅ Git 버전 관리 용이
- ✅ Temperature=0.1로 일관성 충분히 보장
- ✅ OpenAI/Anthropic 공식 예시와 동일한 방식

### ⚠️ 향후 검토 사항

프로젝트 규모 확장 시 다음 방식들을 검토할 수 있습니다:

#### 1. **DSPy 프레임워크** (Stanford NLP)
- **도입 시기**: 프롬프트 20개 이상, Few-Shot 자동 생성 필요 시
- **장점**: 프롬프트 자동 최적화, A/B 테스팅
- **단점**: 러닝 커브, 현재는 오버엔지니어링

```python
# 향후 DSPy 도입 시 예시 (현재 미사용)
import dspy

class BatteryAnalysis(dspy.Signature):
    """배터리 안전 위험 분석 및 리포트 생성"""

    summary_data: str = dspy.InputField(desc="Python 계산 결과 요약")
    physics_validation: str = dspy.InputField(desc="물리 법칙 검증 결과")

    risk_assessment: str = dspy.OutputField(desc="위험도 평가", prefix="## 위험도 평가\n")
    technical_analysis: str = dspy.OutputField(desc="기술 분석", prefix="## 기술 분석\n")
    recommendations: str = dspy.OutputField(desc="조치 권고", prefix="## 조치 권고\n")

# 자동 Few-Shot 학습 예시
from dspy.teleprompt import BootstrapFewShot

trainset = [
    dspy.Example(
        summary_data="...",
        physics_validation="...",
        expected_risk="CRITICAL"
    ).with_inputs("summary_data", "physics_validation")
]

optimizer = BootstrapFewShot(metric=lambda e, p: e.expected_risk in p.risk_assessment)
optimized = optimizer.compile(BatteryAnalyzer(), trainset=trainset)
```

#### 2. **YAML 구조화 템플릿**
- **도입 시기**: 다국어 지원, 프롬프트 변수 관리 복잡도 증가 시
- **장점**: 구조화된 관리, 변수 치환 용이, 버전 관리 명확
- **단점**: 현재는 불필요한 복잡도, 템플릿 엔진 의존성 추가

```yaml
# config/prompts/templates.yaml 예시 (향후 검토)
thermal_safety:
  system_role: |
    당신은 배터리 열적 안전성 전문 엔지니어입니다.

  analysis_objective: |
    온도 변화율(dT/dt)을 분석하여 열폭주 위험을 진단합니다.

  input_format:
    rack_id: "문자열"
    dtdt: "float (°C/min)"
    threshold: "float"

  output_sections:
    - "분석 개요"
    - "검증 결과"
    - "권장 조치사항"

  variables:
    critical_threshold: 5.0
    warning_threshold: 2.0
    temperature_unit: "°C/min"

# Python에서 사용
with open("config/prompts/templates.yaml") as f:
    templates = yaml.safe_load(f)

prompt = templates['thermal_safety']['system_role'].format(
    threshold=templates['thermal_safety']['variables']['critical_threshold']
)
```

#### 3. **Markdown 템플릿** (MD)
- **도입 시기**: 프롬프트 문서화 강화, 버전 관리 가독성 개선 필요 시
- **장점**:
  - Git diff 가독성 우수 (Plain Text보다 구조화)
  - 섹션 구분 명확 (헤더, 코드 블록, 리스트)
  - GitHub/GitLab에서 바로 렌더링 가능
  - 비개발자도 편집 가능 (Plain Text와 유사)
- **단점**:
  - Markdown 파싱 필요 (frontmatter 추출)
  - 현재는 Plain Text로 충분

```markdown
<!-- config/prompts/thermal_safety.md 예시 (향후 검토) -->
---
name: thermal_safety
version: 1.0
model: qwen2.5:32b
temperature: 0.1
---

# SYSTEM ROLE
당신은 배터리 열적 안전성 전문 엔지니어입니다.

## ANALYSIS OBJECTIVE
온도 변화율(dT/dt)을 분석하여 열폭주 위험을 진단합니다.

## INPUT DATA FORMAT
```json
{
  "rack_id": "RACK_042",
  "dtdt": 5.2,
  "threshold": 2.0
}
```

## OUTPUT FORMAT
1. 분석 개요
2. 검증 결과
3. 권장 조치사항

## IMPORTANT RULES
- Markdown 문법 사용 금지 (출력에서)
- 한국어로 작성
```

with open("config/prompts/thermal_safety.md") as f:
    post = frontmatter.load(f)

    metadata = post.metadata  # {'name': 'thermal_safety', 'version': '1.0', ...}

**해결**:
```bash

ollama list
```

**원인**: 파일 인코딩 문제
**해결**:
```python
# phase3_generate_insights.py 확인
with open(output_path, 'w', encoding='utf-8') as f:
    f.write(insight)
