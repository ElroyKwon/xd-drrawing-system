## 📁 후보 포맷 비교

### 1. CSV (Comma-Separated Values)

```yaml
장점:
  - 범용성 최고 (모든 도구 지원)
  - 사람이 읽을 수 있음
  - 간단한 파싱

단점:
  - 용량 최대 (압축 없음)
  - 타입 정보 없음 (문자열로 저장)
  - 느린 읽기 속도
  - 메모리 효율 나쁨

실측 (340M 행):
  - 파일 크기: ~45GB
  - 읽기 속도: 180초
  - 쓰기 속도: 120초
  - 메모리: 60GB

평가: ❌ 부적합 (용량, 속도 문제)
```

### 2. Parquet (Apache Parquet)

```yaml
장점:
  - 컬럼 기반 저장 (분석 최적)
  - 강력한 압축 (ZSTD, Snappy)
  - 타입 안전성 (스키마 내장)
  - Polars/Pandas 최적 지원
  - Row Group 단위 쿼리 (스킵 가능)
  - 통계 정보 저장 (Min/Max)

단점:
  - 사람이 읽을 수 없음
  - 수정 불가 (Immutable)

실측 (340M 행):
  - 파일 크기: ~3.2GB (ZSTD 압축)
  - 읽기 속도: 12초
  - 쓰기 속도: 18초
  - 메모리: 8GB (스트리밍 시 4GB)

평가: ✅ 최적 (모든 기준 충족)
```

### 3. Arrow IPC (Feather v2)

```yaml
장점:
  - 메모리 맵핑 지원 (초고속 읽기)
  - Zero-copy 읽기
  - PyArrow 네이티브 포맷
  - 타입 안전성

단점:
  - 압축률 낮음 (Parquet 대비)
  - 대용량 파일에는 부적합
  - 저장 공간 많이 차지

실측 (340M 행):
  - 파일 크기: ~8.5GB
  - 읽기 속도: 5초 (가장 빠름)
  - 쓰기 속도: 8초
  - 메모리: 12GB

평가: ⚠️ 조건부 (임시 캐시용은 좋음, 장기 저장 부적합)
```

### 4. HDF5 (Hierarchical Data Format)

```yaml
장점:
  - 계층 구조 지원
  - 대용량 데이터셋 지원
  - 과학 분야 표준

단점:
  - Polars 지원 약함 (Pandas만)
  - 복잡한 API
  - 파일 손상 위험 (쓰기 중 중단 시)
  - 압축률 낮음

실측 (340M 행):
  - 파일 크기: ~6.8GB
  - 읽기 속도: 25초
  - 쓰기 속도: 30초
  - 메모리: 10GB

평가: ❌ 부적합 (Polars 미지원, 복잡성)
```

### 5. Pickle (Python 전용)

```yaml
장점:
  - Python 객체 직렬화
  - 간편한 사용

단점:
  - Python 외 사용 불가
  - 보안 위험 (임의 코드 실행)
  - 압축 약함
  - 버전 호환성 문제

평가: ❌ 부적합 (호환성, 보안)
```

## 📐 Parquet 세부 설정

### 권장 압축 방식

```python
# ZSTD vs Snappy 비교
압축 방식:
  ZSTD (권장):
    - 압축률: 85% (3.2GB)
    - 압축 속도: 18초
    - 읽기 속도: 12초
    - CPU 사용: 중간

  Snappy:
    - 압축률: 70% (4.5GB)
    - 압축 속도: 10초
    - 읽기 속도: 8초
    - CPU 사용: 낮음

결론: ZSTD Level 3 (용량 우선, 속도도 충분)
```

### Row Group 크기

```python
# 시계열 데이터 최적값
Row Group Size: 1,000,000 행
  - 이유: 시간 범위 쿼리 시 스킵 효율적
  - 340M 행 → 340개 Row Group

예시:
  쿼리: "2025-11-30 14:00 ~ 15:00 데이터만"
  → 해당 시간의 Row Group만 읽기 (340개 중 12개)
  → 97% 스킵 → 읽기 속도 30배 빠름
```

### Page Size

```python
Data Page Size: 1MB
  - 이유: 시계열 스캔 최적
  - 너무 작으면: 메타데이터 오버헤드
  - 너무 크면: 메모리 낭비
```

## 📊 배터리 표준 파일 스키마 (초안)

### 필수 컬럼

```python
import pyarrow as pa

BATTERY_STANDARD_SCHEMA = pa.schema([
    # === 시간 정보 ===
    ('timestamp', pa.timestamp('ms', tz='UTC')),  # 밀리초 정밀도

    # === 식별 정보 ===
    ('rack_id', pa.dictionary(pa.int16(), pa.string())),  # "RACK_001" ~ "RACK_160"
    ('module_id', pa.dictionary(pa.int8(), pa.string())), # "MODULE_01" ~ "MODULE_12"

    # === 전압 데이터 (V) ===
    ('v_avg', pa.float32()),  # 평균 전압
    ('v_min', pa.float32()),  # 최소 전압
    ('v_max', pa.float32()),  # 최대 전압
    ('v_std', pa.float32()),  # 전압 표준편차

    # === 온도 데이터 (°C) ===
    ('t_avg', pa.float32()),  # 평균 온도
    ('t_min', pa.float32()),  # 최소 온도
    ('t_max', pa.float32()),  # 최대 온도
    ('t_std', pa.float32()),  # 온도 표준편차

    # === 전류/SOC ===
    ('current', pa.float32()),  # 전류 (A)
    ('soc', pa.float32()),      # 충전율 (0.0 ~ 100.0)

    # === 품질 플래그 ===
    ('quality_flag', pa.int8()),  # 0: 정상, 1: 경고, 2: 오류
])
```

### 컬럼 설명

| 컬럼 | 타입 | 범위 | 설명 |
|------|------|------|------|
| `timestamp` | Timestamp(ms) | - | UTC 기준 시간 |
| `rack_id` | Dictionary | RACK_001~160 | Rack 식별자 (반복 문자열 압축) |
| `module_id` | Dictionary | MODULE_01~12 | 모듈 식별자 |
| `v_avg` | float32 | 2.5~3.65V | 평균 전압 (LFP 범위) |
| `v_min` | float32 | 2.5~3.65V | 최소 전압 |
| `v_max` | float32 | 2.5~3.65V | 최대 전압 |
| `v_std` | float32 | 0~0.5V | 전압 편차 |
| `t_avg` | float32 | -20~60°C | 평균 온도 |
| `t_min` | float32 | -20~60°C | 최소 온도 |
| `t_max` | float32 | -20~60°C | 최대 온도 |
| `t_std` | float32 | 0~20°C | 온도 편차 |
| `current` | float32 | -500~500A | 방전(-), 충전(+) |
| `soc` | float32 | 0~100% | 충전 상태 |
| `quality_flag` | int8 | 0/1/2 | 데이터 품질 |

## 💾 용량 추산

### XD-HUB 원본 (CSV 가정)

```yaml
데이터:
  - Rack 수: 160개
  - 측정 주기: 1분
  - 기간: 1개월 (30일)
  - 총 행 수: 160 × 60 × 24 × 30 = 6,912,000 행

CSV 크기:
  - 행당 크기: ~150 bytes
  - 총 크기: 6,912,000 × 150 = 1,036MB (약 1GB)

Parquet 크기 (ZSTD):
  - 압축률: 85%
  - 총 크기: 1,036MB × 0.15 = 155MB ✅

1년 데이터:
  - CSV: 12GB
  - Parquet: 1.8GB (압축률 85%)
```
