## 개요

GEM (Generative Engineering Module)의 모든 Phase에서 일관된 로깅 정책을 적용하여:
- 실행 이력 추적 가능
- 에러 디버깅 용이
- 성능 분석 및 최적화
- AI 호출 비용 계산
- 시스템 모니터링

## 로깅 원칙

### 1. 중앙 집중식 설정
- 모든 로그 파일명은 `config/global/styles/report_style_guide.yaml`에 정의
- 하드코딩 금지

### 2. 로그 레벨 분리
| 레벨 | 용도 | 파일 저장 |
|------|------|----------|
| DEBUG | 상세한 디버깅 정보 | phase3.log |
| INFO | 일반 실행 정보 | phase3.log |
| WARNING | 경고 (재시도, 성능 저하) | phase3.log |
| ERROR | 에러 (복구 가능) | phase3.log + error.log |
| CRITICAL | 치명적 에러 (복구 불가) | phase3.log + error.log |

### 3. 구조화된 로그
- Ollama 호출: JSONL 형식 (파싱 용이)
- 성능 메트릭: JSON 형식
- 실행 로그: 텍스트 형식 (사람이 읽기 쉬움)

## Python logging 모듈 사용

### 설정 (phase3_generate_insights.py)

```python
import logging

def setup_logging(phase3_dir: Path, style_guide: dict):
    """Python logging 모듈 설정"""
    # YAML에서 로그 파일명 읽기
    phase3_log = style_guide['output']['files'].get('phase3_log', 'phase3.log').split('/')[-1]
    error_log = style_guide['output']['files'].get('error_log', 'error.log').split('/')[-1]

    # 로거 생성
    logger = logging.getLogger('phase3')
    logger.setLevel(logging.DEBUG)
    logger.handlers.clear()  # 중복 방지

    # 포맷터
    formatter = logging.Formatter(
        '%(asctime)s - %(name)s - %(levelname)s - %(message)s',
        datefmt='%Y-%m-%d %H:%M:%S'
    )

    # 파일 핸들러 1: 전체 로그
    fh_all = logging.FileHandler(phase3_dir / phase3_log, encoding='utf-8')
    fh_all.setLevel(logging.DEBUG)
    fh_all.setFormatter(formatter)
    logger.addHandler(fh_all)

    # 파일 핸들러 2: 에러만
    fh_error = logging.FileHandler(phase3_dir / error_log, encoding='utf-8')
    fh_error.setLevel(logging.ERROR)
    fh_error.setFormatter(formatter)
    logger.addHandler(fh_error)

    return logger
```

### 사용 예시

```python
logger = setup_logging(phase3_dir, style_guide)

logger.info("Phase 2 JSON 파일 로드 시작")
logger.debug(f"파일 경로: {phase2_dir}")

try:
    # ...
    logger.info("JSON 파일 4개 로드 완료")
except FileNotFoundError as e:
    logger.error(f"JSON 파일 없음: {e}", exc_info=True)
    raise
```

- AI 호출 이력 상세 기록

### Phase 4 (Report)
- Phase 3에 포함 (HTML 생성 단계)

## Ollama 호출 로깅 (JSONL)

### 파일: `ollama_calls.jsonl`

**형식:** JSON Lines (각 줄이 독립적인 JSON 객체)

**예시:**
```jsonl
{"timestamp": "2025-12-26T14:00:01.123", "insight_type": "thermal", "model": "qwen2.5:32b", "temperature": 0.1, "input_bytes": 1024, "output_chars": 2048, "duration_sec": 14.23, "status": "success"}
{"timestamp": "2025-12-26T14:00:15.456", "insight_type": "spc", "model": "qwen2.5:32b", "temperature": 0.1, "input_bytes": 890, "output_chars": 1920, "duration_sec": 12.85, "status": "success"}
{"timestamp": "2025-12-26T14:00:28.789", "insight_type": "integrity", "model": "qwen2.5:32b", "temperature": 0.1, "input_bytes": 756, "output_chars": 0, "duration_sec": 5.12, "status": "failed", "error": "Connection timeout"}
```

**필드 설명:**
| 필드 | 타입 | 설명 |
|------|------|------|
| timestamp | string (ISO 8601) | 호출 시각 |
| insight_type | string | thermal/spc/integrity/correlation/comprehensive |
| model | string | Ollama 모델명 |
| temperature | float | Temperature 값 |
| input_bytes | int | 입력 데이터 크기 (bytes) |
| output_chars | int | 출력 텍스트 길이 (characters) |
| duration_sec | float | 소요 시간 (초) |
| status | string | success/failed |
| error | string (optional) | 에러 메시지 (실패 시) |

### 구현 예시

```python
def log_ollama_call(phase3_dir, insight_type, model, temperature,
                    input_size, output_size, duration, status, error_msg=None):
    """Ollama 호출 기록을 JSONL 파일에 저장"""
    log_entry = {
        "timestamp": datetime.now().isoformat(),
        "insight_type": insight_type,
        "model": model,
        "temperature": temperature,
        "input_bytes": input_size,
        "output_chars": output_size,
        "duration_sec": round(duration, 2),
        "status": status
    }

    if error_msg:
        log_entry["error"] = str(error_msg)

    with open(phase3_dir / 'ollama_calls.jsonl', 'a', encoding='utf-8') as f:
        f.write(json.dumps(log_entry, ensure_ascii=False) + '\n')
```

## 로그 로테이션

### 정책
- 로그 파일이 **10MB 초과** 시 자동 로테이션
- 최대 **5개 백업 파일** 유지
- 오래된 파일 자동 삭제

### Python logging 모듈 사용

```python
from logging.handlers import RotatingFileHandler

# 로테이션 파일 핸들러
fh_rotating = RotatingFileHandler(
    phase3_dir / 'phase3.log',
    maxBytes=10*1024*1024,  # 10MB
    backupCount=5,           # 최대 5개 백업
    encoding='utf-8'
)
fh_rotating.setLevel(logging.DEBUG)
fh_rotating.setFormatter(formatter)
logger.addHandler(fh_rotating)
```

**결과:**
```
phase3.log        # 현재 로그
phase3.log.1      # 이전 로그
phase3.log.2
phase3.log.3
phase3.log.4
phase3.log.5
```

## 로그 분석 및 모니터링

### 1. Ollama 호출 분석

```python
import json

# JSONL 파싱
with open('ollama_calls.jsonl', 'r', encoding='utf-8') as f:
    calls = [json.loads(line) for line in f]

# 성공률 계산
total = len(calls)
success = sum(1 for c in calls if c['status'] == 'success')
print(f"성공률: {success/total*100:.1f}%")

# 평균 소요 시간
avg_duration = sum(c['duration_sec'] for c in calls) / total
print(f"평균 소요 시간: {avg_duration:.2f}초")

# 가장 느린 호출
slowest = max(calls, key=lambda c: c['duration_sec'])
print(f"가장 느린 호출: {slowest['insight_type']} ({slowest['duration_sec']:.2f}초)")
```

### 2. 에러 패턴 분석

```bash
# error.log에서 가장 많이 발생한 에러 찾기
grep "ERROR" error.log | cut -d'-' -f4 | sort | uniq -c | sort -rn | head -5
```

### 3. 성능 추이 분석

```python
import glob
import json

# 모든 performance.json 읽기
perf_files = glob.glob("results/*/phase3_insights/performance.json")
data = []
for f in perf_files:
    with open(f, 'r') as fp:
        data.append(json.load(fp))

# 평균 실행 시간
avg = sum(d['total_duration_sec'] for d in data) / len(data)
print(f"평균 실행 시간: {avg:.1f}초")
```

## 모범 사례 (Best Practices)

### ✅ DO

1. **구조화된 로그 사용**
   ```python
   logger.info(f"JSON 파일 로드 완료: {file_count}개, {total_size}MB")
   ```

2. **에러 시 스택트레이스 포함**
   ```python
   logger.error("Ollama 호출 실패", exc_info=True)
   ```

3. **중요 이벤트 타임스탬프 기록**
   ```python
   logger.info(f"인사이트 생성 시작: {datetime.now().isoformat()}")
   ```

4. **메트릭 수집**
   ```python
   metrics['generate_thermal'] = {"duration_sec": duration, "output_size": len(output)}
   ```

### ❌ DON'T

1. **민감 정보 로깅 금지**
   ```python
   # ❌ 금지
   logger.debug(f"API Key: {api_key}")

   # ✅ 안전
   logger.debug(f"API Key: {'*' * 10}")
   ```

2. **과도한 DEBUG 로그**
   ```python
   # ❌ 너무 많음
   for i in range(10000):
       logger.debug(f"처리 중: {i}")

   # ✅ 적절함
   logger.debug(f"총 {total}개 항목 처리 시작")
   ```

3. **하드코딩된 로그 파일명**
   ```python
   # ❌ 금지
   with open('log.txt', 'w') as f:

   # ✅ YAML에서 읽기
   log_file = style_guide['output']['files']['phase3_log']
   ```
