# GEM (Generative Engineering Module) 프로젝트 전체 요약서

**문서 버전:** v1.0
**작성일:** 2025-12-11
**프로젝트 목표:** 대규모 BESS 데이터(160개+ Rack) 정밀 안전 진단 및 엔지니어 주도형 분석 플랫폼 구축

---

## 📋 프로젝트 개요

### 핵심 컨셉
GEM은 **"지능(SLM)"과 "연산(Python)"의 완벽한 분리**를 통해 제주 BESS 배터리 시스템(160개 Rack, 3.4억 건 시계열 데이터)에 대한 자동 안전 진단을 수행하는 솔루션입니다.

### 주요 특징
1. **Dual Engine Architecture**
   - 🧠 Brain (SLM/Gemma 2:27b): 자연어 해석 및 리포트 생성
   - 💪 Muscle (Python/Pandas): 대용량 병렬 연산 처리

2. **4대 핵심 분석 알고리즘**
   - ✓ **열적 안전성 (Thermal Safety):** dT/dt > 2.0°C/min 탐지
   - ✓ **셀 밸런싱 (SPC):** Z-Score > 3.0σ 이상치 탐지
   - ✓ **내부 단락 (ISC):** 전압 강하 & 온도 상승 동시 발생 탐지
   - ✓ **데이터 무결성:** 센서 고착(Variance=0) 감지

3. **No-DB 전략**
   - 메모리 기반 처리 (AnalysisContext Singleton)
   - Parquet/YAML 파일 시스템 기반 자산화
   - 단일 정적 뷰어 아키텍처

## 🏗️ 시스템 아키텍처

### 3-Layer Architecture

```
┌─────────────────────────────────────────────────┐
│  Layer 1: Control & Interface                   │
│  - FastAPI (API Server)                         │
│  - SecurityMiddleware (인증/권한)                │
│  - PerformanceMonitor (성능 측정)                │
└─────────────────────────────────────────────────┘
                     ↓
┌─────────────────────────────────────────────────┐
│  Layer 2: Computation Engine                    │
│  - PipelineManager (파이프라인 제어)             │
│  - MathEngine (병렬 연산)                        │
│  - PhysicsValidator (물리 검증)                  │
└─────────────────────────────────────────────────┘
                     ↓
┌─────────────────────────────────────────────────┐
│  Layer 3: Data & Intelligence                   │
│  - DataLoader (수집)                             │
│  - SLM Client (Ollama 연동)                     │
│  - AssetManager (저장)                           │
└─────────────────────────────────────────────────┘
```

### 파이프라인 흐름

```
Request → Fetch & Sync → Math Compute → AI Interpret → Asset Save → Response (Viewer URL)
```

## 📅 개발 Phase 구조

### Phase 1: 데이터 수집 및 동기화
- **목표:** 160개 Rack 데이터 안정적 수집 및 시간축 동기화
- **핵심 기술:**
  - Batch Processing (20개 단위)
  - Exponential Backoff Retry
  - Pydantic 데이터 검증
  - Resampling (10초 간격)
- **품질 기준:** 80% 이상 수집 성공 시 진행 (Partial Success 정책)

### Phase 2: 수학적 연산 엔진
- **목표:** 4대 진단 알고리즘 병렬 처리
- **핵심 기술:**
  - ProcessPoolExecutor (멀티코어 활용)
  - Pandas Vectorization (Loop 금지)
  - PhysicsValidator (센서 오류 필터링)
- **성능 목표:** 전체 연산 < 10초 (P95 기준)

### Phase 3: SLM 해석
- **목표:** 수치 데이터 → 전문가 리포트 자동 생성
- **핵심 기술:**
  - Semantic YAML 변환
  - Prompt Guardrail (환각 방지)
  - Timeout & Fallback (60초 제한)
- **안전장치:** AI 실패 시 정적 템플릿 리포트 제공

### Phase 4: 자산화 및 시각화
- **목표:** 결과 무결하게 저장 및 단일 뷰어 제공
- **핵심 기술:**
  - Atomic Write 패턴
  - Chart.js (Zoom/Pan 지원)
  - Deep Linking (리포트↔차트 연결)
- **보안:** Path Traversal 방어

### Phase 5: 통합 및 배포
- **목표:** 운영 API + 개발자 대시보드 구축
- **핵심 기술:**
  - API Key 인증
  - WebSocket 로그 스트리밍
  - 자동 재연결 로직
- **배포:** Windows Server (No Docker)

## 🔒 품질 및 보안 정책

### 에러 핸들링
- **Partial Success:** 데이터 수집률 80% 임계값
- **Retry Strategy:** Exponential Backoff + Jitter
- **Fallback:** AI 타임아웃 시 정적 리포트

### 데이터 검증
1. **입력 검증:** Pydantic Schema (전압/온도 범위)
2. **물리 검증:** dT/dt > 10.0°C/min → Sensor Error
3. **스키마 버전:** YAML/Parquet 메타데이터 버전 관리

### 보안
- API Key 인증 (Bearer Token)
- CORS 제한 (XD-Works 도메인만)
- Path Sanitization (상위 디렉토리 접근 차단)
- Prompt Injection 방어 (Guardrail)

## 🛠️ 기술 스택

### Backend
- **Framework:** FastAPI 0.104+
- **Data Processing:** Pandas 2.1+, PyArrow 14+
- **Concurrency:** asyncio, ProcessPoolExecutor
- **Validation:** Pydantic v2.5+

### AI/LLM
- **Model:** Gemma 2:27b (Ollama)
- **Strategy:** Semantic YAML + Grounding Prompt

### Frontend
- **Core:** Vanilla JS (정적 뷰어)
- **Visualization:** Chart.js + Zoom Plugin
- **Markdown:** marked.js

### Dev Tools
- **Console:** WebSocket 기반 Admin Dashboard
- **Testing:** pytest, Mock CSV
