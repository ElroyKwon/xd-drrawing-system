## 📅 개발 우선순위 (Priority)

### P0 (필수 - Critical)
반드시 구현해야 프로젝트 완료 가능

- ✅ 4대 진단 알고리즘 (SPC, Thermal, ISC, Integrity)

- ✅ Path Sanitization (보안)

- ⚠️ Exponential Backoff + Jitter
- ⚠️ 스키마 버전 관리
- ⚠️ Audit Log (API 접근 기록)
- ⚠️ CORS 제한

### P2 (선택 - Nice to Have)
있으면 좋지만 없어도 동작

- 💡 실시간 스트리밍 모니터링

## 📐 코드 스타일 가이드

### Naming Convention

| 대상 | 규칙 | 예시 |
|------|------|------|
| 모듈 | snake_case | `data_loader.py` |
| 클래스 | PascalCase | `DataLoadResult` |
| 함수 | snake_case | `calculate_zscore()` |
| 상수 | UPPER_SNAKE | `MAX_RETRY_COUNT` |
| 변수 | snake_case | `rack_id` |

### Docstring 규칙

```python
def calculate_zscore(df: pd.DataFrame, column: str) -> pd.Series:
    """
    Z-Score 계산

    Args:
        df: 입력 DataFrame
        column: 계산 대상 컬럼명

    Returns:
        Z-Score Series

    Raises:
        ValueError: 컬럼이 존재하지 않을 경우

    Example:
        >>> z = calculate_zscore(df, 'v_avg')
        >>> print(z.max())
        4.2
    """
```

### Type Hints 필수

```python
# ✅ Good
def load_data(rack_id: str) -> pd.DataFrame:
    pass

# ❌ Bad
def load_data(rack_id):
    pass
```

## 🚫 금지 사항 (Anti-Patterns)

### 1. Pandas Loop 사용 금지

```python
# ❌ 절대 금지
for idx in range(len(df)):
    df.loc[idx, 'result'] = df.loc[idx, 'a'] + df.loc[idx, 'b']

# ✅ Vectorization 사용
df['result'] = df['a'] + df['b']
```

### 2. 하드코딩 금지

```python
# ❌ 절대 금지
API_KEY = "abc123secret"
THRESHOLD = 0.8

# ✅ 설정 파일 또는 환경변수
from config import settings
API_KEY = settings.api_key
THRESHOLD = settings.partial_success_threshold
```

### 3. 전역 변수 사용 금지

```python
# ❌ 절대 금지
current_data = None  # 전역 변수

def process():
    global current_data
    current_data = load()

# ✅ Singleton 또는 DI 사용
class AnalysisContext:
    _instance = None

    @classmethod
    def get_instance(cls):
        if cls._instance is None:
            cls._instance = cls()
        return cls._instance
```

### 4. 예외 무시 금지

```python
# ❌ 절대 금지
try:
    result = risky_operation()
except:
    pass  # 아무 조치 없음

# ✅ 명시적 처리
try:
    result = risky_operation()
except SpecificError as e:
    logger.error(f"Operation failed: {e}")
    raise  # 또는 Fallback 처리
```

## 🚀 빠른 시작 가이드 (Quick Start)

### 첫 개발자를 위한 5단계

**1. 저장소 클론 및 환경 설정**
```bash
git clone <repo_url>
cd GEM_Project
python -m venv venv
venv\Scripts\activate
pip install -r requirements.txt
```

**2. 설정 파일 준비**
```bash
cp config/config.yaml.example config/config.yaml
cp .env.example .env
# .env 파일 편집 (API Key 설정)
```

**3. 테스트 데이터 준비**
```bash
# tests/mock_data/ 폴더에 CSV 복사
# 또는 샘플 데이터 생성 스크립트 실행
python scripts/generate_mock_data.py
```

**4. Dev Console 실행**
```bash
python dev_server.py
# 브라우저: http://localhost:8000/admin
```

**5. 첫 분석 실행**
```
Dev Console에서:
1. Data Source: "Mock CSV" 선택
2. "Run Full Pipeline" 클릭
3. 우측 뷰어에서 결과 확인
```
