## 🏗️ 4-Layer Architecture (Hot/Cold Path 분리)

```
┌─────────────────────────────────────────────────────────────┐
│             Layer 0: Real-time Monitoring (NEW)              │
│  ┌──────────────┐  ┌──────────────┐  ┌──────────────┐      │
│  │   HOT PATH   │  │  Threshold   │  │   Instant    │      │
│  │  임계값 감지  │  │   Checker    │  │    Alert     │      │
│  │   < 1초 SLA  │  │  dT/dt>2.0   │  │  WebSocket   │      │
│  └──────────────┘  └──────────────┘  └──────────────┘      │
└─────────────────────────────────────────────────────────────┘
                     ↓ (위험 감지 시)
                   ┌─────────────┐
                   │ 즉시 알림!  │
                   └─────────────┘

                     ↓ (동시 진행)
┌─────────────────────────────────────────────────────────────┐
│                  Layer 1: Control & Interface                │
│                      (COLD PATH - 기존)                      │
│  ┌──────────────┐  ┌──────────────┐  ┌──────────────┐      │
│  │   FastAPI    │  │  Security    │  │  Performance │      │
│  │   Server     │  │  Middleware  │  │  Monitor     │      │
│  └──────────────┘  └──────────────┘  └──────────────┘      │
└─────────────────────────────────────────────────────────────┘
                            ↓
┌─────────────────────────────────────────────────────────────┐
│                Layer 2: Computation Engine                   │
│  ┌──────────────┐  ┌──────────────┐  ┌──────────────┐      │
│  │  Pipeline    │  │    Math      │  │   Physics    │      │
│  │  Manager     │  │   Engine     │  │  Validator   │      │
│  └──────────────┘  └──────────────┘  └──────────────┘      │
└─────────────────────────────────────────────────────────────┘
                            ↓
┌─────────────────────────────────────────────────────────────┐
│              Layer 3: Data & Intelligence                    │
│  ┌──────────────┐  ┌──────────────┐  ┌──────────────┐      │
│  │    Data      │  │  SLM (vLLM)  │  │    Asset     │      │
│  │   Loader     │  │ Qwen2.5 32B  │  │   Manager    │      │
│  └──────────────┘  └──────────────┘  └──────────────┘      │
└─────────────────────────────────────────────────────────────┘
```

### Hot/Cold Path 차이점

| 경로 | SLA | 목적 | 처리 방식 |
|------|-----|------|----------|
| **HOT Path** | < 1초 | 즉각 위험 감지 | 임계값 단순 비교 |
| **COLD Path** | 15초 | 상세 분석 및 리포트 | 4대 알고리즘 + AI 해석 |

## 🔄 파이프라인 State Machine

### 상태 정의

```
Idle ──┐
       │ Request
       ▼
  Initializing ──┐
                 │ Config OK
                 ▼
         Phase1_Collecting ──┐
                             │ >= 80% Success
                             ▼
                   Phase2_Computing ──┐
                                      │ Success
                                      ▼
                             Phase3_Reasoning ──┐
                                                │ Success
                                                ▼
                                      Phase4_Archiving ──┐
                                                         │ Success
                                                         ▼
                                                    Completed

                           (어느 단계든 Critical Error 발생 시)
                                      ↓
                                   Failed
```

### 상태별 동작

| 상태 | 주요 작업 | 다음 상태 조건 |
|------|----------|---------------|
| **Idle** | 대기 | API 요청 수신 |
| **Initializing** | 설정 로드, 리소스 확인 | 검증 성공 |
| **Phase1** | 데이터 수집 & 동기화 | 80% 이상 수집 |
| **Phase2** | 병렬 연산 & 검증 | 연산 완료 |
| **Phase3** | AI 해석 | 리포트 생성 (Fallback 포함) |
| **Phase4** | 파일 저장 | Atomic Write 성공 |
| **Partial_Success** | 일부 실패 허용 | Phase1 판단 시점 |
| **Failed** | 중단 | - |
| **Completed** | 완료 | - |

## 🔌 데이터 흐름 (Data Flow)

### 1. Request Phase
```
XD-Works (Client)
    │
    │ POST /analyze
    │ {start_time, end_time, target_racks}
    ▼
FastAPI (main.py)
    │
    │ Authorization Check
    │ TraceID 발급
    ▼
PipelineManager
```

### 2. Processing Phase
```
DataLoader
    │ Fetch (Batch 20)
    │ Validate (Pydantic)
    ▼
SyncEngine
    │ Resampling
    │ Broadcasting
    ▼
MathEngine
    │ Parallel Compute
    │ Physics Validation
    ▼
SLM Client
    │ Semantic YAML
    │ Ollama API Call
    ▼
AssetManager
    │ Atomic Write
    │ Directory Structure
```

### 3. Response Phase
```
AssetManager
    │
    │ URL 생성
    ▼
FastAPI Response
    │
    │ {report_url: "..."}
    ▼
XD-Works (Iframe Load)
```

## 🔐 보안 계층

### 인증 체계
```
Request
    │
    ▼
[SecurityMiddleware]
    │ Bearer Token 검증
    │ API Key 매칭
    ▼
    ├─ 성공 → 다음 Handler
    └─ 실패 → 401 Unauthorized
```

### 경로 검증
```
URL Parameter (?source=/data/...)
    │
    ▼
[sanitizeDataPath()]
    │ 정규식 패턴 매칭
    │ /^\/data\/\d{8}_[A-Z0-9]+$/
    ▼
    ├─ 통과 → fetch()
    └─ 차단 → 403 Forbidden
```

## ⚡ 성능 최적화 전략

### 비동기 처리
- **I/O Bound:** asyncio (네트워크, 파일)
- **CPU Bound:** ProcessPoolExecutor (연산)

### 메모리 관리
- **Streaming:** PyArrow Stream (API 응답)
- **Lazy Load:** 뷰어 차트 On-Demand 로드
- **Compression:** Parquet Snappy 압축

### 병렬화
```
160 Rack 분석
    │
    ▼
[ProcessPoolExecutor]
    │ CPU 코어 수 = N
    │ Worker Pool 생성
    ▼
    ├─ Worker 1 → Rack 001-020
    ├─ Worker 2 → Rack 021-040
    ├─ ...
    └─ Worker N → Rack 141-160

    (모두 완료 대기)
```
