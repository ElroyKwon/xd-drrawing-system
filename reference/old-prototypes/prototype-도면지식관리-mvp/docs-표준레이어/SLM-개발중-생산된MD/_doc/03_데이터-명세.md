## 📊 출력 데이터 스키마

### Phase 1 출력 (데이터 변환)

#### 1.1 battery_data.parquet

**목적**: 분석용 전체 데이터 (압축, 고성능)

**Parquet 메타데이터**:
```
compression: ZSTD (level 3)
row_group_size: 1,000,000
schema_version: "1.0"
```

**컬럼 스키마 (Phase 1 생성 후)**:
| 컬럼명 | 타입 | 설명 | 생성 단계 |
|--------|------|------|-----------|
| `timestamp` | datetime64[ns] | 시간 | Phase 1 |
| `rack_id` | str | Rack ID | Phase 1 |
| `v_avg` | float32 | 평균 전압 | Phase 1 |
| `v_min` | float32 | 최소 전압 | Phase 1 |
| `v_max` | float32 | 최대 전압 | Phase 1 |
| `v_std` | float32 | 전압 표준편차 | Phase 1 |
| `t_avg` | float32 | 평균 온도 | Phase 1 |
| `t_min` | float32 | 최소 온도 | Phase 1 |
| `t_max` | float32 | 최대 온도 | Phase 1 |
| `t_std` | float32 | 온도 표준편차 | Phase 1 |
| `current` | float32 | 전류 | Phase 1 |
| `soc` | float32 | SOC (%) | Phase 1 |
| `quality_flag` | int8 | 품질 플래그 | Phase 1 |

**Phase 2에서 추가되는 검증 컬럼**:
| 컬럼명 | 타입 | 설명 | 생성 단계 |
|--------|------|------|-----------|
| `dtdt` | float32 | 온도 변화율 (°C/min) | Phase 2 |
| `z_score` | float32 | Z-Score (σ) | Phase 2 |
| `is_thermal_risk` | bool | 열적 위험 여부 | Phase 2 |
| `is_outlier` | bool | 이상치 여부 | Phase 2 |
| `correlation_coefficient` | float32 | 상관계수 | Phase 2 |
| `sensor_frozen` | bool | 센서 고착 여부 | Phase 2 |
| `out_of_range` | bool | 범위 이탈 여부 | Phase 2 |

**중요**: Phase 2는 Phase 1의 Parquet 파일에 검증 컬럼을 추가하여 동일 시간축을 유지합니다.

- 필수 컬럼만 포함
- 소수점 2자리 반올림 (`round(2)`)
- Boolean → 비트 플래그
- 예상 크기: ~500KB/Rack
