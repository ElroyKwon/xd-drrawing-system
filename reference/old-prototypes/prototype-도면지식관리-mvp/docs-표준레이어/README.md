# docs-비전-의미레이어 — DKS 정체성 재정의 묶음

> **작성 시점**: 2026-04-17
> **계기**: 사용자 명시 — "수요처는 매일 정보 필요한 운영자, 장기 비전은 로봇↔공장 사이 정보·추론·온톨로지 게이트웨이"
> **결론 요약**: DKS = EDMS 아닌 **의미 레이어(Semantic Layer) 회사**. 운영자(사람) + 로봇·외부 AI(기계)가 같은 [[온톨로지]]를 다른 인터페이스로 소비.

---

## 1. 이 폴더는 무엇인가

`docs-시스템분석/`이 8인 페르소나 분석으로 도출한 결론(Layer 0 EDMS부터)에 **두 개의 새 입력**이 추가되어 정체성이 다시 그려진 묶음.

### 두 입력
1. **수요처 명확화** — 건물·공장 등 관리·운영하는 사람, **매일 정보 필요한 사람**
2. **장기 비전** — 공장에 [[로봇]] 도입 시, 로봇과 공장 사이 **정보·추론·온톨로지 게이트웨이** 역할

### 왜 정체성 재정의인가
- `docs-시스템분석`의 8인 페르소나 = *준공 전·도면 유통* 사용자 → 지불의사 0/8
- 본 비전이 가리키는 페르소나 = **운영자**·**로봇 통합 책임자** → 다른 구매자, 다른 가치, 다른 가격 모델
- DKS [[CLAUDE.md]] "맥락 엔진(Context Engine)" 포지셔닝과 정확히 일치하나, 지금까지 그 자리를 비워둠

## 3. `docs-시스템분석`과의 관계

| 항목 | docs-시스템분석 (구) | docs-비전-의미레이어 (신) |
|------|--------------------|--------------------------|
| 페르소나 | 준공 전·도면 유통 8인 | 운영자 + 로봇 통합 책임자 |
| 진입점 | Layer 0 = 도면 찾기 EDMS | Layer 0 = 알람·이상 시 1차 맥락 즉답 |
| 차별점 | 도면 위 주석 / 양방향 태깅 | 운영 데이터 × 도면 [[온톨로지]] 결합 |
| 비전 | AI 메타데이터 자동 추출 | 로봇·외부 AI 의미 게이트웨이 |
| 가격 | per-seat (지불의사 0/8) | 사이트당 월 구독 + 쿼리량 크레딧 |
| 시장 | EDMS (PlanGrid 운명) | 의미 레이어 (선점 가능) |

**핵심**: 폐기 아닌 **위치 재배치**. `docs-시스템분석`의 ISS-034~047 기능들은 본 비전 안에서도 유효하나, **목적·우선순위·소비자**가 달라짐.

## 4. 핵심 통찰 3줄

1. **운영자와 로봇은 같은 코어를 다른 인터페이스로 소비** — §4 모델 ① + ③ 가 한 줄기. 단기·중기·장기 분리 불필요.
2. **의미 레이어 시장 공백** — [[OPC UA]] 통신 표준화, [[ISA-95]]·[[ISO 15926]] 의미 표준 미채택. 도면 자동 의미 추출이 빈자리 진입 도구.
3. **시나리오 #1 운영자 미작성** — DKS [[CLAUDE.md]]에 정작 가장 큰 가치 포인트가 ❌. 본 비전 검증의 첫 작업.
