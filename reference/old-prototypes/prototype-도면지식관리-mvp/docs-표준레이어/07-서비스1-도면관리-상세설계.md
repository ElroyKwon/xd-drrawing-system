# 07. 서비스 1 — 도면 관리 상세 설계

> **목적**
> 프로토타입 코딩 착수 가능한 수준의 상세 명세.
> 기능·데이터·UI·API·기술 스택·MVP 범위·검증·인터뷰 질문지 포함.

> **정체성**
> *"도면이 파일이 아니라, 질문 가능한 살아있는 지식이 됩니다."*

> **타겟 페르소나**
> 공장형 시설관리 책임자·실무자 / 데이터센터형 통합 유지보수 인원

## §1. 서비스 개요

### §1.1 해결 불만 매핑
| 불만 | 해결 방식 |
|------|----------|
| a 도면 최신화 안 됨 | 공사 이벤트 자동 재분석 + HITL 승인 플로우 |
| b 변경 공유·알림 없음 | 온톨로지 기반 영향 분야 자동 통지 |
| 도곽 표준 재작성 요청 | Phase 0 자동 도곽·레이어 정규화 |
| 도면 못 찾음 | 자연어 검색 + 양방향 태깅 |

### §1.2 핵심 차별 1문장
> 기존 EDMS = 파일 저장소. 본 서비스 = **도면이 의미·관계·영향도를 가진 지식 그래프의 시각적 표현**.

## §2. 핵심 사용자 여정 (7 시나리오)

### 시나리오 1 — 현장 긴급 도면 조회 (김○○ 실무자)
- 상황: 새벽 2시 지하 기계실 긴급 호출. 냉동기 배관 확인 필요.
- 기존: NAS 접속 불가 → 사무실 복귀 → 검색 → 다시 현장 (30분+)
- 신규: 폰 PWA → 음성 "냉동기 2호기 냉수 배관" → 30초 내 도면 열람

### 시나리오 2 — 역추적 (박○○ 책임자)
- 상황: 펌프 P-101 교체 견적. 이 펌프가 영향 주는 라인 파악 필요.
- 기존: 엑셀 + 도면 폴더 전수 조사 (2시간)
- 신규: 설비 검색창 "P-101" → "나오는 도면 12개" + "연관 설비 23개" 즉시

### 시나리오 3 — 공사 후 도면 갱신 (이○○ 외주 시공)
- 상황: 냉수 배관 세분화 공사 완료. 변경 반영 필요.
- 기존: 구두 통보 + 담당자가 짬내서 수동 수정 (자주 누락)
- 신규: 수정 도면 업로드 → Phase 0 자동 차이 분석 → 책임자에게 HITL 승인 요청 → 승인 시 온톨로지 갱신·영향 부서 자동 통지

### 시나리오 4 — 자연어 검색 (최○○ 신참)
- 상황: "그 옥상 있는 냉각탑 펌프 어디 도면이지?"
- 기존: 선임에게 물어봄
- 신규: 자연어 입력 → 의미 매칭 → Rev 최신 도면 + 근처 도면 후보

### 시나리오 5 — 도면 위 지식 기록 (박○○ 책임자)
- 상황: "이 밸브는 도면과 달리 실제로는 바이패스됨." 후임에게 전달 필요.
- 기존: 엑셀 메모 or 구두
- 신규: 도면 위 핀 → 메모 + 설비 태그 → 이후 해당 설비 검색 시 자동 노출

### 시나리오 6 — Revision 비교 (정○○ 시설관리)
- 상황: Rev.2와 Rev.3 차이 파악.
- 기존: 두 파일 번갈아 봄
- 신규: diff 뷰 + 변경 영역 하이라이트 + 변경 설비 리스트

### 시나리오 7 — 외부 업체 납품 도면 처리 (윤○○ 접수)
- 상황: 업체마다 도곽·레이어 달라서 재작업 요청.
- 기존: 수동 재작성
- 신규: Phase 0 자동 정규화 + 표준 도곽으로 변환 + 표준 YAML 산출

## §4. 데이터 모델

### §4.1 주요 엔티티 (TypeScript 타입)

```typescript
interface Drawing {
  id: string;
  drawing_number: string;      // "E-SLD-001"
  title: string;
  discipline: "electrical" | "mechanical" | "fire" | "architecture" | ...;
  drawing_type: DrawingType;
  location: string;            // "B1-기계실"
  revision: string;            // "Rev.3"
  revision_date: string;
  revision_note?: string;
  uploader: string;
  uploaded_at: string;
  file_paths: {
    pdf: string;
    png_by_page: string[];     // 페이지별 이미지
    dwg?: string;              // 원본 DWG
  };
  page_count: number;
  ontology_refs: {             // Phase 0 산출 연결
    entities: string[];        // 엔티티 ID 목록
    relations: string[];       // 관계 ID 목록
  };
  source_extracted_at?: string;   // 마지막 Phase 0 분석 시각
  status: "draft" | "under_review" | "approved" | "archived";
}

interface Entity {
  id: string;
  tag: string;                 // "P-101"
  type: string;                // "centrifugal_pump"
  discipline: string;
  location?: string;
  properties: Record<string, any>;
  source_drawings: string[];   // 역추적 인덱스
}

interface Annotation {
  id: string;
  drawing_id: string;
  page: number;
  x: number;                   // 정규화 0~1
  y: number;
  text: string;
  type: "info" | "warning" | "field-note";
  author: string;
  created_at: string;
  linked_entities: string[];   // 설비 태그 연결
}

interface ChangeEvent {
  id: string;
  drawing_id: string;
  event_type: "revision" | "new" | "deprecated";
  before_rev?: string;
  after_rev: string;
  diff: {
    added_entities: string[];
    removed_entities: string[];
    modified_entities: string[];
  };
  approved_by?: string;
  approved_at?: string;
  notified_departments: string[];
}
```

### §4.2 인덱스
- `drawingNumberIdx`: drawing_number → Drawing
- `tagToDrawingsIdx`: entity.tag → Drawing[]
- `disciplineIdx`: discipline → Drawing[]
- `fuseSearchIdx`: 자연어 검색 (제목·메타·태그 통합)

### §4.3 샘플 데이터 소스
- 청주 Phase 0 v2.5 산출물 사용
  - `projects/청주/phase0_v2_0/ontology_readiness/` 결과
  - 283 엔티티, 10개 관계
- JSON으로 프로토타입 초기 부트스트랩

## §5. UI 구조

### §5.1 화면 목록
1. 홈 — 최근 검색·최근 열람·즐겨찾기
2. 검색 — 자연어 + 필터바 + 결과 리스트
3. 도면 상세 — 뷰어 + 메타패널 + 설비 사이드바
4. 설비 상세 — 속성 + 관련 도면 리스트 + 히스토리
5. 업로드 — 드래그앤드롭 + 메타 입력 + Phase 0 진행 상태
6. 변경 영향도 — 그래프 시각화 + 영향 설비·부서 리스트
7. 주석 편집 — 모달 팝오버
8. Revision 비교 — 좌우 분할 + 동기 줌·팬
9. 활동 이력 — 타임라인
10. 설정 — 권한·알림·프로필

### §5.2 주요 컴포넌트

### §5.3 모바일·PWA 구성
- 브레이크포인트 `md` (768px) 미만 시 탭 뷰
- Service Worker: 도면 이미지 cache-first, 메타 network-first
- manifest.json: 홈 화면 추가 가능
- 터치 제스처: 핀치 줌, 스와이프 페이지

### §6.1 REST 엔드포인트

| 메서드 | 경로 | 용도 |
|:---:|------|------|
| GET | `/api/drawings` | 목록 + 필터 |

## §8. MVP 범위 — 4주 프로토타입

### §8.1 포함 기능 (S등급 + 최소 A)
- F-001 자연어 검색 (청주 Phase 0 산출물 기반)
- F-002 설비↔도면 양방향
- F-003 PDF 뷰어 (iframe 또는 react-pdf)
- F-004 메타패널
- F-005 모바일 반응형 (PWA는 후순위)
- F-006 단순 업로드 (Phase 0 실행은 수동 트리거)
- F-009 주석 (기본 핀+텍스트만)

### §8.2 제외 기능 (4주 내)
- Revision diff 시각화
- 공사 이벤트 자동 분석 파이프라인 자동화
- 외부 업체 표준 변환 채널
- 활동 로그·감사
- 권한 관리

### §8.3 주간 계획

| 주차 | 작업 |
|:---:|------|
| 1 | 프로젝트 세팅 + 청주 샘플 데이터 로딩 + 검색·리스트 UI |
| 2 | PDF 뷰어 + 메타패널 + 양방향 네비게이션 |
| 3 | 모바일 레이아웃 + 주석 기본 + 업로드 UI |
| 4 | 통합 테스트 + 샘플 시나리오 시연 스크립트 + 피드백 수집 |

### §8.4 성공 기준
- 청주 샘플 데이터로 시나리오 1·2·4·5 시연 가능
- 모바일에서 1·4 가능
- 비개발자(영업·운영자)가 매뉴얼 없이 5분 안에 시나리오 1 완료

## §10. 인터뷰 질문지 (프로토타입 데모 직후)

### §10.1 경험 평가 (리커트 5점)
1. 기존 방식 대비 **얼마나 빠르게** 도면을 찾았나

### §14.8 모바일 분기 설계 (F-005)

**브레이크포인트**:
- `≥ 1024px` (lg): 3열 고정 (현재와 동일)
- `768~1023px` (md): 좌 aside를 drawer 버튼으로 · 중앙+우측만 병렬
- `< 768px` (sm): **탭 뷰** [리스트 | 뷰어 | 사이드]

**탭 전환 규약**:
```
리스트 탭 항목 클릭 → 뷰어 탭 자동 활성화
뷰어에서 역추적/주석 필요 → 사이드 탭으로 수동 전환
사이드 탭 항목 클릭 → 뷰어 탭으로 복귀 + page override
```

**PWA 구성**:
- `manifest.json` 신규 (홈 화면 추가)
- Service Worker: 도면 이미지 `cache-first` · 메타 `network-first`
- `public/pdf.worker.min.mjs`는 현 서빙 유지 ([[../docs-시스템분석/docs/03-viewer|03 §9]])

**터치 제스처**:
- 핀치 줌: `react-zoom-pan-pinch` 기본 지원 활용
- 스와이프: 탭 전환용 (좌↔우)

### §14.9 서비스 2 연결점 — evidence 링크 수신 URL 규약

```
/drawings/{doc_id}?page={n}&highlight={tag}&from=insight&query_id={id}
```

| 파라미터 | 용도 | 처리 |
|----------|------|------|
| `doc_id` | 필수 | `selectedId` 세팅 |
| `page` | 선택 | `selectedPage` 세팅 |
| `highlight` | 선택 | 뷰어 오버레이 `renderOverlay`에 태그 강조 전달 |
| `from=insight` | 선택 | 헤더에 "← 인사이트로 돌아가기" 브레드크럼 노출 |
| `query_id` | 선택 | 돌아가기 시 원 질의로 복귀 |

→ [[../docs-시스템분석/docs/03-viewer|03 §8 renderOverlay 확장]] 슬롯에 "검색어 박스" 대신 "엔티티 태그 하이라이트" 얹는 형태.

### §14.10 시연 스크립트 — 3단 버전

#### 5분 (핵심 가치 확인)
1. 홈 진입 → "냉동기 냉수 배관" 검색 (10초)
2. 결과 리스트에서 첫 도면 탭 → PDF 뷰어 열림 (5초)
3. 우측 역추적 탭에서 `CH-001` 칩 클릭 → 관련 도면 2건 즉시 표시 (5초)
4. 각 도면 클릭 → 해당 페이지 점프 (10초)
5. 도면 위 "여기 밸브 바이패스" 핀 주석 (30초)
6. **포인트**: NAS+엑셀 대비 시간 압축 체감

#### 10분 (+시나리오 4 자연어 + 5 주석)
- 5분 시나리오 + 한글 자연어 "옥상 냉각탑" 검색
- 핀 주석 3개 찍고 AnnotationList 탭에서 검색·삭제·수정

#### 20분 (+시나리오 1 모바일 + 2 역추적 응용)
- 폰 뷰포트(375px) 에뮬레이션 → 탭 뷰 작동 확인
- `VCB-001` 역추적 → 단선도 도면 p.3 점프 → 주석 확인
- 서비스 2 화면에서 evidence 링크 클릭 → 서비스 1 도면 자동 이동 시연 (§15.5)
- **포인트**: 두 서비스 하나의 쉘로 자연 흐름
