# XD-AI 플랫폼 SaaS 기능 확정 지표: 추상성을 배제한 '결론 도출 근거 (Criteria for Decision)'

기존의 "생산성이 향상된다, 편의성이 좋다"와 같은 정성적이고 추상적인 비전을 배제해야 합니다. 
우리가 특정 SaaS 기능을 도입하거나 다음 타겟으로 결정하기 위해서는 **'실제 수치(Data)'와 '기술적 제약사항'에 기반한 냉정한 결론 도출 근거**가 있어야 합니다. 

어떤 워크플로우를 먼저 SaaS화 할 것인지, 이 기능이 시장에서 통할 것인지 '정확한 결론'을 내리기 위한 정량적/기술적 판단 근거 프레임워크를 수립합니다.

---

## 1. 기술적 실현 가능성 (Technical Feasibility) 판단 근거
"AI가 교차 검증을 해준다"는 환상을 버리고, 현재 파이프라인의 **오차율**을 근거로 제공할 수 있는 서비스의 한계를 결정해야 합니다.

### 평가 지표 (Go/No-go 결정 기준)
*   **Vision Data 매핑 일치율 (Entity Grounding Accuracy):**
    *   [근거 수치]: 도면상의 `UNKNOWN-GRID-H-01` 같은 객체를 Bounding Box로 온전히 인식하고, 텍스트 스키마로 100% 매핑에 성공하는 비율.
    *   **판단 기준:** 일치율이 **95% 미만**이라면, 자동 B.O.M(견적/수량 도출) 솔루션으로 가는 결론은 **기각**해야 합니다(신뢰도 1개 누락=신뢰성 붕괴). 반면 이 수치가 **80% 수준**이라면 완벽함을 요구하지 않는 '탐색 보조(Navigation)' 기능으로 타겟 결론을 선회해야 합니다.
*   **Multi-hop 지식그래프(TypeDB) 쿼리 레이턴시:**
    *   [근거 수치]: "이 밸브와 연결된 평면도, 시방서를 모두 가져와"라는 요청 시나리오에서 DB를 순회하여 UI를 렌더링하기 전까지 걸리는 총 소요 응답 시간(TTFB).
    *   **판단 기준:** 워크플로우 응답 지연이 **3초 초과** 시, 실시간 대화형 챗봇(Agentic Chat) UI 도입은 사용자 피로도로 인해 **기각**해야 합니다. 이 경우 백그라운드에서 동작 후 알림을 주는 '비동기 리포트 생성기(Asynchronous Report Generator)' UI 형태로 결론을 지어야 합니다.

## 2. 경제성 및 비즈니스 (Unit Economics & ROI) 판단 근거
서비스 구축에 드는 컴퓨팅 비용과 절감 비용의 명확한 수치 비교가 되어야 합니다. 

### 평가 지표
*   **도면 1장 당 지식 추출 단가 (Cost Per Page Analysis):**
    *   [근거 수치]: 도면/문서를 토큰화하고 클라우드 AI (Gemini 등) 혹은 1차 에이전트 파이프라인을 통과시켜 온톨로지로 변환하는 데 드는 API 토큰 비용 및 리소스 비용 계산.
    *   **판단 기준:** (절감된 엔지니어 시간 * 엔지니어 시급) $>$ (도면 분석 토큰 비용 * 10배 마진). 이 식을 충족하지 못하면 워크플로우를 단순화하거나 로컬 SLM(소형언어모델) 병행 도입을 근거로 삼아야 합니다.
*   **환각(Hallucination) 수정 소요 시간 (Human-in-the-loop Penalty):**
    *   [근거 수치]: AI가 잘못된 도면 좌표를 출처로 제공했을 때, 엔지니어가 이를 발견하고 교정(수동 태그)하는 데 걸리는 시간.
    *   **판단 기준:** AI 검도 오류를 사람이 고치는 시간이 애초에 사람이 문서를 찾는 시간보다 길어진다면, 해당 워크플로우 SaaS 개발은 즉시 중단해야 합니다.

## 4. 제언: 정확한 판단을 위해 지금 수행해야 할 '하드코어 테스트(Ground Truth Test)'
지금까지의 '목업 시각화(design-lab, insight)' 성과를 사업적 결론으로 전환하려면 아래의 숫자를 뽑아내야 합니다.

**[행동 강령 - Action Item]**
1. **건축/토목 도면 샘플 50페이지 추출 (Ground Truth Set 구축)**
2. 사람이 직접 이 50장 안에 있는 설비/부품(Entity)의 총개수, 범례와의 매핑 관계 정답지(Excel)를 생성.
3. 현재 개발된 **pre_analysis_agentic_v2.0 파이프라인**을 50장에 돌려서 결과(YAML) 도출.
4. **결과 비교:** 
   - `Recall(재현율) = AI가 찾은 정답 수 / 사람이 찾은 전체 정답 수`
   - `Precision(정밀도) = AI가 정답이라고 한 것 중, 실제 정답인 것의 비율`

이 수치가 **Precision 90%, Recall 85% 이상**이라는 **[근거 데이터]**가 나와야만, 비로소 외부 투자자나 사내 고객(사우타, 전기 등)에게 "우리의 워크플로우로 당신들의 시간을 줄일 수 있다"고 확정적으로 결론짓고 서비스를 전개할 수 있습니다. 
이 데이터 검증 이전의 UI 고도화는 허수에 불과할 수 있습니다.
