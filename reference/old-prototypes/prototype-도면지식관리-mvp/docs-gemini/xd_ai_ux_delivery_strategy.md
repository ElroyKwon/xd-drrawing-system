# XD-AI 플랫폼 UX 전달 전략: DWG/PDF와 챗봇의 결합

경제성(비용) 관점을 완전히 배제하고, 오직 **B2B 시장에서의 압도적인 'UX 성능과 체감 가치(편의성)'**라는 본질적인 핵심에만 집중하여 "어떻게 보여줄 것인가?"에 대한 해답을 정리했습니다.

---

## 1. 참고 사이트들은 생산성 결과를 '어떻게' 제공하는가? (UX 분석)

v0, Genspark, Perplexity, Manus 등 차세대 AI SaaS들이 기존 ChatGPT와 구분되는 결정적 차이는 **"대답을 텍스트로 끝내지 않는다"**는 점입니다. 그들은 사용자의 시간(생산성)을 아껴주기 위해 가장 최적화된 형태의 '기능적 결과물(Artifact/Component)'을 뱉어냅니다.

| 참고 서비스 | 생산성 결과물 제공 방식 (UX 핵심) | XD-AI 플랫폼(도면) 적용점의 힌트 |
| :--- | :--- | :--- |
| **v0.app** | **Generative UI (코드 $\rightarrow$ 렌더링 화면)**<br>말을 하면 즉시 상호작용 가능한 화면(버튼, 표)을 그려줍니다. | 사용자가 펌프 스펙을 물어보면, 텍스트가 아니라 **'설비 스펙 카드(위젯)'** 형태로 툭 던져주어야 합니다. |
| **Perplexity** | **투명한 인라인 위젯 (Citation Chips)**<br>답변 중간의 `[1]` 번호를 누르면 새 탭으로 가지 않고, 출처 웹페이지의 중요 문단이 미니 창으로 팝업됩니다. | 챗봇 답변에 명시된 `[도면 A]`를 마우스 단위로 올리면, 뷰어 전체가 이동하지 않고 우측 상단에 **'해당 도면의 잘린 조각(Snippet)'**만 팝업으로 보여줍니다. |
| **Genspark/Claw** | **다중 탭 융합 (Multi-hop Synthesis)**<br>주제를 던지면 10개 사이트를 읽고, 목차를 갖춘 '위키 백과' 스타일의 완성된 보고서 페이지를 생성해 줍니다. | “PUMP-01에 대해 알려줘”라는 질문에, AI가 배관도, 전기도, 시방서를 수집하여 **[PUMP-01 종합 위키 페이지]**라는 완성된 문서를 결과물로 던져닙니다. |
| **Manus** | **작업 도구의 시각화 (Tools in Action)**<br>에이전트가 무슨 브라우저를 열었고 코드를 어떻게 짜는지 오른쪽 창에서 실시간 과정을 중계합니다. | "도면 간섭 체크해줘"라고 할 때 결과만 덜렁 주지 않고, **"1층 건축도면 로드 중 $\rightarrow$ 배관 그리드 매핑 중"**이라는 스피너와 과정을 시각화하여 지루함을 없애고 신뢰를 줍니다. |

## 2. 도면(DWG, PDF)을 사용자에게 어떻게 제공할 것인가? (UX 접근법)

기존 건설/엔지니어링 솔루션의 가장 큰 패착은 "파일을 다운로드하거나, 거대한 전용 뷰어 창을 띄운다"는 것입니다. 우리는 도면을 **'파일(File)'이 아니라 구글 공간 '지도(Map)'**처럼 접근해야 합니다.

### 핵심 채택 UX: "Split-View Spatial Canvas (분할 지도 캔버스)"
*   **좌표와 타일 기반 렌더링:** DWG나 PDF를 통째로 로딩하는 UI는 무겁고 답답합니다. 구글 맵스처럼 도면을 웹 기반 캔버스 화면 배경에 끊김 없이 깔아두어야 합니다(벡터 기반 혹은 고해상도 타일링).
*   **파일 제약의 파괴:** 사용자는 "a.dwg 파일을 연다"고 느끼지 않아야 합니다. "지하 1층"을 선택하면 지하 1층 캔버스가 깔리는 개념입니다.
*   **상호작용성(Interactive Layer):** 캔버스 바로 위에 우리가 TypeDB로 매핑한 `Bounding Box`들이 보이지 않는 레이어(Clickable Poly-lines)로 깔려 있습니다. 단순히 그림을 보는 게 아니라 객체를 클릭할 수 있는 공간 뷰어입니다.

## 3. 대화창(Prompt)을 통해 어떤 결과물을 제공할 것인가?

우리의 챗봇(우측 패널)은 사용자와 대화하는 단순한 비서가 아니라, **'생산성 도구(데이터 위젯)를 소환하는 콘솔'** 창입니다. 텍스트로만 답하지 않고 아래 3가지 형태의 강력한 산출물을 채팅창 안에 렌더링해야 합니다.

### ① 네비게이션 액션 칩 (Actionable Navigation Chips)
*   **입력:** "메인 배전반 위치 어디야?"
*   **출력 (챗봇 결과):** "메인 배전반(E-PANEL-01)은 [지하 1층 전기실(X:120, Y:430)]에 위치해 있습니다."
*   **UX 가치:** 사용자가 채팅창의 **`[지하 1층 전기실...]` 버튼 칩을 클릭하는 순간, 좌측의 거대한 도면 캔버스가 해당 좌표로 스무스하게 줌인(Pan & Zoom)**되며 해당 영역이 1초간 번쩍입니다. (직관성의 극대화)

### ② 동적 데이터 위젯 (Generative Data Tables/Cards)
*   **입력:** "냉각탑(CT-01)이랑 연결된 밸브들 리스트업 해줘."
*   **출력 (챗봇 결과):** (텍스트 대신 표 UI가 즉시 생성됨)
    *   [CT-01 연결 밸브 리스트] (엑셀형 UI 위젯)
        *   `V-001` (상태: ON) | [평면도 보기] [P&ID 보기]
        *   `V-002` (상태: OFF) | [평면도 보기] [P&ID 보기]
*   **UX 가치:** 사용자가 결과를 눈으로만 읽는 게 아니라, 채팅창 안에서 표(Table) 컴포넌트를 조작하거나 CSV로 즉시 다운로드할 수 있는 업무용 산출물을 줍니다. (v0의 Generative UI 방식)

### ③ 종합 인사이트 보드 (Synthesis Dashboard)
*   **입력:** "어제 납품된 2층 도면, 이전 리비전(v1)이랑 스펙 달라진 거 요약해 줘."
*   **출력 (챗봇 결과):** 채팅창이 아니라, 중앙 팝업이나 화면 절반을 덮는 형태의 완성된 **[리비전 비교 보고서]** 아티팩트를 소환합니다. 
*   **UX 가치:** 단순 Q&A 수준을 넘어, 사람 1명이 하루 종일 엑셀에 정리해야 할 결과물을 완성된 '보고서 양식'으로 띄워줍니다. (Genspark Claw 방식)

## 4. 최종 결론: 우리가 만들어야 할 UX의 완성형

결국 사용자가 느끼는 "가치"는 **내 눈이 찾아야 할 것을 AI가 대신 가리켜 주고(Highlight/Pan), 내 손이 쳐야 할 문서를 AI가 다 모아 표로 요약해 주었을 때(Widget)** 폭발합니다.

챗봇은 거들 뿐입니다.
**'도면(공간/지도) 중심의 메인 뷰어' $\leftrightarrow$ 그 위로 날아다니는 '챗봇의 동적 결과물 위젯(Generative UI)'**의 강력한 상호 연동(연결 고리)이야말로 우리 소프트웨어가 보여주어야 할 UX의 차별점입니다.
