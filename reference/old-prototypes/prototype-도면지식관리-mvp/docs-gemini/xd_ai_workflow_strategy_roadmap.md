# XD-AI 플랫폼: 도메인 특화 워크플로우 중심 SaaS 발전 전략

## 1. 선도 AI 소프트웨어(Genspark Workflows 등)의 핵심 장점 분석

Genspark Workflows나 Manus.im과 같은 최신 AI 트렌드의 핵심은 단순한 **'단답형 질의응답(1-hop QA)'**에서 벗어나, AI가 인간을 대신해 연속적이고 복잡한 과제를 수행하는 **'자율형 워크플로우(Autonomous Workflow)'**로 진화했다는 점입니다.

**핵심 장점:**
1.  **다단계 추론 및 태스크 분할 (Multi-step Reasoning):** 사용자 요청을 하위 태스크로 나누어, 문서를 찾고, 내용을 읽고, 교차 검증하는 과정을 스스로 수행.
2.  **투명한 과정 시각화 (Process Transparency):** AI가 "무엇을 찾고 있는지", "어떤 문서를 읽었는지" 실시간으로 보여주어 블랙박스 문제를 해결하고 사용자 신뢰 확보.
3.  **구조화된 최종 산출물 (Structured Synthesis):** 단순 텍스트가 아닌, 마크다운 보고서, 위키, 테이블 등 사용자가 즉시 업무에 활용할 수 있는 형태의 종합 산출물(Artifact)을 제공.

## 2. 우리 플랫폼이 집중해야 할 '워크플로우(Workflow)'와 제공 서비스

우리의 XD-AI (Data Knowledge Studio) 플랫폼은 웹상의 일반 정보가 아닌 **'도면, 구조 계산서, 시방서, 설비 정보'**라는 매우 전문적이고 파편화된 도메인 데이터를 다룹니다. 따라서 우리의 워크플로우는 철저히 **'엔지니어링 데이터의 정합성 검증과 지식그래프 탐색'**에 집중해야 합니다.

### 집중해야 할 핵심 워크플로우 스키마 3가지

1.  **교차 검증 워크플로우 (Cross-Validation Workflow)**
    *   **동작:** 사용자가 "지하 1층 공조설비 스펙과 평면도 위치가 일치하는지 확인해 줘"라고 요청.
    *   **과정:** `시방서 데이터 추출(Text)` $\rightarrow$ `평면도 범례 및 심볼 확인(Vision)` $\rightarrow$ `지식그래프(TypeDB) 매핑` $\rightarrow$ `불일치 항목 도출`.
2.  **설비 이력 및 관계 추적 (Entity Relationship Tracing)**
    *   **동작:** 특정 펌프(`PUMP-01`)에 문제가 생겼을 때의 대응.
    *   **과정:** `PUMP-01` 검색 $\rightarrow$ 연결된 배관/전기 라인 추적 $\rightarrow$ 관련 매뉴얼 및 과거 유지보수 기록 수집 $\rightarrow$ `종합 대응 인사이트 보고서` 발행.
3.  **물량 파악 및 요약 리포트 (BOM & Summary Workflow)**
    *   **동작:** 여러 장의 도면에 흩어진 특정 자재의 수량 도출.
    *   **과정:** 모든 대상 도면 순회 분석 $\rightarrow$ 테이블 형태로 물량 추출 $\rightarrow$ 중복 및 누락 객체(Unknown-Grid 등) 판단 $\rightarrow$ 명세서 자동 생성.

## 3. 마일스톤 및 목표 설정의 근거 (Criteria for Judgment)

마일스톤을 세우고 개발 목표를 잡을 때, **"우리가 제대로 가고 있는가?"**를 판단하기 위한 정량/정성적 근거(지수)가 필요합니다. 

**판단 근거 (핵심 KPI):**
1.  **엔지니어 개입 절감 시간 (Time-saved Index):** 사람이 수동으로 도면 수십 장을 펼쳐놓고 엑셀로 비교할 때 걸리는 시간 대비, AI 워크플로우가 산출물을 내는데 걸리는 시간의 단축률.
2.  **지식그래프 무결성 점수 (Graph Integrity Score):** `Unknown` 엔티티나 환각(Hallucination) 없이 도면과 텍스트 정보가 올바른 관계(Relation)로 매핑된 비율(검도율).
3.  **사용자 행동 전환율:** 사용자가 제공된 인사이트나 citation(출처) 링크를 클릭하여 원본 도면으로 얼마나 자주 진입하고 활용하는지(UX 효용 증명).

## 4. 로드맵 및 단계별 마일스톤 (Milestone)

| 단계 | 마일스톤 (Milestone) | 핵심 산출물 및 서비스 형태 | 판단 근거 (목표 달성 기준) |
| :--- | :--- | :--- | :--- |
| **Phase 1** | **지식 쿼리 & Citation UI 정립 (현재 ~ 단기)** | - 1:1 문서/도면 대상 질의응답<br>- 답변에 명시적 출처(Bounding Box, 페이지 링크) 표기 UI 구축 | - 환각(Hallucination) 발생률 5% 미만<br>- 출처 링크 클릭시 뷰어 이동의 정확도 |
| **Phase 2** | **프리셋 워크플로우 (Pre-set Workflow) 투입** | - "도면 교차 검증", "범례 추출 정리" 등 버튼 하나로 실행되는 정형화된 다단계 에이전트 작업 파이프라인 구축 | - 표준 도면 50장 기준, 교차 검증에 소요되는 시간을 인간 1인 일과(8H) 대비 1/10 로 단축 |
| **Phase 3** | **자율 지식 탐색 워크플로우 (Agentic Knowledge Workflow)** | - Genspark Workflows와 유사한 동적 채팅 형태.<br>- 사용자의 모호한 지시("1층 전기 통신 도면들 이상 없는지 훑어봐")를 AI가 스스로 분석 계획(Plan)을 세우고 실행 후 종합 보고서 제출. | - AI가 제안한 발견(Insight) 중 엔지니어가 '유효함(Useful)'으로 채택한(HITL 적용) 비율 80% 이상 |
| **Phase 4** | **예측 기반 SaaS 프로액티브 알림** | - 사용자가 묻기 전에 플랫폼이 백그라운드에서 주기적으로 도면 개정본을 비교하고 간섭/위험 요소를 대시보드에 Alert 발생. | - 알림 기능을 통한 사전 도면 오류 적발 건수 및 프로젝트 리스크 비용 감소 추정치 |
