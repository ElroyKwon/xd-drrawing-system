# DKS MVP 기능 워크스루 (프레젠테이션 기록)

> 작성일: 2026-04-20  
> 목적: Chrome DevTools MCP를 통한 실시간 시연 기록 — 각 기능의 동작 방식과 UI 포인트 정리  
> 실행 환경: `npm run dev` 후 `http://localhost:3000`

## 1단계 · 홈 — 도면 목록 전체 보기

**URL**: `http://localhost:3000/`

### 화면 구성 (3분할 레이아웃)
| 영역 | 내용 |
|---|---|
| 좌측 사이드바 | 네비게이션 (홈·검색·인사이트·알람·보고서·설정) |
| 중앙 패널 | 도면 카드 목록 + 카테고리/타입 필터 |
| 우측 패널 | 설비 역추적 탭 + 주석 탭 |

### 핵심 포인트
- 전체 도면 26건 카드 목록
- 각 카드: 제목, 카테고리 배지(전기/기계/소방/시설), 스니펫, 문서타입·페이지·날짜
- 우측 패널에 설비 태그 50건이 칩(chip) 형태로 미리 표시

---

## 2단계 · 필터 — 카테고리 및 문서 타입 복합 필터

### 카테고리 필터
- **전체 / 전기 / 기계 / 소방 / 시설** 5개 버튼
- 활성 버튼 **노란색** 강조
- 예: "전기" 선택 → 전기 문서만 표시

### 문서 타입 필터
- **모든 종류 / checklist / code / drawing / manual / reference / spec** 7개 버튼
- 카테고리 필터와 **동시 교차 적용** — "전기" + "drawing" → 전기 DRAWING만 필터링

## 3단계 · 검색 — 홈 퍼지 검색창

### 동작 방식
- **Fuse.js 퍼지 검색** — 제목+내용 스니펫 동시 검색
- 입력 즉시 실시간 필터링, 결과 카운트 `2 / 26 건`

### 시연: "냉동기" 검색
| 결과 | 타입 | 매칭 이유 |
|---|---|---|
| 냉동기 운전 매뉴얼 (Carrier 30XW) | 기계 · MANUAL | 제목 포함 |
| 데이터센터 냉방 계통도 | 기계 · DRAWING | 스니펫 "냉동기 2대" 포함 |

---

## 3-B단계 · 검색 전용 페이지 (`/search`)

### 홈 검색창과의 차이

| 구분 | 홈 검색창 | 검색 전용 페이지 |
|---|---|---|
| URL 상태 | 비URL (공유 불가) | `/search?q=VCB` (공유 가능) |
| 레이아웃 | 좁은 사이드 패널 | **전체 너비 풀스크린** |
| 카드 클릭 | 중앙 뷰어로 열기 | `/drawings/{id}` 상세 이동 |
| 용도 | 열람 중심 | 탐색·비교 중심 |

### 시연: `/search?q=VCB` → 3 / 26 건
- BESS 전기 단선도, 22.9kV 수전설비, 6.6kV 변전설비

> 주의: 검색창에 직접 입력은 React 이벤트 때문에 URL 파라미터로 직접 접근 필요

## 4단계 · PDF 뷰어 — 인라인 도면 열람

### 진입: 좌측 목록 카드 클릭 → 중앙 패널이 뷰어로 전환

### 뷰어 구성
| 요소 | 설명 |
|---|---|
| 상단 제목바 | 문서명 · 타입 · `p.시작페이지` · 작성일 |
| 페이지 네비게이션 | `‹ 12 / 53 ›` 이전/다음 버튼 |
| 원본 PDF 버튼 | 실제 PDF 파일 직접 링크 |
| 줌 컨트롤 | `−` / `+` / 리셋 |
| 뷰어 본체 | react-pdf 렌더링, zoom-pan-pinch 드래그 패닝 |

### 특이사항
- `documents.json`의 `page` 필드로 시작 페이지 자동 점프
- 실 PDF 없을 경우 PNG fallback 자동 적용

### 시연: 냉동기 운전 매뉴얼 → p.12 "일반상세도-3" (M14-004)

## 5단계 · 주석 핀 — 도면 위 위치 기반 메모

### 진입: 뷰어 도면 위 임의 위치 클릭 → 팝오버 등장

### 팝오버 필드
| 필드 | 내용 |
|---|---|
| 제목 | `주석 추가 · p.12 (35.0%, 44.9%)` — 클릭 위치 정규화 좌표 |
| 종류 | **정보**(파랑) / **주의**(노랑) / **현장메모**(초록) |
| 메모 | 자유 텍스트 (필수 — 비어있으면 저장 버튼 disabled) |
| 작성자 | 기본값 "홍길동" |
| 태그 | 쉼표 구분 |

### 저장 결과
- 도면 위 컬러 원형 핀 표시
- 우측 탭 카운트 증가: `주석 (1)` → `주석 (2)`
- `localStorage mvp-annotations-v1` 저장 — 새로고침 유지

## 6단계 · 설비 역추적 — 태그로 관련 도면 역탐색

### 위치: 우측 패널 "설비 역추적" 탭

### 동작
- 입력창에 설비 태그 입력 (예: `CH-001`) → 즉시 매칭 도면 목록
- 또는 태그 칩(50건) 클릭으로 바로 선택
- 결과 클릭 → 해당 도면·페이지 점프

### 시연: CH-001 → 2건 매칭
| 결과 | 타입 | 페이지 |
|---|---|---|
| 냉동기 운전 매뉴얼 | manual · MECHANICAL | p.12 |
| 데이터센터 냉방 계통도 | drawing · MECHANICAL | p.1 |

## 7단계 · 주석 목록 탭 — 전체 주석 한눈에 보기

### 위치: 우측 패널 "주석 (N)" 탭

### 구성
- 상단: 메모·작성자·태그·문서명 통합 검색창
- 본문: 주석을 **문서별 그룹**으로 묶어 표시
- 하단: `N / N 건` 카운트

### 주석 카드
| 항목 | 내용 |
|---|---|
| 색상 점 | 파랑=정보, 노랑=주의, 초록=현장메모 |
| 메모 내용 | 입력 텍스트 |
| 메타 | 종류 · 페이지 · 작성자 |

### 핵심 가치
도면을 하나씩 열지 않고 **전체 현장 메모를 한 화면에서 조회**

## 8단계 · Insight Lab (`/insight/lab`) — 통계 × 온톨로지 AI 분석

> 주의: `/insight`는 404. 진입점은 `/insight/lab`

### 화면 구성 (3분할)
| 영역 | 내용 |
|---|---|
| 좌측 | 통계 이벤트 17건 (CRITICAL/WARNING/NORMAL × 통계/열/무결성) |
| 중앙 | **Bundle** — 통계+온톨로지+도면 3레이어 결합 |
| 우측 | **Bundle Chat** — Bundle 컨텍스트 AI 대화 |

### Bundle 구성
1. **통계 이벤트**: 대상·메트릭·측정값·임계·해석
2. **온톨로지 엔티티**: 설비 태그·타입·스펙 + **관계 확장 최대 2홉**
3. **근거 도면**: 관련 도면 링크 + 미리보기

### 관계 확장 예시 (SHV 수전 VCB 패널 1101)
- 1-HOP: BLK-MOF ← feeds ← 1101
- 1-HOP: 1101 → connects_to → 1102
- 1-HOP: 1101 → feeds → TR-001
- 2-HOP: 1101 → feeds → HV-VCB-TR1 (via TR-001)

### 한계 시연 케이스 (목록 하단 3건)
- BESS 배터리 랙 RACK_05 / 인버터 INVERTER-02 / BMS CH_01
→ 온톨로지 미포함 → 엔티티 매칭 실패 — 솔직하게 노출하는 설계

## 9단계 · 알람 페이지 (`/insight/alarms`)

### 화면 구성
- 상단: `N건 위급 · N건 경고 · 총 N건` 요약
- 본문: 알람 카드 목록 (최신순)

### 알람 카드
- 심각도 색상 점 + 배지 + 제목 + 설비태그·알람코드·발생시각

### 알람 상세 드로어
카드 클릭 → 우측 슬라이드 패널:
- 알람코드, 현재값 vs 임계값, 출처(SCADA), 발생시각, 설명
- **"AI 물어보기"** 버튼 → 알람 AI 분석 페이지 이동

### 시연: VCB-001 과전압
- 현재값 25.1 kV / 임계값 24 kV / 출처 SCADA

## 9-B단계 · 알람 AI 분석 페이지 (`/insight/answers/{id}`)

### 레이아웃: 좌(58%) + 우(42%) 2분할

| 영역 | 내용 |
|---|---|
| 좌측 | AI 1차 분석 결과 (스크롤, 고정) |
| 우측 | **멀티턴 챗 패널** (대화 히스토리 누적) |

### 좌측 AI 1차 분석 구성
- 알람 상태 분석 테이블 (현재값·초과율·심각도)
- 권장 조치 순서 (단계별)
- 가능한 원인 테이블
- **근거 도면·문서** + "도면 열기 →" 링크 (관련도 % 표시)
- **구조화 분석** (AI 신뢰도 % + 권고 조치사항 + 관련 설비 태그)
- **후속 질문 제안** 3개 버튼

### 우측 챗 패널
- 후속 질문 버튼 클릭 → **페이지 이동 없이** 우측 챗으로 전송
- 대화 히스토리 `messages[]`가 API 호출 시 전체 전송 → 멀티턴 유지
- 직접 입력도 가능 (Enter 전송, Shift+Enter 줄바꿈)

### API
- 초기 분석: `POST /api/insights/ask/stream` (alarmId)
- 후속 대화: `POST /api/insights/chat/stream` (alarmId + question + history)

## 10단계 · 보고서 페이지 (`/insight/reports`)

### 목록 화면
- **일일 보고서** 목록 — 상태 배지 + Rev.번호 + 날짜 + 섹션 수
- 상태: `초안`(회색) / `검토됨`(노랑) / `확정`(초록)

### 상세 페이지 레이아웃: 좌(62%) + 우(38%) 2분할

| 영역 | 내용 |
|---|---|
| 좌측 | 보고서 섹션 목록 (수치+AI해석 2열) |
| 우측 | **AI 대화 패널** (보고서 전체 컨텍스트 주입) |

### 섹션 구조 (각 섹션)
| 좌 | 우 |
|---|---|
| `python_data` 자동 계산 수치 (read-only) + 재계산 버튼 | `slm_narrative` AI 해석 (편집 가능, 확정 시 readOnly) |

### 워크플로
```
초안(draft) → [담당자 편집] 검토됨(reviewed) → [확정 버튼] 확정(finalized)
```
- 확정 시 Rev.번호 +1, 텍스트 잠금, "확정" 버튼 사라짐

### AI 대화 (우측 챗)
- 보고서 전 섹션(수치+해석)이 `contextSummary`로 주입
- 예: "코어 스위치 CPU 71% 위험 및 선제 조치" → 보고서 수치 직접 인용해 답변
- `POST /api/insights/chat/stream` (contextSummary 포함)

### 보고서 4종
| 날짜 | 분야 | 상태 |
|---|---|---|
| 4/20 | 전체 알람·KPI·설비건전도·익일주의 | 확정 Rev.2 |
| 4/19 | 소방 설비 정기점검 (스프링클러·배관·유도등) | 확정 Rev.2 |
| 4/18 | 기계 냉방 계통 운전 (냉동기·펌프·AHU·에너지) | 검토됨 Rev.1 |
| 4/17 | 전기 수전설비 + 정보통신 설비 | 초안 Rev.1 |

### 상태 저장
- `localStorage dks-reports-v1` — 새로고침 유지
- 초기 JSON에 없는 신규 보고서도 병합 로드
