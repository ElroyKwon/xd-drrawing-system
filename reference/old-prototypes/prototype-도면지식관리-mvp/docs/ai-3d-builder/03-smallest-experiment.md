# 03. 최소 실험 — 현실 데이터 확보

> **목적**: 02의 아키텍처 선택 전에 "Claude Vision이 내 도면을 얼마나 읽는가"를 **3시간 안에** 실측.
> **성격**: 코드 없는 실험. Claude Console 또는 간단한 스크립트 1개로 실행.

---

## 왜 실험이 먼저인가

01 §1.1의 Claude 40%는 **기계 도면** 기준이다. **한국 건축 도면**에서 이 숫자가 유지되는지 확인 안 하고 아키텍처 선택을 못 한다.

실험 결과에 따라:
- 정확도 >70% → Option A 재평가 가능
- 정확도 40~70% → Option B 확정
- 정확도 <40% → Option C 또는 Gemini 교체 검토

---

## 실험 설계

### 실험 1 — 단일 도면, 단일 뷰 (Claude의 베이스라인)

**입력**: 사용자가 보유한 **실제 도면 1장** (건축 평면도 권장)

**프롬프트** (Claude Console에 붙여넣기):
```
첨부 이미지는 건축 도면이다. 아래 필드를 JSON으로 추출하라. 불확실하면 null.

{
  "sheet_type": "floor_plan | section | elevation | other",
  "sheet_title": "도면 제목 (한글)",
  "scale": "1:100 등",
  "grid_labels_x": ["X1", "X2", ...],
  "grid_labels_y": ["Y1", "Y2", ...],
  "elements": [
    {"type": "wall | column | door | window | equipment",
     "location_grid": "X1-X2 / Y3 라인 등",
     "dimension_mm": "길이 또는 크기",
     "label": "도면상 라벨 (있으면)"}
  ],
  "annotations": ["기타 읽을 수 있는 한글 주기"]
}
```

**측정 항목**:
1. JSON 파싱 성공? (y/n)
2. `sheet_type`·`scale` 정답? (y/n)
3. 그리드 라벨 정답률: 실제/추출
4. 요소 recall: 도면에 있는 벽·문 중 몇 %를 놓쳤나
5. 요소 precision: 추출된 것 중 몇 %가 맞나
6. 치수 오차율: 대표 5개 치수의 (추출값 - 실제값) / 실제값

**목표 소요**: 30분 (이미지 준비 + 호출 + 수동 검증)

---

### 실험 2 — 같은 도면, Gemini 교차 검증

**입력**: 실험 1과 **동일 이미지**
**도구**: Google AI Studio (무료, 로그인만)
**프롬프트**: 실험 1과 동일

**측정**: 같은 지표를 Gemini 2.x Pro에서 측정. **Claude vs Gemini 격차 확인**.

**목표 소요**: 20분

---

### 실험 3 — 다중 뷰 누적 (사용자 시나리오 핵심)

**입력**: 같은 건물의 **평면도 1장 + 입면도 1장 + 단면도 1장** (총 3장)

**Turn 1**: 평면도만 입력, 실험 1의 프롬프트
**Turn 2**: 평면도 + 입면도, "앞 턴 JSON에 높이 정보 추가하라"
**Turn 3**: 3장 모두, "앞 JSON에 단면 정보로 구조적 단면 추가"

**측정 항목**:
1. 턴 2에서 평면도 요소 수가 유지되는가? (Turn 1 요소를 잊지 않는가)
2. 턴 2에서 "높이"가 입면도에서 추출되어 같은 객체에 병합되는가?
3. 턴 3에서 같은 벽이 세 뷰에서 동일 객체로 식별되는가?
4. 좌표 드리프트: 턴 1의 좌표와 턴 3의 같은 객체 좌표 차이

**핵심**: 이 실험이 사용자 원 질문 "20장 순차 누적"의 **축약판**. 3장으로 작동 안 하면 20장은 거의 확실히 실패.

**목표 소요**: 60분

---

### 실험 4 (선택) — Option B 추출 단계 품질

**입력**: 평면도 5장 (5개 층 또는 5개 구역)
**프롬프트**: 실험 1 프롬프트를 5번 실행, 결과를 JSON 배열로 수집
**측정**: 5장 총 recall/precision. 일관성(같은 그리드 라벨 네이밍이 5장에서 유지되는가).

**목표 소요**: 60분

---

## 실험에 필요한 준비물 (사용자 제공)

| 항목 | 필요도 | 비고 |
|---|---|---|
| Anthropic Console API 키 또는 Claude.ai 계정 | 필수 | 실험 1,3에 필요 |
| Google AI Studio 계정 | 필수 | 실험 2에 필요 (무료) |
| 실제 건축 도면 이미지 1~5장 | **필수** | 합성 말고 진짜. 기밀이면 다음 항목 |
| (대안) 공개 샘플 도면 | 대안 | 한국 공공기관·교과서 출간물에서 획득 가능 |

**기밀 도면 주의**: 외부 API로 업로드. 사용자 조직 정책 확인 필수.
- Anthropic: 기본적으로 API 입력은 학습에 사용 안 됨 (정책 링크 확인 필요)
- Google AI Studio 무료 티어: 학습 사용 가능성 있음 → 유료 Gemini API 권장

---

## 결과 해석 가이드

| 결과 | 다음 단계 |
|---|---|
| 실험 1이 recall/precision ≥ 70% | Option A도 재고 가능. 실험 3으로 |
| 실험 1이 40~70% | Option B 확정. 사람 검수 UI가 핵심 |
| 실험 1이 < 40% | Option C 전환 또는 Gemini로 모델 교체 |
| 실험 2에서 Gemini가 Claude보다 **2배 이상** 좋음 | 파이프라인은 Claude + Vision은 Gemini 하이브리드 |
| 실험 3에서 턴 2에서 정보 유실 > 30% | 다중 뷰 누적은 Option B에서도 사람 검수 필수 |
| 실험 3에서 좌표 드리프트 > 10% | 좌표는 LLM에 맡기지 말고 결정론 처리 |

## 실험하지 **않는** 것들 (의식적 제외)

- YAML 스키마 확정 — 결과 보고 결정
- 목업 페이지 UI — 실험이 먼저
- Three.js 코드 생성 테스트 — 추출이 먼저
