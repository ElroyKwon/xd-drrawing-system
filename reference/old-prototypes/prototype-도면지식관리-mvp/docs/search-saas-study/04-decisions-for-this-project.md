# 04 · 현 프로젝트 맥락에 대한 갭 분석과 결정 포인트

> **문서 성격**: 01~03의 조사를 현재 코드베이스에 투영해 "신규 화면 설계를 한다면 어디에, 어떤 스코프로, 어떤 기술로"의 결정 옵션 A/B를 제시. 이 문서는 **사용자 확정 후 실행 계획으로 승격**되는 초안.

## 1. 현재 상태 스냅샷 (2026-04-21 기준)

### 1.1 두 기존 트랙
| 트랙 | 경로 | 성격 | 상태 |
|---|---|---|---|
| 트랙 S1 — Layer 0 MVP | `src/app/(s1)/` | 도면 검색·뷰어·주석·역추적 | W4-T1 완료, W4-T2 대기 |
| 트랙 S2 — Insight Lab | `src/app/(s2)/insight/` | 통계×온톨로지 RAG 시연 | Phase 1 완성 |

### 1.2 `(s1)` 홈이 현재 제공하는 것 (01·02 조사 기준 진단)

| 01/02의 기능·패턴 | `(s1)` 현재 | 갭 |
|---|---|---|
| B1 통합 인덱싱 | JSON 시드 8건 | — |

| B3 패싯 필터 | 학제·종류 2종 칩 | 다중/범위/저장 없음 |
| B4 권한·개인화 | 없음 | 없음 |

## 3. 결정 포인트 3가지

### D1. 구현 깊이

| 옵션 | 내용 | 작업량 | 시연 임팩트 | 장단점 |
|---|---|:---:|:---:|---|
| **A. 동작 프로토타입** | 실제로 클릭→검색, 검색→하이라이트, 2-hop traversal 작동 | 2~3일 | 높음 | 이후 Layer 1 진입 시 재활용 가능 / 데이터 준비 필요 |
| **B. 정적 스크린 모형** | Figma-like HTML 한 페이지, 상호작용 없음 | 반나절 | 중 | 빠르게 "그림" 완성 / 설득력은 A보다 약함 |
| **C. 시나리오 인터랙션** | 정해진 2~3개 클릭 경로만 진짜 동작, 나머지는 mock | 1일 | 중상 | 시연은 스무스 / 확장성은 한정 |

**권장**: **C → A 단계적**. 먼저 C로 데모 경로 완성(1일), 설득 성공 시 A로 확장.

### D2. 데이터 소스

| 옵션 | 내용 | 장단점 |
|---|---|---|
| **A. 기존 `documents.json` + `doc-entity-links.json` 재활용** | 현 8건 도면 그대로, hotspot과 2-hop 관계 데이터만 신규 | 환경 일관 / 도면이 회의용 임팩트가 약할 수 있음 |
| **B. Gemini 예시 기반 새 mock** (P-101, CH-001, PLC 등) | 도면 1~2장 새로 준비 + 전용 시나리오 | "교과서적" 케이스로 설득 용이 / 도면 확보 부담 |
| **C. 하이브리드** | 기존 `mech-system.png`·`fire-plan-3f.png`를 쓰되 hotspot·태그는 새 시나리오용으로 설계 | 도면 재활용 + 시나리오 자유 / 태그·도면 괴리 가능 |

**권장**: **C**. 8건 중 이미지 fallback이 있는 `mech-system`·`fire-plan-3f`·`doc-003`·`doc-004`에 hotspot 3~5개씩 수동으로 찍고, `doc-entity-links.json`에 2-hop 확장 태그(예: `CH-001 → COOL-TOWER-01 → VFD-007`)를 덧댄다.

### D3. 개념 범위 (02의 4패턴 중 몇 개를 담나)

| 옵션 | 포함 패턴 | 코드 규모 | 설득 범위 |
|---|---|:---:|---|
| **A. 4개 전부** | Click-to-Search + Search-to-Highlight + 영향도 2-hop + HUD | 큼 | 종합 |
| **B. 집중 2개** | Click-to-Search + 영향도 2-hop | 중 | 핵심 가치만 |
| **C. 시나리오 하나로 4개 연쇄** | 한 경로 안에서 C2S→S2H→영향도 팝업→HUD가 순차로 등장 | 중 | 서사적 설득 |

**권장**: **C**. "냉동기 CH-001을 클릭 → 속성 HUD 팝업 → '영향받는 하위 설비' 탭 → 2-hop으로 연결된 VFD·CT 목록 → 그 중 하나 검색창에 자동 입력 → 뷰어에서 하이라이트"의 1분 시나리오. 4개 패턴이 한 경로에 자연스럽게 들어간다.

## 4. 권장 조합 (하나로 묶으면)

> **D1=C / D2=C / D3=C**

한 줄 요약: **"기존 도면·태그 자산을 재활용하여, 1분짜리 냉동기 장애 시나리오를 따라가는 동작 프로토타입을 `/concept`에 만든다. 4패턴이 한 경로에 연쇄 등장."**

- **작업량**: 1~1.5일
- **신규 파일 추정**:
  - `src/app/(s3)/concept/page.tsx`
  - `src/app/(s3)/concept/HotspotLayer.tsx`
  - `src/app/(s3)/concept/HUDPanel.tsx`
  - `src/app/(s3)/concept/ImpactGraph.tsx` (간단한 수평 트리)
  - `data/concept/hotspots.json`
  - `data/concept/impact-edges.json`
- **건드리는 기존 파일**: `src/app/(s1)/page.tsx`에 진입 버튼 한 줄 추가 (8~10줄). 보존 6항목 불변.

## 6. 사용자 확정이 필요한 3가지

1. **D1·D2·D3 추천안(C/C/C)** 수락 여부 — 다르면 어느 옵션?
2. **시연 시나리오** — "냉동기 CH-001 장애 경로" 예시로 진행? 아니면 다른 설비(PLC 결선/공조/소방 중)?
3. **진입 버튼 위치** — `(s1)` 좌측 사이드바 검색창 아래 / 또는 우측 패널 탭에 하나 추가 / 또는 루트 경로에 토글?

## 7. 확정 후 후속 작업 흐름

1. `data/concept/hotspots.json`, `impact-edges.json` 초안 작성 (수기)
2. `src/app/(s3)/concept/page.tsx` 레이아웃 — 뷰어 풀스크린 + 플로팅 검색창 + HUD 패널
3. `HotspotLayer` — `AnnotationLayer` 복제 후 클릭 핸들러만 치환
4. `HUDPanel` — 글래스모피즘 스타일, 속성 + 링크 + "영향받는 하위" 버튼
5. `ImpactGraph` — 2-hop traversal 결과를 수평 트리로 (d3 없이 Flex만으로 가능)
6. Search-to-Highlight — 쿼리 변경 시 hotspots에서 매칭 항목 `ring-4 ring-amber-400` 애니메이션
7. 시나리오 End-to-End QA
8. 기존 (s1) `page.tsx`에 진입 버튼 1개 추가
