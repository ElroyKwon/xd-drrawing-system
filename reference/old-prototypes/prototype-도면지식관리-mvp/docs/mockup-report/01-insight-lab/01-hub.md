# 화면 · Insight Lab 화면 1 · 인사이트 홈 (허브)

**경로**: `/insight`
**소속 트랙**: Insight Lab (서비스 2 · AI 인사이트)
**화면 분류**: 랜딩·진입점·대시보드 요약

---

## 1. 화면 개요

![Insight Lab 홈 · 최근 알람·자주 묻는 질문·일일 보고서](../screenshots/il-hub-full.png)

이 화면은 Insight Lab에 들어온 사용자가 처음 마주하는 **허브 대시보드**입니다. "오늘 뭘 봐야 하나"를 3초 안에 판단하도록 설계되었습니다. 상단에 한 줄짜리 타이틀, 그 아래로 **세 개의 카드 섹션**이 세로로 쌓여 있습니다: (1) 최근 알람 3건, (2) 자주 묻는 질문 5건, (3) 최신 일일 보고서 링크.

랜딩 페이지는 **요약 · 입구 · 유도** 세 가지 역할을 동시에 수행해야 합니다. 이 화면은 "지금 이상 상황이 있는가(알람)", "남들은 뭘 물어보는가(질문 예시)", "어제 무슨 일이 있었나(보고서)" 세 축을 한 화면에 배치해 **관심사를 강요하지 않고 열어줍니다**. 각 카드는 독립적으로 해당 상세 경로로 뛰는 입구이고, 허브 자체는 체류를 강요하지 않습니다.

전체 페이지는 **얕고 가볍게**, 카드 3개 이상으로 번잡해지지 않도록 의도적으로 제약되어 있습니다.

## 3. UX 상세 설명

### 3.1 타이틀 블록

- `text-lg font-semibold`로 **"인사이트 홈"** 한 줄. 그 아래 `text-sm text-slate-400`으로 한 줄짜리 부제
- 아이콘 없음, 장식 없음 — 카드 섹션이 시각적 주인공이 되도록 비워둠

### 3.2 최근 알람 카드

- 상단 헤더: 좌측에 "최근 알람" 라벨, 우측에 "전체보기 →" 링크 (`/insight/alarms`로 이동)
- 본문 리스트: `recentAlarms`를 `triggered_at` 역순 정렬해 **상위 3건**만 표시 (`.slice(0, 3)`)
- 각 행 구성:
  - 좌측: 심각도별 색 도트 (위급=`bg-rose-500`, 경고=`bg-amber-400`, 정보=`bg-sky-400`)
  - 중앙: 알람 이름 (`truncate` 1줄)
  - 우측: 설비 태그 (monospace 폰트, `VCB-001` 같은)
- 행 사이 구분선은 `divide-slate-50`로 매우 옅음 — 리스트가 "덩어리"로 보이게

### 3.3 자주 묻는 질문 카드

- 상단 헤더 "자주 묻는 질문" (전체보기 링크 없음 — 별도 라우트 미제공)
- `queryExamples.slice(0, 5)` 상위 5건 텍스트만 렌더
- 클릭 핸들러 없음 — **참고·영감용**으로만 기능. 사용자가 "아, 이런 걸 물어봐도 되는구나"를 인지하는 것이 목적

### 3.4 최신 일일 보고서 카드

- 전체 카드 본문이 **하나의 거대한 링크**. hover 시 `bg-slate-50` 전체 배경 전환
- 좌측: 날짜 한글 표기 ("2026년 4월 20일 일일 보고서") + 섹션 수와 상태 메타 (`6개 섹션 · 초안`)
- 우측: `›` 우측 꺽쇠로 "들어감"을 암시
- 카드 상단 "전체보기 →"로 `/insight/reports` 리스트로 이동 가능
- 정적 import (`import reportJson from "@data/insight/reports/report-daily-20260420.json"`) — 목업이므로 최신 보고서가 하드코딩되어 있음

### 3.5 시각 스타일

- 모든 카드: `rounded-xl border border-slate-200 bg-white` — 흰색 카드, 연한 회색 테두리, 부드러운 모서리
- 전체 배경: `bg-slate-50` — 카드가 떠 보이도록 배경은 아주 연한 회색
- 각 카드 내 헤더/본문 경계: `border-b border-slate-100` — 극히 옅은 구분선

## 4. 이 UX가 만드는 효과

| UX 결정 | 사용 경험에서의 변화 |
|---|---|
| 카드 3개로 제한 | 정보 과부하 없이 "지금 뭘 봐야 하나" 3초 내 판단 |
| 최근 알람을 최상단에 배치 | 운영자가 들어오자마자 **이상 상황 유무**를 먼저 인지 — 위급 색점이 눈에 먼저 들어옴 |
| 알람 3건만 노출 + "전체보기" 링크 | 허브에서 리스트 전체를 스크롤하지 않음, 상세는 별도 페이지에서 |
| 자주 묻는 질문 5건 (클릭 비활성) | 검색창을 비우고 시작할 때의 "뭘 물어봐야 하지" 장벽 제거 — 예시가 **프롬프트 교사** 역할 |
| 일일 보고서를 카드 하나 통째로 링크화 | 어제자 요약을 한 클릭으로 진입 — 아침 첫 루틴이 자연스럽게 연결됨 |
| 심각도별 색 도트 | 텍스트를 읽기 전에 **색만으로도** 위급·경고·정보 구분 가능 |
| 모든 카드에 부가 설명이 없고 데이터만 | 허브는 "어디로 갈지" 선택하는 곳이라는 신호 — 체류가 아닌 분기의 공간 |

## 5. 사용자 동작 흐름

| # | 액션 | 결과 | UX 의도 |
|---|---|---|---|
| 1 | `/insight` 진입 | 3개 카드가 세로로 나타남 | 한 스크롤 안에서 전체 파악 |
| 2 | 최근 알람 카드의 "전체보기 →" 클릭 | `/insight/alarms`로 이동 | 알람 전체 리스트로 깊이 들어감 |
| 3 | 일일 보고서 카드 본문 클릭 | `/insight/reports/{id}`로 직행 | 가장 최근 보고서를 원클릭으로 열기 |
| 4 | 일일 보고서 헤더의 "전체보기 →" | `/insight/reports` 리스트로 이동 | 과거 보고서까지 훑고 싶을 때 |
| 5 | 자주 묻는 질문 항목 응시 | 클릭 비반응, 시선만 머묾 | 질의 예시를 **읽고** 챗 화면에서 직접 입력하도록 유도 |
| 6 | 알람 항목의 설비 태그 (`VCB-001`) 시선 집중 | 심각도와 설비가 한 쌍으로 인지됨 | 알람 상세로 들어가기 전 맥락 형성 |

## 6. 데이터·API 의존성

### 원천 데이터
- `data/insight/alarms-fixed.json` — 고정 알람 이벤트 10건 (심각도·태그·임계값·설명)
- `data/insight/queries-examples.json` — 질문 예시 20여 건
- `data/insight/reports/report-daily-20260420.json` — 가장 최신 일일 보고서 (정적 import)

### 참조하는 lib
- `@/lib/insight/data-loader` — `recentAlarms`, `queryExamples` export
  - `recentAlarms`: `alarms`를 `triggered_at` 내림차순 정렬한 배열
  - `queryExamples`: `queries-examples.json`을 그대로 노출

### 참조하는 타입
- `@/lib/insight/types` — `AlarmEvent`, `AlarmSeverity`, `InsightQuery`

### 실제 LLM 호출 여부
**없음**. 정적 JSON 3개를 읽어 리스트 3개를 렌더하는 순수 프레젠테이션 페이지입니다. 이 화면은 SSR로 빌드 타임에 이미 결정 — 클라이언트 훅 없음 (use client 없음).

### 하위 페이지 링크
- `/insight/alarms` (알람 목록)
- `/insight/reports` (보고서 리스트)
- `/insight/reports/{id}` (보고서 상세)

## 7. 이 화면이 기여하는 서비스 측면

| DKS 서비스 측면 | 이 화면이 맡는 역할 |
|---|---|
| **운영 현황 한눈 파악** (서비스 2 핵심) | 허브 접속 즉시 "오늘 이상 여부"를 색점과 건수로 시각화 |
| **알람 → 분석 → 도면** 파이프라인의 입구 | 알람 리스트 카드가 후속 화면(`/insight/alarms` → 드로어 → `/insight/answers/{id}`)의 시작점 |
| **사용자 질의 유도** (AI 어시스턴트 진입 장벽 낮춤) | 자주 묻는 질문 5건으로 "어떻게 물어봐야 하나" 장벽을 해소 |
| **일일 보고서 루틴화** | 랜딩에 일일 보고서를 노출해 아침·주기 점검 루틴을 자연스럽게 유도 |
| **정보의 계층화** | 허브는 요약 · 하위 페이지는 상세 — 계단식 깊이로 인지 부담 분산 |

**이 화면이 해결하지 않는 것**: 알람 필터링, 보고서 작성/수정, 자유 질의 등 모든 **조작**은 하위 화면의 영역입니다. 허브는 **읽고 선택**하는 공간입니다.

## 8. 의견 수렴 포인트

### 스스로 본 보완 포인트

- **자주 묻는 질문 클릭 비활성**: 목업에서는 텍스트만 나열됨. 실 서비스에서는 클릭 시 `/insight/chat?q={text}` 같은 프리필 링크가 자연스러움
- **알람 3건 고정**: 위급이 많은 날에도 3건만 보임. "위급은 전부 / 나머지는 상위 N건" 조건부 노출이 더 나을 수 있음
- **일일 보고서 카드가 1건 고정 import**: 날짜가 바뀌어도 파일을 교체하지 않으면 오래된 보고서를 노출. 실 서비스에서는 런타임에 최신 보고서 fetch 필요
- **허브에 KPI 위젯이 없음**: 가동률·MTTR 등 일일 숫자 지표가 없어 "오늘 상태" 감각이 약함. 추가 카드 검토 가치 있음
- **주간·월간 보고서 진입점 부재**: 현재는 일일 보고서만 카드화됨. 기간 전환 토글이 필요할 수 있음
- **빈 상태 디자인 없음**: 알람 0건일 때 빈 리스트가 그대로 노출됨 — "오늘 알람 없음, 정상 운전 중" 같은 긍정 메시지 필요
