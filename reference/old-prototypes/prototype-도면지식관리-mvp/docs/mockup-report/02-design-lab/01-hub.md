# 화면 · Design Lab 허브

**경로**: `/design-lab`
**소속 트랙**: Design Lab (2-트랙)
**화면 분류**: 진입·안내 중심 허브

---

## 1. 화면 개요

![Design Lab 허브 · 두 화면 카드 + 목적 구분표](../screenshots/dl-hub-full.png)

이 화면은 Design Lab 트랙의 **진입점**입니다. 목적이 서로 다른 두 개의 신규 화면(화면 1 "카드형 도면 리스트" · 화면 2 "챗 워크스페이스")을 **나란히 비교**하여 선택할 수 있도록, 큰 카드 2개와 하단의 목적 구분 표로 구성되어 있습니다.

허브의 핵심 메시지는 **"두 화면은 경쟁이 아니라 역할이 다르다"**는 점입니다. 따라서 단순히 링크 리스트를 보여주는 대신, 각 화면이 **무엇을 주역으로 삼는가**·**대화를 어떻게 사용하는가**·**사용자가 어떤 목적으로 들어오는가**라는 세 축을 좌우 대조해 이해관계자가 즉시 차이를 파악하게 설계했습니다.

상단에는 **"Design Lab · 목업 전용 트랙"** 배지가 붙어 있어, 이 화면의 결과물이 프로덕션(`/s1`, `/s2`, `/s3`)에 영향을 주지 않음을 명시합니다. 즉 허브는 **탐색·비교의 안전지대**임을 첫눈에 밝힙니다.

## 3. UX 상세 설명

### 3.1 상단 배지 · 그라디언트 제목

- `Design Lab · 목업 전용 트랙` 배지는 **연한 보라 pill**로, 좌측에 작은 보라 닷을 달아 "살아있는 트랙"임을 보여줍니다
- 제목은 `GradientHeading` 공용 컴포넌트로 슬레이트 900 → 보라 600 → 시안 600 그라디언트 텍스트
- 부제는 중간 회색 한 단락 — "무엇을 하는 화면인가 + 프로덕션 분리" 두 정보를 모두 전달

### 3.2 HubCard — 두 화면을 큰 카드 2개로

각 카드는 **독립된 Link 블록**이며, 다음 요소로 구성됩니다.

| 요소 | 위치 | 표현 |
|---|---|---|
| 배지 pill | 좌상단 | `화면 1` · `화면 2` — 각 카드의 그라디언트 컬러로 채색 |
| 부제 | 배지 옆 | 한 줄로 "탐색·브라우징 중심" 등 핵심 성격 |
| 제목 | 배지 아래 큰 폰트 | 24px 볼드, 슬레이트 900 |
| 포인트 리스트 | 제목 아래 | 좌측 작은 회색 닷 + 한 줄씩 3개 |
| "열기 →" | 우하단 | 호버 시 화살표가 **3px 더 벌어져** 클릭 가능성 강화 |
| 블러 장식 | 우상단 `-right-16 -top-16` | 크고 흐린 그라디언트 원, 호버 시 불투명도 20% → 40% |

**그라디언트 구분**:
- 화면 1 카드: `from-violet-500 to-fuchsia-500` (보라·자홍)
- 화면 2 카드: `from-cyan-500 to-blue-500` (시안·블루)

이 컬러는 단순한 장식이 아니라 **두 화면의 정체성 색**으로, LabNav·목적 구분 표·각 화면 내부 강조색까지 일관되게 재등장합니다.

### 3.3 호버 인터랙션

- 카드 전체가 `hover:-translate-y-1`로 **1px 떠오름**
- 그림자가 `shadow-sm → shadow-xl`로 강화
- 블러 원의 opacity가 두 배로 증가해 **카드가 "켜지는" 감각**
- "열기 →" 화살표가 `gap-1.5 → gap-3`으로 벌어짐

네 가지 효과가 동시에 일어나지만 각각은 미세합니다 — 합쳐졌을 때 "이 카드를 누르라"는 암묵적 신호가 만들어집니다.

### 3.4 목적 구분 표 — 3열 CompareRow

하단 `rounded-2xl` 흰 박스 안에 **3개의 CompareRow 카드**가 `md:grid-cols-3`로 나열됩니다.

각 CompareRow는:
- 상단 작은 회색 uppercase 라벨 (`주역` · `대화의 역할` · `진입 목적`)
- 그 아래 **1 배지(보라) + 텍스트** 한 줄, **2 배지(시안) + 텍스트** 한 줄

| 라벨 | 화면 1 | 화면 2 |
|---|---|---|
| 주역 | 카드(도면) | 대화 |
| 대화의 역할 | 검색 보조 (필터) | 주 작업 (질의·요약) |
| 진입 목적 | "뭐가 있나 훑어보기" | "이 질문 답해줘" |

이 표는 의도적으로 **3행×2열 구조**를 벗어나 **3개의 독립 카드**로 만들어졌습니다 — 각 비교 축을 개별 카드로 분리하면 시선이 "라벨 → 1 → 2 → 다음 라벨"로 자연스럽게 흐르고, 옆 비교축과 뒤섞이지 않습니다.

### 3.5 데이터 출처 주석

최하단 한 줄: *"모든 데이터는 drawings.json 실데이터 + 목업 시드. 실제 LLM 호출은 없습니다."*

이 문장은 이해관계자가 시연을 본 뒤 **"그럼 실제로 쓸 때는?"**이라고 묻기 전에 먼저 답해주는 장치입니다. 회색 작은 폰트로 배치하여 본문을 방해하지 않습니다.

## 4. 이 UX가 만드는 효과

| UX 결정 | 사용 경험에서의 변화 |
|---|---|
| 두 화면을 **나란히** 카드로 제시 | 이해관계자가 "어느 게 더 낫지?"가 아니라 **"둘은 역할이 다르다"**로 인지 |
| 목적 구분 표를 카드 아래 별도 섹션으로 분리 | 카드만 보고 들어가도 되고, 고민된다면 표까지 읽어 판단 근거 강화 가능 |
| 컬러 쌍(violet/fuchsia · cyan/blue)을 처음부터 도입 | 각 화면으로 이동한 뒤에도 **컬러로 "여기가 어느 트랙"**인지 지속 인지 |
| 배지·부제·포인트 3단 구조 | 3초(배지) → 10초(부제) → 30초(포인트) 단계적 정보 소비가 가능 |
| 호버 시 4중 미세 인터랙션 | 정적 스크린샷에서는 안 보이는 **"살아있는 UI" 감각** — 시연에서 효과 발휘 |
| "목업 전용 트랙" 배지를 상단에 명시 | 이해관계자가 기능 완성도로 오해해 실서비스 품질 기대를 잘못 갖는 것을 예방 |
| 목적 구분 표의 **1·2 배지 색상**이 카드 배지와 1:1 매칭 | 같은 숫자를 기억하지 않아도 **색으로 어느 화면 이야기인지 즉시 연결** |

|---|---|---|---|

| 5 | LabNav의 "← 프로덕션 복귀" 클릭 | 기존 메인 서비스(`/`)로 복귀 | Design Lab이 **격리된 영역**임을 이탈 경로로 재확인 |

## 6. 데이터·API 의존성

**없음**. 허브는 정적 콘텐츠(제목·카드 텍스트·비교 표)만 렌더합니다. 데이터 로더 호출이나 API 요청이 전혀 없습니다.

### 참조하는 컴포넌트

- `@/components/design-lab/shared` — `GradientHeading` (그라디언트 제목용)
- Next.js 내장: `next/link` (카드 링크)

### 이 화면에서 정의된 로컬 컴포넌트

- `HubCard` — 큰 카드 2개 (page.tsx 내부)
- `CompareRow` — 목적 구분 표의 3개 셀 (page.tsx 내부)

### 실제 LLM 호출 여부

**없음**. 허브는 순수한 안내 페이지이므로 어떤 비동기 작업도 수행하지 않습니다.

## 7. 이 화면이 기여하는 서비스 측면

| DKS 서비스 측면 | 이 화면이 맡는 역할 |
|---|---|
| **의사결정 지원 (목업 비교 검토)** | 서로 다른 UX 가설을 **한 페이지에서 병렬 노출**해 이해관계자가 "어느 방향이 우리에게 맞나"를 즉시 판단하도록 |
| **트랙 분리 신호** | 상단 배지·하단 주석으로 **목업 = 탐색 영역**임을 명시해 프로덕션 품질 기대와 분리 |
| **브랜드·컬러 시스템 앵커** | violet·cyan 두 축의 컬러 페어를 **허브에서 선언**하고, 하위 화면들이 재사용 — 트랙 전체의 시각적 일관성 |
| **진입점 단일화** | Design Lab을 한 곳(`/design-lab`)에서만 시작하도록 유도해 **사이드 네비 혼란 방지** |

**이 화면이 해결하지 않는 것**: 두 화면 중 어느 쪽이 **실제 현장에서 더 효율적인가**는 허브의 몫이 아닙니다. 허브는 비교를 **제시**할 뿐, 결론은 이해관계자 대화에서 도출됩니다.

## 8. 의견 수렴 포인트

### 스스로 본 보완 포인트

- **카드 수가 2개로 고정**: 향후 화면 3이 추가되면 `lg:grid-cols-3`로 바뀌어야 하는데, 허브 내 카드 컬러 팔레트가 이미 2종만 정의되어 있어 **3번째 컬러 축 설계**가 필요
- **모바일 레이아웃**: `lg:grid-cols-2`가 1024px 이하에서 1열로 접히지만, 두 카드를 세로로 쌓으면 **목적 구분 표와의 시각적 연결**이 약해짐 — 모바일용 별도 비교 UI 필요할 수 있음
- **시각 장애 접근성**: 컬러로만 화면 1·2를 구분하는 부분(배지 1·2의 색)이 있어 **아이콘 병기 또는 숫자 병기** 보강 검토
- **"목업 전용" 배지의 강도**: 상단에 한 번만 등장 — 스크롤 후 카드만 보는 사용자는 이 맥락을 놓칠 수 있음. 각 카드 하단에도 작은 라벨 검토
- **진입 경로 안내 부재**: 허브에서 바로 화면 2를 열면 "전체 리스트에서 진입" 상태로 시작하지만, 허브 단계에서는 이 선택지가 노출되지 않음 — 카드 내에 **"기본 진입 모드"** 힌트 추가 검토
- **"열기 →"의 위치**: 우하단이라 카드 전체를 클릭할 수 있다는 신호가 약함. 실제로는 카드 어디를 클릭해도 이동하지만 사용자가 "열기"만 눌러야 한다고 오해할 수 있음
