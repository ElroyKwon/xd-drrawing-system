# 03. 방향성 판단 프레임워크 — DKS는 무엇이 되어야 하는가?

> 작성일: 2026-04-17  
> 전제: 01-본질적 가치 검토 + 02-시장조사 결과를 종합한 판단

---

## 1. 시장이 알려주는 것

### 도면 관리 "만"으로는 죽는다

```
PlanGrid (도면 뷰어)         → Autodesk에 $875M 인수 → 브랜드 소멸
Fieldwire (현장 관리)         → Hilti에 인수 → 독립 종료
Aconex (프로젝트 EDMS)       → Oracle에 $1.2B 인수 → 플랫폼 흡수
Vela Systems (현장 검사)     → Autodesk에 인수 → 소멸
```

**단일 기능 도구의 결말은 예외 없이 "인수 or 소멸"이다.**  
이것이 의미하는 바: DKS가 "도면 관리 시스템"을 목표로 하면, 아무리 잘 만들어도 Procore/Autodesk의 하위 기능으로 대체된다.

### 살아남는 서비스의 공통점

| 생존 조건 | 설명 | 예시 |
|-----------|------|------|
| **도메인 깊이** | 특정 산업의 깊은 문제를 해결 | Nuvolo(의료), UpKeep(현장 정비) |
| **데이터 네트워크** | 쓸수록 데이터가 쌓여서 가치 증가 | Procore(하도급 네트워크) |
| **AI/분석 레이어** | 원시 데이터 위에 지능을 얹음 | OpenSpace(진행률 AI), Fiix(예측정비) |
| **워크플로우 통합** | 일상 업무 프로세스에 끼어들음 | Confluence(개발 문서), Notion(팀 협업) |

## 2. DKS의 위치를 솔직하게 진단

### 있는 것 vs 없는 것

| DKS에 있는 것 | DKS에 없는 것 |
|---------------|---------------|
| 설비 지식 그래프 (125건 mock) | 실 데이터 |
| 영향도 분석 엔진 | 실사용자의 실사용 검증 |
| 4 페르소나 UX | 업로드/등록 기능 |
| 도면 리스트+다운로드 | 도면 안 내용 이해 (AI/OCR) |
| Phase 0 전문가 검증 | 지불 의사 검증 |
| 프로토타입 데모 | 배포 가능한 제품 |

### 치명적 gap

```
[사용자가 원하는 것]          [DKS가 줄 수 있는 것]
DWG 올리면 알아서 다 해줘    →  할 수 없음 (DWG 파싱 불가)
도면 안의 설비를 자동 인식    →  할 수 없음 (AI 미구현)
CRUD가 되게 해줘             →  할 수 없음 (읽기 전용)
하나의 시스템에서 다 하고 싶어 →  할 수 없음 (조회만 가능)
```

**사용자의 기대와 현재 역량 사이에 거대한 간극이 있다.**

## 3. 세 가지 가능한 경로

### 경로 A: "경량 EDMS + 설비 지식" (올인원 축소판)

```
PDF 업로드 → 페이지 분리 → 수동 메타 태깅 → 검색/열람/다운로드
                                    +
              설비 태그 연결 → 관계 그래프 → 영향도 분석
```

**가치 제안**: "EDMS 도입할 예산은 없지만, 도면과 설비를 체계적으로 관리해야 하는 중소 시설관리 조직을 위한 경량 솔루션"

| 장점 | 단점 |
|------|------|
| 지금 당장 만들 수 있음 (AI 불필요) | "수동 태깅"은 사용자가 안 함 |
| 시장 공백 존재 (한국 중소 FM) | 낮은 ARPU ($50-200/월) |
| 설비 그래프가 차별점 | 높은 churn (수동 유지 비용 > 편익이면 이탈) |

**리스크**: 수동 태깅의 노동 대비 시스템이 주는 편익이 부족하면, "그냥 공유 폴더에 넣는 게 낫다"가 됨.

**수익 모델**: per-seat 구독 $30-80/user/월  
**필요 고객**: 100개사 × 5명 × $50 = $25,000/월 = 연 $300K → 1인 사업 가능, 팀 운영은 어려움

---

### 경로 B: "AI 도면 분석 엔진" (기술 중심)

```
PDF 업로드 → AI가 각 페이지 분석 → 설비 태그 자동 추출 → 관계 자동 생성
                                                    → 지식 그래프 자동 구축
                                                    → 검색/영향도 즉시 사용 가능
```

**가치 제안**: "도면 PDF를 올리면, AI가 설비를 인식하고 관계를 만들어준다. 그래서 '이 설비를 끄면 어디까지 영향 받나?'를 즉시 알 수 있다."

| 장점 | 단점 |
|------|------|
| 사용자 기대와 일치 ("올리면 알아서") | AI 정확도가 곧 제품 품질 |
| usage-based 과금 가능 | 개발 난이도 매우 높음 |
| 기존 EDMS/CMMS 어디에도 없는 기능 | 한국어 설비 도면 학습 데이터 부족 |
| 높은 switching cost (데이터 축적) | 초기 투자 필요 (LLM 비용, 학습) |

**리스크**: AI 정확도가 80% 미만이면 "검수하는 데 드는 노력 > 수작업"이 되어 무용지물.

**수익 모델**: usage-based (PDF 페이지당 $0.5-2) + 월 구독  
**시나리오**: 1,000페이지/월 × $1 × 100개사 = $100K/월 = 연 $1.2M → 팀 운영 가능

---

### 경로 C: "CMMS 보완 플러그인" (기생 전략)

```
기존 CMMS(Maximo, UpKeep 등) ← API 연동 → DKS 설비 관계 엔진
"이 설비를 정비할 때, 연관 설비와 영향 범위를 DKS가 알려줌"
```

**가치 제안**: "기존 CMMS에 '설비 관계 지능'을 추가하는 플러그인"

| 장점 | 단점 |
|------|------|
| 기존 시장 위에 올라탐 | CMMS 벤더 종속 |
| 명확한 사용 시나리오 (정비 전 영향도 확인) | 마켓플레이스 수수료 |
| 작은 범위로 빠르게 출시 가능 | 독립 가치 약함 |

**리스크**: CMMS 벤더가 같은 기능을 내장하면 끝. 방어 불가.

**수익 모델**: Procore/Maximo 마켓플레이스 리스팅, $10-30/user/월  
**현실**: 한국에서 CMMS 마켓플레이스 생태계 미성숙

## 4. 냉정한 비교

| 기준 | 경로 A (경량 EDMS) | 경로 B (AI 엔진) | 경로 C (플러그인) |
|------|:---:|:---:|:---:|
| **지금 만들 수 있나** | O | △ (6-12개월) | O |
| **사용자 기대 충족** | △ (수동) | O | △ (부분) |
| **차별화** | 약함 | 강함 | 중간 |
| **수익 잠재력** | 낮음 ($300K/yr) | 높음 ($1M+/yr) | 낮음 |
| **기술 리스크** | 낮음 | 높음 (AI 정확도) | 낮음 |
| **시장 리스크** | 높음 (지불 의사 미검증) | 중간 | 높음 (생태계 미성숙) |
| **방어 가능성** | 약함 | 강함 (데이터 해자) | 없음 |

## 5. 검증해야 할 것 (우선순위순)

어떤 경로든 선택하기 전에 아래가 먼저다:

### 5.1 즉시 검증 (1-2주)

| # | 질문 | 검증 방법 |
|---|------|-----------|
| 1 | 현장에서 "설비 관계 탐색"을 실제로 쓰는 순간이 있는가? | 현장 관찰 (shadowing) |
| 2 | 도면을 찾는 데 실제로 얼마나 시간을 쓰는가? | 사용자 5명 일일 행동 추적 |
| 3 | 지금 도면을 어디에 어떻게 보관하는가? | 현재 도구 스크린샷 수집 |
| 4 | "이 기능에 월 5만 원 내겠는가?" | 직접 질문 (WTP 조사) |

### 5.2 단기 검증 (1-2개월)

| # | 질문 | 검증 방법 |
|---|------|-----------|
| 5 | AI가 한국어 설비 도면에서 태그를 추출할 수 있는가? | 실 도면 10장으로 Claude/GPT Vision 테스트 |
| 6 | 추출 정확도가 실무에서 수용 가능한가? | 현장 엔지니어에게 결과 보여주고 피드백 |
| 7 | 경량 EDMS만으로 "공유 폴더보다 낫다"고 느끼는가? | 프로토타입에 업로드 기능 추가 후 2주 사용 |

## 6. 제안: 지금 해야 할 한 가지

**세 경로를 동시에 추구하지 말 것.**

지금 가장 가치 있는 행동은 **검증 #5**: "AI가 한국어 설비 도면에서 태그를 추출할 수 있는가?"

왜냐하면:
- 이 답이 **Yes**이면 → 경로 B가 열린다 (가장 높은 가치)
- 이 답이 **No**이면 → 경로 B는 닫히고, A 또는 C로 가야 한다
- 이 검증에 드는 비용 = 실 도면 PDF 10장 + Claude Vision API 비용 수만 원
- 1-2일이면 결과를 알 수 있다

**경로 B가 가능한지 먼저 확인하고, 불가능하면 경로 A의 MVP를 검토하는 것이 합리적이다.**

## 7. 부정적·불안정성 종합 평가

### 이 프로젝트가 실패할 수 있는 시나리오

| 시나리오 | 확률 | 결과 |
|----------|------|------|
| AI 추출 정확도 70% 미만 | 중 | "검수 비용 > 편익" → 사용 안 함 |
| 한국 중소 시설관리의 지불 의사 = 0 | 높음 | 매출 없음 |
| 대형사가 유사 기능 출시 | 중 | 차별화 상실 |
| 실 데이터 확보 실패 | 중 | 제품 검증 불가 |
| 1인/소규모 팀으로 EDMS+AI+UX 전부 감당 | 높음 | 번아웃, 품질 저하 |

### 가장 큰 불확실성

**"이 시스템에 돈을 낼 사람이 있는가?"를 아직 한 번도 검증하지 않았다.**

전문가 6인 검증, 사용자 인터뷰 2회는 "이런 기능이 있으면 좋겠다"를 확인했을 뿐, "이것에 돈을 내겠다"를 확인하지 않았다. 이 둘은 완전히 다른 질문이다.
