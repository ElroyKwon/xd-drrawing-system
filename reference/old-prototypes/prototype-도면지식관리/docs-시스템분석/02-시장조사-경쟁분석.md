## 2. 카테고리별 상세 분석

### 2.1 EDMS/도면관리 전문

| 서비스 | 가치 제안 | 수익 모델 | 핵심 기능 | 약점 |
|--------|-----------|-----------|-----------|------|
| **Aconex** (Oracle) | 대형 프로젝트 단일 정보 플랫폼 | 프로젝트 기반 구독 (고가) | 문서 통제, RFI, 감사 추적 | 비싼 가격, Oracle 인수 후 혁신 둔화 |
| **Procore** | 건설 올인원 | 연간 매출 기반 구독 | 도면 비교, RFI, 재무 통합 | 소규모에 과도한 비용 |
| **PlanGrid** → Autodesk Build | 현장 최신 도면 즉시 접근 | 구독 $39-59/user/월 | 모바일 뷰어, 마크업, 이슈 | 독립 브랜드 소멸 |
| **Bluebeam Revu** | PDF 마크업·협업 최강 | 구독 $240/yr~ | Studio 세션, 측정, 펀치리스트 | 클라우드 전환 느림 |
| **ProjectWise** (Bentley) | 설계 데이터 버전관리 | Enterprise 라이선스+사용량 | CAD/BIM 통합 워크플로 | 높은 도입비, 중소기업 부적합 |
| **BIM 360/ACC** | BIM 중심 통합 | 구독 $45-85/user/월 | 모델 협업, Clash Detection | Autodesk 종속 |

**공통점**: 모두 **도면을 "파일"로** 관리. 도면 안의 내용(설비, 관계)은 이해하지 못함.

### 2.2 시설관리(FM) SaaS

| 서비스 | 가치 제안 | 수익 모델 | 약점 |
|--------|-----------|-----------|------|
| **Archibus** | IWMS 시장 선도 | Enterprise 라이선스+클라우드 | 구식 UI, 높은 구현 비용 |
| **Planon** | 스마트빌딩·ESG | Enterprise 구독 | 복잡한 설정, 아시아 인지도 낮음 |
| **FM:Systems** | 직장 경험 관리 | 모듈별 SaaS | JCI 인수 후 방향 불확실 |
| **iOFFICE** | 직관적 공간 관리 | $3-8/user/월 | 제조/인프라 부적합 |
| **Nuvolo** | ServiceNow 기반 FM | ServiceNow 구독 | 플랫폼 비용 높음 |

**시사점**: FM은 "공간"과 "자산" 중심. 도면은 배경 기능. DKS와 직접 경쟁 아님.

### 2.3 CMMS/정비관리

| 서비스 | 가치 제안 | 수익 모델 | 타겟 |
|--------|-----------|-----------|------|
| **Maximo** (IBM) | 엔터프라이즈 EAM 표준 | 라이선스+SaaS 전환 중 | 대기업, 유틸리티 |
| **eMaint** (Fluke) | 유연한 중견 CMMS | $69-120/user/월 | 중견 제조/시설 |
| **Fiix** (Rockwell) | AI 정비 최적화 | Freemium+$45-75/user/월 | 제조업 |
| **UpKeep** | 모바일 퍼스트 | $45-75/user/월 | SMB 현장 |
| **Limble** | 쉬운 도입, 빠른 ROI | $28-69/user/월 | SMB 첫 CMMS |

**시사점**: CMMS에서 도면은 "참조 문서". DKS의 설비 관계 그래프는 CMMS가 제공하지 못하는 것.

### 2.4 지식관리 SaaS

| 서비스 | ARR/규모 | 수익 모델 | 시사점 |
|--------|----------|-----------|--------|
| **Notion** | $250M+ | Freemium + $8-15/user/월 + AI $10/user | 조직 전체 침투가 핵심 |
| **Confluence** | Atlassian $4B+ 중 핵심 | per-user 구독 | 개발자 생태계 lock-in |
| **Guru** | 시리즈 C $40M | $5-15/user/월 | 슬랙 연동으로 침투 |
| **Document360** | 지식베이스 특화 | $149-599/월 (프로젝트 단위) | 고객 지원 지식에 특화 |

**결론: "지식 관리"만으로 수익화하려면 per-seat로 조직 전체 침투 필수. 틈새 도구는 단독 생존 어려움.**

### 2.7 디지털 트윈

| 서비스 | 규모 | 현실 |
|--------|------|------|
| **Autodesk Tandem** | 대형 | 구축 비용 수억 원 |
| **Siemens Xcelerator** | 대형 | 제조 특화 |
| **AVEVA** | 대형 | 플랜트 특화 |
| **Azure Digital Twins** | 플랫폼 | PoC 중단 빈번 |

- 시장 전망: 2024년 $16B → 2030년 $110B (McKinsey)

- 주요 불만: 초기 비용, 통합 난이도, ROI 미입증, "멋진 3D 뷰어"에 그침

## 3. 한국 시장 특수성

### 3.1 구조적 장벽

| 장벽 | 설명 |
|------|------|
| **SI 관성** | 대기업·공공은 "구축형" 선호. SaaS 전환 극히 느림 |
| **IT 예산** | 건설사 IT 예산 = 매출의 0.5-1% (글로벌 대비 극히 낮음) |
| **보안 인증** | ISMS 필수, 클라우드 도입 시 보안 심사 |
| **한글/한국 규격** | 한글 도면, KS 규격, 전자결재 연동 필수 |
| **외산 실패** | Documentum, Aconex 도입 후 활용률 저조 사례 다수 |

### 3.2 한국 기존 플레이어

| 서비스 | 영역 | 현실 |
|--------|------|------|
| **일렉콤** | 플랜트 전기도면 | SaaS 전환 미흡, 특정 분야 한정 |
| **마이다스** | 구조해석+BIM | 도면 관리보다 설계 해석 중심 |
| **스마트건설** (정부) | BIM 의무화 | 민간 확산 느림, 표준 파편화 |

### 3.3 한국 시장의 기회

- BIM 의무화 확대 → 디지털 도면 관리 수요 증가
- 공공 EDMS 클라우드 전환 움직임
- **중소 건설사·시설관리 조직을 위한 한국형 클라우드 도면 SaaS = 시장 공백**
- 단, 이 세그먼트의 ARPU가 극히 낮아 수익화 난이도 높음

## 4. 수익 모델 패턴 정리

시장에서 검증된 수익 모델:

| 모델 | 설명 | 대표 서비스 | DKS 적용 가능성 |
|------|------|-------------|:---:|
| **Per-seat 구독** | 사용자 수 × 월/연 단가 | Procore, Notion, UpKeep | △ |
| **프로젝트 기반** | 프로젝트당 과금 | Aconex | ✕ (운영 단계라 프로젝트 개념 약함) |
| **사용량 기반** | 처리 페이지/문서 수 | Textract, Nanonets | O (PDF 업로드/분석) |
| **Freemium** | 무료 기본 + 유료 확장 | Fiix, Notion | △ |
| **플랫폼 수수료** | 생태계 내 앱/연동 수수료 | Procore Marketplace | ✕ (생태계 구축 비현실적) |
| **매출 연동** | 고객사 매출/규모 기반 | Procore | ✕ |

## 5. 생존과 실패의 패턴

### 5.1 인수/소멸된 서비스들

| 서비스 | 결말 | 원인 |
|--------|------|------|
| **PlanGrid** | Autodesk $875M 인수 (2018) | 단일 기능(도면뷰어)으로 플랫폼 경쟁 불가 |
| **Fieldwire** | Hilti 인수 (2022) | 현장관리 단일 기능의 한계 |
| **Aconex** | Oracle $1.2B 인수 (2017) | 대형 플랫폼 종속 |
| **Vela Systems** | Autodesk 인수 | 현장 검사 단일 기능 |

**공통 패턴:**
```
단일 기능 → 사용자 확보 → 플랫폼 대기업이 유사 기능 출시 → 차별화 상실 → 인수 or 도태
```

### 5.2 생존하는 서비스의 조건

1. **플랫폼화**: 하나의 기능이 아닌 여러 기능을 묶어 switching cost 높임 (Procore)
2. **생태계 lock-in**: 다른 도구와의 통합이 깊어서 빠져나오기 어려움 (Autodesk)
3. **도메인 깊이**: 특정 산업의 깊은 문제를 해결 (Nuvolo = 의료시설 특화)
4. **네트워크 효과**: 사용자가 늘수록 가치 증가 (Procore의 하도급 연결)

### 5.3 소규모 SaaS가 죽는 이유

| 요인 | 설명 |
|------|------|
| **낮은 ARPU** | SMB 대상 $50-200/월, CAC 회수에 18-24개월 |
| **높은 해지율** | 프로젝트 종료 시 해지, churn 5-10%/월 |
| **기능 흡수** | 대형 플랫폼이 유사 기능 출시하면 끝 |
| **IT 리터러시** | SMB 건설사 영업/온보딩 비용 과다 |

## 6. DKS에 대한 시사점

### 6.1 냉정한 현실 인식

| 사실 | 의미 |
|------|------|
| 도면 "관리"만으로는 독립 SaaS 생존 불가 | PlanGrid, Fieldwire 모두 인수로 귀결 |
| 한국 건설 IT 예산 극히 낮음 | per-seat 모델로 돈 벌기 어려움 |
| 대형사는 이미 EDMS 있음 | 신규 진입 공간 없음 |
| 중소는 돈 안 냄 | 무료~초저가만 수용 |

### 6.2 가능성이 있는 방향

| 방향 | 근거 | 리스크 |
|------|------|--------|
| **AI 문서 분석 usage-based** | Textract/Nanonets 모델 검증됨. PDF 업로드 → 설비 자동 추출 → 건당 과금 | AI 정확도가 가치를 좌우 |
| **CMMS 보완 플러그인** | CMMS에 없는 "설비 관계"를 제공하는 연동 모듈 | 단독 가치 약함, 종속적 |
| **중소 시설관리 경량 패키지** | EDMS 없는 조직에 "도면+설비+정비" 올인원 | 낮은 ARPU, 높은 churn |
| **도면 지식화 엔진 (B2B API)** | 도면 PDF를 넣으면 설비 그래프 JSON을 돌려주는 API | 시장 교육 필요 |

### 6.3 하면 안 되는 방향

| 방향 | 이유 |
|------|------|
| EDMS 풀스택 구축 | Procore, Aconex와 경쟁. 자원 대비 승산 없음 |
| DWG 뷰어 자체 개발 | Autodesk, ODA 독점. 품질 경쟁 불가 |
| 디지털 트윈 플랫폼 | 초기 비용 과다, adoption 5% 미만 |
| 범용 지식관리 | Notion, Confluence와 경쟁. 이길 수 없음 |

## 7. 불확실성 평가

### 7.1 확실한 것 (검증됨)

- 현장 엔지니어는 도면을 찾는 데 시간을 낭비하고 있다
- 설비 간 관계를 아는 사람은 경험 많은 소수뿐이다
- 기존 EDMS는 도면 안의 내용(설비, 관계)을 이해하지 못한다
- AI 문서 분석 시장은 실제로 돈이 되고 있다 (Textract 등)

### 7.2 불확실한 것 (검증 필요)

- AI가 한국어 설비 도면에서 태그/관계를 정확히 추출할 수 있는가?
- 추출된 지식 그래프의 품질이 현장에서 신뢰받을 수 있는가?
- 중소 시설관리 조직이 월 얼마까지 지불할 의사가 있는가?
- 설비 관계 탐색·영향도 분석이 실제 의사결정에 사용되는가?
- 한국 시장에서 클라우드 도면 관리에 대한 심리적 장벽은 얼마나 높은가?

### 7.3 부정적 시그널

- 한국 건설사 IT 예산이 매출의 0.5-1% → SaaS 지불 의사 극히 낮음
- 도면 관리 단독 SaaS는 글로벌에서도 대부분 인수/소멸
- Phase 0 프로토타입의 데이터가 125건 mock → 실 데이터 검증 제로
- 사용자 인터뷰 2회로는 시장 수요 검증에 불충분
- DWG/AI 제외 시 남는 기능이 너무 얇음

### 7.4 긍정적 시그널

- 한국 중소 시설관리에 적합한 클라우드 도면 SaaS = 시장 공백
- "설비 지식 그래프"는 기존 EDMS/CMMS 어디에도 없는 기능
- BIM 의무화 → 디지털 도면 관리 수요 구조적 증가
- AI 문서 분석 단가 지속 하락 → 진입 비용 감소
