## 전체 기능 맵

```
┌─ Layer 0: 즉시 고통 해결 ──────────────────────────────────┐
│                                                              │
│  [S] 도면 위 주석/메모          ← 박정민(팀장) 핵심 요구     │
│  [S] 도면 검색 강화             ← 김현수(현장) 핵심 요구     │
│  [S] PDF 인라인 뷰어            ← 전원 기본 기대             │
│  [A] PDF 업로드 + 페이지 분리   ← 이수진(문서) 핵심 요구     │
│  [A] 도면 분류 ���계/필터        ← 전원 기본 기대             │
│  [A] 모바일 반응형              ← 김현수(현장) 핵심 시나리오  │
│                                                              │
├─ Layer 1: 지식 축적 ────────────────────────────────────────┤
│                                                              │
│  [A] 설비↔도면 양방향 태깅      ← Layer 1 핵심 데이터 축적   │
│  [B] 도면 버전/Revision 관리    ← 이수진(문서) 요구          │
│  [B] 도면 비교 (Revision diff)  ← 경쟁 제품 핵심 기능        │
│  [B] 활동 이력/감사 로그        ← 최동훈(IT) 보안 요구       │
│                                                              │
├─ Layer 2: AI 자동화 ────────────────────────────────────────┤
│                                                              │
│  [C] AI 메타데이터 자동 추출    ← 이수진(문서) 꿈의 기능     │
│  [C] AI 설비 관계 자동 구축     ← 제품 비전의 핵심           │
│  [C] AI 중복 도면 감지          ← 이수진(문서) 요구          │
│                                                              │
└──────────────────────────────────────────────────────────────┘
```

### ISS-034 도면 위 주석/메모 (Annotation)

**근거**: 페르소나 분석 — 박정민(팀장): "퇴직자 경험을 도면 위에 기록하고 후임이 볼 수 있으면 가치가 크다." 이 기능이 **NAS+엑셀에 없는 한 가지**이며, Layer 1(지식 축��)의 핵심 데이터 소스.

**기능 정의**:
- 도면 이미지 위에 핀(마커)을 찍고 메모를 남김
- 핀 클릭 시 메모 내용 + 작성자 + 날짜 표시
- 메모에 설비 태그 연결 가능 (예: "이 위치의 밸브 CH-001-V3은 도면에 없지만 실제로 바이패스됨")
- 메모 목록 패널에서 전체 주석 일괄 조회

**구현 방법**:
```
사용 라이브러리: 이미 설치된 react-zoom-pan-pinch 확장
데이터 구조:
  interface Annotation {
    id: string;
    doc_id: string;          // 연결된 도면
    page: number;            // 페이지 번호
    x: number;               // 핀 x좌표 (0-1 정규화)
    y: number;               // 핀 y좌표 (0-1 정규화)
    text: string;            // 메모 내용
    author: string;          // 작성자
    created_at: string;      // 작성일
    tags?: string[];         // 연결된 설비 태그
    type: "info" | "warning" | "field-note";  // 유형
  }

구현 단계:
  1. DrawingViewer.tsx를 확장하여 클릭 시 핀 생성 지원
  2. annotations.json 데이터 파일 생성 (Phase 0: 정적 JSON)
  3. 핀 렌더링 오버레이 (절대 좌표 → 정규화 좌표로 zoom/pan 호환)
  4. 메모 입력 폼 (팝오버)
  5. 메모 목록 사이드 패널
```

**연결 파일**: `src/components/drawing/DrawingViewer.tsx`, `src/components/drawing/DrawingDetailPane.tsx`  
**예상 공수**: 2-3일  
**우선순위**: **S**

### ISS-035 도면 검색 강화

**근거**: 페르소나 분석 — 김현수(현장): "새벽 2시 긴급 콜. '냉동기 2호기 냉각수 배관'으로 검색해서 바로 뜨면 가치 있다." 현재 도면 검색은 리스트 스크롤만 가능. Fuse.js는 설비 검색에만 연결됨.

**기능 정의**:
- 도면 전용 검색바: 도면 제목, 설명(snippet), 학제, 연결 설비 태그로 통합 검색
- 필터 조합: 학제 + 도면 종류 + 위치/층 (다중 선택)
- 설비 역추적: 설비 태그 입력 → 해당 설비가 나오는 도면 목록 (엑셀에서 가장 어려운 것)
- 검색 결과 하이라이트

**구현 방법**:
```
기존 Fuse.js 활용 확장:
  - 현재: entities.json만 검색 대상
  - 확장: documents.json도 Fuse 인스턴스에 포함
  - 검색 키: title, snippet, discipline, 연결 설비 태그

필터 UI:
  - DrawingsPageView.tsx 상단에 필터바 추가
  - discipline 칩 필터 (MECHANICAL | ELECTRICAL | FIRE | FACILITY)
  - 도면 종류 드롭다운 (drawing | manual | spec | checklist | code)

설비→도면 역추적:
  - data-loader.ts에 tagToDocsIdx 인덱스 추가
  - Entity.source_pages → documents.json의 page 매칭
  - EntityCard에서 "관련 도면 보기" 링크 추가
```

**연결 파일**: `src/components/drawing/DrawingsList.tsx`, `src/components/shell/DrawingsPageView.tsx`, `src/lib/data-loader.ts`  
**예상 공수**: 1-2일  
**우선순위**: **S**

**기능 정의**:
- PDF 파일을 브라우저 내에서 직접 렌더링
- 페이지 네비게이션 (이전/다음, 페이지 번호 입력)
- 줌/팬 지원
- 이미지(PNG) 모드와 PDF 모드 ���동 전환

**구현 방법**:
```
방법 1 (간단): <iframe src="/drawings/xxx.pdf">
  - 장점: 구현 0줄. 브라우저 내장 PDF 뷰어 사용.
  - 단점: 스타일 커스텀 불가, 주석 오버레이 불가, 모바일 UX 약함.

방법 2 (권장): react-pdf (pdf.js 래퍼)
  - npm install react-pdf
  - 장점: 페이지 단위 렌더링, 줌/팬, 주석 오버레이 가능
  - 단점: 번들 크기 증가 (~500KB gzipped)
  - ISS-034(주석)과 통합 가능: PDF 페이지 위에 핀 오버레이

방법 3 (미래): @pdfme/viewer 또는 PSPDFKit (상업)
  - 마크업, 측정, 서명 등 고급 기능
  - 현 단계에서는 과잉

추천: Phase 0은 방법 1(iframe), 주석 기능 구현 시 방법 2로 전환
```

**연결 파일**: `src/components/drawing/DrawingViewer.tsx`, `src/components/drawing/DrawingDetailPane.tsx`  
**예상 공수**: 방법 1 = 0.5일, 방법 2 = 1-2일

```

Phase 1 전환 시:
  - S3/R2 파일 스토리지
  - DB 기반 메타데이터
  - 서버리스 이미지 변환 (AWS Lambda + Ghostscript layer)
```

- 다중 필터 UI: 칩/태그 형태로 조합 선택

**구현 방법**:
```
데이터 모델 확장 (types.ts):
  interface DocSnippet {
    // 기존 필드...
    drawing_number?: string;     // 도면 번호 (예: "E-SLD-001")

필터 UI:

  - 각 필터는 드롭다운 또는 칩 형태
```

### ISS-040 설비↔도면 양방향 태깅

**근거**: Layer 1(지식 축적)의 핵심. 사용자가 도면에 설비 태그를 연결하면, 반대로 설비에서 관련 도면을 찾을 수 있다. 이것이 엑셀에서 가장 어려운 것이며, 축적될수록 온톨로지가 된다.

**기능 정의**:
- 도면 상세에서: "이 도면에 나오는 설비" 태그 추가/제거
- 설비 상세(EntityCard)에서: "이 설비가 나오는 도면" 목록 표시
- 태그 자동완성: 기존 entities.json의 태그 목록에서 검색
- 연결 시 근거 페이지 번호 기록

**구현 방법**:
```
데이터 구조:
  interface DocEntityLink {
    doc_id: string;
    entity_tag: string;
    page?: number;         // 해당 설비가 나오는 페이지
    linked_by: string;     // 연결한 사람
    linked_at: string;     // 연결일
  }

양방향 인덱스 (data-loader.ts):
  docToEntitiesIdx: doc_id → entity_tag[]
  entityToDocsIdx:  entity_tag → doc_id[]

UI:
  DrawingDetailPane: 하단에 "연결된 설비" 섹션 + 태그 입력
  EntityCard: "관련 도면" 탭 추가 (현재 source_pages만 사용)
```

**연결 파일**: `src/lib/types.ts`, `src/lib/data-loader.ts`, `src/components/drawing/DrawingDetailPane.tsx`, `src/components/entity/EntityCard.tsx`  
**예상 공수**: 2일

### ISS-047 AI 중복 도면 감지

**근거**: 이수진(문서): "같은 도면 여러 번 받음." 업체가 동일 도면을 ��른 파일명으로 재제출하는 경우.

**기능 정의**:
- 업로드 시 기존 도면과 유사도 비교
- 이미지 유사도 (perceptual hash) + 메타데이터 유사도

```
임계값: 유사도 90% 이상 → 경고
```
