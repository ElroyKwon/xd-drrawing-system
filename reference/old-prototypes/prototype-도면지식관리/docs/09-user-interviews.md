## 회차 #2 — 2026-04-16

### 대상자
실제 현장 사용자 (익명)

### 원문

> 실제 사용자는  
> 업로드한 도면 목록이 있으면 좋겠다.  
> 도면 목록 옆에는 도면 설명 이 잇으면 좋겠다.  
> 좌우 반씩 정보 표시 공간에 이미지 설명 리스트를 보고 내가 선택해서 빠르게 찾아서 클릭해서 볼수 있으면 좋겠다.  
> 그리고 그 정보를 보면서 업로드 해 둔 dwg, pdf 파일들을 다운로드 해줄수 있게 해주면 좋겠다.

> 도면의 리스트 page 가 있을건데 거기서 클릭하면 바로 도면 상세가 보였으면 좋겠다.

> 각 업체들마다 올리는 도면은 제각기 정리되거나 마음대로 정보를 표시하는 것들이 있을거다. AI 그것들을 도면을 한장씩 인식해서 우리회사의 도곽 을 넣어서 다시 그려주거나, 혹은 도면의 그림 스타일을 우리회사가 지정한 형태로 다시 그려주거나 하는 기능들이 포함되었으면 좋겠다고 한다.

> 다시 그리는건 어렵겠지... 근데 아무튼 리스트로 가지고 가자고.

### 정리된 요구사항

| # | 요구사항 | 우선순위 | 연결 ISS |
|---|----------|----------|----------|
| R1 | 업로드 도면 목록 화면 | 높음 | ISS-026 |
| R2 | 좌우 반반 분할뷰 — 좌: 리스트+설명 / 우: 상세·뷰어 | 높음 | ISS-027 |
| R3 | 리스트 클릭 → 바로 상세 (중간 페이지 없이) | 높음 | ISS-029 |
| R4 | 원본 파일(DWG/PDF) 다운로드 버튼 | 높음 | ISS-030 |
| R5 | AI 도곽 삽입 / 회사 표준 스타일 재도시화 | 낮음(장기) | ISS-031 |

### 설계자 메모

- R1~R4: 이번 Phase 1에서 구현. 페르소나별 `/drawings` 라우트로 추가.
- R5(AI 재도시화): 실현 난이도가 높아 이번 단계에서는 **비활성 버튼 + 툴팁**으로 요구사항 존재만 시각화. 실제 기능은 별도 연구 과제.
- PDF 뷰어는 `<iframe>` mock 수준. DWG는 브라우저 직접 렌더 불가이므로 다운로드만 제공.
- 각 업체마다 도면 품질·형식이 다르다는 점은 향후 도면 표준화 파이프라인 설계 시 핵심 요구로 반영.

## 이 파일 사용 방법

- 새 인터뷰가 생기면 **상단에 새 회차 섹션 추가** (최신이 위)
- 대상자는 익명 처리 가능 ("현장 사용자 A", "관리자 B" 등)
- 원문은 **그대로 인용** (맞춤법 수정 없이)
- ISS 번호는 확정 후 기재, 미확정 시 "검토 중" 표기
