# DKS 도면지식관리 시스템 — 시연 발표 스크립트

> 대상: 현장 사용자 / 영업 임원  
> 소요 시간: 약 20분  
> 서버: `npm run dev` → http://localhost:3000  
> 캡쳐 이미지: `docs/test-records/screenshots/`

---

## 여는 말 (1분)

> "현장에서 설비 정보가 필요할 때, 어디서 찾으시나요?
> 도면 PDF를 열어서 페이지를 넘기거나, 전임자한테 전화하거나, 엑셀 대장을 뒤지시죠.
> DKS는 그 모든 걸 한 곳에 모아서 자연어로 물어볼 수 있게 만든 지식 엔진입니다.
> 오늘은 기계, 전기, 소방, 안전 — 4명의 담당자 입장에서 각각 어떻게 쓸 수 있는지 보여드리겠습니다."

> 각 분야의 핵심(CRITICAL) 설비 수도 함께 표시됩니다.

## 6. 마무리 — 데이터 품질 관리 (1분)

**설명 멘트:**
> "마지막으로 데이터 품질 관리에 대해 말씀드립니다.
> 이 시스템의 데이터는 3가지 상태로 관리됩니다.
> 첫째, 초록 신뢰도 80% 이상에 뱃지 없음 — AI 추출 + 인간 검수 완료. 가장 신뢰할 수 있습니다.
> 둘째, 회색 '미검수' 뱃지 — AI가 추출했지만 사람이 아직 확인하지 않은 상태. 참고는 하되 중요한 판단에는 한 번 더 확인하세요.
> 셋째, 노란 'HITL 검수 필요' 뱃지 — AI가 스스로 '이건 이상한 것 같다'고 표시한 것. 우선적으로 검토해야 합니다.
> 이 세 단계를 통해 사용자가 데이터를 맹신하지 않고 올바르게 활용할 수 있습니다."
