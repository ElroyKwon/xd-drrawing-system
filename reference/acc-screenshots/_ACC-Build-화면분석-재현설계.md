# ACC(Autodesk Construction Cloud) Build 화면 분석 — 재현 설계

> 목적: `acc.autodesk.com`(브랜딩상 **[[Autodesk Forma]]** / 제품 **[[Autodesk Build]]**) 화면 37장을 한 장씩 분석하여, **[[도면정보관리]] 서비스**를 동등 수준으로 재현하기 위한 컴포넌트·정보구조·기능 명세 도출.
> 분석 대상 스크린샷: 본 폴더(`수집자료/screenshot/`) 내 37장 (촬영 2026-06-12 10:21~11:42)
> 샘플 프로젝트: `Construction : Sample Project - Seaport Civic Center` (300 Mission St, San Francisco)

---

## 0. 제품 구조 한눈에 — 무엇을 베끼는가

ACC는 **단일 허브 + 모듈 스위처** 구조의 건설 SaaS. 우리가 재현 대상으로 삼는 핵심은 **Build 모듈의 시트(도면)·뷰어·이슈·마크업·파일** 5종.

```
[계정/허브 레벨]  Autodesk Forma 헤더
  ├─ My Home (개인 대시보드)
  ├─ 프로젝트 목록 / 프로젝트 작성 / 프로젝트 템플릿
  └─ Project Admin (구성원·회사·역할·권한)
        │  (프로젝트 선택 → 모듈 스위처로 Build 진입)
        ▼
[프로젝트 레벨]  Build 헤더 (좌측 모듈 사이드바)
  ├─ 홈        ← 위젯 대시보드
  ├─ 시트      ← 도면 목록 + 2D 뷰어 (★ 핵심)
  │     ├─ 마크업 (펜·도형·화살표)
  │     ├─ 이슈 (핀 배치 + 상세 폼)
  │     └─ 시트 비교 (버전 오버레이 diff)
  ├─ 파일      ← 폴더형 문서관리(CDE)
  ├─ 이슈      ← 이슈 전용 목록/필터/지도
  ├─ 양식      ← (체크리스트/점검, 미촬영)
  └─ 사진      ← 앨범/갤러리/지도
```

**DKS 매핑**: 우리의 `도면정보관리 서비스`는 이 중 **시트+2D뷰어+이슈+마크업**이 1순위 재현 대상. 차별점은 ACC에 **없는** 온톨로지 기반 "설비 검색·영향도 분석·AI 채팅"을 얹는 것 → ACC를 베끼되 **그 위에 우리 코어를 더한다**.

### C. 시트(도면) 목록 & 2D 뷰어 ★ 최우선 재현

## 3. 정보구조(데이터 모델) 도출

```
Hub(허브) 1 ── N Project(프로젝트)
Project ── N Member(구성원: 회사·역할·권한)
Project ── N Sheet(시트: 번호·공종·태그·버전세트·썸네일·도면원본)
  Sheet ── N Version(버전: 버전세트로 그룹)
  Sheet ── N Markup(마크업: type[pen|rect|circle|line|arrow|text], geometry, color, author)
  Sheet ── N IssuePin(이슈핀: {x,y} 도면좌표)
Project ── N Issue(이슈)
  Issue: 템플릿, 제목, 상태, 유형(2단 계층), 설명, 담당자, 확인자,
         위치, 마감일, 첨부[], 참조[], 근본원인, 댓글[], audit_log[]
Project ── N File(파일: 폴더트리, 버전, 공유상태, 연결이슈, 크기)
Project ── N Photo(미디어: 앨범, 지오태그)
```

> **DKS 우위 지점**: ACC의 Sheet/Markup/Issue는 전부 **도면좌표(픽셀)** 기반. 우리는 여기에 **온톨로지 엔티티 ID**를 1차 키로 더해, 핀·이슈·마크업을 "픽셀"이 아닌 "**설비 X**"에 묶는다 → 검색·영향도 분석·AI 채팅과 자동 연동.
