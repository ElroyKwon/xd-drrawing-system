# 파일(CDE) 기능요구

## 1. 목적
- 청주 사업장 [[도면관리시스템]]의 **파일 모듈** = 폴더형 [[CDE]](Common Data Environment) 재현 기능요구 정의.
- 벤치마크: [[ACC Build]] 파일 모듈(폴더 트리 + 파일 테이블 + 버전 + 공유).
- 소유 범위(이 문서만): **폴더트리 / 버전 / 공유 / 업로드**. 뷰어·마크업·이슈는 위임.

## 2. 배경/전제
- ★스코프 제약: [[DWG]] 원본 **편집 금지**. 파일 모듈은 "저장·버전·배포" 담당, 렌더는 [[01-3-2_시트-도면뷰어]]에 위임.
- 파일 = 진실원천(SoT). 도면 파싱 산출물(분야·도면번호·태그)은 파일 메타데이터로 연결.
- 1차 = 2D 시트 한정. 3D/BIM Bridge 범위 밖.
- 차별점: [[ACC Build]] 폴더는 픽셀/경로 기반이지만, 우리는 파일 메타를 **온톨로지 엔티티 ID**에 연결 가능(향후, [[TypeDB]]).

---

## 3. 폴더 트리 / 표준 폴더
> 좌측 폴더 패널 + 우측 파일 테이블 2-pane 레이아웃. 채울 것: 트리 구조·표준 폴더 템플릿·권한 경계.

### 3.1 폴더 트리 UI
- 트리 노드: 확장/축소, 선택 시 우측 테이블 필터링. 무엇을 채울지: 가상 스크롤·드래그 이동 여부 명시.

### 3.2 표준 폴더 템플릿
- 프로젝트 생성 시 기본 폴더 골격(예: 분야별/단계별). 채울 것: 청주 사업장 실제 분야 분류(전기/기계/배관 등) 확정 후 매핑.
- ❓TBD: 표준 폴더 = ACC식 "도면영역(Sheet folder)" vs 자유 폴더 — 선택 필요.

### 3.3 폴더 권한
- 폴더 단위 읽기/쓰기/관리 권한 경계. 상세 RBAC는 [[01-3-9_권한-RBAC]]로 위임(존재 시).

### 3.4 폴더 작업(생성/이동/이름변경/삭제)
- 컨텍스트 메뉴 액션 목록. 삭제 = soft-delete(휴지통) 여부 명시.

---

## 4. 파일 테이블 (버전/공유상태/이슈/크기)
> 공통 [[데이터테이블]] 컴포넌트 재사용(정렬·컬럼토글·필터·내보내기). 이 절은 컬럼 정의만 소유.

### 4.1 컬럼 정의
| 컬럼 | 내용 | 비고 |
|---|---|---|
| 이름 | 파일명 + 타입 아이콘 | DWG/PDF/IMG |
| 버전 | 현재 버전(V1, V2…) | 클릭 시 버전 이력 |
| 공유상태 | 공개/제한/링크 | §6 참조 |
| 이슈 | 연결 이슈 수 배지 | [[01-3-5_이슈]] 연동 |
| 크기 | 파일 크기 | |
| 수정일/수정자 | 최근 변경 | |
- 채울 것: 정렬 기본값·컬럼 토글 기본 노출 셋.

### 4.2 행 액션 / 인스펙터
- 행 클릭 → 우측 인스펙터(미리보기·메타·버전·이슈 요약). 공통 인스펙터 패턴은 [[01-3-1_공통-UI-컴포넌트]] 위임.
- 더블클릭 → 뷰어 진입([[01-3-2_시트-도면뷰어]]).

### 4.3 빈 상태 / 로딩
- 빈 폴더·검색 무결과 빈상태 카피. 공통 패턴 위임.

## 5. 업로드 / 버전 관리
> 업로드 플로우 + 동일 파일명 → 버전 누적 규칙. 핵심 비즈니스 로직 절.

### 5.1 업로드 플로우
- 드래그앤드롭 + 파일 선택. 다중 업로드·진행률·실패 재시도. 채울 것: 허용 확장자·최대 크기 TBD.
- 업로드 직후 백엔드 파싱 트리거(DWG→기하 JSON). 파싱 상세는 [[01-3-2_시트-도면뷰어]]/파싱 파이프라인 문서 위임.

### 5.2 버전 관리 규칙
- 동일 폴더·동일 파일명 재업로드 = **신규 버전 누적**(덮어쓰기 아님). 이전 버전 보존.
- 버전 메타: 버전번호·업로더·시각·코멘트. 채울 것: 버전 롤백/현재버전 지정 가능 여부.

### 5.3 버전 이력 뷰
- 버전 목록 + 각 버전 다운로드/뷰어. 두 버전 비교는 [[01-3-6_시트비교]]로 위임(diff 오버레이).

### 5.4 다운로드 / 내보내기
- 원본 DWG/PDF 다운로드. 일괄 다운로드(zip) 여부 명시.

## 7. RAGflow 수집 연계 (향후 / Phase 2)
> CDE에 들어온 파일 → 지식 인덱싱 파이프라인 진입점. 이 문서는 "연계 지점"만 소유, 파이프라인 본체는 위임.

### 7.1 수집 트리거
- 업로드/버전 확정 시점 → [[RAGflow]] 수집 큐 등록(이벤트 발행, [[WebSocket]]/메시지). 채울 것: 동기 vs 비동기.

### 7.2 메타·엔티티 바인딩
- 파일 메타(분야·도면번호·태그) + 향후 [[TypeDB]] 온톨로지 엔티티 ID를 인덱싱 단위로 전달.
- 이것이 [[ACC Build]] 대비 차별점: 파일이 "엔티티 단위 검색·영향도"로 연결됨.

### 7.3 인덱싱 상태 표시
- 파일 테이블에 "지식화 상태" 컬럼(미수집/처리중/완료) 향후 추가. Phase 1 비노출.
