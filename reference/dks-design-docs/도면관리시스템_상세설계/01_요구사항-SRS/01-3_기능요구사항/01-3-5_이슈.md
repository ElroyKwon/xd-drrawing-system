# 이슈 기능요구

## ① 목적
- 청주 사업장 도면관리 시스템의 **이슈(Issue) 모듈** 기능요구 정의.
- [[ACC Build]] Build 제품의 이슈 모듈(핀 배치 + 상세폼 + 목록 + 필터 + 지도 뷰) 동등 재현.
- 소유 범위: **핀 배치 / 상세폼 / 목록 / 필터 / 지도** — 이것만 다룸. 나머지는 위임.

## ② 배경 / 전제
- 핵심 스코프 제약: [[DWG]] 원본 미편집. 흐름 = DWG → 백엔드 파싱 → 기하 JSON → 웹 2D 뷰어 렌더 → 그 위 **오버레이**. 이슈 핀도 픽셀 좌표 오버레이.
- 이슈는 시트(2D 도면) 위 좌표에 핀으로 고정. 뷰어 렌더/좌표계는 [[01-3-2_시트-도면뷰어]] 소유 → 본 문서는 이슈 데이터·UI만.
- 펜/도형/화살표 등 그리기 도구는 [[01-3-4_마크업]] 소유. 이슈 핀은 마크업과 별개 객체(상태·담당자·워크플로 보유).
- 1차 범위 = 2D 시트 한정. 3D BIM 핀 범위 밖.
- ★DKS 차별점: 핀을 픽셀 좌표뿐 아니라 **설비 엔티티 ID**에도 바인딩 가능(Phase 0 파싱 산출물의 분야·도면번호·태그 활용). Phase 2에서 [[TypeDB]] 온톨로지 연동 시 영향도·AI채팅의 앵커가 됨. → 해당 절에서 언급.

## ③ 상세 목차 (채울 골격)

## 이슈 생성 (템플릿 / 유형 2단계층 / 핀)
- 이슈 생성 진입점·기본 플로우 정의.

### 생성 진입점
- 뷰어 툴바 "이슈 추가" 버튼 → 핀 드롭 모드 / 목록 화면 "신규" 버튼(핀 없는 일반 이슈).

### 유형 2단계 계층 (Type / Subtype)
- 상위 유형(예: 품질·안전·설계변경·간섭) → 하위 유형 선택 구조.
- 유형별 기본 템플릿(필수 필드·기본 담당자) 연결.
- ❓ 청주 사업장 표준 유형 분류 체계 = TBD (발주처 협의).

### 이슈 템플릿
- 유형 선택 시 필드 프리필·필수항목 세트 자동 적용.
- ❓ 템플릿 관리(관리자 편집) 1차 포함 여부 = TBD.

### 핀 배치 플로우
- 핀 드롭 모드 → 시트 위 클릭 → 좌표 캡처 → 인라인 상세폼 오픈.
- 핀 좌표 = 뷰어 좌표계(model space) 기준 저장(픽셀 아님, 줌 불변). 좌표 변환은 [[01-3-2_시트-도면뷰어]] 위임.
- (DKS) 클릭 지점 근처 설비 엔티티 자동 후보 제시 → 엔티티 ID 바인딩 옵션.

## 이슈 필드 (상태 / 담당자 / 확인자 / 마감 / 위치)
- 이슈 데이터 모델의 핵심 필드 정의.

### 상태(Status) 워크플로
- 상태 값·전이 규칙(예: 초안→오픈→진행→답변완료→닫힘). ACC 상태 매핑 정리.
- ❓ 상태 전이 권한(누가 닫을 수 있나) = TBD.

### 담당자 / 확인자(Assignee / Reviewer)
- 단일 담당자 + 확인자 분리. 역할 기반 선택(개인/회사/역할).

### 마감일(Due date) / 일정
- 마감일 필드·지연(overdue) 표시 규칙.

### 위치(Location) 필드
- 시트 참조(어느 도면)·핀 좌표·(DKS) 설비 엔티티 ID.
- 영역/구역 태그(존) = TBD.

#### 필드 가시성 / 필수 규칙
- 유형별 필수/선택·읽기전용 매트릭스. 표로 채울 예정.

## 첨부 / 참조 / 근본원인 / 댓글
- 이슈 본문 외 부속 데이터 정의.

### 첨부(Attachments)
- 사진·문서 첨부. 파일 저장은 [[CDE]]/파일 모듈 위임([[01-3-6_파일]]), 본 문서는 연결 메타만.

### 참조(References)
- 다른 시트·파일·다른 이슈 링크. (DKS) 설비 엔티티 참조.

### 근본원인(Root cause)
- 근본원인 분류 필드(유형별 코드 목록). ❓ 분류 체계 = TBD.

### 댓글(Comments / Activity)
- 스레드형 댓글, @멘션, 첨부 인라인.
- 실시간 갱신 = [[WebSocket]] 푸시 여부 TBD(1차는 폴링 가능).

## 목록 / 필터 / 지도 / 내보내기
- 이슈 조회 3종 뷰(목록·지도·보드)와 출력.

### 목록 뷰(Data Table)
- 데이터테이블 공통 컴포넌트 재사용 — 정렬·컬럼토글·페이지네이션은 [[01-3-8_공통UI]] 위임. 본 문서는 이슈 컬럼 셋·기본 정렬만 정의.

#### 우측 인스펙터(상세 패널)
- 행 선택 시 우측 패널에 상세폼 표시(공통 인스펙터 패턴 재사용).

### 필터(Filter)
- 필터 차원: 상태·유형·담당자·마감·시트·생성일. 저장된 필터(saved view) 1차 포함 여부 TBD.
- (DKS) 설비 엔티티/분야 기준 필터 — Phase 2 온톨로지 연동 시 강화.

### 지도 뷰(Map / Sheet pin view)
- 시트 위 핀 분포를 한눈에. 시트 선택 → 핀 클러스터 → 핀 클릭 시 상세.
- 핀 색상 = 상태별. 줌/팬은 뷰어 위임.

### 내보내기(Export)
- CSV/PDF 내보내기. 보고서 양식 = TBD(발주처 양식 확인 필요).
