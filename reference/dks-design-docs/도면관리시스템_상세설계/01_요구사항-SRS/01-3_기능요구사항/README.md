# 기능 요구사항 개요

## ① 목적

- 청주 사업장 납품용 [[도면관리 시스템]] 의 **모듈별 기능요구사항 인덱스** 소유 문서.
- 이 폴더(`01-3_기능요구사항/`)의 **진입점·역인덱스** 역할만 수행. 개별 모듈 상세 FR은 각 하위 문서로 위임(DRY).
- [[Autodesk Construction Cloud]](ACC) Build 제품의 도면관리 핵심 기능을 **동등 재현**하는 1차 범위 FR 전체를 한눈에 매핑.

## ② 배경 / 전제

- **★핵심 스코프 제약 (모든 FR의 대전제)**
  - [[DWG]] 원본을 **편집하지 않는다**. 흐름: DWG → 백엔드 파싱 → 기하 JSON → 웹 2D 뷰어 렌더 → 마크업/이슈/이벤트 **오버레이**.
  - 본 시스템은 CAD 편집기가 **아니라** "뷰어 + 주석/이벤트 오버레이" 시스템.
  - 정밀 CAD 편집(osnap 스냅, 좌표 편집, [[SHX]] 폰트 정밀 렌더)은 **범위 밖**.
  - 1차 범위 = **2D 시트(도면) 한정**. 3D BIM 뷰어/Bridge 범위 밖.
- **벤치마크**: [[ACC Build]] 37화면 분석 → [[_ACC-Build-화면분석-재현설계]].
- **DKS 차별점**: 핀/마크업/이슈를 픽셀 좌표뿐 아니라 **설비 엔티티 ID**에 바인딩. [[TypeDB]] 온톨로지 기반 엔티티 단위 검색·영향도·AI채팅은 **Phase 2** (→ [[Phase2-온톨로지연동]]).
- Phase 0 도면 파싱 산출물(분야·도면번호·태그) 기존 존재 → FR에서 메타데이터 소스로 전제.

## ③ 상세 목차

## 기능 모듈 목록

> 모듈별 1줄 책임 + 하위 FR 문서 링크 표. 각 모듈은 자체 상세 문서를 소유하고 여기선 인덱스만 유지.

### 핵심 모듈 (ACC Build 재현 대상)

#### M1. 홈 (위젯 대시보드)
- 프로젝트 진입 시 위젯 카드 대시보드(최근 시트·이슈 요약·활동 피드).
- 채울 것: 위젯 종류 목록, 사용자별 레이아웃 저장 범위. → [[FR-M1-홈]]

#### M2. 시트 (도면 목록 + 2D 뷰어)
- 도면 목록(데이터테이블) + [[PDF.js]]/Canvas 기반 2D 뷰어. 본 시스템의 중핵.
- 채울 것: 목록 컬럼·필터, 뷰어 팬/줌/회전, 레이어 토글, 측정 도구 범위. → [[FR-M2-시트]]

#### M3. 마크업 (펜·도형·화살표)
- 뷰어 위 주석 오버레이(펜·도형·화살표·텍스트). 벡터 좌표 저장, DWG 비편집.
- 채울 것: 마크업 타입·스타일·버전 귀속·권한. → [[FR-M3-마크업]]

#### M4. 이슈 (핀 + 상세폼 + 목록 + 필터)
- 도면 위 핀 배치 + 이슈 상세폼/목록/필터. **★엔티티 ID 바인딩 후크 위치**.
- 채울 것: 이슈 필드·상태흐름·담당자·핀↔엔티티 연결 필드. → [[FR-M4-이슈]]

#### M5. 시트 비교 (버전 오버레이 diff)
- 두 버전 도면 오버레이 + 변경 diff 하이라이트.
- 채울 것: 정합 방식(좌표 기준), diff 시각화, 비교 대상 선택 UX. → [[FR-M5-시트비교]]

#### M6. 파일 (폴더형 CDE)
- 폴더 트리 기반 [[CDE]](공통데이터환경) 파일 저장소. 업로드/버전/권한.
- 채울 것: 폴더 구조, 버전 이력, 권한 모델, 시트와의 연동. → [[FR-M6-파일]]

### 공통 UI 기능 (전 모듈 횡단)

> 앱셸·데이터테이블·인스펙터 등 횡단 UI는 모듈 FR이 아닌 **공통 UI FR**로 분리하여 중복 제거.

- 앱셸: 모듈 사이드바 + 프로젝트 스위처. → [[FR-공통-앱셸]]
- 데이터테이블: 정렬·컬럼토글·필터·내보내기. → [[FR-공통-데이터테이블]]
- 우측 인스펙터 패널 / 위젯카드 / 모달폼 / 빈상태 / 토스트. → [[FR-공통-UI패턴]]

## 요구사항 작성 양식 (FR-ID)

> 팀이 즉시 따라 쓸 FR 표기 규약. 모든 하위 모듈 문서가 이 양식 준수.

### FR-ID 명명 규칙

- 형식: `FR-<모듈코드>-<3자리>` (예: `FR-M2-001`, `FR-M4-012`).
- 모듈코드: M1~M6, 공통은 `CMN`.
- 채울 것: 번호 채번 정책(절판/재사용 금지 등), 하위 분할 표기(`-a/-b`). ❓ 채번 자릿수 3자리 확정?

### FR 항목 필수 필드

> 각 FR 1건당 작성할 표준 필드 정의(표 양식 1세트).

- `ID` / `요구사항명` / `설명(개조식)` / `우선순위(Must·Should·Could)` / `검증방법` / `ACC 대응화면` / `비고`.
- 채울 것: 우선순위 등급 정의, 검증방법 카테고리(데모/리뷰/테스트).

### 추적성 (Traceability)

> FR ↔ ACC 화면 ↔ 설계 ↔ 테스트 연결 규칙. 양방향 링크로 누락 방지.

- FR → ACC 37화면 분석 항목 역참조 필수.
- 채울 것: 추적 매트릭스 위치, 갱신 책임자. ❓ 매트릭스를 별도 문서로 둘지 표로 둘지 TBD.
