# 마크업 기능요구 (SRS)

## ① 목적
- [[ACC Build]] Build의 마크업(펜·도형·화살표·텍스트 주석) 기능 동등 재현 요구 정의.
- 소유 범위: **주석 객체 자체**의 생성·편집·목록·권한. 핀/이슈 연계는 [[01-3-5_이슈]]에, 렌더 기반은 [[01-3-2_2D뷰어]]에 위임.

## ② 배경 / 전제
- ★스코프: [[DWG]] 원본 비편집. 흐름 = DWG → 백엔드 파싱 → 기하 JSON → 웹 2D 뷰어 렌더 → **그 위에 마크업 오버레이**. 정밀 CAD 편집(osnap 스냅·좌표 편집·SHX 정밀 렌더)은 범위 밖.
- 마크업 = 뷰어 좌표계 위의 독립 주석 레이어. 원본 도면 기하와 분리 저장.
- 1차 범위 = 2D 시트 한정. 3D/BIM 범위 밖.
- 차별점(향후): 마크업을 픽셀 좌표뿐 아니라 **설비 엔티티 ID**에도 바인딩 가능 → [[TypeDB]] 온톨로지 연계는 Phase 2, 본 문서는 좌표 기반 + 엔티티 바인딩 필드 예약만.

## ③ 상세 목차

## 마크업 도구셋
- ACC Build 도구 팔레트 동등 재현 요구. 각 도구별 입력 제스처·옵션·렌더 산출물 정의.

### 펜 (자유 곡선)
- 드래그 경로 점열 → polyline 저장. 색·두께 옵션. (정밀 부드러움 처리는 TBD)

### 도형
#### 사각형 / 원·타원
- 시작-끝 드래그 박스. 채움 on/off, 외곽선 색·두께.
#### 직선 / 폴리라인
- 클릭 점 추가, 더블클릭 종료. 스냅 없음(스코프 제약).
#### 클라우드(구름선)
- 리비전 표기용. 영역 지정 → 구름 외곽 자동 생성. 곡률 파라미터 TBD.

### 화살표
- 시작-끝 벡터. 머리 스타일·크기. 지시선(leader) 용도.

### 텍스트
- 클릭 위치에 텍스트 박스. 폰트·크기·색. **SHX 정밀 폰트 미지원** → 웹 폰트로 대체(스코프 제약 명시).
- 콜아웃(말풍선) 변형: 텍스트 + 지시선 결합. 포함 여부 TBD.

### 공통 스타일 속성
- 색상 팔레트, 선 두께, 불투명도, 채움. 기본값/마지막 사용값 유지 정책 TBD.

### 좌표·바인딩 모델
- 저장 좌표계 = 도면 월드 좌표(뷰어 zoom/pan 무관 고정). 픽셀 아님.
- 예약 필드: `entityId`(설비 엔티티 바인딩, Phase 2), `sheetId`, `pageVersion`. → [[01-3-2_2D뷰어]] 좌표 변환과 정합 필요.

## 객체 편집 (이동/복사/잠금/삭제)
- 생성 후 주석 객체 단위 조작 요구. CAD 편집 아님 — 오버레이 객체의 단순 변환.

### 선택 / 다중 선택
- 클릭 선택, 드래그 박스 다중 선택. 선택 핸들 표시.

### 이동 / 크기조정 / 회전
- 핸들 드래그. 회전 지원 여부 TBD(텍스트·화살표 한정?).

### 복사 / 붙여넣기 / 복제
- 동일 시트 내 복제. 시트 간 복사 지원 여부 TBD.

### 잠금 / 숨김
- 잠금 시 편집 불가·선택만. 레이어 가시성 토글.

### 삭제 / 실행취소
- 단건·다중 삭제. Undo/Redo 스택 범위(세션 한정 vs 영속) TBD.

### 동시 편집 충돌
- 다중 사용자 동시 마크업 시 [[WebSocket]] 실시간 반영. 충돌 해결(last-write-wins vs 객체 락) TBD. → [[01-3-2_2D뷰어]] 실시간 채널 공유.

## 마크업 목록 / 검색
- 시트별·전체 마크업을 데이터테이블로 조회. 공통 UI [[01-2_공통UI]] 데이터테이블 재사용.

### 목록 컬럼
- 작성자·작성일·시트·도구유형·색상·연결 이슈 여부. 컬럼 토글·정렬.

### 필터
- 작성자, 도구유형, 시트, 기간, 이슈 연결 유무. 빈 상태 처리.

### 검색
- 텍스트 마크업 내용 전문 검색. 향후: 엔티티 ID 기준 검색(Phase 2).

### 내보내기
- 마크업 포함 [[PDF]] 출력 / 마크업 메타 CSV. 범위·형식 TBD.

### 뷰어 연동
- 목록 항목 클릭 → 해당 시트·좌표로 점프(zoom-to). 인스펙터 패널에 상세.

## 권한 / 소유권
- 누가 마크업을 만들고·수정·삭제하는지 규칙. [[01-3-5_이슈]]·프로젝트 권한 모델과 정합.

### 소유권 모델

### 역할별 권한
