# 시트(도면목록) 기능요구

## ① 목적
- 청주 사업장 [[도면관리 시스템]] "시트" 모듈 중 **도면 목록 화면**의 기능 요구를 확정.
- 벤치마크 = [[ACC Build]] Sheets 목록 화면 동등 재현. 차별점 = 분야·도면번호·태그 기반 검색 + 향후 [[온톨로지]] 엔티티 바인딩.
- 소유 범위(이 문서만): **도면 목록 / 썸네일 / 공종 / 버전세트 / 검색·필터·정렬 / 격자·리스트 뷰 / 내보내기**.

## ② 배경 / 전제
- 핵심 스코프: [[DWG]] 원본 **편집 금지**. 흐름 = DWG → 백엔드 파싱 → 기하 JSON → 웹 2D 뷰어 렌더 → 오버레이(마크업·이슈). 본 화면은 그 **진입 목록**.
- 1차 범위 = **2D 시트 한정**. 3D BIM/Bridge 범위 밖.
- [[DKS]] Phase 0 도면 파싱 산출물(분야·도면번호·태그)이 이미 존재 → 본 목록의 메타데이터 소스.
- 목록에서 시트 선택 → 뷰어 진입은 [[01-3-3_2D뷰어-렌더링]] 소관(여기선 진입 트리거만 정의).
- 공통 UI(데이터테이블·위젯카드·빈상태·토스트)는 [[01-2_공통UI-컴포넌트]] 위임, 본 문서는 시트 고유 동작만.

## ③ 상세 목차

## 목록 표시(썸네일·번호·공종색·태그)
> 각 시트 행/카드가 보여줄 메타데이터 항목과 표시 규칙 정의.

### 표시 데이터 항목
> 도면번호 / 도면명 / 공종(분야) / 현재 버전 / 업로드일 / 작성자 / 상태 / 태그 / 썸네일. 각 항목의 출처(파싱 산출물 vs 사용자 입력) 1줄 명기.

### 썸네일 생성·표시
> 백엔드 파싱 시 시트별 미리보기 이미지 생성 전략(서버 렌더 vs 뷰어 캡처). 캐싱·해상도·플레이스홀더(미생성 시).

#### 썸네일 미생성/실패 처리
> 파싱 실패·미지원 엔티티 도면의 폴백 아이콘 + 재생성 트리거.

### 공종(분야) 색상 코딩
> 분야별 색상 매핑 규칙(건축/구조/전기/기계/소방 등). 색상 정의 소스는 [[01-2_공통UI-컴포넌트]] 토큰 참조, 매핑 표만 여기 소유.

### 태그 표시
> Phase 0 산출 태그 + 사용자 추가 태그 구분 표시. 태그 클릭 → 필터 연동(아래 검색절).

### 상태 배지
> 시트 상태값 정의(예: 작성중/검토/승인/폐기 — TBD). 배지 색·우선순위.

## 버전세트 그룹핑
> 같은 도면번호의 여러 버전을 하나의 "버전세트"로 묶는 규칙.

### 버전세트 정의·식별 키
> 그룹핑 키 = 도면번호 기준(원칙). 키 충돌·도면번호 누락 시 폴백 규칙(TBD).

### 최신 버전 대표 표시
> 목록 기본은 버전세트당 최신 1개 대표 행. 대표 선정 기준(업로드일 vs 버전라벨).

### 버전 펼치기/이력 보기
> 대표 행 확장 시 하위 버전 목록. 각 버전 → 뷰어/시트비교 진입. 시트비교는 [[01-3-5_시트비교-버전오버레이]] 위임.

### 버전 업로드 시 세트 자동 매칭
> 신규 업로드 도면을 기존 세트에 자동 귀속하는 로직 위치(파싱 단계 vs 목록 단계). ❓ 자동 vs 수동 매칭 확인 필요.

## 검색 / 필터 / 정렬
> 목록 상단 검색바 + 필터칩 + 정렬 컨트롤의 동작 명세.

### 텍스트 검색
> 검색 대상 필드(도면번호·도면명·태그). 부분일치·대소문자·한글 처리. 검색 응답 위치(클라이언트 필터 vs 서버 쿼리) ❓.

#### 엔티티 단위 검색(차별점·향후)
> ACC엔 없는 "설비 엔티티 ID/명" 검색. Phase 2 [[TypeDB]] 온톨로지 연동 시 핀·마크업이 엔티티에 바인딩되는 구조의 진입점. 1차는 자리만 확보.

### 필터
> 공종·상태·태그·버전유무·기간(업로드일) 다중 필터. 필터칩 AND/OR 규칙. 필터 상태 URL/세션 보존 여부.

### 정렬
> 정렬 가능 컬럼(도면번호·도면명·업로드일·공종). 기본 정렬 = TBD. 다중 정렬 지원 여부 ❓.

### 빈 결과 처리
> 검색·필터 결과 0건 시 빈상태 화면 + 필터 초기화 액션. 빈상태 컴포넌트는 [[01-2_공통UI-컴포넌트]] 위임.

## 격자 / 리스트 뷰
> 두 가지 보기 모드 토글과 각 모드의 레이아웃.

### 격자(썸네일 카드) 뷰
> 카드 1개 구성(썸네일 + 도면번호 + 공종색 + 태그). 카드 크기·반응형 컬럼 수.

### 리스트(데이터테이블) 뷰
> 컬럼 구성·정렬·컬럼 토글. 테이블 동작은 [[01-2_공통UI-컴포넌트]] 데이터테이블 상속, 시트 고유 컬럼만 정의.

### 뷰 모드 토글·기본값·상태 유지
> 토글 위치, 기본 뷰 모드(TBD), 사용자 선택 보존(세션/로컬).

### 다중 선택·일괄 액션
> 체크박스 다중 선택 → 일괄 다운로드/내보내기/상태변경. 일괄 액션 범위 ❓(상태변경 포함 여부).

## 내보내기
> 목록·선택 시트를 외부로 추출하는 기능.

### 내보내기 대상·형식
> 목록 메타데이터 CSV/Excel 내보내기 + 시트 원본/[[PDF]] 다운로드 구분. 형식 범위 TBD.

### 내보내기 범위
> 전체 목록 vs 현재 필터 결과 vs 선택 항목. 기본 동작 정의.

### 원본 다운로드 권한
> DWG 원본 다운로드 허용 여부·권한. ❓ 원본 반출 정책(보안) 확인 필요.
