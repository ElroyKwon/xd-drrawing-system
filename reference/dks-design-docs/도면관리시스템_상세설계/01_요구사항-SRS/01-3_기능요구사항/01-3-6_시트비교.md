# 시트 비교 기능요구

## 1. 목적
- 동일 도면(시트)의 **두 버전**을 겹쳐 차이를 시각적으로 식별하는 기능 정의
- ACC Build "Sheet Compare" 동등 재현 — 버전 오버레이 diff(추가/삭제 색상 구분)
- 소유 범위: **버전 오버레이 diff 한정**. 2D 렌더 엔진·시트 목록은 [[01-3-2_시트(도면목록·2D뷰어)]]에 위임, 마크업은 [[01-3-3_마크업]]에 위임

## 2. 배경 / 전제
- 핵심 스코프 제약: [[DWG]] 원본 비편집. 흐름 = DWG → 백엔드 파싱 → 기하 JSON → 웹 2D 뷰어 렌더 → 그 위 오버레이. 비교도 **렌더 결과 위 오버레이**로 처리, 원본 미변경
- 비교 대상 = 2D 시트 한정(3D BIM 비교는 범위 밖)
- 버전 모델은 [[CDE]] 파일 버전 체계 전제 (V1/V2/V3…). 버전 정의·승격은 본 문서 소유 아님 → [[01-3-7_파일(폴더형CDE)]] 위임
- 차별점(향후): 픽셀 diff를 넘어 "설비 엔티티 ID" 단위 변경 감지를 [[TypeDB]] 온톨로지와 바인딩(Phase 2). 본 SRS는 픽셀/벡터 diff 1차 정의, 엔티티 diff는 §6에서 훅만 명시

## 3. 비교 대상 선택 (시트 피커)
> 채울 내용: 두 버전을 고르는 진입 UI/UX와 데이터 흐름

### 3.1 진입 경로
- 시트 상세 뷰어에서 "비교" 버튼 → 비교 모드 진입. 홈 위젯/시트 목록에서의 진입 가능 여부 TBD

### 3.2 좌/우 버전 선택 UI
- 좌측(Base) · 우측(Compare) 2-컬럼 시트 피커. 동일 시트의 버전 드롭다운 + 썸네일 미리보기
- 기본값: 좌=직전 버전, 우=최신 버전 (❓ 기본 매칭 규칙 확정 필요)

### 3.3 시트 매칭 규칙
- 동일 시트 식별 키 = 도면번호(파싱 산출물). 도면번호 누락/변경 시 폴백 규칙 TBD

### 3.4 빈/예외 상태
- 버전 1개뿐 → 비교 불가 빈상태. 페어링 실패 토스트. 공통 UI 패턴은 [[_ACC-Build-화면분석-재현설계]] 참조

## 4. 오버레이 diff (추가/삭제 색상)
> 채울 내용: 두 버전을 겹쳤을 때 차이를 색으로 표현하는 핵심 규칙

### 4.1 diff 색상 규약
- 추가(Compare에만 존재) = 색상 A, 삭제(Base에만 존재) = 색상 B, 동일 = 중립/저채도 (구체 색상값 TBD — [[99. 디자인md]] 토큰 따름)

### 4.2 diff 산출 방식
#### 4.2.1 래스터(픽셀) diff
- 두 버전 렌더 이미지 픽셀 비교. 단순/빠름·정렬 민감. 1차 채택 후보
#### 4.2.2 벡터(기하 JSON) diff
- 파싱 기하 객체 단위 비교(엔티티 add/del/move). 정밀·구현 부담 큼. Phase 분리 검토
- ❓ 1차 출시 = 래스터 vs 벡터 중 택1 결정 필요

### 4.3 정합(Alignment)
- 두 버전 좌표계/원점 정렬. 타이틀블록·도면틀 기준 정렬 vs 좌표 직접 사용 TBD

### 4.4 diff 결과 부가 정보
- 변경 영역 개수·카운트 배지. 변경 목록 패널(우측 인스펙터) 표시 여부 TBD

### 4.5 비편집 보장
- diff는 표시 전용 오버레이. 원본/기하 JSON 미변경 불변식 명시

## 5. 레이어 가시성 토글
> 채울 내용: 비교 화면에서 무엇을 켜고 끄는지

### 5.1 비교 레이어 구성
- Base 레이어 / Compare 레이어 / diff 하이라이트 레이어 — 독립 on·off

### 5.2 표시 모드
- 오버레이(겹침) / 좌우 분할(side-by-side) / 스와이프(슬라이더) 중 지원 범위 확정 TBD
- 투명도 슬라이더(Base↔Compare 페이드)

### 5.3 CAD 레이어 토글 연계
- 파싱 기하의 CAD 레이어(전기/배관 등) 토글 — [[01-3-2_시트(도면목록·2D뷰어)]] 레이어 패널 재사용(DRY). 비교 모드 특화 동작만 본 문서 정의

## 6. 확대 비교 (Zoom)
> 채울 내용: 세밀한 차이를 들여다보는 인터랙션

### 6.1 동기 줌·팬
- 좌우 분할/스와이프 시 양쪽 뷰포트 줌·팬 동기화(잠금 토글). 줌 엔진은 뷰어 공통 [[PDF.js]]/Canvas 기반 재사용

### 6.2 변경점 네비게이션
- 변경 영역 간 이전/다음 점프(diff hotspot 순회). 미니맵/오버뷰 표시 여부 TBD

### 6.3 정밀도 한계 명시
- 정밀 CAD 편집(osnap·좌표 편집·SHX 정밀 렌더) 범위 밖. 확대는 "관찰" 목적, 측정/스냅 비포함

### 6.4 엔티티 diff 훅 (Phase 2, 예고)
- 확대 시 변경된 객체에 설비 엔티티 ID 하이라이트 → [[TypeDB]] 영향도 질의 연동 자리. 1차 미구현, 인터페이스 자리만 확보
