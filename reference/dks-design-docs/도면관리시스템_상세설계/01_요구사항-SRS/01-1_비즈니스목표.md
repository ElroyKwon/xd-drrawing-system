# 비즈니스 목표

> 소유 주제: **왜 이 시스템을 만드는가(Why) + 무엇으로 성공을 판정하는가(KPI)**.
> 기능 명세·아키텍처·UI 재현은 본 문서 범위 밖 → [[링크]]로 위임.

---

## 목적 (이 문서가 답하는 것)
- 청주 사업장 납품 "도면관리 시스템"의 존재 이유와 성공 판정 기준 단일 정의.
- 기능/화면이 아니라 **사업적 정당성과 측정 지표**만 다룸.
- 채울 것: 1차 목표([[ACC]] Build 동등 재현)와 차별점([[DKS]] 온톨로지)의 사업적 우선순위 명시.

---

## 배경 (청주 납품)

### 납품 맥락
- 채울 것: 청주 사업장 도입 배경 — 현행 도면 관리 방식(파일서버/수기), 페인포인트, 납품 트리거.
- ❓TBD: 발주 형태(SI 단발 vs 운영 연계), 납품 일정·검수 기준.

### 현행 한계 (As-Is)
- 채울 것: 버전 혼선, 마크업/이슈 추적 부재, 분야 간 도면 영향도 파악 불가 — 정성 항목 나열.

### 기준 제품 (벤치마크)
- [[ACC]] Build를 동등 재현 대상으로 채택. 37화면 분석 근거 → [[_ACC-Build-화면분석-재현설계]].
- 채울 것: ACC 직접 도입 대비 자체 구축 선택 이유(라이선스 비용, 온프레미스, 온톨로지 연동 여지).

### 핵심 스코프 전제 (목표 해석의 전제 — 상세는 위임)
- **CAD 편집기 아님**: [[DWG]] 원본 무편집 → 백엔드 파싱 → 기하 JSON → 웹 2D 뷰어 렌더 → 마크업/이슈/이벤트 **오버레이**.
- 정밀 CAD 편집(osnap·좌표편집·[[SHX]] 폰트 정밀렌더)·3D BIM 뷰어/Bridge 범위 밖. 1차 = **2D 시트 한정**.
- 채울 것: 이 제약이 사업 목표(비용·납기·검수)에 주는 영향 1~2줄. 상세 정의 → [[01-2_스코프정의]] (TBD).

## 비즈니스 목표

### 1차 목표 — ACC Build 동등 재현 (Phase 1)
- 채울 것: "동등"의 사업적 정의 — 어느 모듈까지 도달하면 납품 합격인지(홈/시트/마크업/이슈/시트비교/파일/사진 중 필수 vs 후순위).
- 채울 것: 측정 가능한 형태로 목표 진술(개조식). 예) "ACC Build 7개 모듈 중 핵심 5개 웹 재현".

### 2차 목표 — DKS 온톨로지 연동 (Phase 2)
- 채울 것: 도면=진실원천. Phase 0 파싱 산출물(분야·도면번호·태그) 재활용.
- 차별점: ACC엔 없는 **설비 엔티티 ID 단위** 검색·영향도·AI채팅. 핀/마크업/이슈를 픽셀이 아닌 엔티티에도 바인딩.
- 채울 것: [[TypeDB]] 온톨로지 연동으로 얻는 사업 가치(영향도 분석, 지식질의). 상세 → [[데이터-지식-스튜디오]].
- ❓TBD: Phase 2를 본 납품 범위에 포함할지, 후속 계약으로 분리할지.

### 비목표 (Non-Goals)
- 채울 것: 명시적 제외 — CAD 편집, 3D BIM, SHX 정밀 렌더, 좌표 편집. 기대치 관리용.

## 단계적 가치 (도면관리 → AI)

### 가치 사다리
- 채울 것: Phase 1(뷰어+오버레이로 ACC 대체) → Phase 2(엔티티 바인딩+영향도) → Phase 3(AI 채팅/지식질의) 가치 누적 흐름.
- 각 단계 "전 단계 산출물을 재료로 다음 가치 창출" 논리 1줄씩.

### 단계별 산출 가치
| 단계 | 핵심 가치 | 측정 연결 |
|------|-----------|-----------|
| Phase 1 | ACC 동등 도면 협업(뷰어·마크업·이슈) | 제품 재현도 KPI |
| Phase 2 | 엔티티 단위 검색·영향도([[TypeDB]]) | 데이터/온톨로지 KPI |
| Phase 3 | AI 채팅·지식질의 | ❓TBD |
- 채울 것: 위 표 각 행에 정량/정성 근거 보강.

### 차별화 논거 (Moat)
- 채울 것: 도면=진실원천 + 온톨로지 바인딩 = ACC 미보유 영역. 사업 제안 시 셀링 포인트로 정리.
