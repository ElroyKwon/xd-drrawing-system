# E2E 시나리오

## ① 목적
- 청주 [[도면관리 시스템]]의 **핵심 사용자 흐름(End-to-End)** 을 사용자 관점 시나리오로 정의.
- 각 시나리오 = 실제 클릭/입력 단계 → 기대 결과(Acceptance). 자동화([[Playwright]] 등) 및 수동 검수 공용 대본.
- 단위/통합 테스트가 아닌 **"전체 흐름이 끊김 없이 동작하는가"** 만 소유. 세부 컴포넌트 테스트는 위임.

## ② 배경/전제
- 시스템 본질 = **[[DWG]] 뷰어 + 주석/이벤트 오버레이** (CAD 편집기 아님).
  - 흐름: [[DWG]] → 백엔드 파싱 → 기하 [[JSON]] → 웹 2D 뷰어 렌더 → 마크업/이슈/이벤트 오버레이.
  - 정밀 CAD 편집(osnap·좌표 편집·[[SHX]] 폰트 정밀 렌더)은 범위 밖 → 시나리오에 포함하지 않음.
  - 1차 범위 = **2D 시트 한정**. 3D BIM 뷰어는 범위 밖.
- 벤치마크 = [[ACC Build]] 37화면. 모듈: 홈/시트/마크업/이슈/시트비교/파일/사진.
- DKS 차별점 = 핀·마크업·이슈를 픽셀뿐 아니라 **설비 엔티티 ID**에 바인딩 (Phase 2 [[TypeDB]] 온톨로지). 해당 시나리오에서 검증 포인트로만 가볍게 언급.
- 시나리오 ID 규약: `E2E-<모듈>-<번호>` (예: `E2E-SHEET-01`).

## 시트 업로드 → 뷰어
> 가장 기본·필수 흐름. 파싱 파이프라인이 뷰어까지 끊김 없이 도달하는지 검증.

### E2E-SHEET-01 단일 DWG 업로드 후 2D 렌더
> 파일 선택 → 업로드 → 백엔드 파싱 → 뷰어 진입까지 happy-path.

#### 단계
- 시트 모듈 진입 → "업로드" → [[DWG]] 1개 선택 → 진행률 표시 확인.
- 파싱 완료 후 시트 목록(데이터테이블)에 신규 행 등장 메모.

#### 기대 결과
- 목록에 도면번호·분야·버전 컬럼 채워짐 (Phase 0 파싱 산출물 매핑).
- 행 클릭 → 2D 뷰어 진입, 기하 [[JSON]] 렌더 완료(빈 화면 아님).

#### 비정상 경로
- 파싱 실패 시 토스트 + 행 상태 "실패" 표기 ❓(상태 라벨 정책 TBD).

### E2E-SHEET-02 대용량/다중 시트 업로드
> 여러 도면 일괄 업로드 시 비동기 처리·진행 표시.
- 진행 상태 폴링 vs [[WebSocket]] 푸시 방식 ❓ → [[11-1_테스트전략]] 또는 설계 문서에 위임.

### E2E-SHEET-03 뷰어 기본 조작
> 렌더 후 팬·줌·핏(fit), 레이어 토글 동작.
- 좌표 편집·스냅은 범위 밖이므로 **불포함**.

## 이슈 생성 → 목록
> 뷰어 위 핀 배치 → 폼 작성 → 목록/필터 반영까지. ACC 이슈 모듈 동등 재현.

### E2E-ISSUE-01 뷰어에서 핀 찍어 이슈 생성
> 도면 위 좌표에 핀 → 상세 폼(모달) → 저장 → 목록 반영.

#### 단계
- 뷰어에서 이슈 도구 선택 → 도면 임의 위치 클릭 → 핀 생성.
- 우측 인스펙터/모달 폼: 제목·담당자·상태·분야 입력 → 저장.

#### 기대 결과
- 핀이 해당 좌표에 고정(팬·줌 시 도면 좌표 따라 이동).
- 이슈 목록 데이터테이블에 신규 행 추가, 핀↔행 양방향 하이라이트.

#### DKS 검증 포인트 (Phase 2)
- 핀이 픽셀 좌표 외에 **설비 엔티티 ID**에도 바인딩되는지 ❓(Phase 2 [[TypeDB]] 연동 시 활성).

### E2E-ISSUE-02 목록 필터·정렬·내보내기
> 데이터테이블 공통 기능을 이슈 데이터로 검증.
- 상태/담당자/분야 필터 → 결과 행 수 변화. 컬럼 토글·정렬·내보내기(CSV/[[PDF]] ❓) 확인.

### E2E-ISSUE-03 이슈 상태 변경 → 이력
> 상태 전이(열림→진행→닫힘)와 변경 이력(activity) 기록.
- 상태 워크플로우 정의는 [[05_이슈]]에 위임. 여기선 흐름 연속성만.

## 마크업
> 뷰어 오버레이 위 펜·도형·화살표·텍스트 주석. 원본 [[DWG]] 불변, 오버레이 레이어에만 저장.

### E2E-MARKUP-01 도형 마크업 생성·저장·재현
> 마크업 도구 → 그리기 → 저장 → 새로고침 후 동일 위치 재현.

#### 단계
- 마크업 도구 패널: 펜/사각형/화살표/텍스트 중 선택 → 도면 위 드로잉.
- 색·굵기 조정 → 저장.

#### 기대 결과
- 마크업이 오버레이 레이어로 저장(원본 기하 [[JSON]] 미변경).
- 새로고침/재진입 시 동일 좌표·스타일 재현, 팬·줌에 정합.

### E2E-MARKUP-02 마크업 표시/숨김·삭제
> 레이어 토글로 마크업 on/off, 개별 삭제. 권한별 편집 가능 여부 ❓.

### E2E-MARKUP-03 마크업 → 이슈 전환
> 마크업 선택 → "이슈로 만들기" 연계 흐름(ACC 패턴). 지원 여부 TBD.

## 시트 비교
> 동일 도면 두 버전 오버레이 후 변경분(diff) 시각화. ACC 시트비교 재현.

### E2E-COMPARE-01 버전 A/B 오버레이 비교
> 두 버전 선택 → 정렬 오버레이 → 추가/삭제/변경 색상 구분.

#### 단계
- 시트 상세에서 버전 선택기로 비교 대상 2개 지정 → "비교" 진입.
- 오버레이 렌더: 기준 버전 + 대상 버전 겹쳐 표시.

#### 기대 결과
- 변경 영역 색상 구분(추가/삭제/유지) 표시.
- 토글로 단일 버전 ↔ 오버레이 전환 가능.

#### 비교 방식 ❓
- 기하 [[JSON]] 객체 단위 diff vs 래스터 픽셀 diff 중 채택안 TBD → 설계 문서 위임.

### E2E-COMPARE-02 비교 결과 공유/캡처
> 비교 뷰 스냅샷 저장 또는 링크 공유. 범위 포함 여부 TBD.
