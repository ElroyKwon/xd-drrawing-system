# 개발 환경 셋업

## ① 목적
- 청주 사업장 납품용 [[도면관리 시스템]] 개발자가 "신규 PC에서 로컬 구동까지" 도달하는 최단 경로 정의
- 소유 범위 = **로컬/도구/온보딩**만. 기술스택 선정 근거·아키텍처는 [[12-1_기술스택]]/[[12-2_프로젝트구조]]로 위임(DRY)
- 전제 스코프: DWG 비편집 — [[DWG]] → 백엔드 파싱 → 기하 JSON → 웹 2D 뷰어 렌더 → 마크업/이슈/이벤트 오버레이. CAD 편집기 아님, 2D 시트 한정

## ② 배경/전제
- 벤치마크 = ACC Build 37화면(상세 [[_ACC-Build-화면분석-재현설계]]). 모듈: 홈·시트·마크업·이슈·시트비교·파일·사진
- Phase 1 = ACC 동등 재현(뷰어+오버레이). Phase 2 = [[TypeDB]] 온톨로지 연동 AI 채팅/지식질의 — 본 문서는 Phase 1 로컬 셋업 우선
- 차별점: 핀/마크업/이슈를 픽셀뿐 아니라 "설비 엔티티 ID"에도 바인딩 → 개발환경도 도면 파싱 산출물(분야·도면번호·태그)을 로컬에서 다룰 수 있어야 함
- 정밀 CAD(osnap·좌표편집·SHX 정밀렌더)·3D BIM·Bridge는 범위 밖 → 관련 무거운 네이티브 의존성 불요
