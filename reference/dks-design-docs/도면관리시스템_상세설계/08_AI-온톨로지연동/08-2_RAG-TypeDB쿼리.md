# RAG & TypeDB 쿼리

## ① 목적
- 사용자의 자연어 질의를 [[TypeDB]] 온톨로지 그래프 질의([[TQL]])로 변환하고, 추론 결과를 자연어 답변([[NLG]])으로 되돌리는 **질의→TQL→NLG 파이프라인** 1개 주제만 소유.
- ACC Build에는 없는 차별점 = "설비 엔티티 단위" 질의·영향도·다단 추론을 도면 뷰어/이슈에 연결하는 AI 채널 (Phase 2).

## ② 배경 / 전제
- **스코프 제약 계승**: DWG는 편집하지 않음. 도면=진실원천. Phase 0 파싱 산출물(분야·도면번호·태그)이 온톨로지 엔티티의 1차 소스.
- 이 문서는 **Phase 2** 산출물. Phase 1(뷰어+마크업+이슈 ACC 재현)이 선행 전제.
- 정형 지식(설비 엔티티·관계)은 [[TypeDB]], 비정형 지식(문서·코멘트·이슈 본문)은 [[RAGflow]] 채널이 담당 → 하이브리드 검색.
- 답변은 항상 **근거 엔티티 ID / 출처 도면**을 동반(환각 억제). 핀/마크업/이슈가 픽셀이 아닌 엔티티 ID에 묶이는 구조를 검색·근거에 재활용.
- LLM 모델 선정·프롬프트 운영은 본 문서 범위 밖 → [[08-3_엔티티-바인딩]] 또는 별도 운영 문서로 위임.

## ③ 상세 목차

## Intent 분석
> 자연어 질의를 받아 의도·대상 엔티티·질의 유형을 분류하는 전처리 단. 무엇을 채울지 = 분류 체계와 슬롯 추출 규칙.

### 질의 유형 분류
> 채울 것: 조회형(단일 엔티티 속성)·관계형(연결 탐색)·영향도형(multi-hop)·비정형형(문서 검색) 분기 기준 표.

### 엔티티/슬롯 추출
> 채울 것: 질의에서 도면번호·분야·설비태그·관계어를 추출해 슬롯에 매핑. Phase 0 태그 사전 재사용.

#### 동의어/약어 정규화
> 채울 것: 설비 약어(CDU/RDHx 등)·현장 호칭을 표준 엔티티 ID로 정규화하는 사전 연동.

### 라우팅 결정
> 채울 것: 분류 결과에 따라 TQL 채널 / [[RAGflow]] 비정형 채널 / 혼합 중 어디로 보낼지 결정 로직.

## TQL 변환
> 정형 질의(Intent)를 실행 가능한 [[TQL]]로 합성하는 단. 무엇을 채울지 = 슬롯→쿼리 템플릿 매핑과 검증.

### 슬롯 → 쿼리 템플릿 매핑
> 채울 것: 질의 유형별 파라미터화된 TQL 템플릿 카탈로그(조회·관계·영향도). 자유 생성 대신 템플릿 우선(안전성).

### 스키마 바인딩
> 채울 것: 추출 슬롯을 실제 온톨로지 타입/롤([[08-1_온톨로지-스키마]] 정의)에 바인딩하는 규칙.

#### 미매칭 처리
> 채울 것: 슬롯이 스키마에 없을 때 폴백(유사 타입 제안·재질문·빈상태 응답).

### 쿼리 검증 / 가드레일
> 채울 것: 실행 전 문법·범위·과도한 fan-out 차단. 읽기 전용 보장(편집 금지 제약과 일관).

## Multi-hop 추론 → NLG
> 다단 관계 추론 결과를 자연어 답변으로 변환하는 단. 무엇을 채울지 = 추론 경로 수집과 근거 동반 NLG.

### 추론 경로 수집
> 채울 것: TypeDB rule 기반 추론으로 N-hop 경로(예: 설비A 고장 → 영향 설비 목록) 수집. 경로=근거.

### 결과 → 자연어 변환
> 채울 것: 그래프 결과를 [[NLG]] 템플릿/LLM으로 서술화. 개조식 요약 + 표/리스트 혼용.

#### 근거 / 출처 표기
> 채울 것: 답변마다 근거 엔티티 ID·출처 도면번호·신뢰도 표기. 뷰어 핀으로 점프 가능한 링크 동반.

### 뷰어/이슈 연결
> 채울 것: 답변 내 엔티티를 2D 뷰어 핀·이슈로 딥링크. 상세 바인딩 규약은 [[08-3_엔티티-바인딩]]로 위임.

## [[RAGflow]] 비정형 채널
> 문서·코멘트·이슈 본문 등 비정형 텍스트 검색 보조 채널. 무엇을 채울지 = 인덱싱 대상과 하이브리드 융합.

### 인덱싱 대상
> 채울 것: 도면 메타·이슈 본문·마크업 코멘트·첨부 문서를 청크/임베딩하는 범위. DWG 기하 자체는 제외.

### 정형+비정형 융합
> 채울 것: TypeDB 결과(정형)와 [[RAGflow]] 검색(비정형)을 하나의 답변으로 합치는 랭킹·중복제거 전략.

### 실시간 동기화
> 채울 것: 신규 이슈/코멘트 생성 시 인덱스 갱신 트리거([[WebSocket]] 이벤트 연동 가능성). TBD.
