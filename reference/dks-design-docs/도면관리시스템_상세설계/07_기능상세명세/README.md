# 기능 상세명세 개요

## ① 목적

- 청주 사업장 납품 [[도면관리 시스템]]의 **동작 수준(behavioral) 기능 스펙 인덱스**를 소유하는 문서.
- 화면/아키텍처/데이터모델이 아닌 "**기능 단위가 무엇을 하는가**(입력→처리→출력, 상태전이, 권한, 예외)"만 책임짐.
- 각 기능별 상세 명세 파일이 이 폴더(`07_기능상세명세/`) 아래 1기능 1파일로 누적되며, 본 README는 그 **목록·작성양식·소유범위**만 관리.

## ② 배경 / 전제

- **벤치마크**: [[Autodesk Construction Cloud]](ACC) Build 제품 동등 재현. 모듈 분해는 [[_ACC-Build-화면분석-재현설계]] 37화면 분석 참조(SoT).
- **★핵심 스코프 제약** (모든 하위 명세의 전제):
  - [[DWG]] 원본을 **편집하지 않음**. 흐름 = DWG → 백엔드 파싱 → 기하 [[JSON]] → 웹 2D 뷰어 렌더 → 마크업/이슈/이벤트 **오버레이**.
  - 즉 **CAD 편집기 아님** → "뷰어 + 주석/이벤트 오버레이" 시스템. 정밀 CAD 편집(osnap, 좌표편집, [[SHX]] 폰트 정밀렌더)은 범위 밖.
  - 1차 = **2D 시트 한정**. 3D [[BIM]] 뷰어/Bridge 범위 밖.
- **Phase 구분**:
  - Phase 1 = ACC Build 동등 재현(본 문서 범위).
  - Phase 2 = [[TypeDB]] 온톨로지 연동 AI 채팅/지식질의 — 명세는 별도 위임([[데이터-지식-스튜디오]]).
- **차별점(우리의 무기)**: 핀/마크업/이슈를 픽셀 좌표뿐 아니라 **설비 엔티티 ID**에도 바인딩. 관련 기능 명세에서 "엔티티 바인딩" 항목으로 명시(아래 §기능 목록 표시).

## ③ 상세 목차

## 구성

> 이 폴더의 파일 구조·명명 규칙·기능 ID 체계를 정의.

### 폴더 구조
> `07_기능상세명세/README.md`(본 문서·인덱스) + `FN-XXX_<기능명>.md`(기능별 1파일). 채울 것: 실제 파일 네이밍 예시.

### 기능 ID 체계
> 모듈 접두어 + 일련번호. 예: `HOME-01`, `SHEET-03`, `MARKUP-02`, `ISSUE-05`. 채울 것: 모듈 접두어 확정 표.

### 상태 표기 규약
> 각 기능 상태(📝초안 / 🔄검토 / ✅확정 / 🅿️Phase2보류). 채울 것: 범례.

### 명세 ↔ 화면분석 추적성
> 각 기능이 [[_ACC-Build-화면분석-재현설계]]의 어느 화면/위젯에서 유래했는지 역참조 컬럼 규칙.

## 명세 작성 양식

> 1기능 1파일에 공통 적용할 템플릿. 아래 H4가 각 명세 파일의 필수 섹션.

#### 1. 개요 / 사용자 스토리
> "누가 / 무엇을 / 왜". 1~2줄. 채울 것: As-a / I-want / So-that 형식.

#### 2. 입력 → 처리 → 출력 (I-P-O)
> 트리거 입력, 백엔드/프런트 처리, 결과 출력. 오버레이 기능은 "원본 DWG 불변" 전제 재확인란 포함.

#### 3. 상태 전이 / 라이프사이클
> 해당 시 상태머신(예: 이슈 Open→In Review→Closed). 마크업/뷰어는 N/A 가능.

#### 4. 권한 / 역할 (RBAC)
> 뷰어/마크업작성/이슈관리/관리자 등 역할별 가능 동작. 채울 것: 역할 매트릭스 행.

#### 5. 데이터 바인딩
> 픽셀 좌표 + (해당 시) **설비 엔티티 ID** 바인딩 여부·키. Phase 2 온톨로지 연동 후크 명시.

#### 6. UI 컴포넌트 매핑
> 사용하는 공통 UI(데이터테이블/인스펙터/모달폼/토스트/빈상태)를 [[_ACC-Build-화면분석-재현설계]] 공통 UI에 위임 링크.

#### 7. 예외 / 엣지 케이스
> 빈 데이터, 파싱 실패, 권한 거부, 버전 불일치 등.

#### 8. 수용 기준 (Acceptance Criteria)
> Given-When-Then 체크리스트. 구현 완료 판정 기준.

#### 9. TBD / 미결
> 기능별 결정 대기 항목.

## 기능 목록

> 모듈별 기능 인덱스 테이블. 컬럼 = ID · 기능명 · 상태 · 엔티티바인딩(○/△/✕) · 명세파일 · 화면분석 출처. 아래는 골격(상태/파일은 채워나감).

### 홈 (HOME)
> 위젯 대시보드. 기능: 프로젝트 요약위젯, 최근 이슈/시트 위젯, 빠른이동. 채울 것: HOME-01~ 행.

### 시트 (SHEET)
> 도면 목록 + 2D 뷰어. 기능: 시트목록(데이터테이블), 버전 관리, **2D 뷰어 렌더**([[PDF.js]]/Canvas 후보 — 렌더링 상세는 별도 위임), 줌/팬, 시트 검색·필터. 엔티티바인딩 △(도면번호·태그). 채울 것: SHEET-01~.

### 마크업 (MARKUP)
> 뷰어 오버레이 주석. 기능: 펜/도형/화살표/텍스트, 색상·굵기, 레이어 표시토글, 저장(원본 DWG 불변). 채울 것: MARKUP-01~.

### 이슈 (ISSUE)
> 핀 + 상세폼 + 목록 + 필터. 기능: 핀 배치(좌표+엔티티ID), 상세폼(모달), 목록 데이터테이블, 상태 전이, 필터/담당자. 엔티티바인딩 ○. 채울 것: ISSUE-01~.

### 시트 비교 (COMPARE)
> 버전 오버레이 diff. 기능: 두 버전 정렬·오버레이, 변경 하이라이트, diff 토글. 채울 것: COMPARE-01~.

### 파일 (FILE / CDE)
> 폴더형 [[CDE]]. 기능: 폴더 트리, 업로드, 파싱 트리거(DWG→기하 JSON), 권한. 채울 것: FILE-01~.

### 사진 (PHOTO)
> 현장 사진. 기능: 갤러리, 메타(위치/일시), 이슈 연결. 채울 것: PHOTO-01~.

### 실시간 / 공통 (COMMON)
> 토스트 알림, 빈상태, 프로젝트 스위처. 실시간 갱신 채널([[WebSocket]] 후보)은 아키텍처 문서에 위임하고 여기선 "어느 기능이 실시간 의존인지"만 표기.
