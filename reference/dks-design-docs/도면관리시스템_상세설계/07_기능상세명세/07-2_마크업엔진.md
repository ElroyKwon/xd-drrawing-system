# 마크업 엔진 명세

## 1. 목적
- [[ACC]] Build "마크업" 모듈(펜·도형·화살표·텍스트)의 동등 재현 명세.
- 소유 범위 = **오버레이 드로잉 그리기/저장(직렬화) 동작** 한정.
- 이 문서는 "어떻게 그리고, 어떻게 저장/복원하며, 어떻게 선택/편집하는가"만 정의.

## 2. 배경 / 전제
- ★핵심 제약: [[DWG]] 원본을 편집하지 않음. 흐름 = DWG → 백엔드 파싱 → 기하 JSON → 2D 뷰어 렌더 → **그 위에 마크업을 오버레이**.
- 마크업은 도면 픽셀/벡터 위에 떠 있는 **별도 레이어**. 원본 기하와 분리 저장.
- 정밀 CAD 편집(osnap 스냅, 좌표 입력, [[SHX]] 폰트 정밀 렌더)은 범위 밖.
- 1차 범위 = 2D 시트 한정. 좌표계·줌/팬 변환은 [[07-1_2D뷰어렌더링]]가 소유 → 본 문서는 그 좌표계를 **소비**만.
- 차별점(향후): 마크업을 픽셀뿐 아니라 "설비 엔티티 ID"에도 바인딩 가능 → [[TypeDB]] 온톨로지 연동(Phase 2). 본 문서는 데이터 모델에 엔티티 ID 슬롯만 예약.

---

## 3. 드로잉 모델 (SVG / Canvas)
> 채울 것: 렌더 백엔드 선택 근거, 도형 타입 정의, 좌표 변환 책임 경계.

### 3.1 렌더 백엔드 선택
- [[SVG]] vs [[Canvas]] 트레이드오프 표(히트테스트 용이성·도형 수 성능·텍스트 렌더 품질).
- 권장 가설: 마크업 오버레이 레이어는 SVG(선택·편집 용이), 베이스 도면은 Canvas/[[PDF.js]] 라스터 — 2계층 분리.
- ❓TBD: 마크업 수백 개 시 SVG DOM 성능 한계 검증 필요.

### 3.2 지원 도형 타입(툴)
> 채울 것: 각 툴의 기하 파라미터·드래그 인터랙션.
#### 3.2.1 펜(자유선, freehand path)
- 포인트 배열 → path. 단순화(Douglas-Peucker) 적용 여부 ❓TBD.
#### 3.2.2 직선 / 화살표
- 시작·끝 점 2개. 화살표는 head 스타일 속성.
#### 3.2.3 사각형 / 타원
- bounding box 기반.
#### 3.2.4 폴리라인 / 폴리곤
- 클릭 누적 + 더블클릭 종료.
#### 3.2.5 텍스트 / 콜아웃(말풍선)
- 인라인 편집. 폰트는 웹폰트 한정(SHX 미지원).
#### 3.2.6 형광펜(highlight)
- 반투명 사각/자유 영역.

### 3.3 스타일 속성 모델
- color / strokeWidth / opacity / fill / dash / arrowHead / fontSize.
- 도형 타입별 적용 가능 속성 매트릭스 표.

### 3.4 좌표계 / 변환 책임
- 마크업은 **도면 월드좌표**로 저장, 화면 표시 시 뷰어 변환행렬 적용.
- 줌/팬 무관 고정 크기 요소(핀류) vs 스케일 요소(도형) 구분 ❓TBD.
- 변환행렬 소스 = [[07-1_2D뷰어렌더링]] (위임, 중복 정의 금지).

## 4. 직렬화 포맷
> 채울 것: 저장 스키마, 버전 관리, 백엔드 전송/영속 방식.

### 4.1 마크업 도큐먼트 스키마(JSON)
- 최상위: sheetId / sheetVersion / schemaVersion / shapes[].
- shape 공통 필드: id / type / geometry / style / createdBy / createdAt / layerId.
- **entityRefs[]**: 바인딩된 설비 엔티티 ID 배열(Phase 2 예약, 1차엔 빈 배열 허용).
- geometry는 도형 타입별 discriminated union.

### 4.2 좌표 직렬화 규약
- 월드좌표(소수점 정밀도 ❓TBD)·단위 표기.
- 펜 path 포인트 압축 포맷 ❓TBD(평문 배열 vs 인코딩).

### 4.3 버전 / 마이그레이션
- schemaVersion 필드로 전방호환. 마이그레이션 함수 위치 메모.

### 4.4 영속 / 동기화
- 저장 단위 = 시트당 마크업 도큐먼트(또는 shape 단위 patch) ❓TBD.
- REST(저장/조회) + 실시간 협업 시 [[WebSocket]] patch 브로드캐스트(Phase 후순위).
- 영속 대상·테이블 정의는 [[06_데이터모델]]로 위임.

### 4.5 내보내기 / 가져오기
- 마크업 포함 [[PDF]] 출력(베이스+오버레이 합성) 범위 여부 ❓TBD.
- ACC 호환 포맷 가져오기는 범위 밖(명시).

## 5. 히트테스트 / 선택
> 채울 것: 클릭→도형 식별 알고리즘, 다중선택·핸들 편집.

### 5.1 히트테스트 전략
- SVG: 네이티브 pointer 이벤트(타깃 element) 활용.
- Canvas 백엔드 시: 바운딩박스 1차 + 정밀 판정(선 거리 임계값) 2차.
- 선(stroke) 히트 허용 오차(px) 정의.

### 5.2 선택 모델
- 단일 클릭 / Shift 다중선택 / 영역 드래그(marquee) 선택.
- 선택 상태 표현(하이라이트 + 핸들).

### 5.3 편집 핸들
- 리사이즈 핸들(코너/엣지), 회전 핸들 지원 여부 ❓TBD, 이동(드래그).
- 펜/폴리라인 정점 편집 범위 ❓TBD.

### 5.4 z-order / 레이어
- 앞으로/뒤로 보내기. 레이어 그룹화 지원 여부 ❓TBD.

### 5.5 엔티티 바인딩 인터랙션(Phase 2 훅)
- 마크업 선택 시 연결된 설비 엔티티 인스펙터 표시 → [[07-3_이슈핀시스템]]·온톨로지 검색과 공유. 본 문서는 선택 이벤트만 발행, 패널은 위임.
