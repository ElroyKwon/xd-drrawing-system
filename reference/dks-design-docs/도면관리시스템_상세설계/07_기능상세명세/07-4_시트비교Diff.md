# 시트 비교 Diff 명세

## 1. 목적
- 같은 도면([[DWG]] 기반)의 **두 버전 간 시각적 차이**를 산출·표시하는 알고리즘 정의
- [[ACC]] Build "시트 비교(Compare)" 모듈 동등 재현 — 구버전/신버전 오버레이 + 변경분 하이라이트
- 소유 범위 = **버전 차이 산출 알고리즘**에 한정. 비교 UI/슬라이더/토글은 [[07-2_2D뷰어]]에 위임, 마크업 표기는 [[07-3_마크업]]에 위임

## 2. 배경 / 전제
- 핵심 스코프 제약: DWG 원본 미편집. 흐름 = DWG → 백엔드 파싱 → 기하 JSON → 웹 2D 뷰어 렌더. Diff는 **렌더 결과(래스터) 또는 기하 JSON 위에서** 계산
- 1차 = 2D 시트 한정. 3D/Bridge 범위 밖
- 1차 구현 = **래스터 픽셀 diff** (구현 단순·도면 종류 무관). 벡터 객체 diff는 Phase 2 후보
- 입력 = 동일 도면번호의 버전 A(base) / 버전 B(compare). 버전 메타·페어링은 [[06_데이터모델_시트버전]] 소유
- 차별점(향후): 변경 영역을 픽셀이 아닌 "설비 엔티티 ID"에 연결 → [[TypeDB]] 온톨로지 기반 영향도 질의. 본 문서는 산출 알고리즘만, 바인딩은 §6에서 훅만 정의

## 3. 래스터 픽셀 diff (1차 구현)
> 두 버전을 동일 해상도 비트맵으로 렌더 후 픽셀 단위 비교. 채울 내용: 파이프라인·임계값·성능.

### 3.1 입력 정규화
- 동일 [[DPI]]/줌/뷰포트로 양 버전 래스터화 (서버 렌더 vs 클라 [[PDF.js]]/Canvas 렌더 중 택1 — TBD)
- 시트 크기/회전 불일치 시 정합 필요 → §6 참조
- 채울 내용: 렌더 해상도 기준, 타일 단위 처리 여부

### 3.2 픽셀 비교 알고리즘
- 채울 내용: 그레이스케일화 → 픽셀 차이 임계값(anti-aliasing 노이즈 무시) → 차이 마스크 생성
- 후보 라이브러리: pixelmatch / resemble.js 류 (선정 TBD)

#### 3.2.1 노이즈 억제
- 채울 내용: 1~2px 미세 시프트·압축 아티팩트 흡수 방법(블러/형태학 연산)

#### 3.2.2 변경 영역 클러스터링
- 채울 내용: 차이 픽셀 → 연결요소(connected component) → 바운딩 박스 묶음. 잔점 필터링 최소 면적

### 3.3 출력 산출물
- 채울 내용: 차이 마스크(오버레이 이미지) + 변경 영역 박스 리스트(좌표·면적). 뷰어가 소비할 JSON 스키마

## 4. 벡터 객체 diff (향후 / Phase 2)
> 기하 JSON 엔티티 단위 비교. 래스터보다 정밀하나 파싱 의존. 후순위.

### 4.1 객체 매칭
- 채울 내용: 엔티티 ID/지오메트리 해시 기반 A↔B 매칭. 매칭 불가 = 추가/삭제 후보

### 4.2 변경 분류
- 채울 내용: 추가 / 삭제 / 이동·변형(geometry delta) / 속성변경(레이어·태그)

### 4.3 엔티티 바인딩 (차별점)
- 채울 내용: 변경 엔티티 → 설비 엔티티 ID 매핑 → [[TypeDB]] 영향도 질의 훅. ACC엔 없는 우리 무기. 상세는 [[06_데이터모델_시트버전]]/온톨로지 문서에 위임

## 5. 색상 규칙 (추가/삭제)
> 변경 의미별 색상 컨벤션. 뷰어 렌더 입력. ACC 관례 준수.

### 5.1 기본 컬러맵
- 채울 내용: 추가=청색 / 삭제=적색 / 변경=황색 등 (ACC 색상 확인 후 확정 — TBD)
- 색약 대비 모드 필요 여부 ❓

### 5.2 블렌딩 모드
- 채울 내용: 두 버전 오버레이 시 투명도·곱연산 등. 슬라이더 연동은 [[07-2_2D뷰어]] 소유

## 6. 정합 / 정렬 (Registration)
> 두 버전의 좌표계·스케일이 어긋날 때 diff 전 정렬. 미정렬 시 전체가 거짓 변경으로 산출.

### 6.1 기준점 정합
- 채울 내용: 시트 외곽/타이틀블록/도면틀 기준 정렬. 자동 vs 수동 기준점

### 6.2 자동 정렬 알고리즘
- 채울 내용: 이동·회전·스케일 보정(phase correlation / 특징점 매칭). 정렬 실패 시 폴백

### 6.3 정합 실패 처리
- 채울 내용: 정합 신뢰도 낮으면 사용자 경고 + 수동 2점 지정 UI (UI는 뷰어 위임, 본 문서는 입력 파라미터만)
