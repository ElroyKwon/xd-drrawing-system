# 물리 아키텍처 (청주 배포)

## ① 목적
- 청주 사업장 납품 [[도면관리 시스템]]의 **온프레/클라우드 물리 구성** 단일 소유 문서
- 서버·스토리지·DB·네트워크 실배포 형상 정의 → 발주/설치/운영 인수인계 기준
- 논리 구성요소(서비스 단위)는 [[02-2_논리아키텍처]]가 소유 → 본 문서는 "어디에 어떤 하드웨어로 올리는가"만 다룸 (DRY)

## ② 배경/전제
- **스코프 전제**: [[DWG]] 원본 미편집. 흐름 = DWG → 백엔드 파싱 → 기하 JSON → 웹 2D 뷰어 렌더 → 마크업/이슈/이벤트 오버레이. CAD 편집기 아님 → 고사양 GPU 워크스테이션 불필요
- **1차 범위**: 2D 시트 한정. 3D BIM 뷰어 범위 밖 → 대용량 테셀레이션/렌더팜 불요
- **배포 성격**: 청주 사업장 **온프레 우선** (사내망 도면 = 보안 자산). 클라우드는 보조/백업 옵션으로만 검토
- **Phase 2 대비**: 향후 [[TypeDB]] 온톨로지 노드 증설 여지 확보 (지금 미설치, 슬롯만 산정) → 상세는 [[02-2_논리아키텍처]] 위임
- **차별점 메모**: 핀/마크업/이슈를 픽셀+"설비 엔티티 ID"에 동시 바인딩 → 엔티티 인덱스 저장소 용량은 §용량 산정에서 별도 슬롯

---

## 배포 토폴로지(온프레 우선)

### 토폴로지 개요
- 청주 사내망 내 단일 서버랙 기준 구성도 채울 것. 외부 인터넷 단절 가정 시 패키지/업데이트 반입 경로도 명기

#### Option A: 단일 서버 올인원 (PoC/초기 납품)
- 1대 물리/가상 서버에 웹·API·파서·DB·스토리지 동거. 청주 1차 규모(사용자 수 TBD)에서 충분한지 판단 근거 채울 것

#### Option B: 역할 분리 2~3노드 (운영 권장)
- 프론트/API 노드 + 파싱 워커 노드 + DB/스토리지 노드 분리. DWG 파싱 부하 격리 목적. 노드별 사양은 §서버 구성에서

#### Option C: 클라우드/하이브리드 (보조 검토)
- 도면 원본은 온프레, 백업/재해복구만 클라우드. 사내 보안정책 충돌 여부 ❓ → [[02-6_보안아키텍처]] 결정 따름

### 가상화/컨테이너 방침
- VM vs 컨테이너([[Docker]]) 채택 결론 채울 것. 파서 워커는 컨테이너 격리 유력 — 근거 메모

### 가용성/이중화 수준
- 1차 납품 단일 노드 허용 여부, 운영 단계 이중화 목표 채울 것. 무중단 요구 수준 ❓

## 서버/스토리지/DB 구성

### 서버 사양 (역할별)
- 노드별 vCPU/RAM/디스크 표 채울 것. 산정 입력은 §용량 산정 결과 참조

#### 웹/API 노드
- 웹 서버 + REST API + [[WebSocket]](실시간 마크업/이슈 협업) 호스팅. 동시접속 기준 사양 채울 것

#### 파싱 워커 노드
- DWG → 기하 JSON 변환 배치/큐 처리 부하 집중 노드. CPU 바운드 가정 — 코어 수 산정 근거 채울 것. 변환 엔진/라이선스는 [[02-2_논리아키텍처]] 위임

#### DB 노드
- 메타데이터 RDB + (Phase 2) [[TypeDB]] 슬롯. 동거/분리 결정 ❓

### 스토리지 구성
- 계층별 용량/RAID 채울 것

#### 원본 도면 저장소 (DWG 원본, 불변)
- DWG 원본 = 진실 원천(DKS 전제). 불변 보관·버전 누적 정책. 용량 증가율 추정은 §용량 산정

#### 파생물 저장소 (기하 JSON / 타일 / 썸네일)
- 뷰어 렌더용 파생 산출물. 재생성 가능 → 저비용 계층 배치 가능 여부 메모. [[PDF.js]]/2D 타일 캐시 포함 여부 ❓

#### 첨부물 저장소 (이슈 사진/마크업 첨부)
- ACC Build 사진/이슈 첨부 대응. 오브젝트 스토리지 vs 파일시스템 결정 채울 것

### 데이터베이스 물리 배치
- RDB 인스턴스 수·복제 채울 것

#### 메타데이터 RDB
- 시트/버전/이슈/마크업/사용자 메타. 엔진 선정은 [[02-2_논리아키텍처]] 위임 — 본 문서는 물리 배치/용량만

#### 엔티티 인덱스 / 온톨로지 슬롯 (Phase 2)
- 설비 엔티티 ID ↔ 도면 좌표 바인딩 인덱스 물리 배치. [[TypeDB]] 증설 시점 노드 계획 채울 것

### 백업/보존
- 백업 주기·보존기한·복구목표(RPO/RTO) 채울 것. 사내 백업 인프라 연동 여부 ❓

## 네트워크/망분리

### 망 구성 (청주 사내망 기준)
- 서비스망/관리망 분리도 채울 것. 도면 자산 = 폐쇄망 가정 검증

#### 접근 경로
- 사내 사용자 브라우저 → 웹/API 노드 경로. 외부 협력사 접근 필요 여부 ❓ (ACC는 클라우드 협업 전제였으나 청주는 폐쇄망일 수 있음)

#### 포트/프로토콜
- HTTPS / [[WebSocket]] / DB 내부 포트 목록 채울 것. 인터넷 인바운드 차단 여부 명기

### 망분리 정책 정합성
- 청주 사업장 망분리 등급/정책 확인 후 채울 것 ❓ → 상세 통제는 [[02-6_보안아키텍처]] 위임 (본 문서는 물리 결선/세그먼트만)

### 패키지/업데이트 반입
- 폐쇄망 시 오프라인 업데이트·컨테이너 이미지 반입 절차 채울 것
