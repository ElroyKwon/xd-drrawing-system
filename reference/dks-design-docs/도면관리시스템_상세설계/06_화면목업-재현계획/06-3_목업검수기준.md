# 목업 검수 기준

## ① 목적
- 목업(정적/저충실 프로토타입)이 "다음 단계(실제 구현)로 넘어가도 되는가"를 판정하는 **합격선** 정의.
- 주관적 "그럴듯함"이 아니라 **체크 가능한 항목**으로 통과/보류 판정.
- 소유 주제 = **완성도 합격선 그 자체**. 화면별 상세 스펙·컴포넌트 정의·구현 방법은 위임([[링크]]).

## ② 배경 / 전제
- 벤치마크 = [[ACC]] Build 37화면. 목업은 그 핵심 모듈(홈/시트/마크업/이슈/시트비교/파일/사진)을 **시각 재현**.
- ★스코프 전제: [[DWG]] 편집 아님 → "[[PDF.js]] 기반 2D 뷰어 + 마크업/이슈/이벤트 **오버레이**". 목업도 이 흐름을 전제로 화면 구성.
- 1차 = 2D 시트 한정. 3D BIM 뷰어는 검수 대상 아님.
- 목업 단계에서 실데이터·실API 불필요. 단, **레이아웃·상태·인터랙션 골격**은 실제 구현으로 전환 가능한 구조여야 함(CLAUDE.md §4 Frontend-First).
- 차별점(엔티티 단위 핀/검색)은 목업에서 **자리(placeholder) 확보** 수준만 검수. 실동작은 Phase 2.

## ③ 상세 목차

## 레이아웃 충실도
> 벤치마크 화면 대비 "구조적 배치"가 맞는지 판정. 픽셀 일치 아님, 구역 일치.

### 앱셸 골격
> 모듈 사이드바 + 프로젝트 스위처 + 상단바 + 본문 영역 4분할이 모든 화면에 일관 적용되었는지.

### 모듈별 본문 레이아웃
> 시트=목록+뷰어 2분할, 이슈=목록+인스펙터, 홈=위젯 그리드 등. 각 모듈 기준 배치 충실도.

#### 우측 인스펙터 패널 위치/토글
> 이슈·마크업 선택 시 우측에서 슬라이드되는 인스펙터 영역이 확보되었는지(내용은 더미 가능).

### 그리드/간격 일관성
> 공통 spacing·카드 정렬이 화면 간 흔들리지 않는지. TBD: 기준 그리드 단위(8px?) → [[06-1_공통UI-컴포넌트목록]] 확정값 참조.

## 컴포넌트 일치
> 공통 UI 컴포넌트가 "있어야 할 화면에 있는지" + "동일 컴포넌트가 동일하게 재사용되는지" 판정.

### 필수 공통 컴포넌트 존재 여부
> 데이터테이블·위젯카드·모달폼·토스트·빈상태·인스펙터 — 화면별 등장 매트릭스로 체크.

#### 데이터테이블 기능 표면
> 정렬·컬럼토글·필터·내보내기 버튼이 UI상 노출되는지(동작은 더미 허용, 버튼 자체는 필수).

#### 이슈 핀 / 마크업 도구 표면
> 뷰어 위 핀 배치 UI, 마크업 툴바(펜·도형·화살표) 노출. 엔티티 바인딩 자리는 placeholder.

### 컴포넌트 재사용 일관성
> 같은 역할의 버튼/배지/태그가 화면마다 다른 스타일로 갈라지지 않는지(DRY 위반 적발).

## 반응형 / 빈상태 포함
> "정상 데이터 화면"만이 아니라 경계 상태까지 목업에 포함되었는지 판정.

### 빈상태(Empty State)
> 도면 0건·이슈 0건·검색 결과 없음 화면이 별도로 존재하는지. 안내 문구+CTA 포함.

### 로딩/에러 상태
> 뷰어 로딩 스켈레톤, 파싱 실패/뷰어 로드 실패 에러 화면 존재 여부. TBD: 에러 카피 톤.

### 반응형 거동
> 최소 데스크톱 기준. 인스펙터 열림 시 본문 리플로우, 사이드바 접힘 거동 확인.
> TBD: 태블릿/모바일 지원 여부 → ❓ 스코프 확정 필요(현장 사용 환경 미정).

## 검수 체크리스트
> 위 항목을 통과/보류로 찍는 실제 운영 체크리스트. 검수자는 이 표만 보고 판정.

### 합격선 정의
> 통과 조건 = (필수 항목 100% PASS) + (권장 항목 N% 이상). TBD: N 임계값.

### 화면별 체크 표(틀)
> 행=화면(홈/시트/마크업/이슈/시트비교/파일/사진), 열=검수 카테고리. 각 셀 PASS/HOLD/N-A.

|------|:--:|:--:|:--:|:--:|:--:|
| 홈 | ☐ | ☐ | ☐ | ☐ | ☐ |

### 판정 등급
> PASS(구현 진행) / HOLD(수정 후 재검) / N-A(스코프 외). 등급별 후속 액션 명시.
