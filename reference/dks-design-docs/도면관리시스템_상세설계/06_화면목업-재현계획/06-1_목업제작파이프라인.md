# 목업 제작 파이프라인

## ① 목적
- 본 문서가 소유하는 단일 주제 = **"목업 1장(화면 1개)을 생산하는 표준 절차"**
- ACC Build 화면을 동등 재현하는 정적 목업을 누가 만들어도 동일 품질/형식으로 나오게 하는 반복 가능한 워크플로우 정의
- 화면 *목록/우선순위*는 본 문서가 다루지 않음 → [[06-2_화면재현-우선순위]]로 위임
- 공통 컴포넌트 사양(앱셸·데이터테이블·인스펙터 등)도 위임 → [[_ACC-Build-화면분석-재현설계]]

## ② 배경 / 전제
- CLAUDE.md §4 Frontend-First 모드: UI/플로우를 먼저, 가상 데이터로 동작 검증 후 백엔드 설계
- 1차 목표 = ACC Build 화면 동등 재현. 산출물은 **정적 HTML 목업**(실제 [[DWG]] 파싱/렌더 미연동)
- 스코프 제약(전 문서 공통 전제): CAD 편집기 아님 → "뷰어 + 마크업/이슈/이벤트 오버레이". 2D 시트 한정. 3D BIM 범위 밖
- 따라서 목업의 2D 뷰어 영역은 **실제 도면 이미지/PNG 또는 placeholder**로 대체(이 단계에선 [[PDF.js]]/캔버스 실렌더 불필요)
- 차별점(향후 바인딩): 핀/마크업/이슈를 픽셀이 아닌 "설비 엔티티 ID"에도 연결 → 목업 단계에선 *데이터 속성 자리만 마련*(예: `data-entity-id`), 실제 [[TypeDB]] 연동은 Phase 2

## ③ 상세 목차

## 입력 (ACC 스크린샷)
- 목업 1장의 출발점이 되는 원천 자료와 그 정규화 규칙을 채울 절

### 원천 스크린샷 위치
- 수집 경로: `수집자료/screenshot/` (ACC Build 37화면 캡처)
- 화면별 분석 근거는 [[_ACC-Build-화면분석-재현설계]] 참조

### 화면 1장 단위 정의
- "목업 1장" = ACC 모듈 1화면(예: 홈 대시보드 / 시트 목록 / 이슈 상세)
- 모달·인스펙터 같은 부분 UI도 독립 1장으로 셀 수 있는지 기준 → TBD ❓

### 입력 체크리스트 (제작 착수 전)
- 채울 내용: 해당 화면의 ⓐ 레이아웃 골격 ⓑ 데이터 항목 ⓒ 상호작용(클릭/필터/토글) ⓓ 빈상태/에러상태 유무 식별
- 재사용 공통 UI는 신규 제작 금지, 기존 컴포넌트 참조만

## 제작 방식 (frontend-design / HTML→캡처)
- 실제 목업을 만드는 손작업 절차를 채울 절

### 도구 / 스택
- `frontend-design` 스킬 활용(고품질 정적 UI 생성)
- 출력 형식: 단일 HTML + 인라인/번들 CSS (런타임 의존 최소화)
- 가상 데이터: 실제 API 스펙으로 전환 가능한 구조(JSON 상수)로 작성 — 추후 [[CDE]]/백엔드 응답으로 치환
- 상태/실시간 요소([[WebSocket]] 이벤트 오버레이 등)는 목업에선 정적 스냅샷으로 표현

### 제작 단계 (표준 순서)
#### 1) 골격 배치
- 앱셸(모듈 사이드바 + 프로젝트 스위처) 적용 → 사양은 [[_ACC-Build-화면분석-재현설계]]
#### 2) 본문 모듈 채우기
- 데이터테이블 / 위젯카드 / 2D 뷰어 영역 / 우측 인스펙터 등 배치
- 2D 뷰어 영역 = 도면 placeholder 이미지 + 오버레이 레이어(핀/마크업 좌표는 가상값)
#### 3) 상호작용 정적 표현
- 필터·정렬·컬럼토글·모달폼·토스트·빈상태를 "보이는 상태"로 고정 표현
#### 4) 검수
- ACC 원본 스크린샷과 1:1 시각 대조

### HTML → 캡처 (산출 이미지화)
- 완성 HTML을 브라우저 렌더 후 스크린샷으로 고정 (리뷰·문서 첨부용)
- 캡처 방식/해상도 표준(뷰포트 px, 배율) → TBD ❓
- 캡처 자동화 도구 선정(chrome-devtools 스크린샷 등) → TBD ❓

### 품질 기준 (DoD)
- 채울 내용: 동등 재현 합격선(레이아웃 일치도/필수 데이터 항목/상태 커버리지)
- 정량 기준 수치화 여부 → TBD ❓
