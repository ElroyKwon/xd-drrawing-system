# 목업: 마크업

## ① 목적
- [[ACC]] Build "마크업(Markup)" 모듈을 동등 재현하기 위한 화면 목업 명세.
- 소유 범위 = **마크업 도구/주석 상태 목업** 한정. 펜·도형·화살표 그리기 도구 UI와 주석(annotation) 상태 모델만 다룸.
- 가상 데이터 기반 Frontend-First 골격 정의 → 의도 검증 후 Backend 마크업 영속 API 설계로 전환.

## ② 배경/전제
- ★핵심 스코프: [[DWG]] 원본 미편집. 흐름 = DWG → 백엔드 파싱 → 기하 JSON → 웹 2D 뷰어 렌더 → 그 위에 마크업 "오버레이".
- 즉 CAD 편집기가 아니라 **뷰어 + 주석 오버레이**. osnap 스냅·좌표 정밀 편집·SHX 폰트 렌더는 범위 밖.
- 1차 범위 = 2D 시트 한정. 마크업은 시트 뷰포트 좌표계 위의 별도 레이어로 존재.
- 마크업 좌표는 픽셀이 아닌 **뷰어 정규화 좌표(시트 기준)**로 저장 → 줌/팬 무관 위치 유지. (상세 좌표계는 [[06-2-2_시트]] 위임)
- 차별점: ACC 마크업은 픽셀 위치만 가짐. 우리는 마크업을 **설비 엔티티 ID**에도 선택 바인딩 가능 → 향후 [[TypeDB]] 온톨로지 영향도/AI채팅 연동. (Phase 2, 본 문서는 데이터 모델에 엔티티 ID 슬롯만 예약)

## ③ 상세 목차

## 대상 화면
- 무엇을 채울지: 마크업 모드가 활성화된 시트 뷰어 화면 레이아웃과 진입 경로를 정의.

### 진입 경로
- 시트 2D 뷰어에서 "마크업" 토글 진입. 시트 뷰어 본체 레이아웃은 [[06-2-2_시트]] 소유 → 여기선 마크업 레이어/툴바만.

### 화면 영역 구성
- 무엇을 채울지: 뷰어 캔버스 위 마크업 레이어 + 상단/좌측 도구 팔레트 + 우측 인스펙터 영역 배치 스케치.

#### 마크업 캔버스 레이어
- 도면 렌더 위에 겹쳐지는 SVG/Canvas 오버레이 레이어. 도면 렌더 자체는 read-only.
- 렌더 방식(SVG vs Canvas) ❓TBD — [[PDF.js]]/시트 뷰어 렌더 스택과 정합 필요.

#### 도구 팔레트 위치
- 펜/도형/화살표/텍스트 선택 팔레트 배치(좌측 세로 vs 상단 가로) ❓TBD.

### 상태별 화면
- 무엇을 채울지: 마크업 없음(빈상태)/그리는 중/선택됨/저장중 4개 상태의 화면 표현.

#### 빈 상태
- 마크업 0건 시 안내 문구 + 첫 도구 유도. 공통 빈상태 패턴은 [[06-1_공통UI패턴]] 위임.

## 펜/도형/화살표 상태
- 무엇을 채울지: 각 마크업 객체의 가상 데이터 상태 모델(공통 필드 + 타입별 필드).

### 공통 마크업 객체 모델
- 무엇을 채울지: 모든 마크업이 공유하는 필드를 가상 JSON으로 정의.

#### 공통 필드 (가상 데이터 스펙)
- `id`, `type`(pen|shape|arrow|text), `sheetId`, `pageVersion`(시트 버전 핀), `points[]`(정규화 좌표), `style`(color·strokeWidth·opacity), `author`, `createdAt`, `entityId`(설비 엔티티 바인딩, nullable·Phase 2 예약).
- 실제 API 스펙 전환 가능한 구조로 작성(표준 response format은 [[06-1_공통UI패턴]] 참조).

### 펜(자유 곡선) 상태
- 무엇을 채울지: 연속 좌표 배열 + 굵기/색 상태. 스무딩 적용 여부 ❓TBD.

### 도형 상태
- 무엇을 채울지: 사각형·원·다각형·구름표시(revision cloud) 지원 범위 확정.

#### 지원 도형 목록
- 사각형/타원/직선/구름표시 — 1차 포함 후보. 폴리곤 자유편집 포함 여부 ❓TBD.
- 채움(fill) 지원 여부 ❓TBD.

### 화살표/지시선 상태
- 무엇을 채울지: 시작·끝 좌표 + 화살촉 방향. 양방향/지시선(leader) 변형 포함 여부 ❓TBD.

### 텍스트/콜아웃 상태 (포함 여부 ❓TBD)
- ACC 마크업의 텍스트 박스 포함할지 결정 대기. 포함 시 폰트 정밀 렌더 범위 밖이므로 웹 기본 폰트로 제한.
