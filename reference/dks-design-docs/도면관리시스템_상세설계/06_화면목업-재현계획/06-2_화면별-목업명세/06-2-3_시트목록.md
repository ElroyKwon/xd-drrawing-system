## 목적
- 청주 사업장 [[도면관리 시스템]]의 "시트(도면) 목록" 화면 목업 명세 단일 소유 문서.
- [[ACC Build]]의 Sheets 모듈(도면 목록 + 진입점) UI/데이터 구조를 동등 재현하기 위한 골격 정의.
- 목록 → 클릭 시 2D 뷰어 진입의 "리스트 측" 책임만 다룸. 뷰어 자체는 [[06-2-4_2D뷰어]]에 위임.

## 배경/전제
- ★스코프: [[DWG]] 원본 비편집. 흐름 = DWG → 백엔드 파싱 → 기하 JSON → 웹 2D 렌더([[PDF.js]]/Canvas). 본 화면은 그 산출물을 "목록"으로 노출.
- Phase 0 파싱 산출물(분야·도면번호·태그) 존재 → 목록 컬럼/필터의 1차 데이터 소스.
- 1차 범위 = 2D 시트 한정. 3D/[[BIM]]/Bridge는 범위 밖.
- 공통 UI([[데이터테이블]]·인스펙터·빈상태·토스트)는 [[06-1_공통UI-패턴]] 재사용. 본 문서는 시트 목록 "고유" 부분만 기술.
- 차별점(향후): 핀/마크업/이슈를 픽셀이 아닌 "설비 엔티티 ID"에 바인딩 → [[TypeDB]] 온톨로지 연동(Phase 2). 본 목록에서는 "엔티티 태그 컬럼/필터" 자리만 예약.

## 대상 화면
> 무엇을 채울지: 화면의 정체성·진입경로·전체 레이아웃 골격을 못박는다.

### 화면 정의
- 앱셸(모듈 사이드바 → "시트") 진입 시 기본 표시 화면.
- 좌: 공종(분야) 트리/필터 · 중앙: 시트 목록(테이블 or 썸네일 그리드) · 우: (선택)인스펙터.
- 상단: 버전세트 스위처 + 검색 + 뷰 토글(테이블/그리드).

### 뷰 모드
- 테이블 뷰: 도면번호·제목·공종·버전·업데이트일 등 컬럼형.
- 썸네일 그리드 뷰: 도면 미리보기 카드 중심. (ACC 기본은 그리드)
- ❓ 기본 뷰 모드 = 그리드 vs 테이블 (TBD: 청주 운영자 선호 확인)

### 진입/이탈 경로
- 시트 카드/행 클릭 → [[06-2-4_2D뷰어]] 진입.
- 버전세트 스위처 변경 → 동일 목록을 해당 버전세트 기준으로 재로딩.

## 구성요소(공종색/썸네일/버전세트)
> 무엇을 채울지: 시트 목록 고유의 3대 시각 구성요소 명세. 공통 테이블 속성은 위임.

### 공종(분야) 색상 체계
- Phase 0 파싱의 "분야" 값 → 색상 매핑(좌측 트리 + 카드 색띠/배지).
- 매핑 예시 골격: 건축/구조/전기/기계/소방/제어 … (실 분야 코드 = Phase 0 산출 기준)
- ❓ 분야 코드 체계·색상 팔레트 확정 (TBD: Phase 0 파싱 결과 분야 enum 수령 후)
- 색상은 [[06-1_공통UI-패턴]] 디자인 토큰 참조(중복 정의 금지).

### 썸네일
- 백엔드 파싱 시 생성된 도면 미리보기 이미지(래스터) 사용. DWG 직접 렌더 아님.
- 그리드 카드: 썸네일 + 도면번호 + 제목 + 공종 배지 + 버전 뱃지.
- ❓ 썸네일 생성 시점/해상도/캐시 전략 (TBD: 파싱 파이프라인 측과 협의 → [[05_백엔드-파싱]] 위임)
- 빈/실패 썸네일 → 플레이스홀더(공종색 기반 placeholder).

### 버전세트(Version Set)
- ACC의 버전세트 = 발행 회차 묶음. 스위처로 "현재 보는 회차" 선택.
- 목록은 선택된 버전세트 기준 "최신 시트" 표시. 과거 회차 비교는 [[06-2-6_시트비교]]에 위임.
- 행/카드에 버전 뱃지(예: V3) + "업데이트됨" 표식.
- ❓ 버전세트 메타 출처(파일명 규칙 vs 별도 발행 메타) (TBD)

### 엔티티 태그 컬럼 (Phase 2 예약)
- 도면이 포함한 설비 엔티티 태그 요약 노출 자리. Phase 1에서는 비활성/숨김.
- 향후 [[TypeDB]] 바인딩 시 "엔티티로 도면 찾기" 역검색 진입점.

## 상호작용 메모
> 무엇을 채울지: 클릭·필터·정렬·검색·다중선택 등 행위별 결과. 공통 동작은 위임.

### 필터/검색
- 좌측 공종 트리 체크 → 목록 필터.
- 텍스트 검색: 도면번호·제목 대상. (엔티티 태그 검색은 Phase 2)
- 공통 필터 칩/정렬/컬럼토글/내보내기 = [[데이터테이블]] 패턴 위임([[06-1_공통UI-패턴]]).

### 선택/일괄 동작
- 다중 선택(체크박스) → 일괄 내보내기/다운로드.
- ❓ 1차에서 일괄 다운로드 범위(PDF만 vs 원본 DWG 포함) (TBD: 청주 보안정책 확인)

### 상태/피드백
- 로딩: 스켈레톤(그리드 카드/테이블 행).
- 빈 상태: 업로드 유도 빈상태 카드(공통 패턴).
- 에러/완료 토스트: 공통 토스트 위임.

### 실시간 갱신 (선택)
