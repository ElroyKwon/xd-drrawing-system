# 목업: 2D 뷰어

## ① 목적
- [[ACC]] Build "시트" 모듈의 2D 도면 뷰어 화면을 동등 재현하는 목업 명세.
- 소유 범위 = **뷰어 셸 + 도면 렌더 표현**(툴바·패널·필름스트립·캔버스 렌더). 마크업/이슈 오버레이의 "도구 동작"은 위임.
- 이 문서는 뷰어가 "무엇을 어떻게 보여주는가"만 소유. 주석 생성 로직은 [[06-2-5_마크업]]·[[06-2-6_이슈]]가 소유.

## ② 배경/전제
- ★스코프 제약: [[DWG]] 원본 무편집. 흐름 = DWG → 백엔드 파싱 → 기하 JSON → 웹 2D 캔버스 렌더 → 마크업/이슈/이벤트 **오버레이**.
- CAD 편집기 아님 = osnap 스냅·좌표 편집·[[SHX]] 폰트 정밀 렌더 범위 밖.
- 1차 범위 = **2D 시트 한정**. 3D [[BIM]] 뷰어/Bridge 제외.
- 렌더 입력 = Phase 0 파싱 산출물(기하 JSON + 분야·도면번호·태그). 렌더 백엔드 후보는 [[06-3_뷰어렌더링-기술검토]]에 위임.
- 차별점: 핀/마크업이 픽셀 좌표뿐 아니라 **설비 엔티티 ID**에도 바인딩 가능(향후 [[TypeDB]] 온톨로지). 본 문서는 "엔티티 하이라이트 표현"만 다루고 바인딩 로직은 [[06-2-9_엔티티바인딩]]에 위임.

## ③ 상세 목차

## 대상 화면(여러 시트)
> 뷰어가 떠야 하는 진입 경로·상태별 화면 변형 카탈로그.

### 진입 경로
> 어디서 뷰어로 진입하는지.
- 시트 목록([[06-2-3_시트목록]]) 행 클릭 → 단일 시트 뷰어.
- 이슈/마크업 목록의 위치 링크 클릭 → 해당 시트 + 핀 포커스 진입.
- 딥링크 URL(`/sheet/{id}?x=&y=&zoom=`) 직접 진입 — 좌표 복원. ❓파라미터 스킴 TBD.

### 화면 레이아웃(앱셸 내 배치)
> 공통 앱셸 대비 뷰어 화면의 영역 분할.
- 좌: 모듈 사이드바(공통 앱셸 — [[06-2-1_공통UI]] 위임).
- 중앙: 도면 캔버스(주 영역).
- 우: 인스펙터 패널(선택 객체/시트 메타).
- 하단: 필름스트립(시트 썸네일 가로 스크롤).
- 상단: 뷰어 툴바.

### 상태별 변형
> 데이터/네트워크 상태에 따른 화면.
- 로딩(기하 JSON 페치 중) — 스켈레톤/프로그레스.
- 빈 상태(파싱 산출물 없음 / 미지원 시트).
- 에러(렌더 실패·파싱 누락) — 폴백 [[PDF.js]] 뷰 후보. ❓폴백 정책 TBD.
- 대용량 도면(레이어 多) — 점진 렌더/가상화 안내.

### 멀티 시트 탐색
> 여러 시트 간 이동 UX.
- 필름스트립 클릭 이동 / 키보드 ←→.
- 현재 시트 인디케이터(n/총).
- 버전 전환 시 [[06-2-7_시트비교]]로 위임(diff는 여기서 안 함).

## 툴바/패널/필름스트립
> 뷰어 크롬(주변 UI) 3종의 구성 요소 명세.

### 뷰어 툴바
> 상단 도구 모음. 표현/동작만 — 마크업/이슈 도구는 위임.
#### 뷰 컨트롤
> 순수 뷰 조작(상태 비저장).
- 줌 인/아웃·맞춤(fit)·1:1·줌박스.
- 팬(스페이스+드래그 / 휠).
- 회전 90° 단위. ❓회전 허용 여부 TBD.
- 측정(거리/면적) — 읽기 전용 표시. ❓1차 포함 여부 TBD.
#### 레이어 토글
> 파싱된 도면 레이어 on/off.
- 레이어 목록 패널(체크박스·색 표시).
- 분야별 그룹핑(전기/배관/구조 등 Phase 0 분야 태그 활용).
#### 모드 전환 진입점
> 마크업/이슈 모드로의 전환 버튼만 소유.
- 마크업 모드 → [[06-2-5_마크업]] 도구 활성.
- 이슈 핀 모드 → [[06-2-6_이슈]] 핀 배치 활성.

### 인스펙터 패널(우측)
> 공통 인스펙터 패턴의 뷰어 특화 콘텐츠.
- 시트 메타(도면번호·분야·버전·갱신일).
- 선택 객체 정보(레이어·엔티티 타입·연결 엔티티 ID).
- 엔티티 ID 바인딩 표시 → 상세 [[06-2-9_엔티티바인딩]] 위임.
- 패널 구조/리사이즈 등 공통 동작은 [[06-2-1_공통UI]] 위임.

### 필름스트립(하단)
> 시트 썸네일 가로 스트립.
- 썸네일 생성 소스(파싱 시 렌더 캐시 vs 온디맨드). ❓TBD.
- 활성 시트 강조·스크롤 동기화.
- 접기/펼치기 토글.

## 렌더 표현(샘플 도면)
> 캔버스가 기하 JSON을 어떻게 그리는지 — 시각 표현 규약.

### 렌더 파이프라인(개념)
> 입력→출력 단계. 기술 선택은 위임.
- 입력: 기하 JSON(라인/폴리라인/원호/텍스트/해치 등 프리미티브).
- 변환: 도면 좌표계 → 화면 좌표(뷰포트 변환).
- 출력: Canvas/SVG 레이어 + 오버레이 레이어 분리.
- 렌더 엔진/라이브러리 선택 = [[06-3_뷰어렌더링-기술검토]] 위임.

### 지원 프리미티브 표현
> 무엇을 어떻게 그릴지.
- 선/폴리라인/원·원호/스플라인(근사)·텍스트·치수선·해치.
- 텍스트는 시스템 폰트 근사 렌더([[SHX]] 정밀 폰트 미지원 — 가독성 우선).
- 미지원 프리미티브 처리 정책. ❓TBD.

### 레이어/색상 표현
> 도면 레이어 → 화면 스타일 매핑.
- 레이어별 색/선종/가중치(파싱 메타 기반).
- 배경(흑/백) 토글.
- 색상 정책: 원본 ACI 색 유지 vs 표준 팔레트. ❓TBD.

### 오버레이 레이어 합성
> 도면 위 주석 레이어 배치 규약(렌더 측면만).
- z-순서: 도면 < 마크업 < 이슈핀 < 선택 하이라이트.
- 줌/팬 시 오버레이 좌표 동기화(도면 좌표계 고정).
- 핀·마크업 실시간 갱신 채널 = [[WebSocket]]. 동기화 상세 [[06-2-8_실시간협업]] 위임.

### 엔티티 하이라이트 표현
> 온톨로지 차별점의 "보이는 부분"만.
- 엔티티 ID 선택 시 해당 기하 강조(외곽선/색).
- 검색·영향도 결과 → 다중 하이라이트.
- 하이라이트 표현만 소유. 검색/바인딩 로직 [[06-2-9_엔티티바인딩]] 위임.

### 샘플 도면(목업용)
> 목업에서 쓸 실제 샘플 — mock 금지, 실제 파싱 산출물 사용.
- 청주 사업장 Phase 0 파싱 결과 중 대표 시트 N개 선정. ❓대상 시트 TBD.
- 분야 다양성 확보(전기/배관/구조 각 1+). ❓TBD.
