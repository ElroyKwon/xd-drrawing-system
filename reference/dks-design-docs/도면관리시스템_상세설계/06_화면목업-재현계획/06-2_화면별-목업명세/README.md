# 화면별 목업 명세 개요

## 목적
- [[ACC]] Build 37화면 분석을 청주 사업장 납품용 [[도면관리 시스템]] "목업 화면 목록"으로 1:1 매핑
- 각 목업의 우선순위·소유 문서·진행 상태를 한 곳에서 추적하는 마스터 인덱스
- 상세 화면별 명세(레이아웃·상태·인터랙션)는 본 폴더 하위 개별 문서로 위임 — 이 문서는 "지도(map)"만 소유

## 배경 / 전제
- ★스코프 제약 (전 목업 공통 전제):
  - [[DWG]] 원본 비편집. 흐름 = DWG → 백엔드 파싱 → 기하 JSON → 웹 2D 뷰어 렌더 → 마크업/이슈/이벤트 "오버레이"
  - CAD 편집기 아님 = "뷰어 + 주석/이벤트 오버레이" 시스템. 정밀 CAD 편집(osnap·좌표편집·SHX 정밀렌더) 범위 밖
  - 1차 범위 = 2D 시트 한정. 3D BIM 뷰어/Bridge 범위 밖
- AI 채팅/지식질의 = Phase 2 ([[TypeDB]] 온톨로지 연동). 본 계획은 Phase 1 뷰어+오버레이 재현 중심
- 차별점 = 핀/마크업/이슈를 픽셀이 아닌 "설비 엔티티 ID"에도 바인딩 ([[DKS]] 도면-진실원천 + Phase 0 파싱 산출물 활용)
- 벤치마크 원본 = `[[_ACC-Build-화면분석-재현설계]]` (37화면 분석)

## 목업 목록(우선순위)
> 우선순위 = P0(MVP 필수) / P1(1차 납품) / P2(차별화·Phase 2 연계). 각 목업은 하위 개별 명세 문서를 가짐.
> 채울 내용: 모듈별 목업 ID·이름·우선순위·한줄 목적. ACC 7대 모듈(홈/시트/마크업/이슈/시트비교/파일/사진) + 공통 UI 셸을 목업 단위로 분해.

### 공통 셸 (전 화면 토대)
- 채울 내용: 앱셸(모듈 사이드바+프로젝트 스위처), 데이터테이블 패턴, 우측 인스펙터, 위젯카드, 모달폼, 빈상태, 토스트 — 재사용 컴포넌트 목업. → 상세 위임 [[06-1_공통UI-컴포넌트]] ❓경로 확정 필요

### P0 — MVP 코어
- 채울 내용: 시트 목록 + 2D 뷰어([[PDF.js]]/Canvas 렌더), 마크업 오버레이, 이슈 핀+상세폼. 뷰어 위 오버레이 좌표계가 모든 후속 기능의 토대이므로 최우선.

### P1 — 1차 납품 완성
- 채울 내용: 홈 위젯 대시보드, 이슈 목록+필터, 파일(폴더형 [[CDE]]), 시트 비교(버전 오버레이 diff), 사진.

### P2 — 차별화 / Phase 2 연계
- 채울 내용: 엔티티 단위 검색·영향도 패널, AI 채팅 바인딩 슬롯. → 상세 위임 [[Phase2-온톨로지-연동]] ❓경로 확정 필요
