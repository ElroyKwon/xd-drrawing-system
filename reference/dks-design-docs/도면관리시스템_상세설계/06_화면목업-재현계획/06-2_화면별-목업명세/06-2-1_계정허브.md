# 목업: 계정/허브

## ① 목적
- ACC Build "프로젝트 진입 이전" 레이어(계정/허브/프로젝트 선택) 5개 화면을 [[목업]]으로 동등 재현하는 명세
- 청주 사업장 납품용 도면관리 시스템의 "진입 동선" 확정 — 로그인 후 어디로 떨어지고, 프로젝트를 어떻게 고르고 만드는지
- 이 문서는 "계정/허브 레이어"만 소유. 프로젝트 내부 모듈(시트·이슈·마크업)은 [[링크]]로 위임

## ② 배경 / 전제
- ★스코프 제약: [[DWG]] 원본 비편집. 흐름 = DWG → 파싱 → 기하 JSON → 웹 2D 뷰어 → 마크업/이슈 오버레이. 이 문서는 그 진입부만 다룸
- 1차 범위 = 2D 시트 한정. 3D BIM/Bridge 범위 밖
- 벤치마크: ACC Build 37화면 분석(같은 vault `screenshot/_ACC-Build-화면분석-재현설계.md`)
- 공통 UI(앱셸·데이터테이블·인스펙터·모달폼·빈상태·토스트)는 [[06-2-2_앱셸-공통UI]]가 소유 → 본 문서는 "어떤 공통 컴포넌트를 쓰는지" 참조만
- [[DKS]] 차별점: 프로젝트 생성 시 Phase 0 도면 파싱 산출물(분야·도면번호·태그)을 메타로 연계 가능 — 향후 [[TypeDB]] 온톨로지 바인딩의 진입점

## 대상 화면(5장)

### 1. 로그인 / 계정 진입
- 무엇을 채울지: 인증 진입 화면. 로그인 폼(ID/PW), SSO 자리(향후), 로딩 상태
- 청주 납품: 사내 계정 체계 연동 여부 ❓TBD

### 2. My Home (계정 대시보드)
- 무엇을 채울지: 로그인 직후 랜딩. "내가 속한 프로젝트" 위젯 + 최근 활동 요약
- ACC의 계정 레벨 홈 ↔ 프로젝트 레벨 홈 구분 명시 (프로젝트 홈은 [[06-2-2_앱셸-공통UI]] 외 별도 문서 소유)

### 3. 프로젝트 목록
- 무엇을 채울지: 허브 내 프로젝트 카드/테이블 목록. 정렬·필터·검색·즐겨찾기
- 공통 데이터테이블 컴포넌트 재사용

### 4. 프로젝트 작성(생성 마법사)
- 무엇을 채울지: 신규 프로젝트 생성 폼/스텝. 기본정보 → 템플릿 선택 → 구성원 초대 → 도면 인입 연계
- ★DKS 연계: 생성 시 Phase 0 파싱 산출물 매핑 단계 포함 여부 ❓TBD

### 5. 프로젝트 템플릿 / 구성원 설정
- 무엇을 채울지: 템플릿 정의(폴더구조·이슈유형·라벨 프리셋) + 구성원/권한 초기 설정
- 템플릿 = 신규 프로젝트 부트스트랩 소스

## 각 화면 구성요소 체크

### My Home
#### 레이아웃
- 무엇을 채울지: 계정 헤더 + 프로젝트 스위처 + 위젯 그리드. 앱셸 차용 범위 명시
#### 위젯 카드
- 내 프로젝트 / 최근 열람 도면 / 할당된 이슈 요약 (위젯카드 컴포넌트 재사용)
- ❓위젯 종류·기본 배치 TBD
#### 빈 상태
- 소속 프로젝트 0건 시 "프로젝트 만들기" CTA

### 프로젝트 목록
#### 보기 전환
- 카드 뷰 / 테이블 뷰 토글
#### 데이터테이블
- 컬럼: 프로젝트명·역할·도면 수·최근 활동·상태. 정렬·컬럼토글·필터·내보내기는 공통 컴포넌트 위임
#### 검색 / 필터
- 이름 검색 + 상태/역할 필터
#### 빈 상태 / 카드 인터랙션
- 카드 클릭 → 프로젝트 진입(앱셸 로드). 즐겨찾기 핀

### 프로젝트 작성(생성 마법사)
#### 스텝 구성
- Step1 기본정보(이름·코드·사업장=청주·기간)
- Step2 템플릿 선택
- Step3 구성원 초대 / 권한
- Step4 도면 인입 연계(Phase 0 파싱 산출물 매핑) ❓포함 여부 TBD
#### 모달 / 폼 검증
- 필수값 검증, 토스트 피드백 (모달폼 컴포넌트 재사용)
#### 완료 후 동선
- 생성 직후 프로젝트 홈 진입 or 목록 복귀 ❓TBD

### 프로젝트 템플릿
#### 템플릿 항목
- 폴더 구조 프리셋([[CDE]] 폴더형) / 이슈 유형·라벨 / 분야 라벨(전기·기계·소방…)
#### 분야 라벨 ↔ 파싱 연계
- Phase 0 도면 "분야" 메타와 템플릿 분야 라벨 정합 ❓매핑 규칙 TBD

### 구성원 / 권한
#### 구성원 목록
- 이름·이메일·역할·상태 테이블
#### 권한 레벨
- 뷰어 / 마크업·이슈 작성 / 관리자 (ACC 역할 모델 참조)
- ❓권한 레벨 세분도 TBD (청주 납품 요구사항 확인 필요)
#### 초대 플로우
- 이메일 초대 or 사내 계정 일괄 등록 ❓TBD

## 목업 메모
- 본 레이어는 "정적/CRUD 위주" — 2D 뷰어·오버레이 부하 없음. 목업 우선순위 중(中), 앱셸·시트 뷰어 이후 착수 권장
- 공통 컴포넌트(데이터테이블·모달폼·위젯카드·빈상태·토스트)는 전부 [[06-2-2_앱셸-공통UI]] 정의 재사용 — 여기서 재정의 금지(DRY)
- 가상 데이터: 청주 프로젝트 1개 + 더미 프로젝트 2~3개로 목록/스위처 검증
- [[WebSocket]] 실시간 갱신은 이 레이어 범위 밖(이슈/마크업 모듈에서 다룸)
- 향후 [[TypeDB]] 온톨로지 연동은 프로젝트 생성 시 도면 엔티티 바인딩 진입점으로만 표시, 상세는 Phase 2 문서로 위임
