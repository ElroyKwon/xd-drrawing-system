# 목업: 시트 비교

## ① 목적
- [[ACC]] Build "시트 비교(Compare)" 기능 동등 재현 목업 명세 소유.
- 같은 도면의 두 버전(또는 두 시트)을 오버레이하여 변경분 시각 diff 제시.
- 소유 범위 한정: **비교 다이얼로그(대상 선택) ~ 오버레이 diff 표현**. 뷰어 본체 인터랙션·마크업은 위임.

## ② 배경/전제
- ★스코프 제약: [[DWG]] 원본 미편집. diff는 백엔드 파싱 산출 기하 JSON 기반 렌더 결과의 시각 대조이며, CAD 형상 비교 엔진이 아님.
- 비교 대상 = 2D 시트 한정. 3D/[[BIM]] 비교 범위 밖.
- 렌더 기반 = [[PDF.js]]/[[Canvas]] 위 2개 레이어 오버레이 (편집 아님).
- 차별점: 픽셀 diff 외에 향후 "설비 엔티티 ID" 단위 변경 추적 가능성(Phase 2, [[TypeDB]] 온톨로지) — 본 문서는 훅 지점만 표기, 상세는 위임.

## ③ 상세 목차

## 대상 화면
> 무엇을 채울지: 비교 모드 진입점·화면 골격·뷰어와의 관계 정의.

### 진입 경로
> 시트 목록/뷰어 툴바의 "비교" 버튼, 버전 히스토리에서 "이 버전과 비교" 등 진입 시나리오 열거.

### 레이아웃 구조
> 전체화면 비교 뷰 vs 분할(좌/우) vs 단일 오버레이 — 채택안 + 이유. 앱셸/모듈사이드바 유지 여부.

#### 상단 비교 바
> 좌측본(Base)·우측본(Compare) 라벨, 버전 뱃지, 스왑 버튼, 닫기. 슬라이더/토글 컨트롤 위치.

#### 메인 뷰포트
> 오버레이 캔버스 영역. 줌·팬은 [[06-2-2_시트목록-2D뷰어]]에 위임(중복 명세 금지), 비교 전용 컨트롤만 여기.

## 다이얼로그/피커
> 무엇을 채울지: 비교 대상 두 항목을 고르는 모달 플로우 명세.

### 비교 다이얼로그 트리거 → 오픈
> 버튼 클릭 시 모달폼 오픈. 공통 UI 모달폼 패턴 재사용([[_ACC-Build-화면분석-재현설계]] 공통 컴포넌트).

### 대상 선택 UI
> 두 슬롯(좌/우) 피커. 각 슬롯에서 시트·버전 선택.

#### 버전 피커
> 동일 시트의 버전 히스토리 드롭다운/리스트. 버전번호·업로드일·작성자 표시.

#### 시트 피커 (선택)
> 서로 다른 시트 간 비교 허용 여부 ❓. 허용 시 시트 검색·선택 컴포넌트.

#### 유효성/제약
> 동일 시트 다른 버전이 기본. 호환 불가 조합(예: 좌표계·해상도 상이) 가드 처리.

### 빈 상태/에러
> 비교 가능한 버전이 1개뿐일 때 빈상태 안내. 파싱 실패본 비교 시도 시 토스트 에러.

## 오버레이 diff 표현
> 무엇을 채울지: 두 레이어 합성 방식과 변경분 시각 인코딩 정의 — 본 문서 핵심 절.

### 오버레이 합성 모드
> ACC 벤치마크의 비교 표현 방식 정리 후 채택안 선정.

#### 색상 오버레이 (추가/삭제/유지)
> Base=한 색, Compare=다른 색, 공통=중립색 합성. 색상값 TBD(접근성 대비 고려).

#### 스와이프/슬라이더 모드
> 가운데 슬라이더로 좌우본 가르기. 인터랙션 상태(드래그) 정의.

#### 토글/온오프 깜빡(blink) 모드
> 두 레이어 교대 표시로 변경 인지. 선택 구현 ❓.

### 변경 영역 표기
> 차이 발생 구역 하이라이트/박스. 픽셀 기반 vs 영역 기반 수준 — 1차는 시각 오버레이만(자동 diff 검출 정밀도는 TBD).

### 비교 뷰 컨트롤
> 투명도(opacity) 슬라이더, 레이어 표시 토글, 색상 범례.

#### 범례(Legend)
> 추가/삭제/유지 색 의미 표기 위젯.

#### 동기화 줌·팬
> 분할 모드 시 좌우 뷰포트 동기 이동. 단일 오버레이 모드면 불필요.

### 결과 활용 (오버레이→액션)
> 변경 영역 위에서 바로 이슈/마크업 생성 연계. 상세는 [[06-2-5_마크업]]·이슈 문서에 위임, 여기선 진입 훅만.

#### 엔티티 단위 변경(Phase 2 훅)
> 픽셀 diff를 넘어 "설비 엔티티 ID" 변경 목록 제시 가능성. [[TypeDB]] 온톨로지 연동 — 자리표시만, 설계는 Phase 2.
