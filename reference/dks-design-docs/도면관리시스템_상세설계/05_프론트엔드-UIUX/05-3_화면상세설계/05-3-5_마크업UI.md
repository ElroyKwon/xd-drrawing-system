# 마크업 UI 화면설계

## ① 목적
- [[ACC Build]] Build 마크업 모듈(펜·도형·화살표 주석)을 동등 재현하는 마크업 도구/편집 인터랙션의 화면설계.
- 소유 범위 = **마크업 도구 선택 + 그리기 인터랙션 + 선택/편집 + 마크업 리스트**. 그 외(뷰어 렌더링, 이슈 핀, 저장 API)는 위임.

## ② 배경 / 전제
- 핵심 스코프: [[DWG]] 원본 비편집. 백엔드 파싱 → 기하 JSON → 2D 뷰어 렌더 위에 마크업을 **오버레이** 레이어로 그림. CAD 편집기 아님.
- 마크업은 뷰어 좌표계(시트 로컬 좌표) 위의 벡터 주석 오브젝트. [[osnap]] 스냅·좌표 정밀 편집은 범위 밖.
- 렌더 기반: 2D 뷰어가 [[PDF.js]]/Canvas/SVG 중 무엇을 쓰는지에 따라 마크업 오버레이 좌표 변환이 달라짐 → [[05-3-2_시트2D뷰어]]에 종속. ❓렌더러 확정 TBD.
- 차별점: 마크업을 픽셀 좌표뿐 아니라 **설비 엔티티 ID**(도면 파싱 산출물)에도 바인딩 가능 → [[TypeDB]] 온톨로지 연동(Phase 2). 본 문서는 바인딩 UI 훅만 명시.

## 도구 선택 UX
> 채울 것: 마크업 툴바 구성, 도구 종류, 선택 상태 표현, 단축키.

### 툴바 레이아웃
- 위치/형태: 뷰어 상단 또는 좌측 플로팅 툴바. ❓위치 TBD(ACC는 상단 가로 툴바).
- 그룹: 선택(커서) · 그리기 도구 · 텍스트 · 스타일 · 실행취소/재실행.

### 도구 종류 (ACC 동등)
- 펜(프리핸드), 직선, 화살표, 사각형, 원/타원, 폴리라인, 텍스트, 클라우드(구름선), 하이라이트, 측정(❓범위 포함 여부 TBD).
- 각 도구 = 아이콘 버튼 + 호버 툴팁(이름+단축키).

#### 도구별 정의 메모
- 채울 것: 도구별 입력 제스처·생성 오브젝트 타입 매핑 표(도구 → 데이터 모델).

### 활성 도구 상태
- 채울 것: 선택된 도구의 시각 강조(배경/테두리), 단일 활성 보장, 그리기 후 자동 커서 복귀 여부 정책.

### 단축키
- 채울 것: 도구별 키맵(예 V=선택, P=펜, A=화살표). ❓전체 키맵 확정 TBD.

## 그리기 인터랙션
> 채울 것: 포인터 이벤트 흐름, 드래그/클릭 제스처, 라이브 프리뷰, 좌표 변환.

### 입력 모델
- 포인터 이벤트(마우스/터치/펜) → 뷰어 좌표 변환 → 마크업 오브젝트 생성.
- 채울 것: 스크린 좌표 ↔ 시트 좌표 변환 책임 위치(뷰어 위임 vs 마크업 레이어 자체). [[05-3-2_시트2D뷰어]] 참조.

### 제스처별 흐름
#### 드래그형 (사각형·원·직선·화살표·하이라이트)
- 채울 것: pointerdown→move(라이브 프리뷰)→up(확정) 상태머신. Shift=정비율/직각 보정 정책.

#### 다중 클릭형 (폴리라인·클라우드·측정)
- 채울 것: 클릭마다 정점 추가, 더블클릭/Esc=종료, 진행 중 고스트 세그먼트 표시.

#### 프리핸드 (펜)
- 채울 것: move 샘플링 간격·포인트 단순화(라인 디시메이션) 정책. 과샘플 방지.

#### 텍스트
- 채울 것: 클릭→인라인 입력 박스→blur/Enter 확정. 폰트 정밀 [[SHX]] 렌더 불필요(웹 폰트 사용).

### 라이브 프리뷰 & 커밋
- 채울 것: 임시 레이어에 프리뷰 후 up 시 확정 오브젝트로 승격. 취소(Esc) 시 폐기.

### 좌표 정밀도 한계 (명시)
- [[osnap]] 미지원. 스냅 없음 = 자유 배치. 픽셀-시트 변환 정밀도만 보장, CAD급 좌표 정밀 편집 미제공.

### 엔티티 바인딩 훅 (Phase 2 대비)
- 채울 것: 마크업 확정 시 그 위치가 어떤 설비 엔티티 영역과 겹치는지 백엔드 질의 → entityId 후보 제시 UI. ❓동기/지연 바인딩 TBD. [[TypeDB]] 연동은 별도 문서 위임.

## 선택 핸들 / 미니툴바
> 채울 것: 선택 상태, 변형 핸들, 컨텍스트 미니툴바, 스타일 편집.

### 선택 모델
- 채울 것: 단일/다중 선택, 클릭 히트테스트, 선택 시 바운딩박스 표시.

### 변형 핸들
- 채울 것: 리사이즈 핸들(코너/엣지), 이동 드래그, 회전(❓지원 여부 TBD), 정점 핸들(폴리라인/펜 점 편집).
- 한계: 정밀 좌표 입력창 미제공(스코프 외).

### 컨텍스트 미니툴바
- 채울 것: 선택 오브젝트 근처 플로팅 바 — 색상·선두께·채움·불투명도·삭제·복제·앞/뒤 순서.
- 채울 것: 도구별 가용 속성 차이(텍스트=폰트크기, 화살표=머리스타일 등).

### 스타일 속성
- 채울 것: 색상 팔레트(ACC 기본색 세트), 선 두께 단계, 점선/실선, 불투명도 슬라이더. 기본값 정책.

### 인라인 편집
- 채울 것: 텍스트 더블클릭 재편집, 폴리라인 정점 추가/삭제 인터랙션.

## 마크업 리스트 패널
> 채울 것: 우측 인스펙터 내 마크업 목록, 가시성 토글, 메타 표시, 동기화.

### 패널 위치/구조
- 우측 인스펙터 패널(공통 UI). [[05-2_공통UI컴포넌트]] 인스펙터 패턴 재사용.
- 채울 것: 헤더(개수·필터) + 리스트 + 빈상태.

### 리스트 아이템
- 채울 것: 타입 아이콘 · 작성자 · 작성시각 · 색상 칩 · 가시성 토글 · 라벨/메모.
- 채울 것: 클릭 시 뷰어에서 해당 마크업으로 팬/줌 + 선택 동기화.

### 필터 / 정렬
- 채울 것: 도구타입·작성자·날짜 필터, 정렬 옵션. [[05-2_공통UI컴포넌트]] 데이터테이블 필터 위임 가능 여부 ❓TBD.

### 가시성 / 레이어
- 채울 것: 전체 마크업 레이어 on/off, 개별 토글, (❓버전별 레이어 분리 TBD).

### 이슈 연계
- 채울 것: 마크업 → 이슈로 승격/링크 동선. 상세는 [[05-3-6_이슈UI]]로 위임(링크만).

### 실시간 동기화
- 채울 것: 다중 사용자 마크업 반영 정책. [[WebSocket]] 실시간(❓Phase 1 포함 여부 TBD) — 통신 상세는 별도 문서 위임.
