# 시트 목록 화면설계

## 1. 목적
- 청주 사업장 도면관리 시스템의 **시트(도면) 목록 화면** 레이아웃·상호작용 단일 소유 명세
- [[ACC Build]] "Sheets" 모듈의 도면 목록 영역을 동등 재현 + DKS 차별점(엔티티 단위 메타) 반영
- 이 문서가 다루는 것: **목록 자체의 표현·탐색·선택**. 뷰어 렌더·마크업은 위임 → [[05-3-4_2D뷰어]]

## 2. 배경 / 전제
- 스코프 제약: DWG 원본 편집 없음. 흐름 = [[DWG]] → 백엔드 파싱 → 기하 JSON → 웹 2D 렌더. **목록 화면은 파싱 산출 메타데이터를 소비**(원본 미접근).
- 1차 범위 = **2D 시트 한정**. 3D BIM 목록 범위 밖.
- 데이터 원천: Phase 0 도면 파싱 산출(분야·도면번호·태그). 목록 컬럼은 이 산출 스키마에 종속 → 상세 스키마는 [[03_데이터모델]]에 위임.
- 공통 컴포넌트(데이터테이블·툴바·빈상태·토스트)는 [[05-2_공통UI컴포넌트]] 재사용. 본 문서는 **시트 목록에 특화된 차이만** 기술.

## 3. 레이아웃 (테이블 / 그리드)
> 채울 내용: 목록의 2개 뷰 모드와 기본 화면 골격.

### 3.1 화면 골격
- 앱셸 안 콘텐츠 영역: 상단 툴바 + 본문 목록 + (선택 시) 우측 인스펙터.
- 인스펙터는 행 선택 시에만 슬라이드인 → 상세 정의는 §6 / [[05-2_공통UI컴포넌트]].

### 3.2 테이블 뷰 (기본)
- 채울 내용: 행=시트 1개, 정렬·컬럼토글·페이지네이션. 밀집 정보용 디폴트 뷰.
- 가상 스크롤 적용 여부 → ❓ (도면 수 규모 확정 후 결정, TBD)

### 3.3 그리드(썸네일) 뷰
- 채울 내용: 카드형 썸네일 배열. 시각적 탐색용. 카드 1개 = 썸네일 + 도면번호 + 제목 + 공종칩.
- 테이블↔그리드 토글 위치 = 툴바 우측(§5).

### 3.4 반응형 / 빈 상태
- 채울 내용: 좁은 폭에서 컬럼 우선순위 축소 규칙(TBD).
- 빈 상태: 파싱 결과 0건 / 필터 결과 0건 2종 분기 → [[05-2_공통UI컴포넌트]] 빈상태 패턴 재사용.

## 4. 컬럼 / 공종색 / 썸네일
> 채울 내용: 목록 정보 구조의 핵심. 어떤 메타를 어떻게 보여줄지.

### 4.1 컬럼 정의
- 채울 내용: 컬럼 목록·라벨·정렬가능여부·기본 표시여부 표.
- 1차 후보: 썸네일 / 도면번호 / 제목 / 분야(공종) / 버전 / 업데이트일 / 이슈수 / 마크업수.
- 각 컬럼 데이터 출처(파싱 메타 vs 시스템 메타) 매핑 → ❓ 일부 미확정(TBD).

#### 4.1.1 DKS 확장 컬럼
- 채울 내용: ACC엔 없는 차별 컬럼. **연결 엔티티 수**(시트에 바인딩된 설비 엔티티 ID 개수) 등.
- 향후 [[TypeDB]] 온톨로지 연동 시 "영향도" 컬럼 추가 여지 → Phase 2, 지금은 자리만(TBD).

### 4.2 공종색(분야) 표현
- 채울 내용: 분야(전기/기계/배관 등)별 색상칩 매핑 규칙.
- 색상 팔레트·접근성(색맹 대비) → ❓ 디자인 토큰 확정 후(TBD). 토큰은 [[05-1_디자인시스템]]에 위임.

### 4.3 썸네일
- 채울 내용: 썸네일 생성 출처(파싱된 기하 JSON 기반 경량 프리뷰 vs 별도 래스터). 생성 책임은 백엔드.
- 로딩 전 플레이스홀더 / 실패 시 폴백 아이콘 규칙.

## 5. 툴바 (검색 / 필터 / 내보내기)
> 채울 내용: 목록 상단 조작 영역. 공통 데이터테이블 툴바의 시트 특화 구성.

### 5.1 검색
- 채울 내용: 도면번호·제목 텍스트 검색. 즉시(디바운스) 필터.
- DKS 차별: **엔티티/태그 검색**(설비 엔티티 ID·파싱 태그로 시트 역검색) → 향후 강화, 1차는 텍스트 우선(TBD 범위).

### 5.2 필터
- 채울 내용: 분야(공종)·버전·이슈유무 등 패싯 필터. 다중 선택.
- 필터 상태 URL 동기화 여부 → ❓(공유 링크 요구 시 필요, TBD).

### 5.3 정렬 / 컬럼 토글 / 뷰 토글
- 채울 내용: 정렬 컨트롤, 컬럼 표시 토글, 테이블↔그리드 토글 배치.

### 5.4 내보내기
- 채울 내용: 목록 CSV 내보내기(공통 패턴 재사용). 도면 파일 자체 내보내기/다운로드 정책 → ❓(원본 [[DWG]] 다운로드 허용 여부 정책 결정 필요, TBD).

## 6. 행 상호작용 → 뷰어
> 채울 내용: 목록에서 도면을 여는 핵심 동선. 이 문서의 "출구".

### 6.1 행 클릭 / 더블클릭 동작
- 채울 내용: 단일 클릭=선택+인스펙터, 더블클릭(또는 행 내 "열기")=뷰어 진입 규칙 확정(TBD).

### 6.2 인스펙터 패널(미리보기)
- 채울 내용: 선택 시 우측 패널에 시트 메타·축소 썸네일·바로가기. 상세 폼 아님(읽기 위주).
- 표시 항목: 분야·버전 이력 요약·연결 엔티티 요약·이슈/마크업 카운트.

### 6.3 뷰어 진입 (핸드오프)
- 채울 내용: 뷰어로 넘길 파라미터(시트 ID·버전·진입 컨텍스트). **렌더링 동작 자체는 본 문서 소유 아님** → [[05-3-4_2D뷰어]].
- 라우팅 경로·딥링크 규약 → ❓(TBD).

### 6.4 행 컨텍스트 액션
- 채울 내용: 행 우측 더보기 메뉴(열기/버전비교/다운로드 등). 시트비교 진입은 → [[05-3-X_시트비교]]에 위임.

## 7. 실시간 / 상태 동기화
> 채울 내용: 목록의 라이브성 범위.
- 이슈/마크업 카운트 실시간 갱신 여부 → [[WebSocket]] 사용 여부 ❓(1차는 폴링/새로고침으로 충분할 수 있음, TBD).
- 파싱 진행 중 시트의 "처리중" 상태 표시 규칙(TBD).
