# 이슈 UI 화면설계

## ① 목적
- [[ACC Build]] Build의 이슈(Issue) 모듈을 동등 재현하기 위한 화면 상세설계.
- 소유 범위 = **핀 배치 인터랙션 + 우측 이슈 상세폼 + 이슈 목록/필터**(작성 모달·지도 보기 포함).
- 본 문서는 UI/인터랙션 골격만 정의. 데이터 모델·API 계약은 백엔드 문서로 위임.

## ② 배경 / 전제
- ★스코프: [[DWG]] 원본 무편집. DWG → 파싱 → 기하 JSON → 2D 뷰어 렌더 → 그 위에 이슈 핀을 **오버레이**. 이슈는 픽셀이 아닌 뷰어 좌표계에 고정.
- 1차 범위 = 2D 시트 한정. 3D/[[BIM]] 핀 배치 범위 밖.
- 핀 좌표는 시트 버전에 종속 → 버전 전환 시 좌표 재투영 필요(❓ 정책 TBD).
- ★차별점: 핀을 좌표뿐 아니라 **설비 엔티티 ID**(DKS 온톨로지)에도 바인딩 → 도면 갱신에도 이슈 추적성 유지. [[TypeDB]] 연동은 Phase 2.
- 뷰어 렌더링 기반 = [[PDF.js]]/Canvas 계열. 상세는 [[05-3-2_2D뷰어UI]] 소유.

## 핀 배치 인터랙션
> 뷰어 위에서 이슈 핀을 찍고/옮기고/선택하는 핵심 인터랙션을 정의.

### 핀 생성 플로우
> "이슈 추가" 모드 진입 → 뷰어 클릭 → 임시 핀 표시 → 작성 모달 오픈까지 단계 기술.
- 진입점: 상단 툴바 "이슈 추가" 버튼 / 우클릭 컨텍스트 메뉴.
- 클릭 지점 = 뷰어 좌표(world coord)로 변환해 보관(스크린 픽셀 아님).

#### 핀 좌표 모델
> 시트 좌표계 ↔ 화면 픽셀 변환 규칙, 줌/팬 시 핀 고정 방식 메모.
- ❓ 좌표 저장 단위: 시트 내부 단위 vs 정규화(0~1) — TBD.

#### 엔티티 바인딩(차별점)
> 클릭 지점 인근 설비 엔티티 자동 추정 → 핀에 entityId 제안. 사용자가 확정/해제.
- Phase 1: 좌표만 저장. entityId 필드는 스키마에 예약(빈 값 허용).

### 핀 선택 / 이동 / 삭제
> 단일/다중 선택, 드래그 이동(편집 권한 시), 삭제 확인 토스트.
- 이동은 "이슈 위치 수정" 권한 한정. 상태가 Closed면 잠금(❓).

### 핀 시각 표현
> 상태별 색/아이콘, 호버 툴팁(번호·제목·상태), 클러스터링 규칙.

#### 상태별 색상 토큰
> Open/In Review/Closed 등 상태 → 색 매핑 표. 디자인 토큰은 [[05-2_공통컴포넌트]] 참조.

#### 핀 클러스터링
> 줌 아웃 시 근접 핀 묶음 표시(숫자 배지). 임계 줌레벨 ❓ TBD.

## 우측 이슈 상세폼(탭)
> 핀/목록 선택 시 우측 인스펙터 패널에 뜨는 상세폼. 공통 인스펙터 셸 위에 구성.

### 패널 셸 / 헤더
> 이슈 번호·제목·상태 뱃지·닫기. 인스펙터 패널 골격은 [[05-2_공통컴포넌트]] 위임.

### 탭 구성
> ACC 기준 탭 분할. 각 탭이 담을 필드 나열.

#### 개요(Overview) 탭
> 제목, 설명, 상태, 유형, 담당자, 기한, 위치(시트·핀), 생성/수정 메타.

#### 활동/댓글(Activity) 탭
> 상태 변경 이력 + 코멘트 스레드. 실시간 갱신은 [[WebSocket]](❓ Phase 1 폴링 대체 가능).

#### 첨부/사진(Attachments) 탭
> 이미지/파일 썸네일 그리드. 업로드는 [[CDE]] 파일 모듈과 연계.

#### 연관(References) 탭
> 연관 마크업·도면·설비 엔티티 링크. 차별점(entityId) 노출 위치.

### 상태 변경 액션
> 상태 전환 버튼/드롭다운, 전환 가능 상태 머신, 권한 가드.

#### 상태 전이 규칙
> Open→In Review→Closed 등 허용 전이만 노출. 전이 매트릭스 ❓ 확정 TBD.

### 인라인 편집 / 저장 UX
> 필드 단위 인라인 편집 vs 일괄 저장. 낙관적 업데이트·실패 롤백·토스트.

## 이슈 목록 / 필터
> 모듈 메인의 데이터테이블 + 필터바. 공통 데이터테이블 위에 이슈 컬럼 구성.

### 데이터테이블 구성
> 컬럼 정의·기본 정렬·행 선택 동작. 정렬/컬럼토글/내보내기 동작은 [[05-2_공통컴포넌트]] 데이터테이블 위임.

#### 컬럼 정의
> #, 제목, 상태, 유형, 담당자, 기한, 시트, 수정일 — 노출/숨김 기본값 표.

#### 행 인터랙션
> 행 클릭 → 우측 상세폼 + 뷰어 핀 포커스(쌍방 하이라이트). 더블클릭 → 해당 시트 이동.

### 필터 바
> 상태/유형/담당자/기한/시트 기준 필터 칩. 저장된 필터(뷰) ❓ TBD.

#### 빠른 필터 프리셋
> "내 이슈", "기한 초과", "미해결" 등 프리셋 칩 정의.

#### 검색
> 제목·번호 텍스트 검색. (Phase 2: 엔티티/온톨로지 의미검색 — 위임 [[TypeDB]].)

### 목록 ↔ 뷰어 연동
> 목록 선택과 뷰어 핀 선택의 양방향 동기화, 스크롤 인투 뷰 규칙.

### 빈 상태 / 로딩 / 에러
> 이슈 0건 빈상태 카드, 스켈레톤 로딩, 로드 실패 재시도. 패턴은 공통 컴포넌트 위임.

---

## 이슈 작성 모달
> 핀 배치 직후 또는 "이슈 추가"에서 뜨는 모달 폼. 모달 골격은 공통 위임.

### 폼 필드 / 검증
> 필수(제목·유형·위치) vs 선택(담당자·기한·설명) 구분, 인라인 검증 규칙.

### 위치 컨텍스트 표시
> 모달 내 미니 뷰어/시트명·핀 좌표 요약. 위치 없이 생성 허용 여부 ❓ TBD.

### 제출 / 취소 UX
> 제출 시 핀 확정·목록 갱신·토스트. 취소 시 임시 핀 제거.

## 지도 보기
> 시트 썸네일 위에 이슈 핀 분포를 한눈에 보는 보조 뷰(ACC "지도/위치" 유사).

### 시트별 핀 분포
> 시트 카드 그리드 + 시트당 이슈 카운트 배지.

### 핀 히트맵 / 밀집도
> ❓ ACC 평가판 범위 확인 필요 — 미존재 시 본 절 축소/삭제 TBD.

### 목록/상세와의 전환
> 지도 핀 클릭 → 상세폼 오픈 + 해당 시트 뷰어 진입.

## ④ 결정 대기 항목 (TBD / ❓)

- ❓ 실시간 갱신: [[WebSocket]] vs 폴링(Phase 1).
- ❓ Closed 이슈 핀 이동/편집 잠금 여부.

- ❓ 저장된 필터(뷰) 기능 1차 포함 여부.
- ❓ 위치 없는 이슈 생성 허용 여부.
- ❓ 지도 보기 핀 히트맵 — ACC 평가판 실존 확인.
