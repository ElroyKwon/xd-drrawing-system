# 앱셸 & 네비게이션 화면설계

## ① 목적
- 청주 [[도면관리 시스템]] 전 모듈(홈/시트/마크업/이슈/시트비교/파일/사진)이 공유하는 **앱셸(App Shell)** 골격 정의.
- 소유 범위 = **헤더 + 모듈 사이드바 + 프로젝트 스위처** 3요소 한정. 그 외 화면 요소는 위임.
- [[ACC]] Build의 모듈 사이드바 + 프로젝트 스위처 패턴을 동등 재현하기 위한 UI 계약 명세.

## ② 배경 / 전제
- 본 시스템은 **CAD 편집기가 아니라 "[[DWG]] 뷰어 + 주석/이벤트 오버레이"** 시스템. 앱셸은 모듈 간 전환 골격만 책임지고, 도면 렌더/편집 로직은 보유하지 않음.
- 1차 범위 = **2D 시트 한정**. 3D BIM 뷰어 진입점은 사이드바에 두지 않음(범위 밖).
- 앱셸은 **데이터 무관(data-agnostic)** 컨테이너. 각 모듈 라우트가 콘텐츠 영역을 채움.
- 향후 Phase 2: [[TypeDB]] 온톨로지 기반 **AI 채팅/엔티티 검색** 진입점이 헤더 또는 사이드바에 추가될 수 있음 → 자리(slot) 예약만 본 문서에서 고려.

---

## 레이아웃
- 앱셸 전체 그리드 구조 정의(고정 헤더 + 좌측 사이드바 + 콘텐츠 영역 + 선택적 우측 인스펙터).
- 영역별 폭/높이, z-index 스택, 스크롤 책임 경계를 채울 것.

### 그리드 골격
- 3분할(헤더 row / 사이드바 col / 콘텐츠) CSS Grid 또는 Flex 구조 채택안 기술.
- 헤더 고정 높이, 사이드바 고정/접힘 폭, 콘텐츠 영역 가변 — 토큰값 채울 것. ❓헤더 높이 px TBD.

### 영역 책임 분리
- 헤더 = 전역 액션(프로젝트 스위처·검색·알림·계정). 사이드바 = 모듈 네비. 콘텐츠 = 모듈 라우트.
- 우측 인스펙터 패널은 [[05-2_공통-UI-컴포넌트]]가 소유 → 본 셸은 슬롯만 제공.

### 스크롤 / 오버플로우 경계
- 헤더·사이드바 고정, 콘텐츠 영역만 독립 스크롤. 2D 뷰어 캔버스의 자체 스크롤/줌과 충돌 회피 규칙 메모.

#### z-index 레이어 규약
- 토스트 > 모달 > 프로젝트 스위처 드롭다운 > 헤더 > 사이드바 > 콘텐츠 순. 수치 토큰 채울 것.

## 모듈 사이드바
- 좌측 세로 모듈 네비게이션 바. ACC Build 모듈 아이콘 레일 재현.
- 모듈 목록·아이콘·활성표시·라우팅 매핑을 채울 것.

### 모듈 항목 정의
- 항목: 홈 / 시트 / 마크업 / 이슈 / 시트비교 / 파일 / 사진. (마크업이 시트 하위 진입인지 독립 항목인지 ❓TBD)
- 각 항목: 아이콘 + 라벨 + 라우트 경로 + 권한 가시성 매핑 테이블 채울 것.
- 범위 밖 항목(3D/Bridge) 비표시 — 명시.

### 활성 상태 / 라우팅
- 현재 라우트 ↔ 사이드바 active 하이라이트 동기화 규칙.
- 딥링크(특정 시트/이슈 URL) 진입 시 올바른 모듈 활성화 보장.

### 아이콘 / 라벨 규약
- 아이콘 세트 출처(❓아이콘 라이브러리 TBD), 라벨 노출 모드(아이콘만 vs 아이콘+텍스트).

#### 향후 확장 슬롯
- Phase 2 AI 채팅/엔티티 검색 진입점 위치 예약(사이드바 하단 분리 영역 후보). 차별점: 핀/이슈가 픽셀이 아닌 **설비 엔티티 ID**에도 바인딩되므로, 엔티티 검색 진입이 셸 레벨에 필요해질 수 있음 — 자리만 예약.

## 프로젝트 / 모듈 스위처
- 헤더 좌측 프로젝트 전환 드롭다운. ACC Build 프로젝트 스위처 재현.
- 현재 프로젝트 표시 + 전환 + 검색 UX를 채울 것.

### 프로젝트 스위처 (헤더)
- 현재 프로젝트명 표시 → 클릭 시 프로젝트 목록 드롭다운/패널.
- 다중 프로젝트 가정 여부 ❓TBD(청주 단일 사업장이면 단순화 가능). 단일이면 비대화형 라벨로 축소.

### 전환 시 상태 처리
- 프로젝트 전환 시 모듈 컨텍스트 리셋/유지 정책, 진행 중 마크업·미저장 이슈 폼 경고 처리.

### 검색 / 필터
- 프로젝트 수 많을 때 드롭다운 내 검색 박스. (단일 프로젝트 확정 시 생략)

#### 권한 / 가시성
- 사용자 역할별 접근 가능 프로젝트만 노출. 권한 모델은 [[06_백엔드-API]]/인증 문서에 위임 → 본 셸은 표시만.

## 반응형 / 접힘
- 화면 폭에 따른 사이드바 접힘 및 헤더 축약 동작.
- 데스크톱 우선(도면 업무 = 대형 모니터). 태블릿/모바일 지원 범위를 채울 것.

### 사이드바 접힘 (collapse)
- 아이콘 전용 ↔ 아이콘+라벨 토글. 접힘 상태 영속(localStorage) 여부 ❓TBD.

### 브레이크포인트 정책
- 데스크톱 우선. 태블릿 이하 지원 수준 ❓TBD(현장 태블릿 열람 요구 확인 필요).
- 2D 뷰어 작업 특성상 모바일 편집은 비대상 — 명시.

### 헤더 축약 / 우선순위
- 좁은 폭에서 헤더 액션(검색·알림) 오버플로우 메뉴로 접는 우선순위 규칙.
