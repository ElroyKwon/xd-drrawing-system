# 화면 상세설계 개요

## ① 목적
- 청주 사업장 [[도면관리 시스템]] 프론트엔드 **화면별 상세설계 문서의 진입점(인덱스)** 제공.
- [[ACC Build]] 37화면 분석을 기준으로 "재현 대상 화면 ↔ 우리 설계 문서"를 1:1 매핑.
- 이 문서는 **인덱스만 소유** — 개별 화면의 상세 와이어프레임/상호작용/데이터바인딩은 각 화면 문서로 위임(DRY).

## ② 배경 / 전제
- **★스코프 제약(모든 화면 공통 전제)**: [[DWG]] 원본 비편집. 흐름 = DWG → 백엔드 파싱 → 기하 JSON → 웹 2D 뷰어 렌더 → 마크업/이슈/이벤트 **오버레이**. CAD 편집기 아님, **뷰어 + 주석/이벤트 오버레이** 시스템.
- 정밀 CAD 편집(osnap·좌표편집·SHX 정밀렌더)은 범위 밖. 1차 = **2D 시트 한정**, 3D BIM 뷰어/Bridge 제외.
- 벤치마크 모듈: 홈 / 시트 / 마크업 / 이슈 / 시트비교 / 파일([[CDE]]) / 사진.
- 차별점(Phase 2): ACC엔 없는 **온톨로지 엔티티 단위** 바인딩 — 핀/마크업/이슈를 픽셀 좌표뿐 아니라 **설비 엔티티 ID**에도 묶음. [[TypeDB]] 연동·AI 채팅은 향후. (해당 화면 문서에서 바인딩 지점만 명시)
- 상세 분석 원천: [[_ACC-Build-화면분석-재현설계]].

## ③ 상세 목차

## 화면 목록
> 재현 대상 화면 전수 + 우선순위 + 담당 설계 문서 경로를 표로 관리. 이 절이 "단일 진실 위치".

### 화면 인벤토리 (ACC 모듈 기준)
> 모듈별 화면을 ID로 부여하고 상태(예정/작성중/완료)·우선순위(P0~P2)·설계문서 링크 컬럼으로 채움.

| 화면 ID | 모듈 | 화면명 | 핵심 기능 | 우선순위 | 설계 문서 |
|---|---|---|---|:--:|---|
| SCR-01 | 홈 | 위젯 대시보드 | 프로젝트 요약 위젯 카드 | P1 | TBD `[[화면-홈]]` |
| SCR-02 | 시트 | 도면 목록 | 데이터테이블(분야·도면번호·버전) | P0 | TBD `[[화면-시트목록]]` |
| SCR-03 | 시트 | 2D 뷰어 | 기하 JSON 렌더 + 팬/줌 | P0 | TBD `[[화면-2D뷰어]]` |
| SCR-04 | 마크업 | 주석 오버레이 | 펜·도형·화살표·텍스트 | P0 | TBD `[[화면-마크업]]` |
| SCR-05 | 이슈 | 이슈 핀/상세 | 핀 배치 + 상세폼 + 목록/필터 | P0 | TBD `[[화면-이슈]]` |
| SCR-06 | 시트비교 | 버전 diff | 두 버전 오버레이 비교 | P1 | TBD `[[화면-시트비교]]` |
| SCR-07 | 파일 | 폴더형 CDE | 폴더트리 + 파일 데이터테이블 | P1 | TBD `[[화면-파일]]` |
| SCR-08 | 사진 | 사진 갤러리 | 그리드 + 위치 태깅 | P2 | TBD `[[화면-사진]]` |

> ❓ ACC 37화면 중 1차 재현 제외(평가판 미포함·범위 밖) 화면 명시 목록 필요 — 3D/Bridge 등.

### 모듈 ↔ 우선순위 정책
> P0(뷰어 핵심 흐름) / P1(보조 모듈) / P2(후순위) 기준선 1~2줄. 스프린트 배치 근거는 [[06_개발계획]]로 위임.

## 와이어프레임 규약
> 각 화면 문서가 따를 **공통 작성 규약**. 형식이 흩어지지 않도록 여기서 한 번만 정의.

### 화면 문서 표준 섹션 구조
> 모든 SCR-xx 문서가 가질 고정 목차 템플릿 정의.
- 화면 목적 / 진입 경로 / 레이아웃(앱셸 영역 표기) / 컴포넌트 배치 / 상호작용 시나리오 / 데이터 바인딩 / 빈상태·에러·로딩 / 엔티티 바인딩 지점(Phase 2 hook).

### 레이아웃 영역 표기법
> 앱셸 기준 영역 명칭 고정 — 모듈사이드바 / 프로젝트스위처 / 메인캔버스 / 우측 인스펙터 / 툴바 / 토스트레이어.
> 와이어프레임은 ASCII 박스 or Excalidraw 중 무엇으로 통일할지 ❓ TBD.

### 상태 표기 규약
> 빈상태 / 로딩(스켈레톤) / 에러 / 권한없음 4종 표기 기호 통일. 실제 컴포넌트는 [[05-2_공통컴포넌트]]로 위임.

### 반응형·브레이크포인트
> 데스크톱 우선(현장 PC). 태블릿/모바일 지원 범위 ❓ TBD.

## 화면-컴포넌트 매핑
> 각 화면이 재사용하는 **공통 컴포넌트**를 역인덱스. 신규 컴포넌트 필요 여부를 조기 식별.

### 공통 UI 컴포넌트 카탈로그 (참조)
> 컴포넌트 정의 자체는 [[05-2_공통컴포넌트]] 소유. 여기선 화면별 사용 매핑만.
- 앱셸 / 데이터테이블(정렬·컬럼토글·필터·내보내기) / 우측 인스펙터 패널 / 위젯카드 / 모달폼 / 빈상태 / 토스트.

### 화면 × 컴포넌트 매트릭스
> 행=화면 ID, 열=공통 컴포넌트. 사용(O)/미사용으로 채워 재사용률·누락 식별.

| 화면 | 앱셸 | 데이터테이블 | 인스펙터 | 위젯카드 | 모달폼 | 빈상태 | 토스트 |
|---|:--:|:--:|:--:|:--:|:--:|:--:|:--:|
| SCR-01 홈 | O | - | - | O | - | O | O |
| SCR-02 시트목록 | O | O | O | - | - | O | O |
| SCR-03 2D뷰어 | O | - | O | - | - | O | O |
| SCR-04 마크업 | O | - | O | - | O | - | O |
| SCR-05 이슈 | O | O | O | - | O | O | O |
| SCR-06 시트비교 | O | - | O | - | - | O | O |
| SCR-07 파일 | O | O | O | - | O | O | O |
| SCR-08 사진 | O | O | O | - | O | O | O |

> 위 매트릭스는 초안 — 각 화면 상세설계 확정 시 갱신.

### 뷰어 전용 컴포넌트 (오버레이 계층)
> 2D 뷰어 위 오버레이 컴포넌트는 일반 UI와 분리 관리. 렌더링 기술( [[PDF.js]] / Canvas / SVG ) 선정은 [[03_2D뷰어-아키텍처]]로 위임.
- 마크업 레이어 / 이슈 핀 레이어 / 엔티티 하이라이트 레이어(Phase 2) / 실시간 협업 커서( [[WebSocket]] ) ❓ 1차 포함 여부 TBD.

### 엔티티 바인딩 매핑 (Phase 2 예약)
> 핀/마크업/이슈가 픽셀 좌표 외에 설비 엔티티 ID로 묶이는 화면(SCR-03/04/05) 표시. 바인딩 스키마는 [[04_백엔드-아키텍처]]·[[TypeDB]] 문서로 위임.
