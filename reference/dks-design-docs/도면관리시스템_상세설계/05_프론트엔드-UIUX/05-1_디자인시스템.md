# 디자인 시스템

## ① 목적
- 청주 [[도면관리 시스템]] 전 화면(홈/시트/마크업/이슈/시트비교/파일/사진)이 공유하는 **단일 시각 언어** 정의.
- 소유 범위(이 문서만): **색 · 타이포 · 간격 · 아이콘 · 톤**. 컴포넌트 동작/조합은 [[05-2_컴포넌트-라이브러리]]에 위임.
- 산출물 = CSS 변수(토큰) 1세트 → 모든 컴포넌트가 토큰만 참조(하드코딩 색·px 금지).

## ② 배경 / 전제
- 벤치마크 = [[ACC]] Build 톤(차분한 뉴트럴 + 단일 액센트). 단, 픽셀 카피 아님 — "동등 재현" 수준의 톤만 차용.
- ★스코프 제약: 본 시스템은 **뷰어 + 주석/이벤트 오버레이** UI 대상. CAD 편집기 UI(좌표 입력 패널 등) 토큰은 정의하지 않음.
- 다크모드: 1차 범위 밖(Phase 2 ❓). 라이트 1테마 우선.
- 토큰 전달 매체 = CSS Custom Properties. 프레임워크 결정은 [[05-2_컴포넌트-라이브러리]] 소유.

## ③ 상세 목차

## 컬러 토큰 (ACC 톤 참고)
> 의도: 전 화면이 참조할 컬러 팔레트를 의미 기반 토큰으로 정의. raw HEX는 여기서만 등장.

### 뉴트럴 스케일
> 의도: 배경/보더/텍스트용 그레이 9~10단계(예: `--gray-50` ~ `--gray-900`) 표. 앱셸·테이블·패널 배경이 여기서 파생.

### 브랜드 / 프라이머리
> 의도: 주 액션·링크·활성탭 색. 1차 = ACC 블루 계열 차용 ❓ vs LS사우타 브랜드 컬러 적용 여부.

### 시맨틱 토큰 (역할 기반)
> 의도: `--bg-surface`, `--bg-canvas`, `--text-primary`, `--text-muted`, `--border-default`, `--border-focus` 등 raw→의미 매핑. 컴포넌트는 시맨틱 토큰만 사용.

#### 뷰어 캔버스 전용
> 의도: 2D 도면 캔버스 배경(흰/연그레이), 그리드선, 선택 하이라이트, 오버레이 위 핀/마크업 대비색. [[PDF.js]]/Canvas 렌더 배경과 충돌 없는 값.

## 타이포그래피
> 의도: 폰트 패밀리·스케일·웨이트 토큰. 도면 라벨 가독성과 UI 텍스트 모두 커버.

### 폰트 패밀리
> 의도: 한글 본문(예: Pretendard/Noto Sans KR ❓) + 숫자/도면번호용 모노 후보. [[SHX]] 폰트 정밀 렌더는 범위 밖 — UI 폰트와 무관.

### 타입 스케일
> 의도: `--text-xs` ~ `--text-2xl` 크기/행간 표. 데이터테이블 셀·위젯카드 제목·모달 헤더 매핑.

### 웨이트 / 라인하이트
> 의도: regular/medium/semibold 3단 + 행간 규칙. 개조식 라벨은 medium 기준.

## 간격 / 그리드
> 의도: 여백·간격의 단일 척도와 레이아웃 그리드 정의. (앱셸 구조 자체는 [[05-3_레이아웃-앱셸]] 소유, 여기선 토큰만.)

### 스페이싱 스케일
> 의도: 4px 베이스 스텝(`--space-1`=4 … `--space-8`=32 등) 표. 패딩/갭 전부 이 토큰으로.

### 레이아웃 그리드 / 컬럼
> 의도: 콘텐츠 영역 컬럼·거터. 모듈사이드바·메인·우측 인스펙터 3분할의 폭 변수(`--sidebar-w`, `--inspector-w`).

### 라운드 / 그림자 / 보더
> 의도: `--radius-*`, `--shadow-card`, `--shadow-modal`, 보더 두께 토큰. 위젯카드/모달/인스펙터 일관 적용.

### 반응형 브레이크포인트
> 의도: 데스크톱 우선 BP. 태블릿 현장 열람 대응 범위 ❓.

## 아이콘 / 버튼 / 배지
> 의도: 액션 표현 3요소의 시각 규격. 동작 스펙은 컴포넌트 문서, 여기선 룩앤필 토큰.

### 아이콘 세트 / 규격
> 의도: 아이콘 라이브러리 후보(Lucide/Tabler ❓), 그리드 사이즈(16/20/24), 스트로크. 모듈사이드바·툴바·이슈상태 아이콘 출처 통일.

### 버튼 위계
> 의도: primary/secondary/ghost/danger 4종 × 사이즈 × 상태(hover/active/disabled) 토큰 매핑. 마크업 툴바 아이콘버튼 포함.

### 배지 / 칩 / 핀
> 의도: 상태 배지·필터 칩·이슈 핀 마커의 색·크기·형태 규격. 핀은 픽셀 좌표뿐 아니라 **설비 엔티티 ID 바인딩**(우리 차별점) 시각 구분 변형 고려 ❓.

## 상태색 (공종 / 이슈상태)
> 의도: 도메인 의미를 색으로 인코딩. 색맹 대비 보조(아이콘/패턴) 병행 규칙.

### 공종(분야) 색 매핑
> 의도: Phase 0 파싱 산출물의 **분야(공종)** 값 → 색 토큰. 시트목록·시트비교 레이어 구분에 사용. 분야 코드 목록은 도면 메타 표준 문서에 위임 ❓.

### 이슈 상태 색
> 의도: 미해결/검토중/해결/닫힘 등 상태별 색(ACC 이슈 톤 참고). 이슈 핀·목록 배지·필터 동일 토큰 공유.

### 시트비교 diff 색
> 의도: 버전 오버레이 시 추가/삭제/변경 레이어 색(예: 추가=청록, 삭제=빨강 반투명). 캔버스 오버레이 대비 확보.

### 접근성 / 대비
> 의도: WCAG AA 본문 대비 목표. 상태색 단독 의존 금지(아이콘 동반) 원칙 명시.
