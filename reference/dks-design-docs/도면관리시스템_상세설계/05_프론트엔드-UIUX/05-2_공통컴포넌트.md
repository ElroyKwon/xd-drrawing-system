# 공통 컴포넌트 라이브러리

## ① 목적
- 전 모듈(홈/시트/마크업/이슈/시트비교/파일/사진)이 재사용하는 UI 컴포넌트 카탈로그 정의
- [[ACC Build]] 공통 UI 패턴을 동등 재현하기 위한 단일 컴포넌트 소스(SoT)
- 모듈별 화면설계는 [[05-1_화면설계]]가 소유 → 이 문서는 "조립 부품"만 소유 (DRY)

## ② 배경/전제
- 스코프 제약: 본 시스템은 CAD 편집기가 아닌 "[[2D 뷰어]] + 주석/이벤트 오버레이". 컴포넌트는 뷰어 위 오버레이 UI와 데이터 관리 UI에 집중 (정밀 CAD 편집 위젯 없음)
- 1차 범위 = 2D 시트 한정. 3D BIM 관련 컴포넌트 범위 밖
- 뷰어 캔버스 자체(렌더·줌·팬)는 별도 소유 → [[05-4_뷰어컴포넌트]]로 위임. 본 문서는 뷰어를 감싸는 셸/패널/툴바만 다룸
- DKS 차별점: 핀/마크업/이슈는 픽셀 좌표뿐 아니라 "설비 엔티티 ID"에도 바인딩 가능 → 인스펙터·모달폼에 엔티티 선택 슬롯 예약 (Phase 2 [[TypeDB]] 온톨로지 연동)
- 기술 스택 가정: 컴포넌트 프레임워크/스타일 시스템 = ❓TBD (vault FMS 목업과 정합성 검토 필요)

## ③ 상세 목차

### 0. 컴포넌트 설계 공통 규약
- 토큰 기반 스타일(색·간격·타이포·라운드·그림자) 참조: [[05-5_디자인토큰]]로 위임
- 명명 규칙·폴더 구조·스토리북 유무: ❓TBD
- 접근성 기준선(키보드 포커스·ARIA·대비): 채울 항목 — 각 컴포넌트 최소 a11y 요구

### 1. 앱셸 (AppShell)
- 전 화면을 감싸는 최상위 레이아웃. 헤더 + 모듈 사이드바 + 콘텐츠 영역 3분할 골격 정의

#### 1.1 헤더 (TopBar)
- 채울 항목: 프로젝트 스위처(드롭다운), 전역 검색 진입점, 사용자 메뉴, 알림 벨
- DKS 슬롯: 향후 AI 채팅 진입 버튼 위치 예약 (Phase 2)

#### 1.2 모듈 사이드바 (ModuleNav)
- 채울 항목: 모듈 아이콘 목록(홈/시트/마크업/이슈/비교/파일/사진), 활성 상태 표시, 접힘(collapse) 동작
- 모듈 ↔ 라우트 매핑 표는 [[05-1_화면설계]] 참조

#### 1.3 프로젝트 스위처 (ProjectSwitcher)
- 채울 항목: 현재 프로젝트 표시, 프로젝트 목록 전환 드롭다운, 권한별 노출 범위 ❓TBD

#### 1.4 반응형/브레이크포인트
- 채울 항목: 데스크탑 우선. 태블릿 이하 사이드바 거동 ❓TBD (납품 환경 해상도 확인 필요)

### 2. 데이터테이블 (DataTable)
- 시트목록·이슈목록·파일목록·사진목록이 공유하는 핵심 테이블. 가장 재사용도 높음

#### 2.1 컬럼 모델
- 채울 항목: 컬럼 정의 스키마(키·라벨·정렬가능·너비·렌더러), 컬럼 토글(표시/숨김), 컬럼 순서

#### 2.2 정렬·필터
- 채울 항목: 단일/다중 정렬, 컬럼 필터 + 상단 빠른 필터 바, 필터 상태 직렬화(URL 동기화 여부 ❓TBD)

#### 2.3 행 상호작용
- 채울 항목: 행 클릭 → 우측 인스펙터 오픈, 다중 선택(체크박스), 행 액션 메뉴(⋯)

#### 2.4 내보내기 (Export)
- 채울 항목: CSV/Excel 내보내기 범위(전체/필터결과/선택행), [[PDF]] 보고서 내보내기 가능 여부 ❓TBD

#### 2.5 페이징·가상화
- 채울 항목: 서버 페이징 vs 무한스크롤, 대용량 행 가상 스크롤 적용 기준

#### 2.6 빈/로딩/에러 상태
- 빈 상태는 §6 EmptyState 위임. 로딩 스켈레톤·에러 행 표현 정의

### 3. 우측 인스펙터 패널 (InspectorPanel)
- 행/핀/마크업/이슈 선택 시 우측에서 슬라이드 인하는 상세/편집 패널. 모듈 공통 컨테이너

#### 3.1 패널 셸
- 채울 항목: 헤더(제목·닫기), 본문 슬롯, 푸터 액션 바, 너비/드래그 리사이즈 여부 ❓TBD

#### 3.2 콘텐츠 슬롯 규약
- 채울 항목: 모듈별 본문(이슈 상세폼·파일 메타·사진 정보)을 끼우는 슬롯 인터페이스
- DKS 슬롯: "연결된 설비 엔티티" 섹션 자리 예약 (엔티티 ID 표시·이동 링크, Phase 2)

#### 3.3 탭형 인스펙터
- 채울 항목: 상세/활동(코멘트·이력)/첨부 등 탭 구성 필요 모듈 식별

### 4. 위젯카드 (WidgetCard)
- 홈 대시보드 위젯 + 인스펙터 내 정보 블록 공용 카드 컨테이너

#### 4.1 카드 셸
- 채울 항목: 헤더(타이틀·액션), 본문, 빈/로딩 상태, 클릭 시 모듈 딥링크

#### 4.2 위젯 변형(variant)
- 채울 항목: 카운트형(이슈 수)·리스트형(최근 시트)·차트형 여부 ❓TBD. 홈 위젯 카탈로그는 [[05-1_화면설계]] 홈 절 참조

### 5. 모달폼 (ModalForm)
- 이슈 생성·필터 저장·파일 업로드 등 다이얼로그 + 폼 공용 래퍼

#### 5.1 모달 셸
- 채울 항목: 오버레이·포커스 트랩·ESC 닫기·크기(sm/md/lg), 스택(중첩 모달) 정책 ❓TBD

#### 5.2 폼 필드 키트
- 채울 항목: 입력/셀렉트/날짜/담당자선택/우선순위/첨부 등 공통 필드 컴포넌트 목록
- DKS 슬롯: "설비 엔티티 선택" 필드 예약 (이슈/핀 ↔ 엔티티 바인딩용, Phase 2)

#### 5.3 검증·제출
- 채울 항목: 클라이언트 검증 규칙 위치, 제출 중 로딩·실패 처리, 낙관적 업데이트 여부

### 6. 빈상태 (EmptyState)
- 데이터 없음/검색결과 없음/권한없음 등 공통 빈 화면 표현

#### 6.1 변형
- 채울 항목: 최초빈(생성 CTA 포함)·검색결과없음·에러형 구분, 일러스트/아이콘 사용 기준

### 7. 토스트 (Toast)
- 저장/삭제/업로드 완료·실패 등 비차단 피드백 알림

#### 7.1 토스트 시스템
- 채울 항목: 타입(성공/경고/에러/정보), 자동 닫힘 시간, 액션 버튼(실행취소 등), 스택·위치
- 실시간 이벤트([[WebSocket]]) 수신 시 토스트 트리거 연동 여부 ❓TBD → [[05-3_상태관리]] 위임

### 8. 컴포넌트 ↔ 모듈 사용 매트릭스
- 채울 항목: 행=컴포넌트, 열=모듈, 셀=사용 여부. 재사용도/우선 구현 순서 도출용 표
