# 상태관리 & 라우팅

> 소유 범위: **클라이언트 상태/라우팅 설계만**. 컴포넌트 트리·디자인 토큰은 [[05-1_앱셸-공통UI]], 뷰어 렌더 파이프라인은 [[05-3_2D뷰어-렌더링]], API 계약은 [[04_백엔드-API설계]]에 위임.

## 1. 목적
- 청주 [[도면관리 시스템]] 웹 클라이언트의 **URL ↔ 화면 매핑 규칙**과 **상태 보관 위치(서버/클라이언트/뷰어)** 경계를 확정.
- [[ACC]] Build 37화면(홈/시트/마크업/이슈/시트비교/파일/사진)을 동등 재현하는 라우팅 골격 제공.

## 2. 배경 / 전제
- ★스코프: [[DWG]] 비편집 — 흐름은 `DWG → 백엔드 파싱 → 기하 JSON → 2D 뷰어 렌더 → 마크업/이슈/이벤트 오버레이`. CAD 편집기 아님 = **편집 상태머신·undo 그래프 불필요**.
- 1차 = 2D 시트 한정. 3D/Bridge 라우트 없음.
- 멀티 프로젝트 구조: 프로젝트 스위처가 모든 라우트의 상위 컨텍스트.
- 차별점(Phase 2): 핀/마크업/이슈를 픽셀 좌표뿐 아니라 **설비 엔티티 ID**에도 바인딩 → 상태 모델에 `entityId?` 필드 자리 예약. [[TypeDB]] 온톨로지 연동은 향후.

## 라우팅 맵
> 채울 것: 전체 URL 트리 + 각 라우트의 진입 데이터·가드·딥링크 규칙.

### URL 트리 (초안)
- `/projects` — 프로젝트 선택 (스위처 랜딩)
- `/p/:projectId/home` — 위젯 대시보드
- `/p/:projectId/sheets` — 시트 목록(데이터테이블)
- `/p/:projectId/sheets/:sheetId` — 2D 뷰어 (마크업/이슈 오버레이 진입점)
- `/p/:projectId/issues` — 이슈 목록 + 필터
- `/p/:projectId/issues/:issueId` — 이슈 상세 (인스펙터 패널)
- `/p/:projectId/compare/:sheetId` — 시트 버전 비교 diff
- `/p/:projectId/files` — 폴더형 [[CDE]]
- `/p/:projectId/photos` — 사진
> 메모: 모듈사이드바 7항목 ↔ 위 라우트 1:1 대응. 확정 후 [[05-1_앱셸-공통UI]] 사이드바와 동기화.

### 딥링크 / 쿼리 파라미터
> 채울 것: 뷰어 좌표·핀·필터를 URL에 보존하는 규칙 (공유 링크 재현성).
- 뷰어 뷰포트: `?x=&y=&zoom=` 또는 `?fit=sheet` — ❓ 정밀도/반올림 정책 TBD
- 이슈 핀 포커스: `/sheets/:sheetId?issue=:issueId`
- 목록 필터 상태: `?status=&assignee=&q=` (테이블 필터 ↔ URL 동기화 여부 TBD)

### 라우트 가드 / 진입 로딩
> 채울 것: 프로젝트 권한 확인, 미선택 시 `/projects` 리다이렉트, 시트/이슈 404 처리.
- 모달 라우트(이슈 생성폼) = 배경 라우트 유지하는 오버레이 라우팅 — ❓ 채택 여부 TBD

### 상태 분류 (먼저 합의)
- **서버 상태**: 시트·이슈·마크업·파일 목록 — 원천은 백엔드, 클라이언트는 캐시.
- **URL 상태**: 현재 모듈·선택 항목·필터·뷰포트 (위 라우팅 맵).
- **클라이언트 UI 상태**: 인스펙터 열림, 활성 마크업 도구, 토스트 큐, 컬럼 토글.
- **뷰어 임시 상태**: 렌더 중 팬/줌/그리는 중 마크업 — § 뷰어 상태 격리 참조.

## 뷰어 상태 격리
> 채울 것: 2D 뷰어 내부 상태를 전역 스토어에서 **분리**하는 경계. (렌더 알고리즘 아님 — 그건 [[05-3_2D뷰어-렌더링]])

### 격리 원칙
- 팬/줌/마우스 드래그 등 고빈도 상태는 전역 스토어에 넣지 않음 (리렌더 폭발 방지).
- 뷰어는 자체 로컬 상태 + 명령형 핸들(ref) 보유, 외부와는 **선택/저장 시점에만** 동기화.

### 좌표계 / 변환 상태
> 채울 것: 도면좌표 ↔ 화면픽셀 변환 행렬 보관 위치, 핀·마크업 좌표 저장 단위(도면좌표 권장) — 상세 변환식은 [[05-3_2D뷰어-렌더링]] 위임.
- ❓ 마크업/핀 영속 좌표를 도면좌표로 저장(권장) vs 픽셀 — 결정 후 백엔드 스키마 영향

### 오버레이 레이어 상태
> 채울 것: 마크업 레이어·이슈 핀 레이어·시트비교 diff 레이어의 표시 토글·z-order 상태 관리.

### 엔티티 바인딩 (Phase 2 자리)
> 채울 것: 핀/마크업의 `entityId` 부착·하이라이트 상태. 지금은 필드만 예약, 동작은 [[TypeDB]] 연동 시.

## 캐시 / 낙관적 업데이트
> 채울 것: 서버 상태 캐시 무효화·마크업/이슈 생성 시 낙관적 반영 규칙.

### 캐시 무효화 규칙
> 채울 것: 이슈 생성/수정 → 목록·상세·뷰어 핀 캐시 동시 갱신 매트릭스.

### 낙관적 업데이트 대상
- 마크업 그리기 완료 → 즉시 화면 반영 후 저장 확정/롤백.
- 이슈 핀 드롭 → 즉시 표시.
> 채울 것: 롤백 UX(토스트), 충돌 처리.
