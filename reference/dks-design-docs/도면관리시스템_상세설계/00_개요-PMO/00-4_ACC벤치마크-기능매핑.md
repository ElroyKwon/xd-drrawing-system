## 목적

- ACC Build 37화면 분석 결과를 자사 [[도면관리 시스템]] 기능으로 **역매핑**하는 마스터표 소유
- 각 화면이 어느 모듈/기능으로 귀결되는지, **우선순위(P0~P3)** 부여
- 1차 목표 = ACC Build 동등 재현. 차별점(온톨로지 바인딩)은 어디에 끼울지 표기
- ※ 이 문서는 "무엇을 만들지 목록"만 소유. 화면별 UI 상세 재현은 [[_ACC-Build-화면분석-재현설계]]에 위임

## 배경/전제

- 스코프 제약(모든 매핑의 전제):
  - [[DWG]] 원본 **편집 금지** → DWG → 백엔드 파싱 → 기하 JSON → 웹 2D 뷰어 렌더 → 그 위 마크업/이슈/이벤트 **오버레이**
  - CAD 편집기 아님 = "뷰어 + 주석/이벤트 오버레이"
  - 정밀 CAD(osnap, 좌표 편집, [[SHX]] 폰트 정밀 렌더) **범위 밖**
  - 1차 범위 = **2D 시트 한정**. 3D BIM/Bridge **범위 밖**
- 벤치마크 모듈: 홈 / 시트 / 마크업 / 이슈 / 시트비교 / 파일([[CDE]]) / 사진
- 공통 UI: 앱셸 / 데이터테이블 / 우측 인스펙터 / 위젯카드 / 모달폼 / 빈상태 / 토스트
- 우선순위 정의:
  - **P0** = ACC 동등 재현 1차 필수(MVP)
  - **P1** = 1차 범위 내 보강
  - **P2** = Phase 2(온톨로지/AI 연동)
  - **P3** = 후순위/범위 밖 후보

## 모듈별 기능 역매핑표

> 의도: ACC 모듈 → 자사 기능명 → 담당 화면 수 → 비고(스코프 제약). 팀이 모듈 단위로 작업 범위를 한눈에.

| ACC 모듈 | 자사 기능명(가칭) | 핵심 동작 | 스코프 비고 |
|---|---|---|---|
| 홈 | 대시보드 | 위젯카드 집계·최근활동 | 위젯 구성 TBD |
| 시트 | 도면목록 + 2D뷰어 | 목록 → [[PDF.js]]/Canvas 렌더 | 편집 없음, 렌더만 |
| 마크업 | 주석 오버레이 | 펜·도형·화살표 | 픽셀좌표 저장 |
| 이슈 | 이슈 핀/폼/목록 | 핀 + 상세폼 + 필터 | 엔티티ID 바인딩(P2) |
| 시트비교 | 버전 diff | 버전 오버레이 비교 | 래스터 diff 우선 |
| 파일 | 폴더형 CDE | 폴더 트리 + 버전 | 권한모델 TBD |
| 사진 | 사진 갤러리 | 업로드·태그·핀연결 | 1차 P2 후보 |

#### 채울 것
- 각 행에 담당 화면 번호(분석문서 화면 ID) 매핑 컬럼 추가
- 자사 기능명 가칭 → 확정 시 [[도면관리 시스템]] 용어집과 동기화

## 화면→기능→우선순위(P0~P3)

> 의도: 37화면을 1행씩 풀어 우선순위·담당모듈·차별화여부 부여하는 **마스터 행 테이블**. 이 문서의 심장.

### P0 (MVP 동등 재현 필수)

| 화면 | 모듈 | 기능 | 차별화 |
|---|---|---|---|
| 앱셸(사이드바+프로젝트스위처) | 공통 | 네비게이션 골격 | - |
| 도면 목록(데이터테이블) | 시트 | 정렬·컬럼토글·필터 | - |
| 2D 뷰어 | 시트 | 기하JSON 렌더·줌·팬 | - |
| 마크업(펜·도형·화살표) | 마크업 | 오버레이 그리기 | - |
| 이슈 핀 + 상세폼 | 이슈 | 핀 생성·상세 입력 | 엔티티 바인딩 훅(P2) |
| 이슈 목록 + 필터 | 이슈 | 목록·상태필터 | - |

#### 채울 것
- 위 6행은 예시 골격 → 분석문서 화면 ID로 실제 행 확정
- 각 행 "수용기준(acceptance)" 1줄 컬럼 추가

### P1 (1차 범위 내 보강)

> 의도: 시트비교, 파일 CDE, 빈상태·토스트 등 UX 완결 항목. 표로 나열.

#### 채울 것
- 시트비교 diff 방식(래스터 vs 벡터) 결정 후 행 확정 → ❓ 아래 참조
- 파일 CDE 권한/버전 행

### P2 (Phase 2 — 온톨로지/AI)

> 의도: [[TypeDB]] 온톨로지 연동, 엔티티 단위 검색·영향도, AI채팅 바인딩 항목.

#### 채울 것
- 핀/마크업/이슈 ↔ 설비 엔티티 ID 바인딩 행
- AI채팅 → 뷰어/이슈 컨텍스트 주입 행

### P3 (후순위 / 범위 밖 후보)

> 의도: 사진 모듈 고도화, 3D, Bridge 등. 범위 밖 명시 보관용.

## 자사 차별화 포인트(온톨로지 바인딩)

> 의도: ACC에 **없는** 우리 무기만 모아서, 어느 P0 기능에 "훅(hook)"을 미리 심어야 하는지 명시. 1차 구현 시 미래 확장 자리 확보가 핵심.

- 핀/마크업/이슈를 **픽셀좌표 + 설비 엔티티 ID** 이중 바인딩 (ACC는 픽셀만)
- Phase 0 도면 파싱 산출물(분야·도면번호·태그) → 엔티티 검색 기반
- 엔티티 단위 **영향도 분석**(연관 도면·설비 추적) — [[TypeDB]] 질의
- 뷰어/이슈에 **AI채팅** 컨텍스트 바인딩(Phase 2)

#### 채울 것
- P0 단계에서 미리 심을 "엔티티ID 필드" 데이터 스키마 훅 위치 명시
- 차별화 기능별 ACC 대비 우위 1줄

## 미수집 항목(양식/Bridge/3D)

> 의도: 평가판 화면에 없어 분석 못 한 항목 = 추정 금지, 보류 목록. 추후 수집 트리거.

- **양식(Forms)**: 점검 체크리스트류 — 미수집, 1차 범위 외 추정
- **Bridge**: 프로젝트 간 데이터 연동 — 평가판 미포함, 범위 밖
- **3D BIM 뷰어**: 평가판 미포함, 범위 밖
- 마크업 권한/협업 동시편집([[WebSocket]] 실시간) — 수집 깊이 부족

#### 채울 것
- 추가 평가판/문서 확보 시 항목별 "재현 필요 여부" 판정
