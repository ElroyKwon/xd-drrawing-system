# 범위정의 (Scope)

> 본 문서는 청주 사업장 납품용 [[도면관리 시스템]]의 **범위 헌법(Constitution)**. In/Out of Scope를 못박아 모든 하위 설계 문서의 전제가 된다. 분쟁/요구 추가 시 이 문서가 기준선(baseline).

## 목적
- 1차 납품 범위와 비범위를 명문화 → 스코프 크리프 차단.
- 모든 모듈/기능 설계는 본 문서의 In Scope 정의를 상속한다.
- (의도) 1~2줄로 "왜 이 경계인지" 근거까지 기록. 빈 선언 금지.

## 배경/전제
- 벤치마크 = [[ACC Build]](Autodesk Construction Cloud) Build 제품, 37화면 분석 존재 → [[_ACC-Build-화면분석-재현설계]].
- 1차 목표 = ACC Build **동등 재현**. AI 채팅/지식질의는 Phase 2.
- ★대전제: [[DWG]] 원본을 **편집하지 않는다**. 흐름 = DWG → 백엔드 파싱 → 기하 JSON → 웹 2D 뷰어 렌더 → 마크업/이슈/이벤트 **오버레이**.
- 즉 본 시스템은 **CAD 편집기가 아니라 "뷰어 + 주석/이벤트 오버레이"**.
- [[DKS]] 차별점: 도면=진실원천. 핀/마크업/이슈를 픽셀뿐 아니라 **설비 엔티티 ID**에도 바인딩(Phase 2 [[TypeDB]] 온톨로지).
- (의도) 위 전제 중 변경 불가 항목과 협의 가능 항목 구분 표기.

## In Scope (2D 뷰어 + 오버레이 5모듈)
- (의도) ACC Build 모듈 중 1차 재현 대상만 확정. 각 모듈 = 별도 상세설계 문서로 위임(DRY), 여기선 "포함된다"만 선언.

### 코어 파이프라인
- DWG 업로드 → 백엔드 파싱 → 기하 JSON 산출 → 2D 뷰어 렌더.
- (의도) 파싱 산출물 스키마/렌더 책임선은 [[00-2_아키텍처개요]]로 위임.

### 모듈 1: 홈 (위젯 대시보드)
- 프로젝트 요약 위젯 카드. (의도) 위젯 종류 목록은 모듈 문서에서.

### 모듈 2: 시트 (도면목록 + 2D 뷰어)
- 도면 목록(데이터테이블) + 단일 시트 2D 렌더 뷰어. [[PDF.js]]/Canvas 계열 렌더 검토.
- (의도) 뷰어 인터랙션(팬/줌/시트전환) 범위만 명시, 정밀편집은 Out.

### 모듈 3: 마크업 (펜·도형·화살표)
- 뷰어 위 오버레이 주석: 펜/도형/화살표/텍스트. 원본 DWG 불변.

### 모듈 4: 이슈 (핀 + 상세폼 + 목록 + 필터)
- 도면 위 핀 배치 + 이슈 상세폼/목록/필터. ★핀을 픽셀 좌표 + (Phase 2) 엔티티 ID에 바인딩.

### 모듈 5: 시트 비교 (버전 오버레이 diff)
- 두 버전 시트 오버레이 diff. (의도) diff 알고리즘 깊이는 모듈 문서 위임.

### 보조 모듈 (범위 포함 여부 ❓ TBD)
- 파일(폴더형 [[CDE]]) / 사진 모듈 — 1차 포함 여부 결정 대기.

### 공통 UI 컴포넌트
- 앱셸(모듈 사이드바 + 프로젝트 스위처), 데이터테이블(정렬·컬럼토글·필터·내보내기), 우측 인스펙터 패널, 위젯카드, 모달폼, 빈상태, 토스트.
- (의도) 컴포넌트 명세는 공통 UI 설계 문서로 위임.

## Out of Scope (DWG 편집 / 정밀 CAD / 3D BIM / Bridge)
- (의도) 명시적 비범위. 과거 초안의 과한 가정을 폐기 근거와 함께 못박는다.

### DWG 원본 편집
- DWG 직접 수정/저장 안 함. 모든 변경은 오버레이 레이어에만 기록.

### 정밀 CAD 편집
- osnap 스냅, 좌표 직접 편집, [[SHX]] 폰트 정밀 렌더 등 제외.
- 근거: 과거 `Development_Design` 초안의 CAD-editor 가정 **폐기**.

### 3D BIM 뷰어 / Bridge
- 3D BIM 뷰어, ACC Bridge 연동 제외. (ACC 평가판에도 미포함.)

### 1차 범위 = 2D 시트 한정
- 3D 모델/IFC 등 비-2D 자산 제외.

## Phase 구분
- (의도) Phase별 산출물 경계와 전환 게이트 명시.

### Phase 1 — 도면관리 (ACC Build 동등 재현)
- 위 In Scope 5모듈 + 코어 파이프라인 + 공통 UI.
- 완료 기준 ❓ TBD (수용 기준 문서로 위임 예정).

### Phase 2 — AI 채팅 / 지식질의 ([[TypeDB]] 온톨로지)
- Phase 0 도면 파싱 산출물(분야·도면번호·태그) + [[TypeDB]] 온톨로지 연동.
- 엔티티 단위 검색·영향도·AI 채팅을 뷰어/이슈에 바인딩.
- 실시간 채널([[WebSocket]]) 검토 ❓ TBD.
- (의도) Phase 1↔2 데이터 연속성(엔티티 ID 바인딩 설계)은 Phase 1에서 미리 훅만 확보.
