# 용어집 (Glossary)

## 목적
- 청주 사업장 [[도면관리 시스템]] 전 문서가 공유할 **단일 용어 기준** 정의.
- [[ACC]](Autodesk Construction Cloud) Build 용어와 자사/[[DKS]](데이터 지식 스튜디오) 용어 간 **혼선 방지** 및 매핑 고정.
- *의도 메모*: 다른 문서가 용어를 재정의하지 않고 이 문서를 인용하도록 SoT(Single source of Truth) 역할.

## 배경/전제
- 핵심 스코프: [[DWG]] 원본 **비편집**. 흐름 = DWG → 백엔드 파싱 → 기하 JSON → 웹 2D 뷰어 렌더 → 마크업/이슈/이벤트 **오버레이**.
- 본 시스템 = "CAD 편집기"가 아니라 **뷰어 + 주석/이벤트 오버레이** 시스템. 용어도 이 전제에 종속.
- 1차 범위 = **2D 시트 한정**. 3D/BIM/Bridge 용어는 범위 밖 → 등재하더라도 "(범위 밖)" 명시.
- 차별점: ACC엔 없는 **온톨로지 엔티티 ID 바인딩**(핀/마크업/이슈를 픽셀 아닌 설비 엔티티에 결속) → 관련 용어 별도 표기.

## 1. 용어 정의표 (핵심 도메인)

> *의도 메모*: 아래 표를 팀이 합의된 정의로 채운다. "정의" 칸은 1~2문장, "본 시스템 적용" 칸은 우리 스코프 안에서의 의미를 적는다.

| 용어 | 정의 | 본 시스템 적용 | 비고 |
|---|---|---|---|
| 시트(Sheet) | 한 장의 도면 페이지 | 파싱된 기하 JSON 1건 = 1시트 | ACC Sheet 대응 |
| 도면(Drawing) | DWG/PDF 원본 문서 | 업로드 원본, 비편집 | 1도면 N시트 가능 ❓ |
| 버전세트(Version Set) | 동일 도면군의 발행 묶음 | 시트비교 기준 단위 | TBD: 명명 규칙 |
| 버전(Version) | 도면/시트의 리비전 | 시트비교 오버레이 diff 대상 | |
| 마크업(Markup) | 뷰어 위 펜·도형·화살표 주석 | 오버레이 레이어, 원본 불변 | 픽셀+엔티티 바인딩 |
| 이슈(Issue) | 위치 결부 작업/하자 항목 | 핀 + 상세폼 + 목록 + 필터 | ACC Issue 대응 |
| 핀(Pin) | 이슈를 도면 좌표에 고정한 마커 | 좌표 OR 엔티티 ID 결속 | ★자사 무기 |
| 오버레이(Overlay) | 렌더 위에 얹는 주석/이벤트 레이어 | 마크업·핀·이벤트 공통 개념 | 원본 비편집 보장 |
| [[CDE]] | 공통 데이터 환경(폴더형 파일 저장소) | 파일 모듈 = 폴더형 CDE | ACC Files 대응 |
| 공종(Discipline) | 도면 분야 분류(전기/기계/배관 등) | Phase 0 파싱 산출 분야값 | 필터·분류 키 |
| 이벤트(Event) | 도면/엔티티에 결부된 상태 변경 기록 | 오버레이 타임라인 항목 | TBD: 범위 |
| 엔티티(Entity) | 온톨로지 설비 단위 객체 | [[TypeDB]] 엔티티 ID | Phase 2 바인딩 |

## 2. ACC ↔ 자사 용어 매핑

> *의도 메모*: 벤치마크([[ACC]] Build) 화면·기능 용어를 자사 용어로 1:1 고정. 재현설계 문서와 충돌 없게 유지.

| ACC Build 용어 | 자사 용어 | 매핑 메모 |
|---|---|---|
| Project | 사업장/프로젝트 | 청주 사업장 단위 |
| Sheets | 시트 | 2D 뷰어 대상 |
| Version Set | 버전세트 | 발행 묶음 |
| Markups | 마크업 | 오버레이 주석 |
| Issues | 이슈 | 핀+폼+목록 |
| Pin | 핀 | +엔티티 ID(자사 확장) |
| Compare | 시트비교 | 버전 오버레이 diff |
| Files | 파일(CDE) | 폴더형 저장소 |
| Photos | 사진 | 현장 사진 첨부 |
| Home(Widgets) | 홈/위젯 대시보드 | 위젯카드 |
| Bridge | (범위 밖) | 미구현 명시 |
| Model(3D) | (범위 밖) | 2D 한정 |

### 2.1 자사 고유 용어(ACC 무대응)
- *의도 메모*: ACC에 없는 DKS/온톨로지 용어 — "대응 없음" 사유 기재.
- 엔티티 바인딩 / 영향도(Impact) / AI채팅 / 지식질의 → Phase 2, [[TypeDB]] 연동. 상세는 [[Phase2-온톨로지연동]]에 위임.

## 3. 약어표

> *의도 메모*: 본문 전체에서 쓰는 약어를 알파벳/가나다 순으로 정리. 첫 등장 풀이 1회.

| 약어 | 원어 | 의미 |
|---|---|---|
| ACC | Autodesk Construction Cloud | 벤치마크 제품 |
| CDE | Common Data Environment | 공통 데이터 환경 |
| DKS | Data Knowledge Studio | 데이터 지식 스튜디오 |
| DWG | Drawing | AutoCAD 원본 포맷 |
| SoT | Single source of Truth | 단일 진실 원천 |
| BIM | Building Information Modeling | (범위 밖) |
| SHX | Shape/Font (AutoCAD) | (정밀 렌더 범위 밖) |
| TBD | To Be Determined | 결정 대기 |

### 3.1 기술 스택 약어
- *의도 메모*: 아키텍처 문서와 중복 정의 금지 — 약어 표기만, 상세는 위임.
- [[PDF.js]] / [[WebSocket]] / [[TypeDB]] → 상세 정의는 [[00-3_아키텍처개요]].
