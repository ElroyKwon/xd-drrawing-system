## 목적

[[ACC]] Build의 이슈 핀·마크업을 동등 재현하기 위한 **좌표 저장 모델**과 그 좌표가 가리키는 **객체(시트/설비 엔티티) 바인딩 규격**, 그리고 생성·이동·해소 시 흐르는 **이벤트 페이로드 규격**을 정의한다.

> 이 문서가 소유하는 것 = "어디에(좌표) + 무엇에(객체) 묶이는가 + 그 변화가 어떻게 전파되는가". 핀/마크업 UI·렌더링 자체는 [[04_뷰어렌더링-설계]], 이슈 워크플로는 [[05_이슈-마크업-기능설계]]에 위임.

## 배경/전제

- 흐름: [[DWG]] → 백엔드 파싱 → 기하 JSON → 웹 2D 뷰어 렌더 → 그 위에 마크업/이슈/이벤트 **오버레이**. DWG 원본은 편집하지 않음.
- 좌표는 **뷰어 좌표계**가 아니라 **시트 고유 좌표계**에 저장 → 줌/팬/뷰포트 변경에 불변(invariant). (메모: 픽셀 저장 금지)
- 1차 범위 = 2D 시트 한정. 3D/BIM 좌표는 범위 밖.
- ★차별점: 핀/마크업을 픽셀뿐 아니라 **설비 엔티티 ID**에도 묶을 수 있음 → ACC엔 없는 영향도·검색·AI채팅 바인딩의 토대([[TypeDB]] 연동은 Phase 2).

## 이슈핀 좌표 모델 {sheet, x, y}

> 채울 내용: 핀 1개를 유일하게 식별·위치시키는 최소 필드 집합과 좌표계 정의.

### 좌표 기준계(Coordinate Reference)
> 시트 원점·축 방향·단위(도면 단위 vs 정규화 0~1). 권장: 시트 바운딩박스 기준 정규화 좌표(해상도 독립). 메모: DWG model-space 좌표 보존 여부 TBD.

### 핀 레코드 스키마
> `{ pin_id, sheet_id, sheet_version, x, y, anchor_type, created_by, created_at }` 골격. anchor_type = `pixel | entity | hybrid`.

#### 멀티 시트 / 멀티 버전 처리
> 같은 이슈가 여러 시트에 핀을 가질 수 있는지, 시트 버전 변경 시 핀 좌표 승계 정책. 메모: [[03-3_버전관리]]와 연동.

### 핀-이슈 카디널리티
> 핀 1 : 이슈 1 기본. 핀 없는 이슈(시트 미연결) 허용 여부 TBD.

## 마크업 지오메트리 저장

> 채울 내용: 펜·도형·화살표·텍스트 등 마크업 종류별 기하 직렬화 포맷.

### 지오메트리 타입 정의
> `pen(polyline)`, `rect`, `ellipse`, `arrow`, `text`, `cloud` 등. 각 타입별 좌표 배열·스타일(색·두께·불투명도) 필드 표.

### 직렬화 포맷
> 권장: 시트 정규화 좌표 기반 JSON. SVG path 채택 여부 TBD. 메모: [[PDF.js]]/Canvas 오버레이 렌더와 동일 좌표계 공유 필수.

#### 좌표 정밀도 / 압축
> float 자리수, 펜 스트로크 포인트 다운샘플링 정책. 대용량 마크업 페이로드 한계 TBD.

### 마크업 그룹·레이어
> 마크업 묶음(group), 레이어 가시성 토글 메타. 메모: 렌더 z-order는 [[04_뷰어렌더링-설계]] 소유.

## Handle ID ↔ 엔티티 가교

> 채울 내용: DWG 객체와 우리 온톨로지 설비 엔티티를 잇는 핵심 가교 테이블. 우리 차별점의 데이터적 근거.

### DWG Handle 의 정의·안정성
> [[DWG]] entity handle(객체 고유 16진 ID)의 버전 간 안정성·재발급 리스크. 메모: 파싱 산출물에 handle 보존되는지 확인 TBD.

### 가교 매핑 테이블
> `{ sheet_id, dwg_handle, entity_id, tag, discipline, confidence }`. Phase 0 파싱 산출물(분야·도면번호·태그)을 입력으로 사용.

#### 좌표 → Handle 역참조(hit-test)
> 핀 좌표에서 가장 가까운 DWG 객체 handle 찾기(서버 측 공간 인덱스). 정밀 osnap 스냅은 범위 밖 — "근접 객체 추정" 수준.

#### Handle → 엔티티 해상(resolution)
> 1 handle 이 다수 엔티티에 매핑되는 모호성 처리, 미매핑 handle 폴백. 메모: 미매핑 시 anchor_type 자동 강등(`entity`→`pixel`).

### 엔티티 바인딩 활용(향후)
> 엔티티 단위 검색·영향도·AI채팅 바인딩 지점만 명시. 상세는 [[06_온톨로지연동-Phase2]]에 위임(Phase 2, [[TypeDB]]).

## 이벤트 페이로드 규격

> 채울 내용: 핀/마크업/이슈 생성·이동·수정·해소 시 실시간 전파되는 메시지 규격.

### 이벤트 종류(Event Catalog)
> `pin.created / pin.moved / pin.deleted`, `markup.created / markup.updated / markup.deleted`, `issue.status_changed`, `binding.updated`. 표로 정리.

### 공통 페이로드 봉투(Envelope)
> `{ event_id, type, sheet_id, project_id, actor, ts, payload, version }` 골격. CLAUDE.md 표준 response format 정합성 확인.

#### 좌표 변경 델타 vs 전체 스냅샷
> moved 이벤트에 델타만 보낼지 전체 좌표 보낼지. 메모: 충돌 해소·재생(replay) 고려.

### 전송 채널
> 실시간 협업 = [[WebSocket]] 푸시, 영속화 = REST 저장. 단방향/양방향, 재연결 시 동기화(catch-up) 정책 TBD.

#### 멱등성·순서 보장
> event_id 기반 중복 제거, 시퀀스 번호로 순서 보장. 동시 편집 충돌 정책(last-write-wins vs CRDT) TBD.

### 권한·감사(Audit) 연동
> actor·timestamp 기록으로 감사 로그 생성. 상세 권한 모델은 [[03-5_권한-감사설계]]에 위임.
