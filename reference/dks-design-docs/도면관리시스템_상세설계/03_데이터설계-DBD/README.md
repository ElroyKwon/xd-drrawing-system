# 데이터베이스 설계서 (DBD) 개요

## 1. 목적

- 청주 사업장 [[도면관리 시스템]]의 **데이터 저장 구조 전체를 조망하는 인덱스 문서**.
- ACC Build 동등 재현(시트·마크업·이슈·시트비교·파일·사진)에 필요한 데이터 모델 + DKS 차별점(엔티티 단위 바인딩) 저장 전략을 한곳에서 위임·연결.
- 이 문서는 **데이터 설계 인덱스만 소유** — 개별 스키마 DDL·ERD 상세·온톨로지 스키마는 하위 문서로 위임([[DRY]]).

## 2. 배경 / 전제

- **스코프 제약(상위 문서 전제 그대로)**:
  - [[DWG]] 원본은 편집하지 않음. 흐름 = DWG → 백엔드 파싱 → 기하 [[JSON]] → 웹 2D 뷰어 렌더 → 마크업/이슈/이벤트 **오버레이**.
  - 정밀 CAD 편집(좌표 편집·[[SHX]] 폰트 정밀 렌더)은 범위 밖 → DB에 CAD 편집 상태를 저장하지 않음.
  - 1차 범위 = **2D 시트 한정**. 3D BIM 데이터 모델 범위 밖.
- **Phase 0 산출물 존재**: 도면 파싱 결과(분야·도면번호·태그)가 이미 있음 → 시트 메타 테이블의 초기 적재 소스.
- **차별점**: 핀·마크업·이슈를 픽셀 좌표뿐 아니라 **설비 엔티티 ID**에도 묶음 → RDB 트랜잭션 데이터 ↔ 그래프 엔티티의 연결 키 설계가 핵심.
- **Phase 구분**: Phase 1 = [[RDB]] 중심(ACC 동등). Phase 2 = [[TypeDB]] 온톨로지 연동(영향도·AI 채팅).

## 3. 상세 목차

### 3.1 DBD 구성

> 이 절: DBD 문서군이 어떤 영역으로 나뉘는지, 각 영역 경계와 SoT를 명시.

#### 3.1.1 데이터 영역 분류
- 채울 내용: 5개 영역으로 분류 — ① 프로젝트/사용자/권한 ② 시트·도면 메타(파싱 산출물 적재) ③ 오버레이(마크업·이슈·핀·이벤트) ④ 파일/버전([[CDE]] 폴더 트리) ⑤ 엔티티 바인딩(그래프 연결 키).

#### 3.1.2 저장소 유형 매핑
- 채울 내용: 영역별 저장 기술 매핑 표 — 관계형 데이터(RDB), 기하 JSON(파일/Blob 스토리지 또는 JSONB), 그래프(TypeDB, Phase 2). 각 셀에 "왜 이 저장소인지" 1줄.

#### 3.1.3 명명·공통 규약
- 채울 내용: 테이블/컬럼 네이밍, PK 전략(UUID vs 시퀀스 ❓), 공통 감사 컬럼(created_at·updated_at·created_by), soft-delete 정책.

### 3.2 RDB vs 그래프 역할

> 이 절: 두 저장소의 책임 경계 — 무엇을 RDB가 쥐고 무엇을 그래프가 쥐는지, 연결 키 규약.

#### 3.2.1 책임 경계
- 채울 내용: RDB = 트랜잭션·이력·권한·오버레이 인스턴스(이슈/마크업 레코드). 그래프 = 설비 엔티티·관계·영향도·온톨로지 분류. 경계 원칙 = "사실(facts)은 RDB, 의미·관계는 그래프".

#### 3.2.2 연결 키 설계 (entity_id 바인딩)
- 채울 내용: 오버레이 레코드가 `entity_id`(그래프 IID/외부 키)를 nullable FK로 보유. 픽셀 좌표 바인딩과 엔티티 바인딩의 **공존** 모델. 동기화 방향(RDB→그래프 단방향? ❓).

#### 3.2.3 Phase별 적용 범위
- 채울 내용: Phase 1은 entity_id 컬럼만 예약(값 비어도 동작). Phase 2에서 TypeDB 채우고 영향도/AI 채팅 쿼리 활성화. 마이그레이션 무중단 전략.

#### 3.2.4 기하 JSON 저장 위치 결정
- 채울 내용: 파싱된 시트 기하 JSON을 RDB JSONB로 둘지, 오브젝트 스토리지 + 메타만 RDB로 둘지 비교(용량·뷰어 로딩 성능). 뷰어 렌더 경로([[PDF.js]]/커스텀 캔버스)와의 정합.

### 3.3 하위 문서 인덱스

> 이 절: DBD 하위 상세 문서로의 라우팅 — 각 문서가 무엇을 소유하는지 1줄 요약 + 링크.

#### 3.3.1 스키마 상세 문서
- 채울 내용: 영역별 DDL/ERD 문서 링크 표(문서명 · 소유 테이블군 · 상태). 예: `01_ERD.md`, `02_시트도면-스키마.md`, `03_오버레이-스키마.md`, `04_파일버전-CDE-스키마.md` (파일명 TBD).

#### 3.3.2 온톨로지/그래프 스키마 (Phase 2)
- 채울 내용: TypeDB 스키마 정의 문서 위임 링크 → [[온톨로지-서비스-기획]]. 엔티티 타입·관계·룰은 여기서 소유하지 않고 링크만.

#### 3.3.3 데이터 적재·마이그레이션
- 채울 내용: Phase 0 파싱 산출물 → 시트 메타 테이블 적재 절차 문서, 시드/마이그레이션 정책 문서 링크.

#### 3.3.4 인덱스·성능·정합성
- 채울 내용: 뷰어 로딩·이슈 목록 필터·시트비교 조회에 필요한 인덱스 전략, 대용량 기하 JSON 캐싱 문서 링크.
