# 비기능/품질 개요 (Non-Functional Requirements)

## ① 목적
- 청주 사업장 [[도면관리 시스템]]의 **품질 속성(Quality Attributes)** 을 한 곳에 모은 **상위 인덱스** 문서.
- 각 품질 속성의 **정의·목표치·측정 방법**을 정리하고, 상세 설계는 소유 문서로 위임(DRY).
- "[[ACC]] Build 동등 재현"이 기능 목표라면, 본 문서는 **"얼마나 잘"** 동작해야 하는지의 기준선.

## ② 배경/전제
- 핵심 스코프 제약: **[[DWG]] 원본 편집 없음**. 흐름 = DWG → 백엔드 파싱 → 기하 JSON → 웹 2D 뷰어 렌더 → 마크업/이슈/이벤트 **오버레이**.
  - → 품질 기준도 **"뷰어 + 주석 오버레이"** 관점으로 잡음. CAD 편집기 수준 정밀도(osnap·[[SHX]] 폰트 정밀 렌더)는 **품질 목표에서 제외**.
- 1차 범위 = **2D 시트 한정**. 3D/[[BIM]] 성능 기준은 범위 밖.
- Phase 2 차별점([[TypeDB]] 온톨로지 엔티티 단위 검색·영향도·AI 채팅)은 별도 품질 절에서 **향후 항목**으로만 표기.
- 측정 가능성 원칙: 모든 목표치는 **검증 가능한 수치 또는 TBD**. 추정·과장 금지.

## 품질 속성 목록

> 각 속성 = [정의 / 목표치(또는 TBD) / 측정 절 참조 / 소유 상세 문서] 4요소로 채운다.

### 성능 (Performance)
- 뷰어/오버레이 응답성 중심 핵심 속성. 상세 목표·튜닝은 [[07_성능-스케일]]로 위임.

#### 시트 로딩 시간
- DWG 파싱 산출 기하 JSON → 2D 뷰어 초기 렌더까지의 시간. 목표치 TBD(❓ 대표 도면 1장 기준 P95).

#### 뷰어 인터랙션 지연
- 팬/줌/마크업 드로잉 입력 → 화면 반영 지연(프레임율/입력 latency). 목표치 TBD.

#### 오버레이 렌더 부하
- 핀·마크업·이슈 다수 시 동시 렌더 한계. "도면 1장당 오브젝트 N개까지 60fps" 형태로 채울 것(N=TBD).

#### 검색/필터 응답
- 시트 목록·이슈 데이터테이블 정렬·필터·검색 응답. 목표치 TBD.

### 확장성 (Scalability)
- 데이터 규모 증가 대응. 상세 용량 산정은 [[07_성능-스케일]] 위임.

#### 동시 사용자
- 청주 사업장 예상 동시 접속 규모. ❓ 목표 동시 사용자 수(현장 규모 확인 필요).

#### 데이터 볼륨
- 프로젝트당 시트 수·버전 수·이슈/마크업 누적 한계. 목표치 TBD.

### 가용성/신뢰성 (Availability / Reliability)
- 단일 사업장 납품 기준 운영 가용성.

#### 가동률 목표
- 업무시간 기준 가용성. ❓ SLA 수준(현장 운영 정책 미정).

#### 데이터 무결성
- 마크업/이슈/버전 오버레이 데이터 유실 방지. DWG 원본 불변(read-only)이 무결성 1차 방어선.

#### 백업/복구
- 오버레이 데이터·메타 백업 주기·복구 목표(RPO/RTO). 목표치 TBD.

### 보안 (Security)
- 도면=민감 자산. 상세 정책은 소유 문서(❓ `10_보안-권한` 신설 여부 TBD)로 위임 검토.

#### 인증/권한
- 프로젝트·모듈 단위 접근 제어. [[CDE]] 폴더 권한 모델과 연계.

#### 도면 자산 보호
- 원본 다운로드·반출 통제, 워터마크 여부. ❓ 청주 사업장 보안 요건 확인 필요.

#### 감사 로그
- 이슈/마크업 변경 이력, 이벤트 추적성.

### 사용성/접근성 (Usability / Accessibility)
- ACC Build 공통 UI 패턴(앱셸·데이터테이블·인스펙터·토스트) 재현 품질.

#### 일관성
- 공통 UI 컴포넌트 동작 일관성. 상세는 [[_ACC-Build-화면분석-재현설계]] 공통 UI 절 위임.

#### 학습 용이성
- ACC 사용 경험자가 추가 학습 없이 사용 가능한 수준 목표.

#### 접근성
- 키보드 내비/대비/포커스 수준. ❓ WCAG 적용 등급 TBD.

### 호환성 (Compatibility)
- 납품 환경 제약.

#### 브라우저 지원
- 지원 브라우저·최소 버전. ❓ 현장 표준 브라우저 확인 필요.

#### DWG 버전/포맷
- 파싱 지원 DWG 버전 범위. 미지원 포맷 처리 정책 TBD.

### 유지보수성 (Maintainability)
- 향후 Phase 2 온톨로지 연동 확장 대비.

#### 모듈성
- 뷰어/오버레이/데이터 계층 분리. 상세 구조는 [[02_아키텍처]] 위임.

#### 확장 포인트
- 핀/마크업/이슈를 픽셀 좌표뿐 아니라 **설비 엔티티 ID**에도 바인딩하는 확장 슬롯 — Phase 2 TypeDB 연동의 품질 전제(데이터 모델 침습 최소화 목표).

## 측정 방법

> 위 각 속성을 "어떻게 검증하는가"를 채운다. 합격/불합격 판정 가능한 형태 지향.

### 측정 환경 기준
- 기준 하드웨어·네트워크·대표 도면 셋 정의. ❓ 청주 현장 대표 도면 샘플 확보 필요.

### 성능 측정
#### 측정 지표
- 응답시간 P50/P95, 프레임율(fps), 입력 latency. [[WebSocket]] 이벤트 왕복 지연 포함 여부 TBD.

#### 측정 도구/방법
- 브라우저 성능 프로파일링([[PDF.js]]/Canvas 렌더 기준), 합성 부하 시나리오. 구체 도구 TBD.

#### 부하 시나리오
- 시트 N장·오버레이 N개·동시 사용자 N명 조합 시나리오 표(값 TBD).

### 신뢰성/가용성 측정
- 장애 주입·복구 시간 측정 절차. RPO/RTO 실측 방법 TBD.

### 사용성 측정
- ACC 경험자 대상 과업 완료율·소요시간 측정(소규모 정성 평가). 방법 TBD.

### 합격 기준 게이트
- 납품 전 NFR 합격선 체크리스트. 각 속성 목표치 확정 후 채움.
