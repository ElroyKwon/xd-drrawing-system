# 로깅 & 감사

## ① 목적
- 도면관리 시스템의 모든 활동/감사 이벤트를 **표준 포맷**으로 기록·보존하는 단일 기준 정의.
- "누가·언제·무엇을·어디에" 했는지 추적 가능하게 만들어 [[ACC]] Build 수준의 활동 이력(Activity Log) 동등 재현.
- 소유 범위 = **활동/감사 로그 표준만**. 권한 정책 자체는 [[09-1_보안-권한]]에 위임, 성능/저장 용량은 [[09-2_성능-확장성]]에 위임(DRY).

## ② 배경/전제
- 핵심 스코프: [[DWG]] 원본 미편집. 흐름 = DWG → 백엔드 파싱 → 기하 JSON → 웹 2D 뷰어 → 마크업/이슈/이벤트 오버레이.
  - ⇒ 감사 대상은 "원본 CAD 편집"이 아니라 **오버레이 객체(마크업·이슈·핀)와 파일/버전 행위**가 중심.
- DKS 차별점: 핀/마크업/이슈가 픽셀 좌표뿐 아니라 **설비 엔티티 ID**에도 바인딩 → 감사 로그도 엔티티 ID를 컨텍스트로 함께 기록(향후 [[TypeDB]] 영향도 추적 연계).
- Phase 1 = 2D 시트 한정. 3D/Bridge 로그 대상 없음.

## ③ 상세 목차 (스캐폴드)

## 로그 레벨/포맷
> 시스템 로그(운영자용)와 감사 로그(규정·추적용)를 **분리**한다는 원칙을 명확히 할 절.

### 로그 분류 체계
> 3계층 구분 의도 메모 — 채울 내용:
> - **시스템 로그**: 앱/서버 동작(에러·경고·디버그). 운영자·개발자 대상.
> - **활동 로그(Activity)**: 사용자 행위 피드(ACC Build 활동 탭 동등). UI 노출 대상.
> - **감사 로그(Audit)**: 변조 방지·법적 추적용. 영구/장기 보존, 별도 저장소.

#### 로그 레벨 표준
> TRACE/DEBUG/INFO/WARN/ERROR/FATAL 매핑. 각 레벨이 어느 분류로 가는지 표로 정의.
> - 환경별 기본 레벨 — prod=INFO / dev=DEBUG (TBD: 운영 합의)

#### 감사 로그 심각도/카테고리
> 감사 이벤트는 레벨이 아닌 **카테고리**(인증/권한변경/데이터변경/내보내기 등)로 분류한다는 메모.

### 표준 로그 포맷
> 구조화 로그([[JSON]] 한 줄/이벤트) 채택. 공통 필드 스키마를 여기서 확정.

#### 공통 필드 스키마
> 채울 필드 목록(초안):
> - `timestamp`(ISO8601·UTC), `level`, `category`, `actor_id`, `actor_role`
> - `action`(예: issue.create, markup.update, sheet.export, file.upload)
> - `target_type` / `target_id`(이슈·마크업·시트·파일·버전)
> - `project_id`, `entity_ids[]`(설비 엔티티 바인딩 — DKS 차별 필드)
> - `result`(success/fail), `ip`, `user_agent`, `trace_id`(요청 상관관계)
> - `before` / `after`(데이터 변경 시 diff 페이로드 — 마스킹 규칙 적용)

#### action 네임스페이스 규약
> `<도메인>.<동작>` 동사형. 도메인 = sheet / markup / issue / file / version / photo / auth / admin.
> 전체 action 사전(enum)은 [[04_데이터-모델]]과 정합 — TBD: 최종 enum 확정.

#### 상관관계 추적(trace_id)
> 동일 요청/[[WebSocket]] 세션 흐름을 묶는 trace_id·session_id 전파 규칙. 분산 추적 도입 여부 ❓.

## 감사 대상 이벤트
> "무엇을 반드시 남기는가"의 완전 목록. 누락=감사 구멍이므로 도메인별 매트릭스로 작성.

### 인증/세션 이벤트
> 로그인 성공/실패, 로그아웃, 세션 만료, 권한 거부(403). 권한 모델은 [[09-1_보안-권한]] 참조.

### 도면/시트 이벤트
> 시트 열람(조회 로깅 범위 ❓ — 전수 vs 샘플), 버전 업로드, 시트 비교 실행, 분야/도면번호 메타 변경.

### 오버레이(마크업·이슈·핀) 이벤트
> 핵심 절. 채울 내용:
> - 마크업 create/update/delete, 이슈 create/상태변경/할당/코멘트
> - 핀 위치 변경, **엔티티 ID 재바인딩**(DKS 고유 — 누가 어떤 설비로 연결을 바꿨나)
> - before/after diff 보존 범위 정의.

### 파일/[[CDE]] 이벤트
> 폴더형 CDE에서 업로드·이동·삭제·복원·다운로드. 권한 변경은 admin 이벤트로 분리.

### 내보내기/공유 이벤트
> [[PDF.js]] 뷰 내보내기, 도면/이슈 목록 export, 외부 공유 링크 발급·접근. 데이터 유출 추적 핵심.

### 관리/설정 이벤트
> 사용자 초대·역할 변경, 프로젝트 생성/보관, 시스템 설정 변경. 권한 상승 행위 필수 기록.

#### 감사 매트릭스 (이벤트 × 보존등급)
> 위 이벤트별로 보존등급(단기/장기/영구)·UI 노출 여부·민감도(개인정보 포함)를 매핑하는 표. TBD.

#### 민감정보 마스킹 규칙
> before/after 페이로드 내 개인정보·좌표 등 마스킹/해시 처리 기준. [[09-1_보안-권한]] 정책과 정합.
