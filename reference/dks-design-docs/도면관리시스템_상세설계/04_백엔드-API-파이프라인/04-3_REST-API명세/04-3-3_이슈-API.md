# 이슈 API

> 소유 범위: **이슈(Issue) 및 부속 엔드포인트(댓글·첨부·참조·필터/내보내기)** 만.
> 마크업 도형 자체는 [[04-3-2_마크업-API]] 소유, 시트/뷰어 좌표계는 [[04-3-4_시트-API]] 소유 → 본 문서는 링크로 위임.

## 1. 목적
- [[ACC]] Build "이슈" 모듈(핀 + 상세폼 + 목록 + 필터)을 동등 재현하는 REST 엔드포인트 계약 정의
- 이슈를 2D 시트 위 **오버레이 핀**으로 표현 — DWG 원본 미편집, [[기하JSON]] 뷰어 좌표 + 향후 설비 엔티티 ID에 바인딩
- (의도 메모) 프론트 이슈 패널·핀·목록테이블이 호출하는 모든 경로의 단일 진실 계약

## 2. 배경/전제
- 흐름: [[DWG]] → 백엔드 파싱 → 기하 JSON → 웹 2D 뷰어 렌더 → 이슈 핀 오버레이. 이슈는 도면 위 "주석/이벤트" 레이어.
- 범위: 2D 시트 한정. 3D/BIM 범위 밖.
- **차별점(DKS)**: ACC 핀은 픽셀 좌표만 → 우리는 `entity_id`(설비 엔티티)에도 결속 가능 → 영향도·온톨로지 질의 연동(Phase 2, [[TypeDB]]). 본 문서는 필드 예약까지만.
- 인증/공통 응답 포맷(`success/data/error/meta`)·페이지네이션 규약은 [[04-3-1_API공통규약]] 위임. ❓ 해당 문서 경로 확정 필요(TBD).

## 3. 상세 목차

### 3.1 이슈 CRUD
- (의도 메모) 핵심 리소스 `Issue`의 생성·조회·수정·삭제 6개 엔드포인트 + 요청/응답 스키마.

#### 3.1.1 리소스 모델 (Issue 엔티티)
- (채울 것) 필드 표: `id, project_id, sheet_id, title, description, status, priority, type, assignee_id, due_date, created_by, created_at, updated_at`.
- 위치 필드: `pin{ x, y, page }`(뷰어 좌표) + `entity_id`(설비 엔티티, nullable — DKS 예약).
- 스키마 본체(타입·필수·길이)는 [[04-2_데이터모델]] 참조, 본 절은 API 직렬화 형태만.
- ❓ `issue_number`(사람이 읽는 일련번호) 자동발번 규칙 TBD.

#### 3.1.2 POST `/api/issues` — 생성
- (채울 것) 요청 바디 예시, 필수 필드, 핀 좌표/시트 결속, 생성 시 기본 status.
- ❓ 핀 없이(목록 전용) 이슈 생성 허용 여부 TBD.

#### 3.1.3 GET `/api/issues/{id}` — 단건 조회
- (채울 것) 단건 + 인스펙터 패널용 확장(`?expand=comments,attachments,refs`) 응답.

#### 3.1.4 GET `/api/issues` — 목록 조회
- (채울 것) 페이지네이션·정렬 기본값. 필터 쿼리는 §3.4 위임.
- 뷰어 핀 렌더용 경량 모드(`?fields=id,pin,status`) — N+1 회피.

#### 3.1.5 PATCH `/api/issues/{id}` — 부분 수정
- (채울 것) 부분 업데이트 허용 필드. 상태 변경은 §3.2 전이 규칙 경유.

#### 3.1.6 DELETE `/api/issues/{id}` — 삭제
- (채울 것) soft delete vs hard delete. ❓ 권한(작성자/관리자) TBD.

### 3.2 상태전이
- (의도 메모) 이슈 라이프사이클 상태기계 + 전이 엔드포인트. ACC: Open → Answered → Closed (+ Void).

#### 3.2.1 상태 정의
- (채울 것) 상태 enum 표 + 의미. ❓ ACC 정확 매핑 vs 자사 단순화(Open/In-Progress/Resolved/Closed) 결정 TBD.

#### 3.2.2 전이 규칙 (상태기계)
- (채울 것) 허용 전이 표(from→to)·금지 전이. 다이어그램 placeholder.

#### 3.2.3 POST `/api/issues/{id}/transition` — 상태 변경
- (채울 것) `{ to_status, comment? }`. 전이 시 자동 댓글/이벤트 로그 기록.
- ❓ 전이 전용 엔드포인트 vs PATCH status 통합 — 결정 대기.

#### 3.2.4 전이 권한/검증
- (채울 것) 역할별 허용 전이(담당자만 Resolve 등). 검증 실패 에러코드.

### 3.3 댓글/첨부/참조
- (의도 메모) 이슈에 종속된 하위 컬렉션 3종. 모두 `/api/issues/{id}/...` 하위 경로.

#### 3.3.1 댓글 (Comments)
- (채울 것) `GET/POST /api/issues/{id}/comments`, `DELETE .../comments/{cid}`.
- 필드: `body, author_id, created_at`. ❓ 멘션/스레드 답글 지원 여부 TBD.

#### 3.3.2 첨부 (Attachments)
- (채울 것) `POST /api/issues/{id}/attachments`(멀티파트 업로드), 목록·삭제.
- 파일 실저장·CDE 연계는 [[04-3-5_파일-API]] 위임 — 본 절은 이슈↔파일 연결 메타만.
- ❓ 사진 모듈([[ACC]] 사진) 연동 방식 TBD.

#### 3.3.3 참조 (References)
- (채울 것) 이슈를 다른 리소스에 결속: `sheet_ref`(시트/페이지), `markup_ref`(마크업 ID), `entity_ref`(설비 엔티티 ID — DKS 무기).
- (의도) entity_ref가 향후 영향도·AI채팅 바인딩 진입점. Phase 2 [[TypeDB]] 질의 연동은 [[06_온톨로지-연동]] 위임.

### 3.4 필터/내보내기
- (의도 메모) 목록 데이터테이블의 필터·정렬·컬럼·내보내기. ACC 공통 데이터테이블 패턴 재현.

#### 3.4.1 필터 쿼리 파라미터
- (채울 것) `status, priority, assignee_id, type, sheet_id, due_before/after, created_by, q(전문검색)`.
- 다중값(`status=open,answered`)·조합(AND) 규약.
- ❓ `entity_id` 필터(특정 설비 관련 이슈) — DKS 차별 항목, 우선순위 TBD.

#### 3.4.2 정렬/페이지네이션
- (채울 것) `sort, order, page, page_size` 기본값. 공통 규약 [[04-3-1_API공통규약]] 참조.

#### 3.4.3 내보내기
- (채울 것) `GET /api/issues/export?format=csv|xlsx`. 현재 필터 상태 반영.
- ❓ PDF 리포트(이슈+핀 스냅샷) 범위 포함 여부 TBD.

#### 3.4.4 집계/위젯용 카운트
- (채울 것) `GET /api/issues/stats` — 상태별/담당자별 카운트(홈 위젯 대시보드용).
