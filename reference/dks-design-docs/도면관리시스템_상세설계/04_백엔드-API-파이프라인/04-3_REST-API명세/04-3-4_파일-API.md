# 파일 API

## ① 목적
- 폴더형 [[CDE]](Common Data Environment) 파일 저장소의 REST 엔드포인트 명세 소유 문서
- ACC Build "파일" 모듈(폴더 트리 + 파일 목록 + 버전 히스토리 + 공유) 동등 재현
- 소유 범위 한정: **폴더/파일/버전 엔드포인트**. 뷰어 렌더·시트 메타·이슈/마크업은 위임([[04-3-1_시트-API]], [[04-3-3_이슈-API]])

## ② 배경/전제
- 흐름 전제: [[DWG]] 원본은 "편집하지 않음". 파일 API는 **원본 바이너리 저장·버전·공유**만 담당, 파싱/기하 JSON 생성은 별도 파이프라인이 트리거 (→ [[04-1_파싱-파이프라인]])
- 파일 저장소 = 진실원천(SoT) 원본 보관소. 업로드된 [[DWG]]/[[PDF]]가 시트로 승격되는 시작점
- 1차 범위 = 2D 시트 원본 한정. 3D/[[BIM]] 파일은 범위 밖
- 응답 구조는 표준 포맷 준수 (`success`/`data`/`error`/`meta`) → 공통 규약 [[04-3-0_API-공통규약]]에 위임
- DKS 차별점: 파일 메타에 Phase 0 파싱 산출물(분야·도면번호·태그) 및 향후 설비 엔티티 ID 바인딩 가능 → 상세는 [[04-2_온톨로지-바인딩]]에 위임

## 폴더 트리
> 폴더 계층 CRUD 및 트리 조회. ACC "파일" 좌측 폴더 사이드바 재현.

### GET /api/projects/{projectId}/folders
- 의도 메모: 프로젝트 루트 기준 폴더 트리 반환. 평면 리스트 + parentId 방식 vs 중첩 트리 방식 결정 필요(❓)

#### 요청 파라미터
- 의도 메모: `depth`(트리 깊이 제한), `includeFileCount`(폴더별 파일 수 포함 여부) 채울 것

#### 응답 스키마
- 의도 메모: `folderId`, `name`, `parentId`, `path`, `childCount`, `fileCount`, `createdAt`, `createdBy` 필드 정의

### POST /api/projects/{projectId}/folders
- 의도 메모: 폴더 생성. body = `{ name, parentId }`. 동일 부모 내 이름 중복 정책 채울 것

### PATCH /api/folders/{folderId}
- 의도 메모: 폴더 이름변경/이동(parentId 변경). 순환 참조(자기 하위로 이동) 차단 검증 명세

### DELETE /api/folders/{folderId}
- 의도 메모: 폴더 삭제. 하위 파일 존재 시 거부 vs 캐스케이드 vs 휴지통 이동 — 정책 결정(❓)

## 파일 업로드/다운로드
> 원본 바이너리 입출력. ACC 데이터테이블(파일 목록) 재현.

### GET /api/folders/{folderId}/files
- 의도 메모: 폴더 내 파일 목록. 정렬·필터·페이지네이션은 공통 데이터테이블 규약([[04-3-0_API-공통규약]]) 따름

#### 응답 스키마
- 의도 메모: `fileId`, `name`, `ext`, `sizeBytes`, `currentVersion`, `mimeType`, `thumbnailUrl`, `sheetId`(승격된 경우), `updatedAt`, `updatedBy`

### POST /api/folders/{folderId}/files
- 의도 메모: 신규 파일 업로드(v1 생성). 멀티파트 vs presigned-URL 직접 업로드 방식 결정(❓)

#### 업로드 방식
- 의도 메모: 대용량 [[DWG]] 고려. 청크/멀티파트 업로드, 재개 가능 여부 채울 것

#### 업로드 후 파싱 트리거
- 의도 메모: 업로드 완료 이벤트 → 파싱 파이프라인 비동기 호출. 진행 상태 알림은 [[WebSocket]] 사용(상세는 [[04-4_실시간-이벤트]]에 위임)

### GET /api/files/{fileId}/download
- 의도 메모: 원본 바이너리 다운로드. 최신 버전 기본, `?version=N`으로 특정 버전. presigned-URL 리다이렉트 vs 스트리밍 결정

### GET /api/files/{fileId}/thumbnail
- 의도 메모: 목록/카드용 썸네일. 생성 시점(업로드 직후 vs 온디맨드) 채울 것

### DELETE /api/files/{fileId}
- 의도 메모: 파일 삭제. 휴지통(soft delete) 채택 여부 — ACC는 휴지통 보유(❓)

## 버전관리
> 동일 파일명 재업로드 시 버전 누적. ACC 우측 인스펙터 "버전 히스토리" 재현.

### POST /api/files/{fileId}/versions
- 의도 메모: 기존 파일에 새 버전 추가(v2, v3…). 동일 폴더 동일명 재업로드를 신규파일이 아닌 버전으로 인식하는 규칙 명세

### GET /api/files/{fileId}/versions
- 의도 메모: 버전 목록(번호·업로더·일시·크기·코멘트). 시트비교(버전 오버레이 diff) 입력원 → diff 자체는 [[04-3-1_시트-API]]에 위임

#### 버전 메타 스키마
- 의도 메모: `versionNumber`, `uploadedBy`, `uploadedAt`, `sizeBytes`, `comment`, `parseStatus`(pending/done/failed)

### GET /api/files/{fileId}/versions/{n}
- 의도 메모: 특정 버전 상세. 해당 버전의 파싱 산출물(기하 JSON) 참조 링크 포함

### POST /api/files/{fileId}/versions/{n}/restore
- 의도 메모: 과거 버전을 최신으로 복원(새 버전으로 승격 vs 포인터 변경) — 방식 결정(❓)

#### 버전 보존 정책
- 의도 메모: 무제한 보존 vs N개 제한. 원본은 SoT이므로 보존 우선 — TBD

## 공유
> 외부/내부 공유 링크 및 권한. ACC "공유" 모달 재현.

### POST /api/files/{fileId}/share
- 의도 메모: 공유 링크 생성. 내부 멤버 지정 vs 토큰 기반 외부 링크 구분

#### 권한 레벨
- 의도 메모: 뷰어/다운로드 가능/만료일 등. 프로젝트 멤버 권한 모델과의 관계 → [[04-3-5_권한-API]]에 위임

### GET /api/files/{fileId}/shares
- 의도 메모: 활성 공유 링크 목록 조회

### DELETE /api/shares/{shareId}
- 의도 메모: 공유 해제(링크 무효화)

#### 외부 공유 정책
- 의도 메모: 청주 사업장 보안 정책상 외부 토큰 링크 허용 여부 — TBD(❓ 고객 확인 필요)
