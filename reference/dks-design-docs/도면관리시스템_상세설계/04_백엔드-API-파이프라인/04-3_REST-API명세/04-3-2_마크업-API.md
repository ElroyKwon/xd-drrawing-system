# 마크업 API

## ① 목적

- [[ACC]] Build 마크업 모듈(펜·도형·화살표·텍스트)을 동등 재현하는 백엔드 REST 엔드포인트 명세
- 2D 시트 뷰어 위에 그려지는 주석(마크업)의 생성·수정·삭제·조회·검색 계약 정의
- 이 문서가 소유하는 것 = **마크업 CRUD 엔드포인트만**. 렌더링·뷰어 좌표계는 [[04-3-1_시트-API]] 및 프론트 뷰어 문서로 위임

## ② 배경 / 전제

- ★핵심 스코프: [[DWG]] 원본은 편집하지 않음. 마크업은 파싱된 기하 JSON 렌더 위에 얹는 **오버레이 레이어**
- 마크업 = 픽셀/시트좌표 기반 벡터 주석. CAD 엔티티가 아니므로 osnap·정밀좌표 편집 범위 밖
- 좌표계: 시트 정규화 좌표(0~1) 기준 저장 → 줌/팬 무관 재현 (구체 변환 규칙은 TBD)
- 차별점: 마크업을 픽셀뿐 아니라 **설비 엔티티 ID**에도 바인딩 가능(Phase 2 [[TypeDB]] 온톨로지 연동). 본 문서는 필드 자리만 예약, 로직은 위임
- 인증/권한/공통 응답 포맷은 [[04-3-0_API-공통규약]]에 정의 → 여기서는 재참조만

## ③ 상세 목차

## 마크업 생성/수정/삭제

> 단일 마크업 리소스의 쓰기 계약. ACC 펜·도형·화살표·텍스트·구름형(revision cloud) 타입 커버.

### 데이터 모델 (Markup 리소스)

> 저장 스키마 필드 정의. 채울 항목 아래.

#### 공통 필드

- `id`, `sheet_id`, `sheet_version`, `author_id`, `created_at`, `updated_at`
- `type` enum: `pen` | `line` | `arrow` | `rect` | `ellipse` | `cloud` | `text` (확정 TBD❓)
- `style`: 색상·선두께·투명도·채움 (구조 TBD)
- `geometry`: 정규화 좌표 배열 (타입별 스키마 분기 — 표로 채울 것)

#### 엔티티 바인딩 필드 (Phase 2 예약)

- `entity_refs`: 마크업이 가리키는 설비 엔티티 ID 목록 — 자리만 예약, 해석 로직은 [[04-4_온톨로지-바인딩]]로 위임
- Phase 1에서는 null 허용/무시

### POST /api/v1/sheets/{sheetId}/markups — 생성

> 신규 마크업 1건 생성. 요청/응답 바디 예시, 필수 필드, 검증 규칙 채울 것.

#### 요청 바디

> type·geometry·style 최소 필수셋. 좌표 범위(0~1) 검증.

#### 응답 (201)

> 생성된 리소스 전체 반환. 표준 응답 래퍼(success/data/error/meta).

### PATCH /api/v1/markups/{markupId} — 수정

> 부분 수정(스타일·좌표·텍스트). 동시편집 충돌 처리 정책 TBD❓(낙관적 락 vs last-write-wins).

### DELETE /api/v1/markups/{markupId} — 삭제

> 소프트 삭제 vs 하드 삭제 결정 TBD❓. 권한(작성자 본인/관리자) 규칙 채울 것.

### 실시간 동기화 훅

> 마크업 변경 시 [[WebSocket]] 브로드캐스트로 동접 뷰어 갱신. 이벤트 페이로드 스펙은 [[04-5_실시간-동기화]]로 위임, 여기서는 트리거 지점만 명시.

## 시트별 조회

> 특정 시트(+버전)에 속한 마크업 목록 읽기. 뷰어 진입 시 1회 로드 + 증분 갱신.

### GET /api/v1/sheets/{sheetId}/markups — 목록

> 해당 시트의 마크업 전체/페이지 반환. 쿼리 파라미터 표로 채울 것.

#### 쿼리 파라미터

- `version`: 시트 버전 지정 (미지정 시 최신)
- `type`: 타입 필터
- `author_id`: 작성자 필터
- `since`: 증분 동기화용 변경시각 기준 (구현 여부 TBD❓)

#### 응답 구조

> 좌표는 정규화값 그대로 반환. 프론트에서 뷰포트 변환. 페이지네이션 정책은 [[04-3-0_API-공통규약]] 따름.

### GET /api/v1/markups/{markupId} — 단건 조회

> 딥링크/이슈 연동 시 단일 마크업 상세. 필요성 검토 후 채울 것.

## 검색

> 시트 경계를 넘는 마크업 횡단 검색. ACC엔 약한 영역 → 우리 차별 포인트 연결점.

### GET /api/v1/projects/{projectId}/markups/search — 횡단 검색

> 프로젝트 전체에서 마크업 검색. 필터 조합 채울 것.

#### 검색 필터

- 텍스트 마크업 본문 키워드
- 작성자·기간·타입·시트(분야/도면번호)
- 도면 메타(분야·도면번호·태그) 연계 — Phase 0 파싱 산출물 활용

#### 엔티티 기반 검색 (Phase 2)

> "이 설비 엔티티에 달린 마크업 전부" 질의 — `entity_refs` 인덱스 기반. ACC 미지원 차별 기능. 상세 로직은 [[04-4_온톨로지-바인딩]]로 위임, 본 문서는 API 진입점만 예약.
