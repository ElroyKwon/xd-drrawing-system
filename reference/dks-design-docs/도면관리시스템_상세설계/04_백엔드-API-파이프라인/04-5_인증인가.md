# 인증/인가

## ① 목적
- 청주 사업장 [[도면관리시스템]] 백엔드의 **로그인·토큰·역할기반 접근제어([[RBAC]])** 단일 소유 문서.
- [[ACC Build]] 동등 재현 1차 범위 내에서 "누가 어떤 프로젝트의 어떤 시트/이슈/마크업에 접근·편집 가능한가"를 결정하는 보안 계층 정의.

## ② 배경/전제
- 흐름 전제: [[DWG]] 원본은 편집하지 않음 → 뷰어 + 오버레이 시스템. 따라서 보호 대상은 **원본 도면 파일 다운로드**, **마크업/이슈/이벤트 쓰기 권한**, **프로젝트 멤버십**.
- 1차 범위 = 2D 시트 한정. 3D/Bridge 권한 모델은 범위 밖.
- 멀티테넌시 여부 TBD — 우선 단일 사업장(청주) 단일 조직 가정. 프로젝트 단위 격리는 1차부터 필수.
- Phase 2: [[TypeDB]] 온톨로지 연동 시 "설비 엔티티 단위" 접근제어 확장 여지 → 본 문서 RBAC 모델이 엔티티 권한의 토대(상세는 [[06_온톨로지-연동]]에 위임).

## 인증 방식
> 채울 내용: 로그인 진입점과 자격증명 검증 방식 확정.

### 로그인 플로우
- ID/PW 기본 로그인 폼 → 백엔드 검증 → 토큰 발급 시퀀스 다이어그램 채우기.
- 비밀번호 해시 알고리즘(예: bcrypt/argon2) 선택 기재. ❓ 알고리즘 결정 TBD.

### 외부 IdP 연동 (선택)
- 사내 SSO/LDAP/AD 연동 필요 여부. ❓ 청주 사업장 계정 체계 확인 필요 → TBD.
- 연동 시 OIDC/SAML 중 무엇인지 위임 결정.

### 계정 프로비저닝
- 초기 관리자 계정 생성·사용자 초대 방식. ACC의 "프로젝트 멤버 초대" 대응 모델 메모.

## 토큰/세션
> 채울 내용: 발급 토큰의 형식·수명·갱신·저장 위치.

### 토큰 형식
- [[JWT]] access token 클레임 설계(sub, role, projectIds, exp). 페이로드 필드 표 채우기.
- 서명 방식(HS256 대칭 vs RS256 비대칭) 선택. ❓ TBD.

### 토큰 수명 / 갱신
- access / refresh token 수명 정책. refresh 회전(rotation) 여부.
- [[WebSocket]] 실시간 이벤트 채널 인증 시 토큰 전달·만료 처리 방식 메모(연결 시 핸드셰이크 인증).

### 저장 위치
- 클라이언트 저장 전략(httpOnly Cookie vs 메모리/스토리지) 트레이드오프 표 채우기. ❓ CSRF 대응과 연동 결정 TBD.

### 로그아웃 / 무효화
- 토큰 블랙리스트·refresh 폐기 처리. 강제 로그아웃 시나리오 정의.

## RBAC
> 채울 내용: 역할 정의 + 리소스별 권한 매트릭스. 본 문서의 핵심.

### 역할 정의
- ACC Build 참조 역할 매핑: 프로젝트 관리자 / 멤버(편집) / 뷰어(읽기전용) 3단 기본안. ❓ 청주 운영 역할명 확정 TBD.
- 시스템 관리자(전체 프로젝트) 별도 분리.

### 권한 매트릭스
- 리소스 × 동작 표 채우기. 행 = {시트, 마크업, 이슈, 파일/[[CDE]] 폴더, 시트비교, 사진, 멤버관리}, 열 = {조회, 생성, 수정, 삭제, 다운로드}.
- 마크업/이슈 "본인 작성분만 수정" vs "전체 수정" 구분 규칙 명시.

### 프로젝트 스코프 격리
- 모든 권한은 **프로젝트 멤버십 전제**. 비멤버는 프로젝트 리소스 일체 불가.
- 토큰 클레임의 projectIds와 요청 리소스의 projectId 대조 로직 위치 메모.

#### 엔티티 단위 권한 (Phase 2 예고)
- 픽셀/시트 단위를 넘어 "설비 엔티티 ID" 단위 접근제어 확장 가능성 — 우리 차별점. 상세 설계는 [[06_온톨로지-연동]]에 위임, 본 절은 훅 지점만 표시.

## API 보호
> 채울 내용: 실제 엔드포인트에 인증/인가를 거는 메커니즘.

### 인증 미들웨어
- 토큰 검증 미들웨어 위치·순서(라우팅 전 전역 적용). 공개 엔드포인트(로그인/헬스체크) 예외 목록.

### 인가 가드 / 데코레이터
- 라우트별 역할·권한 요구 선언 방식(데코레이터/가드). 권한 부족 시 403 vs 404(존재 은닉) 정책. ❓ TBD.

### 리소스 소유권 검사
- 이슈/마크업 수정·삭제 시 작성자 일치 검사 위치. 프로젝트 격리 검사와 결합 순서.

### 파일 다운로드 보호
- 원본 [[DWG]]·기하 JSON·첨부 다운로드 시 서명 URL(presigned) 또는 토큰 게이트 방식. 저장소 연동은 [[04-4_파일-저장소]]에 위임, 본 절은 권한 판정만.
