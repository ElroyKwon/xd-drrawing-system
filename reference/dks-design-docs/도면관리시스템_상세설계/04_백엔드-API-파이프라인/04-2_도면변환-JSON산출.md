# 도면 변환 JSON 산출

## ① 목적
- [[DWG]] 원본을 편집하지 않고, 파싱 결과를 웹 [[2D 뷰어]]가 렌더 가능한 **기하 JSON**으로 변환하는 파이프라인 정의.
- 산출물 = 뷰어 렌더 입력 + 마크업/이슈/이벤트 오버레이의 좌표 기준계.
- 소유 범위: **파싱 → 기하 JSON 변환만**. (DWG 디코딩 자체는 [[04-1_DWG파싱-기하추출]]에 위임.)

## ② 배경 / 전제
- 흐름: DWG → 파싱(위임) → **기하 JSON 변환(이 문서)** → 뷰어 렌더 → 오버레이.
- 비-편집 전제: 좌표/엔티티를 재계산해 표시용 JSON으로 굳힌다. 원본 역기록 없음.
- 1차 범위 = **2D 시트 한정**. 3D/BIM 변환 범위 밖.
- 정밀 CAD 렌더(osnap, [[SHX]] 폰트 정밀 등) 범위 밖 → 표시용 근사 허용.
- [[DKS]] 차별점: 파싱 산출물(분야·도면번호·태그)에 더해, 기하 엔티티에 **설비 엔티티 ID**를 부착할 수 있는 자리를 JSON 스키마에 예약(향후 [[TypeDB]] 온톨로지 바인딩). → 상세는 [[06_온톨로지-바인딩]]에 위임.

## 변환 단계
> 파싱 산출(원시 엔티티 목록)을 받아 표시용 기하 JSON으로 굳히는 단계별 처리 정의.

### 입력 계약
> 상류(04-1)가 넘기는 원시 엔티티 형식·필수 필드(엔티티 타입, 좌표, 레이어, 핸들 등) 명세. 실제 필드 목록은 04-1 확정 후 채움(TBD).

### 단계 파이프라인
> 단계 순서를 한 줄씩 의도와 함께 나열.

#### 1단계 — 엔티티 필터/분류
> 렌더 대상 엔티티(LINE/ARC/CIRCLE/POLYLINE/TEXT/INSERT 등) 선별, 비표시·헬퍼 엔티티 제외 규칙.

#### 2단계 — 레이어/스타일 매핑
> CAD 레이어 → 뷰어 스타일(색·선두께·가시성)로 매핑. 레이어 토글 메타 보존.

#### 3단계 — 블록(INSERT) 처리
> 블록 참조 전개 여부 결정. 전개 vs 참조 유지 트레이드오프. ❓전개 시 JSON 크기 폭증 우려 → 정책 TBD.

#### 4단계 — 텍스트/치수 처리
> 텍스트는 위치·내용·정렬만 보존(폰트 정밀 렌더 비범위). 치수는 표시 문자열로 평탄화.

#### 5단계 — 기하 JSON 직렬화
> 최종 출력 스키마로 직렬화. 좌표는 RTC 정규화 후 값 사용(아래 절).

### 출력 스키마 (기하 JSON)
> 뷰어가 소비할 JSON 루트 구조 초안. 시트 메타 + 엔티티 배열 + 바운딩박스 + 좌표계 정보.

#### 엔티티 표현
> 엔티티 1건 표현(type, 좌표 배열, styleRef, layer, sourceHandle, **entityId(온톨로지용·nullable)**). entityId 부착 시점·규칙은 위임.

#### 스타일 테이블
> 색/선두께/대시 패턴을 엔티티에서 분리해 styleRef로 참조하는 정규화 테이블. 중복 제거 목적.

## 좌표 정규화 (RTC)
> CAD 모델 좌표 → 뷰어 픽셀/논리 좌표 변환 규칙. 마크업/이슈 핀의 좌표 기준이 되므로 **버전 간 안정성**이 핵심.

### RTC 정의
> RTC(Root/Reference Transform Context) — 변환의 의미를 한 줄 정의. ❓명칭 확정 필요(약어 의미 TBD).

### 좌표계 기준
> 원점·축 방향(CAD y-up vs 화면 y-down) 처리, 단위(mm 등) 보존 여부.

### 변환 행렬
> 스케일·평행이동(필요 시 회전) 행렬 정의. 시트별 바운딩박스 기반 정규화.

### 오버레이 좌표 계약
> 마크업/이슈 핀이 저장하는 좌표가 RTC 좌표인지 픽셀인지 결정 → 버전이 바뀌어도 핀 위치 유지되도록. (소비처: [[03_2D뷰어-렌더링]], [[05_이슈-마크업]].)
> ❓ 좌표 저장 단위 결정 = 시스템 전체 영향 큰 핵심 결정. **선결 TBD.**

## 썸네일 생성
> 시트 목록(데이터테이블)·홈 위젯·시트 비교 후보에 쓸 미리보기 이미지 산출.

### 렌더 방식
> 서버사이드 래스터화 vs 헤드리스 뷰어 렌더 중 택1. 트레이드오프(폰트/품질 vs 의존성). ❓ 방식 TBD.

### 사이즈/포맷
> 목록용 소형 + 상세용 중형 등 다중 해상도. 포맷(WebP/PNG) TBD.

### 저장/캐시 키
> 시트ID+버전 기반 캐시 키. 버전 변경 시 무효화 규칙. 저장 위치는 [[04-4_스토리지-CDE]]에 위임.

## 버전 비교용 산출
> 시트 비교(버전 오버레이 diff) 모듈이 소비할 추가 산출물. diff 계산 자체는 [[04-3_시트비교-버전diff]]에 위임 — 이 문서는 **비교 가능한 형태로 굳히는 책임**만.

### 비교 안정 키
> 버전 간 동일 엔티티를 매칭할 키(sourceHandle 등) 보존 규칙. 핸들 부재 시 대체 전략 TBD.

### 정렬/정규화 보장
> 동일 도면이면 동일 순서·동일 좌표가 나오도록 결정론적 직렬화(엔티티 정렬 규칙) — diff 노이즈 최소화.

### diff 입력 패키지
> old/new 두 기하 JSON + 좌표 정합 메타를 묶은 비교 입력 형태 정의.

## 배치 / 큐
> 다건 도면 변환을 비동기로 처리하는 작업 큐 운영. Phase 0 파싱 산출 재처리 포함.

### 잡 모델
> 변환 잡 1건 = (도면ID, 버전, 단계셋). 상태(대기/진행/완료/실패) 정의.

### 큐 / 워커
> 큐 기술 선택(TBD), 워커 동시성·재시도·idempotency. 대량 일괄 업로드 시 백프레셔.

### 진행 알림
> 변환 진행률을 프런트로 푸시([[WebSocket]] 또는 폴링). 토스트/위젯 갱신 연동. ❓ 푸시 방식 TBD.

### 실패 처리
> 부분 실패(엔티티 일부 변환 실패) 시 시트 단위 정책 — 건너뛰고 표시 vs 잡 실패.
