# DWG 파싱 엔진

## ① 목적
- [[DWG]] 바이너리 원본을 **편집 없이** 해독 → 객체 순회 → **기하 [[JSON]]** 산출
- 웹 [[2D뷰어]]가 렌더 가능한 중간표현 생성. 마크업·이슈·이벤트는 이 위에 **오버레이** (본 엔진 책임 아님)
- CAD 편집기 아님 — "읽기 전용 파서". osnap/좌표편집/[[SHX]] 정밀 폰트 렌더는 범위 밖

## ② 배경 / 전제
- 핵심 흐름: `DWG → [본 엔진] 파싱 → 기하 JSON → 뷰어 렌더 → 오버레이`
- 진실 원천 = 도면. [[DKS]] Phase 0 파싱 산출물(분야·도면번호·태그)이 선행 존재 → 본 엔진은 **기하 레벨** 추출 담당 (메타 추출과 분리)
- 차별점: 추출 엔티티에 향후 [[TypeDB]] **설비 엔티티 ID** 바인딩 가능하게 식별자 보존 (ACC엔 없는 무기). 단 바인딩 로직 자체는 [[온톨로지-바인딩]]에 위임
- 1차 범위 = 2D 시트 한정. 3D BIM/Bridge 제외

## [[ACadSharp]] 리더 흐름
> 채울 내용: 바이너리 진입점부터 문서 객체 트리 확보까지의 단계.

### 지원 포맷 / 버전 매트릭스
- [[ACadSharp]]가 읽는 DWG 버전 범위, DXF 동시 지원 여부 정리 (TBD: 청주 납품 도면 실제 버전 샘플 확인)

### 문서 로드 시퀀스
- 스트림 오픈 → `CadDocument` 획득 → ModelSpace 진입 순서
- 손상/암호/미지원 버전 시 실패 분기 의도 메모

### 단위 / 좌표계 처리
- 도면 단위($INSUNITS) → 뷰어 픽셀 좌표 변환 기준점. WCS 기준 정규화 의도

### 레이어 / 표준 메타
- 레이어명·색상·on/off·동결 상태를 기하 JSON에 보존하는 방식 (분야 매핑 힌트)

## BLOCK / INSERT 추출
> 채울 내용: 블록 정의와 인스턴스 분리·전개 전략.

### BLOCK 정의 카탈로그
- 블록 정의를 1회 파싱 → 참조 라이브러리화 (DRY, 중복 전개 방지)

### INSERT 인스턴스 전개
- INSERT의 위치·회전·스케일 변환행렬 적용 방식
- ❓ 결정 대기: 블록을 서버에서 **완전 전개(flatten)** vs 뷰어에서 참조 합성? (JSON 크기 ↔ 렌더 비용 트레이드오프)

#### 중첩 블록 (nested INSERT)
- 재귀 전개 깊이 한도, 순환참조 방어 의도 메모

#### 속성(ATTRIB) 추출
- 블록 속성 텍스트(태그=값) 추출 → 설비 식별자 후보. [[온톨로지-바인딩]] 입력으로 핸드오프

## 엔티티 필터링 (TEXT / LINE / ARC)
> 채울 내용: 2D 시트 렌더에 필요한 엔티티만 화이트리스트.

### 지원 엔티티 화이트리스트
- 1차 지원: LINE, LWPOLYLINE/POLYLINE, ARC, CIRCLE, TEXT/MTEXT, INSERT
- TBD: ELLIPSE, SPLINE, HATCH, DIMENSION 포함 여부 (렌더 난이도 ↔ 도면 충실도)

### 미지원 엔티티 처리 정책
- 스킵 vs 바운딩박스 플레이스홀더 vs 경고 로그 — 정책 결정 필요 (❓)

#### TEXT / MTEXT
- 문자열·삽입점·높이·정렬 추출. [[SHX]] 정밀 폰트는 범위 밖 → 웹폰트 근사 렌더 전제 명시

#### LINE / POLYLINE
- 정점 배열 추출, 폐합 여부 플래그

#### ARC / CIRCLE
- 중심·반지름·각도 → 뷰어 호 렌더용 파라미터 보존 (다각형 근사는 뷰어 책임)

### 좌표 정밀도 / 경량화
- 좌표 소수 자릿수 라운딩, 미세 세그먼트 단순화로 JSON 용량 제어 (TBD: 허용 오차 기준)

## PDF 경로 ([[PyMuPDF]]) 대안
> 채울 내용: DWG가 없거나 PDF로만 제공될 때의 보조 파이프라인.

### 적용 조건 (분기 기준)
- 원본이 [[PDF]]뿐인 경우 / DWG 파싱 실패 폴백 시에만 사용. 기본 경로는 어디까지나 DWG

### 벡터 PDF 추출
- [[PyMuPDF]]로 벡터 패스·텍스트 추출 → 동일 기하 JSON 스키마로 정규화

### 래스터 PDF 처리
- 스캔 도면은 이미지 타일로 처리, 벡터 추출 불가 — 오버레이만 지원. 한계 명시

### DWG 경로와의 산출물 정합
- ❓ 결정 대기: PDF 경로 산출물도 BLOCK/엔티티 ID 부여 가능한가? (불가 시 온톨로지 바인딩 제한)

## 산출물 핸드오프
> 채울 내용: 다운스트림(뷰어·이슈·온톨로지)으로 넘기는 계약.

### 기하 JSON 스키마 (참조)
- 스키마 본체는 [[04-2_기하JSON-스키마]]가 소유. 여기선 **생산자 관점** 필드 충족 책임만 명시

### 엔티티 안정 식별자 (Stable ID)
- 재파싱 시에도 동일 엔티티 = 동일 ID 보장 전략 → 오버레이·온톨로지 바인딩 영속성 핵심 (❓ ID 생성 규칙 결정 필요)

### 저장 / 캐싱
- 산출 JSON 저장 위치([[CDE]] 연동), 버전별 캐시 키. 시트 비교(diff)용 버전 보존 의도

### 비동기 처리 / 진행 통지
- 대용량 도면 비동기 파싱, 진행률을 [[WebSocket]]로 프론트 통지 (상세 API는 [[04_백엔드-API-파이프라인]] 위임)

### 실패 / 부분성공 리포트
- 미지원 엔티티·누락 블록 목록을 핸드오프 메타에 포함 → 뷰어 빈상태/경고 표시 근거
