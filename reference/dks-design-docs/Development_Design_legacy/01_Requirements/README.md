# 01. 요구사항 정의서 (SRS)

## 1. 비즈니스 목표 및 기대효과 (SRS-01) — [[01_Requirements/01_business_goals]]
* **목표**: 상용 오토데스크 [[클라우드]] API(APS)에 대한 원천적 의존성을 탈피하여 **도면 데이터 주권**을 획득하고 서비스 운영 비용을 최소화하는 독자적인 웹 CAD 도면관리 및 지식화 플랫폼을 구축합니다.
* **비즈니스 정량적 기대효과**:
    * **과금 장벽 배제**: APS의 Model Derivative API 호출당 요금(클라우드 뷰어 변환 당 요금 발생) 및 파일 저장 비용을 0원으로 차단합니다.
    * **자체 IP(지식재산권) 확보**: 도면 메타데이터 추출 알고리즘과 렌더러 소스 코드를 내재화하여, B2B 기업 고객 대상 폐쇄형(On-Premise) 또는 프라이빗 클라우드 망 내 배포가 가능해집니다.
    * **RAG 시너지**: 도면 내 객체 Handle ID와 텍스트를 [[TypeDB]] [[온톨로지]] 계통망과 결합해 지능형 시설 관리(FMS) 시장의 독점적 솔루션을 확보합니다.

## 2. 사용자 페르소나 및 유스케이스 정의 (SRS-02)

### ① 도면 관리자 / 엔지니어 (Admin & Knowledge Engineer)
* **목적**: 신축 사업장의 설계 도면([[DWG]]/[[DXF]])을 업로드하고, AI에 의해 파싱된 속성 데이터의 무손실율을 검증한 후 [[TypeDB]] 온톨로지에 동기화합니다.
* **핵심 유스케이스**:
    * 대량의 도면 일괄 업로드 및 레이어 자동 분류.
    * AI 파싱 결함(Defect) 리포트 모니터링 및 수동 보정(HITL).

### ② 현장 유지보수 운영자 (Operator)
* **목적**: 현장에서 태블릿이나 PC 웹 화면을 통해 특정 설비의 계통과 선후 연결 관계를 질문하고 도면 상의 정확한 물리 위치를 즉시 확인합니다.
* **핵심 유스케이스**:
    * 모바일/웹 자연어 채팅 질의: *"[[항온항습기]] AHU-01에 연결된 전원 [[배전반]]과 기계 [[냉동기]] 배관 라인을 표시해줘."*
    * 뷰어 연동: 챗봇 답변을 클릭하면 웹 도면 뷰어 상에서 해당 장비와 연결 배관 라인이 실시간 하이라이트되며 포커싱.

## 3. 기능 요구사항 명세 (SRS-03) — [[01_Requirements/03_functional_requirements]]

| 요구사항 ID | 기능명 | 상세 설명 및 기술적 충족 조건 |
| :--- | :--- | :--- |
| **FR-001** | DWG/DXF 파일 수집 및 업로드 | 사용자가 브라우저 UI에서 도면 파일을 드롭하면, 백엔드 파일 스토리지가 멀티파트(Multipart) 및 S3 Direct-to-Upload를 이용하여 무손실 업로드를 처리함. |
| **FR-002** | 백엔드 속성 및 블록 계층 파싱 | 업로드된 [[DWG]] 바이너리를 분석하여 도면 내 `AcDbBlockTable` (블록 정의)과 `INSERT`(블록 참조) 객체의 Handle ID, 삽입점 X/Y 좌표, 텍스트 속성을 [[JSON]] 및 [[SQLite]]로 자동 추출함. |
| **FR-003** | 폰트 매핑 및 선형 패턴 복원 | AutoCAD 고유 폰트인 SHX 및 일반 TTF 폰트를 웹 호환 [[WOFF]]/[[WOFF2]]로 매핑하고, 점선/쇄선 등의 선종류(Linetype) 정의를 웹에서 그릴 수 있는 픽셀 대시 배열로 분해(Explode)하여 렌더러에 공급함. |
| **FR-004** | 웹 브라우저 벡터 렌더링 | HTML5 Canvas 또는 WebGL(Three.js) 기반의 자체 뷰어를 통해 웹 브라우저에서 플러그인 설치 없이 도면을 60fps 이상으로 렌더링함. 레이어별 ON/OFF 필터링을 지원해야 함. |
| **FR-005** | 자석 스냅(Osnap) 및 측정 기능 | 마우스 포인트 반경 내 가장 가까운 끝점(Endpoint), 중간점(Midpoint)을 KD-Tree 인덱스 기반으로 짚어내고(Snap), 두 스냅 점 사이의 공간 거리(Distance) 및 닫힌 다각형 면적(Area)을 계산하여 화면에 그려줌. |
| **FR-006** | AI 대화 및 뷰어 동기화 액션 | 챗봇의 질의 결과 JSON 내에 타겟 장비의 Handle ID가 동봉될 시, 뷰어 이벤트 버스가 이를 감지하여 해당 객체를 파란색 반짝임 효과로 하이라이트하고 해당 좌표로 뷰를 자동 피팅함. |

## 4. 비기능 요구사항 명세 (SRS-04)

| 요구사항 ID | 품질 속성 | 기술적 제약 및 목표 성능 |
| :--- | :--- | :--- |
| **NFR-001** | 시각화 성능 및 FPS | 엔티티 수가 100,000개 수준인 대형 도면 로딩 시 초기 렌더링 속도 3초 이내, 마우스 드래그/줌 동작 시 화면 끊김 없는 초당 프레임 수 60fps 이상 유지 (OffscreenCanvas 및 Web Worker 연산 필수). |
| **NFR-002** | WebGL 좌표계 정밀도 | 좌표가 수십만 단위(예: $X=350,000.123$)로 구성된 거대 좌표 도면 줌인 시 렌더링 지터링(Jittering, 선 찢어짐 및 흔들림)이 발생하지 않아야 함. (상대 좌표 변환 적용 필수). |
| **NFR-003** | 라이선스 리스크 제어 | 상용 상업 배포 시 소스 코드 공개 의무를 전파시키는 GPLv3 / AGPLv3 계열 오픈소스의 웹 클라이언트 직접 탑재 및 링킹을 철저히 금지함. (MIT/Apache 2.0 라이선스 라이브러리 위주 편성). |
| **NFR-004** | CAD 데이터 호환성 | AutoCAD Release 14(AC1014)부터 최신 AutoCAD 2018+(AC1032) 포맷까지의 [[DWG]] 및 [[DXF]] 바이너리 포맷 규격을 백엔드 변환 파이프라인에서 무손실 해독할 수 있어야 함. |
