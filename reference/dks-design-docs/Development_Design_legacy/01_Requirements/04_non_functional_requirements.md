# SRS-04. 비기능 요구사항 명세서 (Non-Functional Requirements)

> **목적**: 시스템이 기능을 제공함에 있어 어떤 수준의 성능, 품질, 보안, 호환성을 갖추어야 하는지 제약 조건 및 품질 기준(NFR)을 정의합니다. 독자 구축된 웹 CAD 렌더러와 백엔드 파이프라인의 기준점이 됩니다.

## 1. 성능 (Performance)

### 1.1 렌더링 프레임 레이트 (FPS)
* **목표**: 100MB 규모 이하의 일반 2D 플랜트 계통도(P&ID, 단선도) 렌더링 시 **최소 30fps 이상, 권장 60fps**의 화면 주사율을 보장해야 한다.
* **구현 지침**: Three.js WebGL 파이프라인에서 수많은 선분(`LineBasicMaterial`)이 그려질 때 드로우 콜(Draw Call)을 줄이기 위해 `BufferGeometry`를 병합(Merge)하거나 `InstancedMesh`를 적극 적용하여 오버헤드를 막는다.

### 1.2 자석 스냅(Osnap) 응답 속도
* **목표**: 마우스 포인터가 도면 위를 움직일 때 가까운 정점(Vertex)을 탐색하여 스냅하는 연산 시간이 **16ms(1 프레임) 이내**여야 한다.
* **구현 지침**: 수십만 개의 정점 배열에 대해 브라우저 메인 스레드에서 선형 탐색($O(N)$)을 수행하면 렉이 발생한다. 따라서 도면 로딩 직후 `flatbush` 라이브러리를 활용해 KD-Tree(또는 R-Tree) 형태의 공간 분할 인덱스를 구축하여 $O(\log N)$ 성능으로 스냅을 검색해야 한다.

### 1.3 도면 로딩 및 API 레이턴시
* **목표**: 캐싱된 JSON 도면 데이터가 프론트엔드에 도달하고 초기 렌더링이 완료되기까지 **네트워크 다운로드 시간을 제외하고 3초 이내**에 화면에 표출되어야 한다. 

---

## 2. 보안 및 데이터 주권 (Security)

### 2.1 폐쇄망(On-Premise) 완전 격리 지원
* **목표**: Autodesk Forge/APS와 같은 외부 퍼블릭 클라우드 통신을 단 1바이트도 허용하지 않고, 사내 방화벽 내부(망분리 환경)에서 100% 독자 가동될 수 있어야 한다.
* **구현 지침**: 도면 파싱 라이브러리(ACadSharp)와 렌더러 모두 외부 서버 요청이 필요 없는 스탠드얼론(Standalone) 컨테이너 구조여야 한다.

### 2.2 도면 원본 파일(DWG) 유출 통제
* **목표**: 사용자가 웹 뷰어로 도면을 볼 때, CAD 원본 바이너리 파일(.dwg)이 클라이언트의 로컬 PC에 직접 다운로드되지 않아야 한다.
* **구현 지침**: 웹 브라우저에는 오직 기하학적 렌더링을 위한 좌표 캐시 데이터(JSON)만 전달되며, 원본 파일의 직접 다운로드 버튼은 관리자 권한(`role: 'admin'`)에게만 노출된다.

## 3. 호환성 및 기술 제약 (Compatibility)

### 3.1 파일 포맷 지원 범위
* **목표**: 업계 표준인 **AutoCAD 2013 ~ 2018 (및 그 이상 최신 버전)** 규격의 `DWG` 및 `DXF` 포맷을 무손실 해독할 수 있어야 한다.
* **구현 지침**: 백엔드 파서는 `DwgVersion` 해더를 읽어내어 알맞게 파싱하며, 만약 전용 3D CAD(예: Civil 3D 프록시) 요소가 포함되어 해독 불가능한 경우 사용자에게 명확한 에러 로그("Proxy Entity 미지원")를 반환해야 한다.

### 3.2 브라우저 호환성
* **목표**: 최신 **WebGL 2.0**을 지원하는 모든 모던 데스크톱/모바일 브라우저(Chrome, Edge, Safari)에서 동작해야 한다.
* **구현 지침**: IE11 등 레거시 브라우저는 공식적으로 지원을 중단(Not Supported)하며, 접속 시 차단 안내문을 띄운다.

## 4. 신뢰성 및 가용성 (Reliability & Availability)

### 4.1 Git-like 데이터 무결성 보장
* **목표**: 마크업 커밋과 도면 커밋은 변경이나 훼손이 불가능한 '불변(Immutable) 레코드' 원칙을 준수해야 한다.
* **구현 지침**: PostgreSQL 데이터베이스 설계 시, 도면 레코드(`drawings` 테이블)가 갱신될 때 기존 레코드를 `UPDATE` 하는 것이 아니라 무조건 새로운 `drawing_id`로 `INSERT` 되도록 애플리케이션 레벨의 로직(Append-only)을 통제한다.
