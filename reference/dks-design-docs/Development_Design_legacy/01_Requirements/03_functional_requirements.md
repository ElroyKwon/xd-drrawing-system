# SRS-03. 데이터 지식 스튜디오 상세 기능 목록 (Deep-Dive)

> **문서 기준**: 본 문서는 `대화-001-윤비서-사내ERP-AI에이전트-기능목록`의 구조와 퀄리티를 레퍼런스로 삼아, 단순 기능 나열을 넘어 **'데이터 지식 스튜디오(웹 CAD 및 도면관리 플랫폼)'의 모든 화면, 백엔드 로직, AI 연동, 파이프라인 흐름을 상세하게 리스트화하고 구조화**한 마스터 기능 요구사항 정의서입니다.

---

## 1. 데이터 지식 스튜디오(DKS) 플랫폼의 큰 구조

본 플랫폼은 단순한 도면 파일 뷰어가 아니라, **도면 데이터를 지식화하여 한곳에 모으고 AI가 추론·검색·뷰잉·협업까지 보조하는 산업 현장형 운영 OS**에 가깝습니다. 

구조를 단순화하면 다음 4계층으로 나뉩니다.

| 구분 | 기능 |
| --- | --- |
| **입력층** | DWG/DXF 파일 업로드, 현장 엔지니어 마크업(펜/텍스트), 사용자 채팅(자연어 질의) |
| **데이터층** | Git-like 도면 버전(커밋) 이력, 파싱된 기하 JSON, 마크업 커밋 레코드, TypeDB 온톨로지 지식망 |
| **처리층** | ACadSharp 백엔드 파서, Three.js 렌더링 엔진, flatbush KD-Tree 스냅 엔진, LLM RAG 추론 엔진 |
| **액션층** | 화면 내 타겟 설비 점멸/줌인, Auto Diff 차이점 시각화, 도면 기반 작업 지시서 생성, 후속 조치 알림 |

---

## 2. 실제 메뉴 / 코어 업무 기능 16개

사용자가 플랫폼에 접속했을 때 접근 가능한 **핵심 업무 메뉴 16개**입니다.

| 번호 | 기능 | 상세 의미 |
| -: | --- | --- |
| 1 | **대시보드** | 내 프로젝트, 최근 수정된 도면, 확인해야 할 마크업 피드백을 한눈에 조회 |
| 2 | **프로젝트 관리** | 공장/라인/섹터별로 프로젝트를 생성하고 도면 그룹핑 및 권한 배정 |
| 3 | **도면함 (Repository)** | 특정 프로젝트 내의 전체 도면 리스트, 최신 버전, 업로드 일자 관리 |
| 4 | **웹 CAD 뷰어** | 브라우저 내에서 도면을 렌더링하고 패닝/줌/스냅 제어 |
| 5 | **버전 히스토리** | 특정 도면의 과거부터 현재까지의 Git-like 커밋 이력 조회 및 롤백 |
| 6 | **Auto Diff (차분 대조)** | 두 도면 버전을 겹쳐서 추가/삭제된 설비를 초록/빨강으로 자동 색상 대조 |
| 7 | **레이어 관리** | 도면 내 포함된 레이어(Layer) 가시성 On/Off 제어 패널 |
| 8 | **블록/속성 브라우저** | 도면 내 삽입된 장비 블록(INSERT) 리스트 및 내부 속성값(TEXT) 트리 뷰어 |
| 9 | **치수 측정 도구** | 거리(Distance), 반경(Radius), 면적(Area) 측정 도구 툴바 |
| 10 | **마크업 오버레이** | 도면 위에 지시선, 도형, 텍스트 주석을 그리는 드로잉 툴보드 |
| 11 | **이슈 / 피드백 보드** | 마크업 커밋 단위로 등록된 피드백 리스트 및 상태(Open/Merged) 관리 |
| 12 | **글로벌 AI 검색** | 도면 밖 메인 화면에서 자연어로 전체 설비/도면을 검색하는 챗봇 인터페이스 |
| 13 | **로컬 AI 세션** | 특정 도면 뷰어 내에 종속되어, 열려 있는 도면에 대해서만 질의하는 챗봇 |
| 14 | **지식 맵핑 모니터** | (관리자) Handle ID와 TypeDB 엔티티 간의 매핑 정합성 모니터링 메뉴 |
| 15 | **API 연동 콘솔** | 외부 FMS, BMS 등과 데이터 연동을 위한 토큰 발급 및 API 문서 조회 |
| 16 | **시스템 로그** | 도면 변환 성공/실패율, 렌더링 에러, AI 추론 이력 등 시스템 감사 로그 |

## 3. 대시보드 및 프로젝트 관리 기능

| 기능 | 설명 |
| --- | --- |
| **전체 현황 요약** | 내가 속한 프로젝트 수, 총 도면 수, 해결 안 된 마크업 이슈 카운트 표시 |
| **최근 활성 도면** | 최근 3일 이내 업데이트된 도면 또는 내가 마지막에 본 도면 리스트 출력 |
| **프로젝트 워크스페이스** | 프로젝트별로 멤버를 할당하고 독립된 도면함 환경(Repository) 제공 |
| **프로젝트 노트** | 프로젝트 특이사항, 설계 변경 히스토리, 현장 공지사항 텍스트 기록 |

## 4. 도면 수집 및 파싱 파이프라인 (백엔드)

도면 파일 업로드 시 **백엔드 서버에서 백그라운드로 작동하는 자동화 기능**입니다.

| 기능 | 설명 |
| --- | --- |
| **DWG/DXF 업로드** | 클라이언트에서 바이너리 파일을 S3로 안전하게 다이렉트 업로드 |
| **ACadSharp 파싱 트리거** | 업로드 완료 이벤트를 캐치하여 백엔드 Worker 파싱 큐에 등록 |
| **기하학적 객체 해독** | LINE, ARC, CIRCLE, SPLINE, LWPOLYLINE 좌표 데이터를 순회 추출 |
| **블록 전개 (Explode)** | `AcDbBlockTable` 참조를 해독하여 중첩된 블록을 로컬 좌표로 전개 |
| **문자열 속성 추출** | TEXT, MTEXT 요소와 블록 내부의 ATTRIB 값을 JSON 텍스트 배열로 적재 |
| **JSON 최적화 캐싱** | 웹에서 Three.js가 즉각 읽어들일 수 있는 최적화된 JSON 구조로 직렬화하여 S3 저장 |
| **변환 상태 웹소켓 푸시** | 파싱 0% ~ 100% 진행률을 프론트엔드로 실시간 알림 (완료 시 뷰어 진입 활성화) |

## 5. Git-like 버전 관리 및 차분 대조 (Auto Diff) 기능

단순 파일 덮어쓰기가 아닌, **이력 추적과 델타(Delta) 뷰를 지원하는 핵심 관리 기능**입니다.

| 기능 | 설명 |
| --- | --- |
| **도면 커밋 (Commit)** | 동일 파일명으로 재업로드 시 구버전을 지우지 않고 신규 버전(자식) 레코드 생성 |
| **커밋 메시지 등록** | "펌프 P-101 위치 변경 반영" 등 리비전(Revision) 사유 강제 작성 |
| **버전 트리 조회** | 특정 도면의 타임라인(V1 -> V2 -> V3) 이력을 트리 형태로 확인 |
| **과거 버전 롤백** | 최신 버전 문제 발생 시 과거 버전 상태로 강제 복구(체크아웃) 가능 |
| **Diff 연산 엔진** | 백엔드에서 V1/V2의 Handle ID와 기하 좌표를 비교하여 `added`, `deleted` 목록 도출 |
| **Auto Diff 오버레이** | 뷰어에서 두 버전을 겹치고, 추가된 선은 초록색, 삭제된 선은 붉은 점선으로 렌더링 |

## 6. 웹 CAD 시각화 및 렌더링 엔진 (프론트엔드)

오토데스크 뷰어 없이 **자체 구현되는 WebGL 브라우저 렌더러 기능**입니다.

| 기능 | 설명 |
| --- | --- |
| **WebGL 캔버스 초기화** | Three.js 씬(Scene), 카메라, 렌더러 프로파일 초기화 및 최적화 설정 |
| **무한 팬(Pan) / 줌(Zoom)** | 마우스 드래그 및 휠 이벤트를 통한 부드러운 2D 공간 이동 및 축소/확대 |
| **바운딩 박스 핏(Fit-to-View)** | 단축키 또는 버튼 클릭 시 전체 도면이 화면에 딱 맞게 카메라 자동 리셋 |
| **GPU 인스턴싱 렌더링** | 동일한 블록(의자, 밸브 등 수백 개)은 `InstancedMesh`로 묶어 프레임 드랍 방지 |
| **선종류(Linetype) 셰이더** | 점선, 쇄선 등을 Three.js 셰이더(ShaderMaterial) 단에서 패턴으로 끊어 그리기 구현 |
| **폰트/글씨체 렌더러** | 도면 내 한글 및 CAD 전용 SHX 폰트를 웹폰트 패스로 매핑하여 텍스트 깨짐 없이 출력 |

## 7. 스냅(Osnap) 및 설계 상호작용 도구

현장 엔지니어가 도면을 조작하고 실측할 수 있는 **인터랙션 기능**입니다.

| 기능 | 설명 |
| --- | --- |
| **KD-Tree 공간 인덱싱** | 로딩 즉시 도면의 모든 좌표 포인트를 `flatbush` 등 공간 검색 인덱스로 빌드 |
| **자석 스냅 십자선** | 마우스 포인터가 선분이나 끝점 반경 10px 내 진입 시, 정확한 기하 정점으로 강제 흡착 |
| **스냅 아이콘 오버레이** | 끝점(네모), 중간점(세모), 교차점(엑스) 등 오토캐드와 동일한 스냅 심볼 렌더링 |
| **거리 및 치수 측정** | 두 점을 스냅 클릭하면 유클리디안 거리에 도면 스케일을 곱해 실제 M/mm 치수 표시 |
| **다각형 면적 계산** | 3개 이상의 점을 찍어 닫힌 폴리곤을 만들면 그린의 정리(Shoelace)로 내부 면적 실시간 산출 |
| **객체 속성 팝업(Tooltip)** | 도면 내 밸브나 설비 텍스트를 클릭하면 추출된 메타 속성(Layer, Block Name)을 툴팁으로 표시 |

## 8. 마크업 오버레이 및 협업 기능

원본 DWG 파일을 건드리지 않고 **SVG 투명 레이어로 협업하는 기능**입니다.

| 기능 | 설명 |
| --- | --- |
| **펜 드로잉 툴보드** | 색상, 굵기를 선택하여 도면 화면 위에 마우스로 자유 곡선 그리기 |
| **도형 및 지시선 도구** | 사각형, 원형 치기 및 화살표 지시선 삽입 |
| **텍스트 주석 삽입** | 텍스트 박스를 열어 수정 요청사항 직접 타이핑 |
| **뷰포트 동기화 행렬변환** | 사용자가 도면을 줌/팬 할 때 마크업 SVG 레이어의 Matrix도 동일하게 변환시켜 위치 고정 |
| **마크업 커밋 등록** | 작성한 드로잉을 JSON 좌표 패스로 직렬화하여 S3에 저장하고 마크업 상태를 'OPEN'으로 등록 |
| **마크업 상태 관리** | 작성된 피드백에 대해 타겟 담당자가 조치 완료 후 'MERGED' 또는 'CLOSED' 처리 |
| **사용자별 마크업 필터** | 우측 패널에서 특정 일자, 특정 작성자의 마크업만 체크박스로 껐다 켤 수 있는 레이어 기능 |

## 9. AI 비서 연동 및 하이브리드 검색 (가장 중요한 코어)

윤비서의 '자연어 질의' 철학을 가져온 **도면 맞춤형 AI 추론 연동 메커니즘**입니다.

| 기능 | 설명 |
| --- | --- |
| **글로벌 자연어 설비 검색** | 메인 화면에서 "청주공장 2층 펌프 목록 찾아줘" 검색 시 RAG가 해당 도면 리스트 응답 |
| **세션 전이 파라미터 전달** | 검색 결과 도면 클릭 시 `?drawing_id=A&highlight=Handle_123` 파라미터로 뷰어 강제 호출 |
| **로컬 도면 문답 세션** | 열려 있는 도면 뷰어 옆 사이드바에서 챗봇 가동. "이 도면에서 V-101 밸브 어딨어?" 질의 |
| **TypeDB 온톨로지 추론** | 챗봇이 도면 기하가 아닌, 백엔드 TypeDB 지식망에 TQL 쿼리를 날려 계통적 답변과 Handle ID 획득 |
| **이벤트 버스 하이라이트** | 챗봇이 반환한 `target_handles` 페이로드를 프론트엔드가 수신하여 즉각 해당 객체 파란색 점멸 |
| **자동 줌-투-핏(Zoom-to-fit)** | 점멸되는 타겟 설비들의 바운딩 박스를 계산하여 한 화면에 들어오도록 뷰어 카메라 자동 패닝 |

## 10. 현장 발생 이슈 → 자동 파이프라인 흐름

현장의 상황이 업무 데이터로 자연스럽게 흘러가는 **자동화 파이프라인**입니다.

| 단계 | 기능 흐름 |
| -- | --- |
| 1 | **현장 발견**: 현장 오퍼레이터가 패드로 도면을 열어보고 실제 설치된 밸브 위치가 다름을 발견 |
| 2 | **마크업 생성**: 패드 화면 위에 펜으로 동그라미 치고 "밸브 위치 도면 불일치. 수정 요망" 마크업 커밋 저장 |
| 3 | **AI 알림 및 할일 자동화**: 마크업 커밋 이벤트를 감지한 AI가 설계 담당자에게 사내 메신저(Slack)로 자동 알림 발송 |
| 4 | **설계 담당자 조치**: 담당자가 알림 링크를 타고 들어와 마크업 확인 후, AutoCAD 원본에서 파일 수정 |
| 5 | **버전 업로드 (V2)**: 수정한 파일을 플랫폼에 재업로드 (새 커밋 해시 생성) |
| 6 | **Auto Diff 검증**: 시스템이 V1과 V2를 대조하여 밸브 위치가 실제로 이동(삭제선+추가선)했음을 시각적으로 증명 |
| 7 | **마크업 클로즈**: 오퍼레이터가 Diff 결과를 확인하고 해당 마크업 이슈를 'CLOSED' 처리하여 업무 종료 |

## 11. 관리자 권한 및 시스템 운영 / API 제공

| 기능 | 설명 |
| --- | --- |
