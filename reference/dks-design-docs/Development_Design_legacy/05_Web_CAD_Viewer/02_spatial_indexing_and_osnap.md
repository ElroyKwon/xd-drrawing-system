# 05-2. 자석 스냅(Osnap) 및 공간 인덱싱 알고리즘 설계 명세

> **목적**: 사용자가 마우스를 움직일 때 도면상의 수십만 개의 선분, 교차점, 중심점 중 가장 가까운 객체로 마우스 포인터가 달라붙는 '자석 스냅' 현상을 $O(\log N)$의 고속 성능으로 구현하기 위한 수학적 모델을 설계합니다.

---

## 1. 공간 분할 인덱싱 (Spatial Indexing)

매 프레임마다 마우스 좌표와 10만 개의 도면 정점을 일일이 비교 연산($O(N)$)하면 브라우저는 즉시 멈춥니다. 이를 방지하기 위해 도면 로딩 직후 정점 데이터 구조를 최적화합니다.

### 1.1 `flatbush` 라이브러리 도입 (초고속 R-Tree)
자바스크립트 환경에서 정적(Static) 데이터를 위한 가장 빠른 R-Tree/KD-Tree 구현체인 `flatbush`를 활용합니다.
1. JSON이 프론트엔드에 로드되면, 모든 선분의 시작점(Start), 끝점(End), 중심점(Center) 데이터를 추출합니다.
2. `flatbush` 인덱스를 생성하고 포인트를 삽입(`bush.add(x, y, x, y)`)한 뒤 트리 구조를 압축/빌드(`bush.finish()`)합니다. 빌드 소요 시간은 통상 10만 개 기준 수 밀리초 이내입니다.

---

## 2. 고속 스냅 쿼리 로직

마우스가 움직일(`mousemove`) 때마다 아래의 탐색 파이프라인이 가동됩니다.

1. **마우스 좌표 변환**: 스크린 좌표 $(clientX, clientY)$를 Three.js 카메라 행렬의 역산(`unproject`)을 통해 도면의 월드 2D 좌표 $(W_x, W_y)$로 변환합니다.
2. **반경 검색 (Radius Search)**: 마우스 커서 주변 10픽셀 반경에 해당하는 가상의 Bounding Box(MinX, MinY, MaxX, MaxY)를 계산하고, `flatbush.search()`를 호출합니다.
3. **유클리디안 거리 연산**: 트리가 반환한 후보 노드(통상 2~5개)들에 대해서만 피타고라스 정리에 의한 실제 거리를 연산합니다.
   $$ Distance = \sqrt{(P_x - W_x)^2 + (P_y - W_y)^2} $$
4. **마커 스냅**: 가장 거리가 짧은 노드의 좌표로 뷰어 상의 스냅 심볼(십자선 또는 네모 마커) 위치를 강제로 덮어씌웁니다.

이 과정을 통해 스냅 연산 복잡도는 사실상 $O(\log N)$이 되어, 마우스를 아무리 빠르게 움직여도 1ms 이내에 스냅 타겟을 찾아냅니다.

---

## 3. 치수 및 면적 측정 알고리즘 수학 모델

### 3.1 거리 측정 (Distance Measurement)
사용자가 스냅된 두 점 $A(x_1, y_1)$과 $B(x_2, y_2)$를 클릭하면, 유클리디안 공간 거리를 구합니다.
이후 도면의 전역 스케일 팩터(Scale Factor, 예를 들어 1 Unit = 1mm)를 곱하여 최종 수치를 구하고, 두 점을 잇는 보조 치수선과 직교하는 보조선(Extension Line) 벡터를 삼각함수(Sin, Cos)로 계산하여 캔버스에 그립니다.

### 3.2 신발끈 공식 (Shoelace Formula) 기반 면적 산출
사용자가 연속으로 $n$개의 점 $P_1(x_1, y_1), P_2(x_2, y_2), \dots, P_n(x_n, y_n)$ 을 닫힌 다각형 형태로 스냅할 경우, 복잡한 내부 적분 없이 **그린의 정리(Green's Theorem)**에 기반한 다각형 면적 공식을 사용하여 $O(N)$의 속도로 정확한 면적을 구합니다.

$$ Area = \frac{1}{2} \left| \sum_{i=1}^{n-1} (x_i \cdot y_{i+1} - x_{i+1} \cdot y_i) + (x_n \cdot y_1 - x_1 \cdot y_n) \right| $$

구해진 면적(Area) 값은 $m^2$ 또는 $mm^2$ 단위로 환산되어, 생성된 폴리곤의 무게중심(Centroid) 좌표에 텍스트 라벨로 렌더링됩니다.
